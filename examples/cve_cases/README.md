# CVE 风格漏洞审计案例（工业实战项目）

3 个经典 C 语言安全漏洞的**漏洞代码**、**修复代码**和**防御方案**。

## 案例索引

| 案例 | 漏洞类型 | 对标 CVE | 核心错误 |
|------|---------|---------|---------|
| 1 | 缓冲区过度读取 | CVE-2014-0160 (HeartBleed) | 信任客户端提供的长度字段 |
| 2 | Use-After-Free / 返回栈地址 | CVE-2019-1010023 风格 | 释放后使用、返回局部地址 |
| 3 | 整数溢出 → 缓冲区溢出 | CVE-2018-1000001 风格 | 乘法溢出、有符号/无符号混合 |

## 编译运行

```bash
make
./cve_cases
```

> 建议同时用 AddressSanitizer 编译验证：
> `gcc -fsanitize=address -g cve_cases.c -o cve_asan`

## 使用方式

1. **教育培训**：展示漏洞代码 → 问学员问题 → 揭示修复方案
2. **代码审计**：作为检查清单，审查实际项目中的同类模式
3. **CTF 训练**：漏洞代码可作为练习题目

## 防御检查清单

- [ ] 所有外部输入的长度都经过验证
- [ ] 乘法/加法运算检查溢出
- [ ] 分配失败时保持对象一致性
- [ ] 不返回局部变量地址
- [ ] 有符号数先检查负数再转无符号
- [ ] 使用 `-fsanitize=address` 测试
- [ ] 使用 `-D_FORTIFY_SOURCE=2` 编译
- [ ] 静态分析：Cppcheck, clang-analyzer
