# 性能优化案例（工业实战项目）

可量化的 C 语言性能优化技术演示，每个案例包含**慢版本**和**快版本**的对比基准测试。

## 案例索引

| 案例 | 技术 | 原理 | 典型加速比 |
|------|------|------|-----------|
| 1 | 缓存行对齐 | 避免 False Sharing | 10x-100x（多线程） |
| 2 | 分支预测友好化 | 排序数据、消除分支 | 2x-5x |
| 3 | 循环展开 + 指针遍历 | 减少循环开销 | 1.2x-2x |

## 编译运行

```bash
make
./performance
```

## 关键优化原则

| 原则 | 说明 |
|------|------|
| **数据局部性** | 一起访问的数据放在一起，跨缓存行的数据分开 |
| **分支可预测** | 让条件分布有模式（排序、分区），或用位运算消除分支 |
| **减少循环开销** | 展开循环、用指针替代索引、合并循环 |
| **避免过早优化** | 先测量，再优化；编译器 `-O3` 能做很多 |

## 编译器优化对比

```bash
# 无优化（教学用）
gcc -O0 performance.c -o perf_debug

# 标准优化
make

# 极致优化 + LTO
gcc -O3 -flto -march=native performance.c -o perf_fast
```

> 注意：在 `-O3` 下，编译器可能自动展开循环和向量化，缩小手动优化的差距。
