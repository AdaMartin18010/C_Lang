#### ❌ 问题四："伪形式化"严重

**表象**: 很多文件声称"形式化"
**实质**:

- 大多数"形式化"只是贴了几个数学符号，没有严格的定义
- 缺少公理系统、推导规则、证明
- 没有与ISO标准对应的形式化规范
- CompCert Clight的对应关系几乎为零

**示例批评**:

```markdown
❌ 当前内容（伪形式化）:
"指针可以定义为地址和类型的二元组"
→ 没有形式化语法
→ 没有语义函数
→ 没有公理系统
→ 无法验证

✅ 应该的形式化:
"定义地址空间 𝒜 = ℕ_{2^32} (32位系统)
 定义类型集合 𝒯 = {int, char, float, ...}
 指针空间 𝒫 = {(a, τ) | a ∈ 𝒜, τ ∈ 𝒯, align(a, τ)}
 语义函数: load: 𝒫 → Value ⊥
 ..."
```
