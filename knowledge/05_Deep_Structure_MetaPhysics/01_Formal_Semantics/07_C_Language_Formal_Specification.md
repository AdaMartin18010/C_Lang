## 1. 表达式指称语义

### 1.1 表达式抽象语法

```
e ::= n                     (整数常量)
  |   f                     (浮点常量)
  |   x                     (变量)
  |   *e                    (解引用)
  |   &e                    (取地址)
  |   e₁ + e₂               (算术运算)
  |   e₁ - e₂
  |   e₁ * e₂
  |   e₁ / e₂
  |   e₁ % e₂
  |   e₁ == e₂              (关系运算)
  |   e₁ != e₂
  |   e₁ < e₂
  |   e₁ <= e₂
  |   e₁ > e₂
  |   e₁ >= e₂
  |   e₁ && e₂              (逻辑运算)
  |   e₁ || e₂
  |   !e
  |   e₁ = e₂               (赋值)
  |   e₁ += e₂              (复合赋值)
  |   e++ | ++e             (自增/自减)
  |   e₁ ? e₂ : e₃          (条件表达式)
  |   (τ)e                  (类型转换)
  |   e₁, e₂                (逗号表达式)
  |   f(e₁, ..., eₙ)        (函数调用)
  |   sizeof(τ) | sizeof e  (大小运算)
  |   alignof(τ)            (对齐运算)
```

### 1.2 表达式求值语义

**配置**：`⟨e, η, μ⟩` 其中 η 是环境，μ 是存储

**常量**：

```
⟨n, η, μ⟩ ⇓ ⟨n, η, μ⟩          (整数值)
⟨f, η, μ⟩ ⇓ ⟨f, η, μ⟩          (浮点值)
```

**变量访问**（左值转右值）：

```
η(x) = l    μ(l) = v
───────────────────────────────
⟨x, η, μ⟩ ⇓ ⟨v, η, μ⟩
```

**算术运算**：

```
⟨e₁, η, μ⟩ ⇓ ⟨n₁, η', μ'⟩    ⟨e₂, η', μ'⟩ ⇓ ⟨n₂, η'', μ''⟩    n = n₁ + n₂
────────────────────────────────────────────────────────────────────────────
                     ⟨e₁ + e₂, η, μ⟩ ⇓ ⟨n, η'', μ''⟩
```

**除法（含零检查）**：

```
⟨e₁, η, μ⟩ ⇓ ⟨n₁, η', μ'⟩    ⟨e₂, η', μ'⟩ ⇓ ⟨0, η'', μ''⟩    n₁ ≠ 0
─────────────────────────────────────────────────────────────────────────────
                     ⟨e₁ / e₂, η, μ⟩ ⇓ ⊥_divzero

⟨e₁, η, μ⟩ ⇓ ⟨n₁, η', μ'⟩    ⟨e₂, η', μ'⟩ ⇓ ⟨n₂, η'', μ''⟩    n₂ ≠ 0    n = n₁ / n₂
──────────────────────────────────────────────────────────────────────────────────────
                     ⟨e₁ / e₂, η, μ⟩ ⇓ ⟨n, η'', μ''⟩
```

### 1.3 指针运算

**取地址**：

```
⟨e, η, μ⟩ ⇓_L ⟨l, η', μ'⟩      (e 作为左值求值得位置 l)
────────────────────────────────
⟨&e, η, μ⟩ ⇓ ⟨l, η', μ'⟩
```

**解引用**：

```
⟨e, η, μ⟩ ⇓ ⟨l, η', μ'⟩    l ∈ dom(μ')    μ'(l) = v
─────────────────────────────────────────────────────
⟨*e, η, μ⟩ ⇓ ⟨v, η', μ'⟩

⟨e, η, μ⟩ ⇓ ⟨l, η', μ'⟩    l ∉ dom(μ')
───────────────────────────────────────
⟨*e, η, μ⟩ ⇓ ⊥_undefaddr
```

**指针算术**：

```
⟨e₁, η, μ⟩ ⇓ ⟨l, η', μ'⟩    ⟨e₂, η', μ'⟩ ⇓ ⟨n, η'', μ''⟩    size(τ) = k
───────────────────────────────────────────────────────────────────────────
           ⟨e₁ + e₂, η, μ⟩ ⇓ ⟨l + n×k, η'', μ''⟩       (若 e₁ : τ*)
```

### 1.4 赋值表达式

```
⟨e₁, η, μ⟩ ⇓_L ⟨l, η', μ'⟩    (左操作数求值为位置)
⟨e₂, η', μ'⟩ ⇓ ⟨v, η'', μ''⟩  (右操作数求值为值)
────────────────────────────────────────────────────
⟨e₁ = e₂, η, μ⟩ ⇓ ⟨v, η'', μ''[l ↦ v]⟩
```

### 1.5 序列点与未指定行为

```
序列点: 完整表达式结束、&&、||、?:、逗号运算符处

限制: 同一标量对象在两个不同序列点之间被修改超过一次是未定义行为

示例: i = ++i 是 UB
      i = i + 1 是良好定义
```

## 2. 语句操作语义

### 2.1 语句语法

```
S ::= ;                           (空语句)
  |   e;                          (表达式语句)
  |   { S₁ ... Sₙ }               (复合语句/块)
  |   if (e) S₁ else S₂          (条件语句)
  |   if (e) S₁                  (简写: else 为空语句)
  |   switch (e) { case ... }    (开关语句)
  |   while (e) S                (while 循环)
  |   do S while (e);            (do-while 循环)
  |   for (e₁; e₂; e₃) S         (for 循环)
  |   break;                      (跳出)
  |   continue;                   (继续)
  |   return e;                   (返回)
  |   return;                     (void 返回)
  |   goto label;                 (跳转)
  |   label: S                    (标签)
```

### 2.2 语句语义规则

**表达式语句**：

```
⟨e, η, μ⟩ ⇓ ⟨v, η', μ'⟩
────────────────────────────
⟨e;, η, μ⟩ → η', μ'
```

**复合语句**（块作用域）：

```
⟨{ D S }, η, μ⟩ → ⟨S, η', μ'⟩    其中 ⟨D, η, μ⟩ 扩展环境为 η', μ'
```

**If 语句**：

```
⟨e, η, μ⟩ ⇓ ⟨true, η', μ'⟩    ⟨S₁, η', μ'⟩ → η₁, μ₁
─────────────────────────────────────────────────────
⟨if (e) S₁ else S₂, η, μ⟩ → η₁, μ₁

⟨e, η, μ⟩ ⇓ ⟨false, η', μ'⟩   ⟨S₂, η', μ'⟩ → η₂, μ₂
─────────────────────────────────────────────────────
⟨if (e) S₁ else S₂, η, μ⟩ → η₂, μ₂
```

**While 循环**：

```
⟨e, η, μ⟩ ⇓ ⟨false, η', μ'⟩
────────────────────────────────
⟨while (e) S, η, μ⟩ → η', μ'

⟨e, η, μ⟩ ⇓ ⟨true, η', μ'⟩    ⟨S, η', μ'⟩ → η₁, μ₁    ⟨while (e) S, η₁, μ₁⟩ → η₂, μ₂
─────────────────────────────────────────────────────────────────────────────────────────
                         ⟨while (e) S, η, μ⟩ → η₂, μ₂
```

**For 循环**（desugaring）：

```
for (e₁; e₂; e₃) S  ≡  e₁; while (e₂) { S e₃; }
```

**Break**：

```
break 将控制转移到最内层循环或 switch 之后的语句

标记为异常: ⟨break, η, μ⟩ ⇓ ⟨break, η, μ⟩
在循环中捕获:
  ⟨S, η, μ⟩ ⇓ ⟨break, η', μ'⟩   (S 执行了 break)
  ─────────────────────────────────────────────
  ⟨while (e) S, η, μ⟩ → η', μ'   (若未捕获，继续传播)
```

**Return**：

```
⟨e, η, μ⟩ ⇓ ⟨v, η', μ'⟩
────────────────────────────
⟨return e;, η, μ⟩ ⇓ ⟨return v, η', μ'⟩

return 异常传播直到函数边界
```

### 2.3 非局部控制流

```
goto label:
  跳转到标记语句
  限制: 不能跳过变长数组的声明

setjmp/longjmp:
  保存/恢复执行上下文
  实现非局部跳转
```

## 3. 声明的语义

### 3.1 声明语法

```
D ::= τ x;                        (变量声明)
  |   τ x = e;                    (带初始化)
  |   τ f(τ₁ x₁, ..., τₙ xₙ) { S }  (函数定义)
  |   typedef τ T;                (类型别名)
  |   struct T { ... };           (结构体)
  |   union T { ... };            (联合体)
  |   enum T { ... };             (枚举)
```

### 3.2 变量声明语义

**无初始化**：

```
⟨τ x;, η, μ⟩ → η[x ↦ l], μ[l ↦ undef_τ]    l = newloc(μ), align(l, τ)
```

**带初始化**：

```
⟨e, η, μ⟩ ⇓ ⟨v, η', μ'⟩    l = newloc(μ')    v : τ
─────────────────────────────────────────────────────
⟨τ x = e;, η, μ⟩ → η'[x ↦ l], μ'[l ↦ v]
```

### 3.3 作用域规则

```
块作用域: { ... τ x ... } 中 x 仅在块内可见

函数作用域: 标签在整个函数内可见

文件作用域: 全局声明，整个翻译单元可见

extern: 链接到外部定义
static: 内部链接
```

### 3.4 存储类别语义

```
auto:    自动存储期，块结束时释放
static:  静态存储期，程序整个运行期存在
extern:  引用外部定义
register: 建议存储在寄存器中（优化提示）
_Thread_local: 线程存储期
```

### 3.5 类型定义语义

```
⟨typedef τ T;, η, μ⟩ → η[T ↦ typealias(τ)], μ

后续 T 出现时展开为 τ
```

## 4. 类型的形式化定义

### 4.1 类型语法

```
τ ::= void                      (空类型)
  |   char                      (字符)
  |   short                     (短整)
  |   int                       (整数)
  |   long                      (长整)
  |   long long                 (长长整)
  |   float                     (单精度浮点)
  |   double                    (双精度浮点)
  |   long double               (扩展精度)
  |   _Bool                     (布尔)
  |   τ *                       (指针)
  |   τ [n]                     (数组)
  |   τ ()                      (函数)
  |   struct { τ₁ x₁; ... }     (结构体)
  |   union { τ₁ x₁; ... }      (联合体)
  |   enum { c₁, c₂, ... }      (枚举)
  |   const τ                   (const 限定)
  |   volatile τ                (volatile 限定)
  |   restrict τ                (restrict 限定, 指针)
```

### 4.2 类型构造语义

**指针类型**：

```
τ* : 指向 τ 类型对象的指针
值: 有效内存位置或 NULL
操作: * (解引用), & (取地址), 算术运算
```

**数组类型**：

```
τ[n] : n 个 τ 类型元素的数组
大小: sizeof(τ[n]) = n × sizeof(τ)
对齐: alignof(τ[n]) = alignof(τ)
索引: a[i] 等价于 *(a + i)
```

**函数类型**：

```
τ (τ₁, ..., τₙ) : 接受参数 (τ₁, ..., τₙ) 返回 τ 的函数
不能作为值类型（仅指针）
```

**结构体**：

```
struct { τ₁ x₁; τ₂ x₂; ... }

大小: ≥ Σ sizeof(τᵢ) （含填充）
对齐: max alignof(τᵢ)
成员偏移: offsetof(struct, xᵢ) = 前面成员大小之和（考虑填充）
```

### 4.3 类型等价

```
名等价 (C 规则):
  struct S { int x; } 和 struct T { int x; } 是不同的类型
  typedef 创建别名，与原类型等价

结构体自引用:
  struct Node { int data; struct Node *next; }  ✓
  struct Node { int data; struct Node next; }   ✗ (无限大小)
```

### 4.4 类型转换

```
隐式转换 (提升):
  char → int → long → long long
  float → double → long double
  整数提升: 小于 int 的类型提升为 int

显式转换:
  (τ)e : 将 e 的值转换为类型 τ

转换合法性:
  指针 ↔ 整数: 实现定义
  指针 ↔ 指针: 要求兼容类型或 void*
  浮点 ↔ 整数: 截断小数部分
```

## 5. 内存模型形式化

### 5.1 内存抽象

```
存储 S : Loc → Byte*
位置 l : (地址, 大小) 对
字节: 8 位值

有效指针条件:
  valid(l) ⇔ l 指向已分配且未释放的内存区域
```

### 5.2 对象表示

```
类型 τ 的对象在地址 a 的表示:

rep(τ, a) = S(a) ... S(a + sizeof(τ) - 1)

字节序:
  小端: 低字节在低地址
  大端: 高字节在低地址
```

### 5.3 有效类型 (Effective Type)

```
规则: 通过指针访问对象时，访问类型必须与对象的"有效类型"兼容

写入创建有效类型:
  int x;
  *(float*)&x = 1.0f;  // 违反严格别名规则，UB

严格别名例外:
  - char* 可以别名任何类型
  - 包含共同前缀的结构体
  - union 成员
```

### 5.4 对齐要求

```
alignof(τ) : 类型 τ 的对齐要求

有效地址:
  valid_align(a, τ) ⇔ a mod alignof(τ) = 0

对齐宏:
  _Alignas(n) : 指定对齐
  _Alignof(τ) : 查询对齐要求
```

### 5.5 内存生命周期

```
分配:
  - 定义时分配 (自动/静态)
  - malloc/calloc/realloc (动态)
  - 复合字面量

有效期间:
  - 自动: 块执行期间
  - 静态: 整个程序执行期
  - 动态: malloc 到 free 之间
  - 线程局部: 线程存在期间

释放:
  - 自动: 块退出
  - 动态: free
  - 使用后释放 (use-after-free): UB
```

## 6. 未定义行为分类

### 6.1 行为分类

```
明确定义 (Defined):
  标准明确规定的行为

实现定义 (Implementation-defined):
  由实现选择，必须文档化
  例如: char 的符号性、int 大小

未指定 (Unspecified):
  从允许的行为中选择，无需文档化
  例如: 函数参数求值顺序

未定义 (Undefined):
  无限制，可能导致任何结果
  例如: 空指针解引用、有符号溢出
```

### 6.2 未定义行为清单

**算术运算**：

```
- 有符号整数溢出: INT_MAX + 1
- 除零: x / 0
- 移位越界: 1 << 32 (32位 int)
- 移位负数: x << -1
```

**指针操作**：

```
- 空指针解引用: *NULL
- 野指针解引用: 使用已释放内存
- 越界访问: 数组边界外访问
- 无效对齐: *(int*)((char*)&x + 1)
- 指针比较: 不同对象指针比较（非 ==/!=）
```

**内存访问**：

```
- 访问未初始化变量
-  strict aliasing 违规
- 修改字符串字面量
- 重叠的 memcpy
```

**控制流**：

```
- 无 return 的非 void 函数
- 非自愿的无限递归（栈溢出）
- 非法跳转 (longjmp 到已返回的函数)
```

**其他**：

```
- 修改 const 对象
- 两次释放同一内存
- 标准库前置条件违规
```

### 6.3 未指定行为清单

```
- 子表达式求值顺序 (除 &&, ||, ?:, , 外)
- 函数参数求值顺序
- 位域分配顺序
- 整数除法的截断方向（向零还是向负无穷）
```

### 6.4 形式化语义中的 UB 表示

```
配置加入错误状态:
  ⟨e, η, μ⟩ ⇓ ⊥_UB   (未定义行为)
  ⟨e, η, μ⟩ ⇓ ⊥_impldef   (实现定义行为)

语义规则中检查:
  ⟨e₁ / e₂, η, μ⟩ ⇓ ⊥_divzero    若 e₂ ⇓ 0

  ⟨*e, η, μ⟩ ⇓ ⊥_nullptr         若 e ⇓ NULL

  ⟨e₁ + e₂, η, μ⟩ ⇓ ⊥_overflow   若有符号溢出
```

### 6.5 形式化验证考虑

```
良形程序条件:
  well_formed(P) ⇒ P 无未定义行为

类型安全子集:
  限制语言特性，确保:
  - 所有指针访问边界检查
  - 无未初始化读取
  - 整数运算检查溢出

示例: MISRA C, CERT C
```

## 附录：C 语义概览

| 特性 | 形式化挑战 | 处理方式 |
|------|-----------|----------|
| 未定义行为 | 非确定性 | 错误状态 ⊥ |
| 未指定行为 | 非确定性选择 | 集合语义 |
| 实现定义 | 参数化语义 | 配置参数 |
| 指针算术 | 复杂有效性 | 显式有效谓词 |
| 类型双关 | 别名分析 | 有效类型追踪 |
| 内存模型 | 低层细节 | 字节级抽象 |
| 序列点 | 求值顺序 | 偏序约束 |
