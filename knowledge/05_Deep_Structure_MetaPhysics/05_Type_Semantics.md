## 2. 类型作为命题 (Propositions as Types)

### 2.1 Curry-Howard对应基础

Curry-Howard同构建立了直觉主义逻辑与简单类型λ演算之间的对应：

| 逻辑 | 类型理论 | 编程 |
|-----|---------|-----|
| 命题 A | 类型 A | 数据规范 |
| 证明 p : A | 项 t : A | 程序/值 |
| A ⊃ B (蕴含) | A → B (函数类型) | 函数 |
| A ∧ B (合取) | A × B (积类型) | 对/记录 |
| A ∨ B (析取) | A + B (和类型) | 变体/枚举 |
| ∀x:A. B | 依赖函数 Πx:A. B | 泛型函数 |
| ∃x:A. B | 依赖对 Σx:A. B | 存在类型 |
| 矛盾 ⊥ | 空类型 ∅ | 不可能值 |
| 重言式 ⊤ | 单位类型 1 | 单位值 |

### 2.2 证明与程序的对应

**蕴含引入 (⊃I) ↔ λ抽象**:

```text
[x:A]                x:A ⊢ t:B
───────              ───────────
t:B                  λx.t : A→B
─────── (→I)
λx.t : A→B
```

**蕴含消去 (⊃E) ↔ 函数应用**:

```text
f:A→B    a:A         f:A→B    a:A
─────────────        ─────────────
f(a):B               f a : B
```

### 2.3 归一化与证明简化

类型化λ演算中，β归约对应于证明的切消（cut elimination）：

$$(λx. t)\ u →_β t[u/x]$$

这表明：可计算性理论中的求值过程对应于证明论中的证明简化。

## 3. Curry-Howard对应

### 3.1 经典Curry-Howard

对于简单类型λ演算（STLC）：

$$
\text{STLC} + \text{类型} \quad \cong \quad \text{直觉主义命题逻辑} + \text{自然演绎}
$$

### 3.2 扩展对应

**带积类型的λ演算** ↔ 合取逻辑：

```haskell
-- 合取引入 (×I)
pair   : A → B → A × B

-- 合取消去 (×E₁, ×E₂)
fst    : A × B → A
snd    : A × B → B
```

**带和类型的λ演算** ↔ 析取逻辑：

```haskell
-- 析取引入 (+I₁, +I₂)
inl    : A → A + B
inr    : B → A + B

-- 析取消去 (+E)
case   : A + B → (A → C) → (B → C) → C
```

### 3.3 构造性逻辑与计算内容

Curry-Howard对应表明：

1. **证明即程序**：每个证明都包含计算内容
2. **程序提取**：从存在性证明可提取见证
3. **验证即类型检查**：程序正确性通过类型检查验证

**示例**：从证明提取程序

```text
证明: ∀n∈ℕ. ∃m∈ℕ. m > n  →  程序: λn. n+1
```

## 4. System F (多态λ演算)

### 4.1 语法

System F扩展了简单类型λ演算，引入类型抽象和应用：

**项**:

```text
e ::= x | λx:T.e | e e           -- 简单λ演算
    | Λα.e | e [T]               -- 类型抽象和应用
```

**类型**:

```text
T ::= α | T → T | ∀α.T           -- 类型变量、函数、全称量词
```

### 4.2 类型规则

**类型抽象**:

$$\frac{Γ ⊢ e : T \quad α \notin \text{FV}(Γ)}{Γ ⊢ Λα. e : ∀α. T}$$

**类型应用**:

$$\frac{Γ ⊢ e : ∀α. T}{Γ ⊢ e [U] : T[U/α]}$$

### 4.3 编码数据类型

System F具有极强的表达能力，可在类型系统中编码常用数据类型。

**布尔值**:

```haskell
type Bool = ∀α. α → α → α

true  = Λα. λt:α. λf:α. t
false = Λα. λt:α. λf:α. f

if = Λα. λb:Bool. λt:α. λf:α. b [α] t f
```

**自然数（Church编码）**:

```haskell
type Nat = ∀α. (α → α) → α → α

0 = Λα. λs:α→α. λz:α. z
1 = Λα. λs:α→α. λz:α. s z
2 = Λα. λs:α→α. λz:α. s (s z)

succ = λn:Nat. Λα. λs:α→α. λz:α. s (n [α] s z)
add  = λm:Nat. λn:Nat. m [Nat] succ n
```

**积类型**:

```haskell
type Pair A B = ∀α. (A → B → α) → α

pair = ΛA. ΛB. λa:A. λb:B. Λα. λf:A→B→α. f a b
fst  = ΛA. ΛB. λp:Pair A B. p [A] (λa:A. λb:B. a)
snd  = ΛA. ΛB. λp:Pair A B. p [B] (λa:A. λb:B. b)
```

### 4.4 逻辑对应

System F对应于**二阶直觉主义逻辑**：

$$
\text{System F} \quad \cong \quad \text{二阶命题逻辑}
$$

全称量词 ∀ 对应于逻辑中的全称量词，提供了参数多态性。

### 4.5 正规化定理

**定理** (Girard, 1971): System F是强正规化的——所有良类型项都有有限的归约序列终止于范式。

这一结果保证了：

1. 程序终止性（对纯计算）
2. 证明一致性
3. 逻辑一致性

## 5. 子类型 (Subtyping)

### 5.1 子类型关系

子类型建立类型间的层次结构：若 A <: B，则任何需要 B 的地方都可用 A 代替。

**自反性**:

$$\overline{A <: A}$$

**传递性**:

$$\frac{A <: B \quad B <: C}{A <: C}$$

### 5.2 记录子类型

**宽度子类型**（添加字段）:

$$\overline{\{l_1:T_1, ..., l_n:T_n, l_{n+1}:T_{n+1}\} <: \{l_1:T_1, ..., l_n:T_n\}}$$

**深度子类型**（字段协变）:

$$\frac{∀i. S_i <: T_i}{\{l_i:S_i\}_{i \in I} <: \{l_i:T_i\}_{i \in I}}$$

### 5.3 函数子类型

函数类型在参数上逆变，在返回上协变：

$$\frac{T_1 <: S_1 \quad S_2 <: T_2}{S_1 → S_2 <: T_1 → T_2}$$

**记忆口诀**: "参数逆变，返回协变"

### 5.4 子类型的类型规则

**子类型引入**:

$$\frac{Γ ⊢ e : A \quad A <: B}{Γ ⊢ e : B}$$

### 5.5 顶类型与底类型

**顶类型** (⊤): 所有类型的超类型

$$\overline{A <: ⊤}$$

**底类型** (⊥): 所有类型的子类型（无值）

$$\overline{⊥ <: A}$$

## 6. 递归类型

### 6.1 递归类型的必要性

许多数据结构是递归定义的：

```text
List A = Unit + (A × List A)
Tree A = A + (Tree A × Tree A)
```

### 6.2 同构递归类型 (Iso-recursive)

通过显式的roll/unroll操作在同构类型间转换：

```haskell
data ListF a r = Nil | Cons a r
type List a = μr. ListF a r

roll   : ListF a (List a) → List a
unroll : List a → ListF a (List a)
```

### 6.3 等递归类型 (Equi-recursive)

类型与其展开自动等价：

$$μα. T = T[μα.T/α]$$

无需显式转换，但类型相等判定更复杂。

### 6.4 展开与折叠

**展开 (unfold)**:

$$\text{unfold} : μα.T → T[μα.T/α]$$

**折叠 (fold)**:

$$\text{fold} : T[μα.T/α] → μα.T$$

### 6.5 余递归类型与惰性数据结构

**余递归类型** (ν) 表示最大不动点，支持无限数据结构：

```haskell
type Stream a = νr. (a, r)

ones : Stream Nat
ones = ν(s:Stream Nat). (1, s)
```

### 6.6 递归类型与归纳原理

递归类型对应归纳定义，支持结构归纳：

```haskell
-- List的fold（foldr）
fold : ∀a b. (a → b → b) → b → List a → b
fold f z Nil         = z
fold f z (Cons x xs) = f x (fold f z xs)
```

## 7. 高级类型语义主题

### 7.1 依赖类型

依赖类型允许类型依赖于项，提供极强的表达能力：

```haskell
-- 向量：类型包含长度信息
data Vec (A:Type) : Nat → Type where
  Nil  : Vec A 0
  Cons : {n:Nat} → A → Vec A n → Vec A (n+1)

-- 安全的不匹配head/tail
head : {A:Type} → {n:Nat} → Vec A (n+1) → A
tail : {A:Type} → {n:Nat} → Vec A (n+1) → Vec A n
```

### 7.2 线性类型

线性类型追踪资源使用，确保精确的单次使用：

```haskell
-- !A 表示可复制的类型
-- A ⊸ B 表示线性函数（消耗A产生B）

alloc   : Int → Handle
read    : Handle ⊸ (Handle, Byte)  -- 消耗句柄，返回新句柄
close   : Handle ⊸ ()              -- 消耗句柄
```

### 7.3 效果系统

效果系统追踪计算效果：

```haskell
-- 类型标注效果
readFile : String → IO FileContent
pureFunc : Int → Pure Int

-- 多效果组合
compose : (a → e b) → (b → e c) → (a → e c)
```

## 8. 类型语义的范畴论解释

### 8.1 笛卡尔闭范畴 (CCC)

类型语义可在笛卡尔闭范畴中解释：

| 类型构造 | 范畴构造 |
|---------|---------|
| 积类型 A × B | 范畴积 |
| 函数类型 A → B | 指数对象 |
| 单位类型 1 | 终对象 |

### 8.2 范畴论语义

- 类型 → 对象
- 项 → 态射
- 类型构造子 → 泛构造

## 9. 总结

类型语义理论建立了类型系统、逻辑和代数结构之间的深刻联系：

| 理论 | 核心洞见 | 应用 |
|-----|---------|-----|
| Curry-Howard | 证明即程序 | 证明助手、程序提取 |
| System F | 参数多态的逻辑基础 | 泛型编程 |
| 子类型 | 类型层次与替换原理 | 面向对象语言 |
| 递归类型 | 归纳/余归纳数据 | 代数数据类型 |

这些理论为现代编程语言的设计和验证提供了坚实的数学基础。
