### 2.2 帧类型形式化定义

**定义 2.1 (CAN帧集合)**

```
CANFrame ::= DataFrame | RemoteFrame | ErrorFrame | OverloadFrame

DataFrame ::= SOF × ArbitrationField × ControlField ×
              DataField × CRCField × ACKField × EOF

RemoteFrame ::= SOF × ArbitrationField × ControlField ×
                (NoDataField) × CRCField × ACKField × EOF

ErrorFrame ::= ErrorFlag × ErrorDelimiter

OverloadFrame ::= OverloadFlag × OverloadDelimiter
```

### 2.3 帧类型决策树

```
CAN 帧类型选择决策
│
├── 需要发送数据?
│   ├── YES → 数据帧 (Data Frame)
│   │           ├── 标准ID? → CAN 2.0A 数据帧
│   │           └── 扩展ID? → CAN 2.0B 数据帧
│   │
│   └── NO → 请求数据?
│       ├── YES → 远程帧 (Remote Frame)
│       │           └── 用于请求特定ID的数据
│       │
│       └── 错误处理?
│           ├── YES → 错误帧 (Error Frame)
│           │           ├── 主动错误标志 (6个显性位)
│           │           └── 被动错误标志 (6个隐性位)
│           │
│           └── 延迟发送?
│               └── 过载帧 (Overload Frame)
│                   └── 用于插入延迟
```

## 4. 非破坏性仲裁机制形式化分析

### 4.1 仲裁原理

```
┌─────────────────────────────────────────────────────────────────┐
│              CAN 非破坏性仲裁 (Nondestructive Arbitration)       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  核心原理: 线与 (Wired-AND) 逻辑                                 │
│                                                                 │
│  总线逻辑:                                                       │
│    显性位 (Dominant) = 0  >  隐性位 (Recessive) = 1             │
│    0 & 1 = 0  (显性覆盖隐性)                                    │
│                                                                 │
│  仲裁过程示例:                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  时序    Node A (ID=0x100)    Node B (ID=0x200)   总线   │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │  t0      1 (ID10)            1 (ID10)            1      │   │
│  │  t1      0 (ID9)             0 (ID9)             0      │   │
│  │  t2      0 (ID8)             0 (ID8)             0      │   │
│  │  t3      0 (ID7)             0 (ID7)             0      │   │
│  │  t4      0 (ID6)             0 (ID6)             0      │   │
│  │  t5      0 (ID5)             0 (ID5)             0      │   │
│  │  t6      0 (ID4)             0 (ID4)             0      │   │
│  │  t7      0 (ID3)             0 (ID3)             0      │   │
│  │  t8      0 (ID2)             0 (ID2)             0      │   │
│  │  ─────────────────────────────────────────────────────  │   │
│  │  t9      0 (ID1)  ──────┐    1 (ID1)  ──────┐    0      │   │
│  │                         │                   │           │   │
│  │                    A检测到总线为0            │           │   │
│  │                    与自己发送一致            │           │   │
│  │                    继续发送                  │           │   │
│  │                                             │           │   │
│  │                                         B检测到总线为0  │   │
│  │                                         与自己发送不同  │   │
│  │                                         → B退避         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  结果: Node A (ID=0x100) 赢得仲裁，继续发送完整帧               │
│        Node B (ID=0x200) 在t9退避，总线空闲后重试               │
│                                                                 │
│  关键特性:                                                      │
│  1. 仲裁期间不会丢失数据                                        │
│  2. 高优先级(ID值小)的消息自动获得总线                          │
│  3. 低优先级消息自动退避，无数据损坏                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 优先级排序定理

**定理 4.1 (CAN优先级排序)**

```
定理: CAN消息的优先级由其标识符(ID)唯一确定

给定: 两个消息 M1 和 M2，ID分别为 ID1 和 ID2

则:   ID1 < ID2  ⟹  Priority(M1) > Priority(M2)

证明:
  1. 仲裁从最高位(ID10或ID28)开始
  2. 比较对应位:
     - 若 ID1[i] = 0, ID2[i] = 1，则总线为0
     - M1检测到自己发送0，总线为0，一致，继续
     - M2检测到自己发送1，总线为0，不一致，退避
  3. 因此，在第一个不同的位上，发送0的节点获胜
  4. 数值小的ID有更大的概率在高位为0
  5. 结论: ID越小，优先级越高

证毕。
```

### 4.3 优先级设计决策树

```
CAN ID分配策略
│
├── 周期性消息
│   ├── 高速控制 (1-10ms)
│   │   └── 分配低ID (0x001-0x010) - 最高优先级
│   │
│   ├── 中速控制 (10-100ms)
│   │   └── 分配中低ID (0x011-0x050)
│   │
│   └── 低速状态 (100-1000ms)
│       └── 分配中高ID (0x051-0x100)
│
├── 事件驱动消息
│   ├── 紧急故障
│   │   └── 分配最低ID (0x000) - 最高优先级
│   │
│   ├── 重要告警
│   │   └── 分配低ID (0x001-0x010)
│   │
│   └── 一般事件
│       └── 分配中ID (0x100-0x200)
│
└── 诊断/配置消息
    └── 分配高ID (0x300-0x7FF) - 低优先级
```

## 6. CAN与C语言实现映射

### 6.1 数据结构形式化定义

```c
/* CAN帧结构体 - 统一表示标准帧和扩展帧 */
typedef struct {
    /* 标识符 */
    uint32_t id;                /* 11位标准ID 或 29位扩展ID */
    uint8_t  is_extended;       /* 0=标准帧, 1=扩展帧 */
    uint8_t  is_remote;         /* 0=数据帧, 1=远程帧 */

    /* 数据场 */
    uint8_t  dlc;               /* 数据长度码 (0-8, CAN FD: 0-15) */
    uint8_t  data[64];          /* 数据 (CAN FD最大64字节) */

    /* CAN FD特有 */
    uint8_t  is_fd;             /* 0=CAN 2.0, 1=CAN FD */
    uint8_t  brs;               /* Bit Rate Switch (数据段加速) */
    uint8_t  esi;               /* Error State Indicator */
} CAN_Frame;

/* 滤波器配置 - 硬件滤波 */
typedef struct {
    uint32_t filter_id;         /* 滤波器ID */
    uint32_t filter_mask;       /* 掩码 (0=忽略, 1=必须匹配) */
    uint8_t  filter_mode;       /* 0=掩码模式, 1=列表模式 */
    uint8_t  fifo_assignment;   /* 0=FIFO0, 1=FIFO1 */
} CAN_Filter;

/* 错误计数器 */
typedef struct {
    uint8_t  tec;               /* Transmit Error Counter (0-255) */
    uint8_t  rec;               /* Receive Error Counter (0-255) */
    uint8_t  error_state;       /* 0=Active, 1=Passive, 2=BusOff */
    uint32_t error_flags;       /* 错误标志寄存器 */
} CAN_ErrorCounters;
```

### 6.2 位定时计算递归算法

```c
/* 位定时参数计算
 *
 * 目标: 给定CAN时钟频率和目标波特率，计算最佳位定时参数
 *
 * 约束条件:
 *   - 位时间 = 同步段(1) + 传播段 + 相位段1 + 相位段2
 *   - 位时间范围: 8-25 Tq (Time quanta)
 *   - 采样点推荐: 75%-87.5%
 *   - 传播段 ≥ 2 Tq
 *   - 相位段2 ≥ SJW (同步跳转宽度)
 */

typedef struct {
    uint16_t prescaler;     /* 预分频器 (1-1024) */
    uint8_t  sync_seg;      /* 固定1 */
    uint8_t  prop_seg;      /* 传播段 (1-8) */
    uint8_t  phase_seg1;    /* 相位段1 (1-8) */
    uint8_t  phase_seg2;    /* 相位段2 (1-8) */
    uint8_t  sjw;           /* 同步跳转宽度 (1-4) */
    float    sample_point;  /* 采样点位置 (%) */
} CAN_BitTiming;

/* 计算最佳位定时参数 */
int calculate_bit_timing(
    uint32_t can_clock,      /* CAN时钟频率 (Hz) */
    uint32_t target_bitrate, /* 目标波特率 (bps) */
    float    target_sample,  /* 目标采样点 (0.75-0.875) */
    CAN_BitTiming *result
) {
    uint32_t best_error = UINT32_MAX;

    /* 遍历所有可能的预分频器 */
    for (uint16_t prescaler = 1; prescaler <= 1024; prescaler++) {
        /* 计算每位的Tq数 */
        uint32_t tq_per_bit = can_clock / (prescaler * target_bitrate);

        /* 检查是否在有效范围 (8-25 Tq) */
        if (tq_per_bit < 8 || tq_per_bit > 25) {
            continue;
        }

        /* 分配时间段:
         * tq_per_bit = 1(sync) + prop + phase1 + phase2
         * 剩余: prop + phase1 + phase2 = tq_per_bit - 1
         */
        uint8_t remaining = tq_per_bit - 1;

        /* 推荐: prop_seg = 2-3, 剩余分配给phase1和phase2 */
        for (uint8_t prop = 2; prop <= 3 && prop <= remaining - 2; prop++) {
            uint8_t phase_total = remaining - prop;

            /* phase1 应该大于 phase2，且phase2 ≥ 2 */
            for (uint8_t phase2 = 2; phase2 <= phase_total / 2; phase2++) {
                uint8_t phase1 = phase_total - phase2;

                /* 检查范围 */
                if (phase1 < 1 || phase1 > 8 || phase2 > 8) {
                    continue;
                }

                /* 计算采样点 */
                float sample_point = (1.0f + prop + phase1) / tq_per_bit;

                /* 计算与目标采样点的误差 */
                float error = fabs(sample_point - target_sample);

                /* 选择最佳参数 */
                if (error * 1000 < best_error) {
                    best_error = error * 1000;
                    result->prescaler = prescaler;
                    result->prop_seg = prop;
                    result->phase_seg1 = phase1;
                    result->phase_seg2 = phase2;
                    result->sjw = (phase2 > 4) ? 4 : phase2;
                    result->sample_point = sample_point;
                }
            }
        }
    }

    return (best_error == UINT32_MAX) ? -1 : 0;
}
```
