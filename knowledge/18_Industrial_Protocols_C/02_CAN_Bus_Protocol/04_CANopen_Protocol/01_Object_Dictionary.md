> **设备配置**: 电子数据表 (EDS) / 设备配置文件 (DCF)

### 1.2 COB-ID分配矩阵

**定义 1.1 (COB-ID通信对象标识符)**

```
COB-ID = FunctionCode × 2^7 + NodeID

其中:
  FunctionCode: 4-bit 功能码
  NodeID: 7-bit 节点地址 [1-127]

标准CANopen COB-ID分配 (11-bit标准帧):

| 通信对象   | 功能码 | COB-ID范围      | 说明                  |
|------------|--------|-----------------|----------------------|
| NMT        | 0x0    | 0x000           | 网络管理 (广播)       |
| SYNC       | 0x1    | 0x080           | 同步对象              |
| TIME       | 0x2    | 0x100           | 时间戳对象            |
| EMCY       | 0x1    | 0x081-0x0FF     | 紧急报文 (节点1-127)  |
| PDO1 (Tx)  | 0x3    | 0x181-0x1FF     | 发送PDO1              |
| PDO1 (Rx)  | 0x2    | 0x201-0x27F     | 接收PDO1              |
| PDO2 (Tx)  | 0x5    | 0x281-0x2FF     | 发送PDO2              |
| PDO2 (Rx)  | 0x4    | 0x301-0x37F     | 接收PDO2              |
| PDO3 (Tx)  | 0x7    | 0x381-0x3FF     | 发送PDO3              |
| PDO3 (Rx)  | 0x6    | 0x401-0x47F     | 接收PDO3              |
| PDO4 (Tx)  | 0x9    | 0x481-0x4FF     | 发送PDO4              |
| PDO4 (Rx)  | 0x8    | 0x501-0x57F     | 接收PDO4              |
| SDO (Tx)   | 0xB    | 0x581-0x5FF     | SDO响应               |
| SDO (Rx)   | 0xA    | 0x601-0x67F     | SDO请求               |
| NMT Error  | 0xE    | 0x701-0x77F     | 节点守护/心跳         |
```

## 2. 对象字典 (OD) 形式化定义

### 2.1 对象字典结构

**定义 2.1 (对象字典)**

```
对象字典 = { Index × SubIndex → Object }

其中:
  Index: 16-bit 主索引
  SubIndex: 8-bit 子索引 (0为对象计数)
  Object: (DataType, AccessType, DefaultValue, Value)

对象字典区域划分:

┌─────────────────────────────────────────────────────────────────────────────┐
│                     对象字典索引区域                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  0x0000-0x0FFF: 数据类型定义                                                 │
│    ├── 0x0001-0x001F: 标准数据类型 (BOOLEAN, INTEGER8, etc.)                │
│    ├── 0x0021-0x003F: 复杂数据类型 (结构体定义)                              │
│    └── 0x0041-0x005F: 制造商特定数据类型                                     │
│                                                                             │
│  0x1000-0x1FFF: 通信对象参数                                                 │
│    ├── 0x1000-0x1029: 设备信息 (设备类型、错误寄存器等)                      │
│    ├── 0x1200-0x12FF: SDO参数 (COB-ID、超时等)                               │
│    ├── 0x1400-0x15FF: RPDO通信参数                                           │
│    ├── 0x1600-0x17FF: RPDO映射参数                                           │
│    ├── 0x1800-0x19FF: TPDO通信参数                                           │
│    ├── 0x1A00-0x1BFF: TPDO映射参数                                           │
│    └── 0x1F00-0x1FFF: 其他通信参数                                           │
│                                                                             │
│  0x2000-0x5FFF: 制造商特定区域                                               │
│    └── 设备制造商定义的对象                                                  │
│                                                                             │
│  0x6000-0x9FFF: 标准化设备区域 (CiA规范)                                     │
│    ├── 0x6000-0x6FFF: 标准化服务器 (CiA 401 I/O)                            │
│    ├── 0x7000-0x7FFF: 标准化客户端                                           │
│    └── 0x8000-0x9FFF: 预留                                                   │
│                                                                             │
│  0xA000-0xFFFF: 保留/扩展                                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

| 属性 | 值 | 说明 |
|------|-----|------|
| **Object Type** | VAR | 单一值变量 |

## 3. SDO (服务数据对象) 通信

### 3.1 SDO协议形式化定义

**定义 3.1 (SDO通信)**

```
SDO = 基于客户端-服务器模型的对象字典访问服务

SDO传输模式:
  1. 加速传输 (Expedited): 数据 ≤ 4字节
  2. 分段传输 (Segmented): 数据 > 4字节，逐段传输
  3. 块传输 (Block): 批量数据传输优化

SDO请求帧结构 (客户端→服务器):
  COB-ID: 0x600 + NodeID

  Byte 0: ccs (1) | n (2) | e (1) | s (1) | 命令 (3)
    ccs = 1 (客户端命令码)
    n = 0 (加速传输时有效字节数)
    e = 1 (加速传输) / 0 (分段传输)
    s = 1 (指示数据大小)
    命令: 1=下载请求, 2=上传请求

  Byte 1-2: Index (小端)
  Byte 3: SubIndex
  Byte 4-7: Data (加速传输) / 数据长度 (分段传输)

SDO响应帧结构 (服务器→客户端):
  COB-ID: 0x580 + NodeID

  Byte 0: scs (1) | n (2) | e (1) | s (1) | 保留 (3)
    scs = 0 (服务器命令码)
    其他字段与请求对应

  Byte 1-2: Index
  Byte 3: SubIndex
  Byte 4-7: Data / 错误码
```

### 3.2 SDO传输状态机

```
SDO加速上传状态机

状态定义:
  S0: IDLE
  S1: INITIATE_UPLOAD_REQUEST_SENT
  S2: UPLOAD_RESPONSE_RECEIVED
  S3: ERROR

状态转换:
  S0 ──[准备请求]──→ S1
                     │
                     │ 发送SDO请求
                     │ COB-ID = 0x600 + node_id
                     │ Data = 0x40 | Index | SubIndex
                     │
                     ↓
                    S1 ──[超时]──→ S3 (错误: 超时)
                     │
                     │ 等待响应
                     │ COB-ID = 0x580 + node_id
                     │
                     ↓ 响应有效
                    S2 ──[解析数据]──→ S0 (完成)
                     │
                     ↓ 响应错误
                    S3 ──[错误处理]──→ S0 (终止)

SDO错误码定义:
  0x05030000: 切换位未改变
  0x05040000: SDO协议超时
  0x05040001: 非法或未知的客户端/服务器命令
  0x05040005: 内存溢出
  0x06010000: 不支持对象访问
  0x06010001: 尝试读只写对象
  0x06010002: 尝试写只读对象
  0x06020000: 对象不存在
  0x06090011: 子索引不存在
  0x06090030: 值范围参数过高
  0x06090031: 值范围参数过低
  0x06090036: 最大值小于最小值
```

### 3.3 SDO分段传输

```
SDO分段上传协议流程

客户端                    服务器
   │                        │
   │  初始化请求            │
   │  ccs=2, e=0, s=1      │
   │ ─────────────────────→ │
   │                        │
   │                        │  准备数据块
   │                        │
   │  初始化响应            │
   │  scs=0, e=0, s=1      │
   │ ←───────────────────── │
   │                        │
   │  分段请求 (块0)        │
   │  ccs=3, t=0, n=0      │
   │ ─────────────────────→ │
   │                        │
   │  分段响应 (块0)        │
   │  scs=0, t=0, c=0,     │
   │  数据[0..6]           │
   │ ←───────────────────── │
   │                        │
   │  分段请求 (块1)        │
   │  ccs=3, t=1, n=0      │
   │ ─────────────────────→ │
   │                        │
   │  分段响应 (块1)        │
   │  scs=0, t=1, c=1,     │
   │  数据[7..10] (最后)   │
   │ ←───────────────────── │
   │                        │
   │  完成                  │

切换位 (toggle bit): 交替0/1确保顺序
结束位 (c): 1表示最后一段
```

## 4. PDO (过程数据对象) 通信

### 4.1 PDO与SDO对比矩阵

| 特性 | SDO | PDO |
|------|-----|-----|
| **用途** | 参数配置 | 实时数据传输 |
| **数据量** | 任意 (分段支持) | 固定 0-8字节 |
| **通信模型** | 客户端-服务器 (请求-响应) | 生产者-消费者 (发布-订阅) |
| **实时性** | 低 (有确认) | 高 (无确认) |
| **优先级** | 低 | 高 |
| **传输类型** | 按需 | 异步/同步/循环 |
| **COB-ID** | 0x600+node/0x580+node | 0x180+node 等 |

### 4.2 PDO映射机制

**定义 4.1 (PDO映射)**

```
PDO映射 = 将对象字典条目映射到PDO数据字节

RPDO映射配置 (0x1600-0x17FF):
  子索引0: 映射条目数
  子索引1-n: 映射条目 (32-bit编码)

映射条目格式:
  Bit 31-16: Index
  Bit 15-8:  SubIndex
  Bit 7-0:   位长度

示例 (RPDO1映射):
  0x1600.00 = 2              (2个条目)
  0x1600.01 = 0x60400110    (索引0x6040, 子索引0x01, 16位)
  0x1600.02 = 0x60410108    (索引0x6041, 子索引0x01, 8位)

总数据长度 = 16 + 8 = 24位 = 3字节

RPDO1数据帧结构:
  Byte 0-1: 控制字 (0x6040.01)
  Byte 2:   模式选择 (0x6041.01)

PDO映射约束:
  - 总位数 ≤ 64 (8字节)
  - 映射对象必须存在且可写 (RPDO)
  - 映射变更需进入预操作状态
```

### 4.3 PDO传输类型矩阵

| 类型 | 值 | 触发条件 | 应用场景 |
|------|-----|----------|----------|
| **Synchronous** | 0 | SYNC接收后，周期性 | 同步采集 |
| **Synchronous** | 1-240 | 每N个SYNC触发 | 降低带宽 |
| **Synchronous** | 254 | 制造商特定 | 自定义 |
| **Synchronous** | 255 | 异步，事件驱动 | 事件报告 |
| **Asynchronous** | - | 值变化/时间触发 | 状态更新 |
| **RTR only** | - | 仅远程帧触发 | 按需查询 |

## 5. NMT (网络管理) 状态机

### 5.1 NMT状态转换

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       NMT节点状态机                                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                        ┌─────────────────┐                                  │
│                        │   初始化状态     │                                  │
│                        │   (Initializing)│                                  │
│                        └────────┬────────┘                                  │
│                                 │ 自动转换                                   │
│                                 ↓                                           │
│                        ┌─────────────────┐                                  │
│                        │  预操作状态      │←─────────────────────────┐      │
│                        │  (Pre-operational)│                        │      │
│                        └────────┬────────┘                        │      │
│                                 │ Start_Remote_Node               │      │
│                                 │ (NMT命令 0x01)                  │      │
│                                 ↓                                  │      │
│  ┌───────────────┐     ┌─────────────────┐     ┌───────────────┐   │      │
│  │    停止状态    │←────│    操作状态      │────→│   复位应用    │   │      │
│  │  (Stopped)    │     │  (Operational)  │     │ Reset Comm    │   │      │
│  └───────────────┘     └─────────────────┘     └───────────────┘   │      │
│          ↑                      │                        │         │      │
│          │                      │                        │         │      │
│          └──────────────────────┴────────────────────────┘         │      │
│                   Stop_Remote_Node (0x02)                          │      │
│                   Enter_Pre-Operational (0x80)                     │      │
│                   Reset_Node (0x81)                                │      │
│                   Reset_Communication (0x82)                       │      │
│                                                                             │
│  状态通信能力:                                                              │
│  ────────────                                                               │
│  初始化:        无通信                                                      │
│  预操作:        SDO + NMT + 心跳                                            │
│  操作:          所有通信对象 (SDO + PDO + NMT + 心跳)                       │
│  停止:          仅NMT + 心跳                                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 NMT命令帧

```
NMT命令格式 (COB-ID = 0x000):
  Byte 0: 命令码 (CS)
    0x01 = Start_Remote_Node
    0x02 = Stop_Remote_Node
    0x80 = Enter_Pre-Operational
    0x81 = Reset_Node
    0x82 = Reset_Communication

  Byte 1: 节点ID
    0 = 广播 (所有节点)
    1-127 = 特定节点

示例:
  启动所有节点:  COB-ID=0x000, Data=0x01 0x00
  停止节点5:    COB-ID=0x000, Data=0x02 0x05
  复位节点10:   COB-ID=0x000, Data=0x81 0x0A
```
