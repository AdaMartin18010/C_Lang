# J1939 协议基础与PGN深度解析

> **应用领域**: 商用车、重型机械、船舶、农业设备
> **物理层**: CAN 2.0B (29-bit扩展标识符)
> **数据链路**: SAE J1939/21, 网络层: SAE J1939/31
> **最后更新**: 2026-03-29

## 2. PDU (协议数据单元) 格式

### 2.1 29-bit标识符形式化定义

**定义 2.1 (J1939 PDU标识符)**

```
PDU格式 (CAN扩展帧标识符):

Bit 28-26: Priority (3-bit)
  值越小优先级越高
  范围: 0-7
  默认: 6 (大部分消息)
  控制: 3 (高优先级控制)

Bit 25: Extended Data Page (EDP) / Reserved
  必须为0 (保留)

Bit 24: Data Page (DP)
  0 = 第0页 (标准PGN)
  1 = 第1页 (扩展PGN)

Bit 23-16: PDU Format (PF) (8-bit)
  < 240: PDU1格式 (目标地址特定)
  ≥ 240: PDU2格式 (全局广播)

Bit 15-8: PDU Specific (PS) (8-bit)
  PDU1格式: 目标地址 (Destination Address)
  PDU2格式: 组扩展 (Group Extension)

Bit 7-0: Source Address (SA) (8-bit)
  发送节点地址
  0-253: 有效地址
  254: 空地址 (广播声明)
  255: 全局广播

标识符组成公式:
  CAN ID = (Priority << 26) | (EDP << 25) | (DP << 24) |
           (PF << 16) | (PS << 8) | SA

PGN组成公式:
  PGN = (DP << 16) | (PF << 8) | (PS if PF >= 240 else 0)
```

### 2.2 PDU1 vs PDU2格式对比

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       PDU1与PDU2格式对比                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  PDU1格式 (PF < 240, 点对点通信)                                            │
│  ───────────────────────────────                                            │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  Bit 28-26 │ Bit 25 │ Bit 24 │ Bit 23-16 │ Bit 15-8  │ Bit 7-0   │   │
│  │   Priority │  Rsvd  │   DP   │    PF     │    DA     │    SA     │   │
│  │    (3bit)  │ (1bit) │ (1bit) │  (8bit)   │  (8bit)   │  (8bit)   │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │                        │         │ 目标地址 │ 源地址  │               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  PGN = (DP << 16) | (PF << 8)  (PS=0 for PGN计算)                           │
│                                                                             │
│  示例: 请求地址声明 (PF=238 = 0xEE)                                          │
│    PGN = 0x00EE00 = 60928                                                   │
│    CAN ID = 0x18EEFFxx (广播请求)                                           │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  PDU2格式 (PF ≥ 240, 广播通信)                                              │
│  ─────────────────────────────                                              │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  Bit 28-26 │ Bit 25 │ Bit 24 │ Bit 23-16 │ Bit 15-8  │ Bit 7-0   │   │
│  │   Priority │  Rsvd  │   DP    │    PF     │    GE     │    SA     │   │
│  │    (3bit)  │ (1bit) │ (1bit) │  (8bit)   │  (8bit)   │  (8bit)   │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │                        │         │ 组扩展  │ 源地址  │               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  PGN = (DP << 16) | (PF << 8) | GE                                          │
│                                                                             │
│  示例: 车辆位置 (PGN = 65267 = 0x00FEF3)                                     │
│    PF = 0xFE = 254                                                          │
│    GE = 0xF3 = 243                                                          │
│    CAN ID = 0x0CFEF3xx                                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 PGN分类矩阵

| PGN范围 | 说明 | 示例 |
|---------|------|------|
| 0x0000-0x00EF | PDU1格式 (点对点) | 0x00E0 (请求), 0x00EE (地址声明) |
| 0x00F0-0x00FE | PDU1格式专用 | 0x00F0 (传输协议-连接管理) |
| 0x0100-0x01FF | PDU1格式 | 命令类消息 |
| 0x0200-0xEFFF | PDU2格式 | 广播消息 (大多数PGN) |
| 0xF000-0xFEFF | PDU2格式 | 专用广播 |
| 0xFF00-0xFFFF | 保留 | - |

### 3.2 PGN数据字段定义

**示例: PGN 61444 (EEC1 - 发动机控制 #1)**

```
PGN: 0x00F004 = 61444
SPN: 190 - 引擎转速

数据字段布局:
┌───────────────────────────────────────────────────────────────┐
│ Byte 0 │ 引擎扭矩模式 (4bit) │ 实际引擎扭矩百分比 (1%)          │
│        │ 0=无需求           │ 范围: -125% 到 +125%            │
│        │ 1=驾驶员需求        │ 偏移: -125                       │
│        │ 2=巡航控制         │                                  │
├────────┴─────────────────────┴─────────────────────────────────┤
│ Byte 1 │ 驾驶员需求引擎扭矩百分比 (1%)                          │
│        │ 范围: -125% 到 +125%, 偏移: -125                      │
├────────┴───────────────────────────────────────────────────────┤
│ Byte 2 │ 实际引擎扭矩百分比 (1%)                                │
│        │ 范围: -125% 到 +125%, 偏移: -125                      │
├────────┴───────────────────────────────────────────────────────┤
│ Byte 3-4 │ 引擎转速 (SPN 190)                                  │
│          │ 分辨率: 0.125 rpm/bit                               │
│          │ 偏移: 0                                             │
│          │ 范围: 0 到 8031.875 rpm                             │
├──────────┴─────────────────────────────────────────────────────┤
│ Byte 5 │ 引擎启动控制模式 (4bit) │ 引擎转矩限制特性 (4bit)       │
├────────┴─────────────────────────┴───────────────────────────────┤
│ Byte 6 │ 不适用                                                │
├────────┴───────────────────────────────────────────────────────┤
│ Byte 7 │ 不适用                                                │
└────────────────────────────────────────────────────────────────┘

引擎转速解码公式:
  RPM = (Byte3 + Byte4 × 256) × 0.125

示例:
  Byte3 = 0x00, Byte4 = 0x64 (100 in decimal)
  RPM = (0 + 100 × 256) × 0.125 = 3200 rpm
```

## 4. 网络管理

### 4.1 地址声明流程

```
J1939地址声明状态机

状态定义:
  S0: NO_ADDRESS - 无地址
  S1: CLAIMING - 声明中
  S2: CLAIMED - 已声明
  S3: CANNOT_CLAIM - 声明冲突

状态转换:
  S0 ──[通电/复位]──→ S1
                     │
                     │ 发送地址声明 (Address Claim)
                     │ PGN=60928, 源地址=目标地址
                     │ 数据=NAME (8字节)
                     │
                     ↓
                    S1 ──[收到冲突声明]──→ S1 (比较NAME)
                     │   (更高优先级NAME?)
                     │   ├── 是 → S1 (重试新地址)
                     │   └── 否 → S3 (放弃此地址)
                     │
                     ↓ 250ms无冲突
                    S2 ──[正常运行]──→ S2
                     │
                     │ 收到其他节点对本报文的请求
                     ↓
                    S1 (重新声明)

NAME (64-bit设备标识)优先级规则:
  1. 仲裁地址能力 (1bit): 可仲裁=0 > 不可仲裁=1
  2. 行业组 (3bit): 数值小优先级高
  3. 车辆系统实例 (4bit): 数值小优先级高
  4. 车辆系统 (7bit): 数值小优先级高
  5. 功能 (8bit): 数值小优先级高
  6. 功能实例 (5bit): 数值小优先级高
  7. ECU实例 (3bit): 数值小优先级高
  8. 制造商代码 (11bit): 数值小优先级高
  9. 序列号 (21bit): 数值小优先级高

地址范围:
  0-127:   行业标准地址分配
  128-247: 特定协议定义
  248-253: 专有地址
  254:     空地址 (声明过程中)
  255:     全局广播
```

## 5. 传输协议 (TP)

### 5.1 BAM (广播通知消息)

**定义 5.1 (BAM协议)**

```
BAM: 用于多包广播传输

连接管理帧 (CM, PGN=60416):
  Byte 0: 控制字节 = 32 (0x20)
  Byte 1-2: 消息总大小 (字节)
  Byte 3: 总包数 (1-255)
  Byte 4: 保留 (0xFF)
  Byte 5-7: PGN (小端)

数据传输帧 (DT, PGN=60160):
  Byte 0: 序列号 (1-255)
  Byte 1-7: 数据 (7字节/包)

示例 (发送100字节数据):
  步骤1: 发送CM
    CM: [0x20, 0x64, 0x00, 0x0F, 0xFF, PGN_L, PGN_M, PGN_H]
        (100字节, 15包, PGN=...)

  步骤2: 发送15个DT帧
    DT1: [0x01, data0..data6]
    DT2: [0x02, data7..data13]
    ...
    DT15: [0x0F, data98..data99, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF]

时序约束:
  - 包间隔: 10-200ms (默认50ms)
  - 总传输时间 ≤ 750ms (15包×50ms)
```

### 5.2 RTS/CTS (请求发送/清除发送)

```
RTS/CTS连接建立流程 (点对点可靠传输)

发送方                          接收方
   │                              │
   │ 1. RTS (请求发送)           │
   │    CM: 16 (0x10)            │
   │    [16, 大小L, 大小H, 包数, │
   │     最大包数, 下一包号,     │
   │     PGN...]                │
   │ ──────────────────────────→ │
   │                              │
   │ 2. CTS (清除发送)           │
   │    CM: 17 (0x11)            │
   │    [17, 可接收包数,         │
   │     下一包号, 0xFF,         │
   │     0xFF, 0xFF, PGN...]    │
   │ ←────────────────────────── │
   │                              │
   │ 3. DT (数据传输)            │
   │    [序列号, 7字节数据]      │
   │ ──────────────────────────→ │
   │    ... (重复直到完成)        │
   │                              │
   │ 4. End of Message ACK       │
   │    CM: 19 (0x13)            │
   │    [19, 0xFF×6, PGN...]     │
   │ ←────────────────────────── │
   │                              │
   │ 5. 连接关闭                 │

超时处理:
  T1: CTS等待超时 = 1250ms
  T2: 发送方等待CTS超时 = 1250ms
  T3: 发送方等待ACK超时 = 1250ms
  T4: 接收方等待数据超时 = 1050ms
```
