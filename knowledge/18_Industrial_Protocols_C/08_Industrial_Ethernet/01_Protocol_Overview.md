# 工业以太网协议深度概述

> **核心特性**: 实时性、确定性、高可靠性
> **主流协议**: EtherCAT、PROFINET、EtherNet/IP、Modbus TCP
> **技术路线**: 标准以太网改良 vs 专用实时通道
> **最后更新**: 2026-03-29

## 2. EtherCAT 深度解析

### 2.1 EtherCAT数据帧处理原理

**定义 2.1 (EtherCAT帧结构)**

```
EtherCAT帧 = Ethernet Header + EtherCAT Header + 多个EtherCAT PDU + FCS

Ethernet Header (14 bytes):
  - Destination MAC: 主站地址 或 广播
  - Source MAC: 从站返回地址
  - EtherType: 0x88A4 (EtherCAT)

EtherCAT Header (2 bytes):
  - Length: 后续数据长度
  - Reserved: 0
  - Type: 1 (EtherCAT命令)

EtherCAT PDU结构:
  ┌─────────┬─────────┬─────────┬─────────┬─────────┐
  │ CMD     │ IDX     │ ADP     │ ADO     │ LEN     │
  │ (1B)    │ (1B)    │ (2B)    │ (2B)    │ (2B)    │
  ├─────────┴─────────┴─────────┴─────────┴─────────┤
  │ DATA (0-1486 bytes)                              │
  ├─────────────────────────────────────────────────┤
  │ IRQ (2B)                                         │
  └─────────────────────────────────────────────────┘

命令类型 (CMD):
  0x01: APRD (Auto Increment Physical Read)
  0x02: APWR (Auto Increment Physical Write)
  0x04: FPRD (Configured Address Physical Read)
  0x05: FPWR (Configured Address Physical Write)
  0x0D: LRD (Logical Read)
  0x0E: LWR (Logical Write)
  0x0F: LRW (Logical Read Write)

处理-on-the-fly原理:
  帧传播时间 = 传输延迟 + 处理延迟

  传统以太网:
    从站接收完整帧 → 处理 → 转发
    延迟 = 帧大小/速率 × 跳数

  EtherCAT:
    帧流经从站时边接收边处理
    延迟 ≈ 几十纳秒 (硬件处理)

  100站点理论周期:
    帧大小: 100 × 8bytes = 800bytes + overhead ≈ 1KB
    传输时间 @100Mbps: 1KB × 8 / 100M = 80μs
    处理延迟: 100 × 50ns = 5μs
    总周期: <100μs
```

```

├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  状态定义:                                                                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │  Init    │  │Pre-Op    │  │ Safe-Op  │  │   Op     │  │  Boot    │      │
│  │  初始化  │  │预操作    │  │安全操作  │  │ 操作     │  │ 引导     │      │

│                                                                             │
│  状态转换:                                                                   │
│                                                                             │
│   Init ──[配置AL控制寄存器]──→ Pre-Op ──[配置PDO映射]──→ Safe-Op            │
│     │                            │                         │               │
│     │                            │ 可读取输入              │ 可写入输出    │
│     │                            ▼                         ▼               │
│     │                      [SDO通信可用]              [PDO通信启动]         │
│     │                                                                  │    │
│     │                           Safe-Op ──[启动PDO]──→ Op              │    │
│     │                              │                      │            │    │
│     │                              │ 输出保持安全状态    │ 全功能运行  │    │
│     │                              ▼                      ▼            │    │
│     │                          [检查配置]            [实时监控]        │    │
│     │                                                                  │    │
│     └──[固件更新]──→ Boot ─────────────────────────────────────────────┘    │
│                                                                             │
│  各状态功能:                                                                 │
│  ────────────                                                                │
│  Init:     无通信，仅SDO信息对象可读                                          │
│  Pre-Op:   邮箱通信 (SDO) 可用，PDO未激活                                     │
│  Safe-Op:  PDO输入传输，输出保持安全状态                                       │
```

## 3. PROFINET 技术解析

### 3.1 PROFINET实时等级

**定义 3.1 (PROFINET实时性)**

```
PROFINET通信类别:

Class 1: RT (Real Time)
  协议栈优化:
    - 跳过TCP/IP部分处理
    - 直接访问以太网层
    - VLAN优先级 (IEEE 802.1p)

  性能指标:
    周期时间: 1-10ms
    抖动: < 1ms
    适用: 标准工厂自动化

Class 2: IRT (Isochronous Real Time)
  硬件支持:
    - ASIC专用芯片 (ERTEC)
    - 时间同步 (IEEE 1588)
    - 带宽预留 (IEEE 802.1Qbv)

  性能指标:
    周期时间: < 1ms
    抖动: < 1μs
    适用: 运动控制、高精度同步

Class 3: 等时IRT
  特性:
    - 主站控制所有通信时序
    - 从站动作与总线周期严格同步
    - 确定性最高

  性能指标:
    周期时间: 250μs - 1ms
    抖动: < 100ns
    适用: CNC、机器人、印刷机械
```

### 3.2 PROFINET与EtherCAT对比

| 维度 | PROFINET IRT | EtherCAT |
|------|--------------|----------|
| **拓扑** | 线型/星型/环型/树型 | 线型/分支/环型 |
| **设备数/网段** | 64 (IRT) / 无限 (RT) | 65535 |
| **同步精度** | < 100ns | < 100ns |
| **循环周期** | 250μs起 | 100μs起 |
| **标准芯片** | ERTEC 200/400 | ET1100/LAN9252 |
| **主站实现** | 专用硬件或高性能PC | 标准以太网卡 |
| **从站成本** | 较高 | 较低 |
| **诊断功能** | 丰富 (PROFIenergy) | 标准 (EoE/FoE) |
| **IP兼容性** | 完全兼容 | 通过隧道支持 |

## 4. 工业以太网选择决策树

```
工业以太网协议选择
│
├── 实时性需求?
│   ├── 极高 (<100μs, 运动控制)
│   │   ├── 多轴同步?
│   │   │   ├── 是 → EtherCAT或SERCOS III
│   │   │   └── 否 → PROFINET IRT
│   │   │
│   │   └── 现有PLC品牌?
│   │       ├── 西门子 → PROFINET IRT
│   │       ├── Beckhoff → EtherCAT
│   │       └── 其他 → EtherCAT (开放性)
│   │
│   ├── 高 (1-10ms, 标准控制)
│   │   └── PROFINET RT或EtherNet/IP
│   │       └── 现有系统兼容性决定
│   │
│   └── 一般 (>10ms, 监控)
│       └── Modbus TCP或标准EtherNet/IP
│           └── 成本最低
│
├── 拓扑灵活性?
│   ├── 需要混合拓扑
│   │   └── PROFINET (支持任意拓扑)
│   │
│   └── 简单线型足够
│       └── EtherCAT (最优性能)
│
├── 开放性要求?
│   ├── 必须开源/免授权
│   │   └── EtherCAT (技术开放) 或 POWERLINK
│   │
│   └── 可接受商业授权
│       └── 任何协议
│
└── 现有基础设施?
    ├── 西门子主导
    │   └── PROFINET (最佳兼容性)
    │
    ├── Rockwell主导
    │   └── EtherNet/IP
    │
    └── 混合/新建
        └── EtherCAT (通用性最强)
```
