# Modbus 协议概览

> **协议层**: 应用层（OSI第7层）
> **通信模式**: 主从（Master-Slave）
> **最后更新**: 2026-03-29

---

## 1. 协议历史与发展

### 1.1 起源

- **1979年**: Modicon公司开发，用于PLC通信
- **原始目的**: 替代专有通信协议，实现不同厂商设备互联
- **物理层**: 最初基于RS-232，后扩展至RS-485

### 1.2 标准化进程

| 年份 | 里程碑 |
|------|--------|
| 1979 | Modicon发布Modbus协议 |
| 1999 | Modbus-IDA（现为Modbus Organization）成立 |
| 2004 | Modbus TCP被纳入IEC 61158标准 |
| 2006 | 中国发布GB/T 19582国家标准 |
| 2012 | Modbus/TCP Security扩展发布 |

### 1.3 现状

- 全球超过**700万**台设备支持
- 工业自动化领域**最广泛**使用的协议
- 几乎所有工业设备厂商支持

## 3. 核心概念

### 3.1 主从架构

```
        ┌─────────────────────────────────────────┐
        │           单主多从架构                    │
        │                                         │
        │    Master                               │
        │   ┌─────┐                               │
        │   │ 主站 │                               │
        │   └──┬──┘                               │
        │      │                                  │
        │      ▼                                  │
        │   ┌──┴──┐                               │
        │   │ 总线 │ RS-485 / Ethernet             │
        │   └──┬──┘                               │
        │      │                                  │
        │   ┌──┼──┬──┬──┐                        │
        │   ▼  ▼  ▼  ▼  ▼                        │
        │ ┌──┐┌──┐┌──┐┌──┐                       │
        │ │S1││S2││S3││Sn│  Slaves (1-247)       │
        │ └──┘└──┘└──┘└──┘                       │
        │                                         │
        └─────────────────────────────────────────┘
```

**特点**:

- 一主多从，最多247个从机
- 主站发起所有通信
- 从机仅在收到请求时响应
- 广播模式（地址0）无响应

### 3.2 数据模型

| 数据类型 | 访问 | 大小 | 功能码范围 | C语言映射 |
|----------|------|------|------------|-----------|
| **线圈** (Coils) | 读写 | 1位 | 01, 05, 15 | `uint8_t` 位数组 |
| **离散输入** (Discrete Inputs) | 只读 | 1位 | 02 | `uint8_t` 位数组 |
| **保持寄存器** (Holding Registers) | 读写 | 16位 | 03, 06, 16 | `uint16_t` 数组 |
| **输入寄存器** (Input Registers) | 只读 | 16位 | 04 | `uint16_t` 数组 |

```c
/* 数据模型内存布局示例 */
typedef struct {
    /* 线圈: 00001-09999 */
    uint8_t coils[125];           /* 1000个线圈 = 125字节 */

    /* 离散输入: 10001-19999 */
    uint8_t discrete_inputs[125]; /* 1000个输入 */

    /* 输入寄存器: 30001-39999 */
    uint16_t input_regs[125];     /* 125个寄存器 */

    /* 保持寄存器: 40001-49999 */
    uint16_t holding_regs[125];   /* 125个寄存器 */
} ModbusDataModel;
```

## 4. 传输模式对比

| 特性 | Modbus RTU | Modbus ASCII | Modbus TCP |
|------|------------|--------------|------------|
| **编码方式** | 二进制 | ASCII字符 | 二进制 |
| **帧结束** | 3.5字符间隔 | CR LF字符 | 长度字段 |
| **错误检测** | CRC16 | LRC | TCP校验 |
| **效率** | 高（紧凑） | 低（2倍大小） | 中 |
| **可读性** | 差 | 好（可打印） | 差 |
| **现代应用** | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| **物理层** | RS-485 | RS-485 | Ethernet |

## 5. 地址空间映射

### 5.1 传统地址模型

```
地址范围        数据类型            功能码
─────────────────────────────────────────
00001-09999    线圈(Coils)         01, 05, 15
10001-19999    离散输入            02
30001-39999    输入寄存器          04
40001-49999    保持寄存器          03, 06, 16
```

### 5.2 协议地址模型（实际使用）

```
地址范围        数据类型            协议地址（0-based）
────────────────────────────────────────────────────
00001-09999    线圈                0x0000 - 0x270E
10001-19999    离散输入            0x0000 - 0x270E
30001-39999    输入寄存器          0x0000 - 0x270E
40001-49999    保持寄存器          0x0000 - 0x270E
```

**注意**: 协议传输时使用**0-based**地址，但HMI/SCADA通常显示**1-based**地址。

## 6. C语言视角的关键挑战

### 6.1 字节序问题

```c
/* Modbus使用大端序（Big-Endian） */
/* 发送寄存器值前的字节序转换 */
uint16_t value = 0x1234;
uint8_t data[2];

data[0] = (value >> 8) & 0xFF;  /* 高字节: 0x12 */
data[1] = value & 0xFF;          /* 低字节: 0x34 */

/* 或小端主机上的通用转换宏 */
#define HTONS(x) ((((x) & 0xFF) << 8) | (((x) >> 8) & 0xFF))
```

### 6.2 时序敏感性（RTU模式）

```
时序要求:
├── 字符间间隔: ≤ 1.5字符时间
├── 帧间间隔:   ≥ 3.5字符时间
└── 波特率依赖: 9600bps时 ≈ 3.65ms

C语言实现要点:
├── 精确计时（定时器/RTOS tick）
├── 状态机处理接收
└── 超时检测机制
```

### 6.3 缓冲区管理

```c
/* 循环缓冲区设计 - 中断安全 */
typedef struct {
    uint8_t  buffer[512];
    volatile uint16_t head;
    volatile uint16_t tail;
    volatile uint16_t count;
} RingBuffer;

/* 中断服务程序中写入 */
void USART_IRQHandler(void) {
    uint8_t data = USART->DR;
    if (ringbuf_write(&rx_buffer, data)) {
        /* 成功写入，更新帧超时定时器 */
        frame_timer_reset();
    }
}
```
