### 1.2 形式化定义

**定义 1.1 (Modbus RTU 帧)**
```
设 Σ = {0x00, 0x01, ..., 0xFF} 为字节字母表

ModbusRTUFrame ::= Address × FunctionCode × Data × CRC

其中:
  Address ∈ {0x00, 0x01, ..., 0xF7}  (0x00为广播地址)
  FunctionCode ∈ {0x01, 0x02, ..., 0x7F}
  Data ∈ Σ*  (长度范围: 0 ≤ |Data| ≤ 252)
  CRC ∈ Σ²  (16位循环冗余校验)

总帧长度约束: 4 ≤ |Frame| ≤ 256 字节
```

**定义 1.2 (主从状态机)**
```
MasterState ::= IDLE → TRANSMITTING → WAITING → PROCESSING → IDLE
                        ↑_________________________________|

SlaveState ::= LISTENING → RECEIVING → VALIDATING → EXECUTING → 
               TRANSMITTING → LISTENING
                          ↖______________________________↙
                              (异常时跳过TRANSMITTING)

状态转换触发条件:
  - 字符接收完成 (T1.5超时)
  - 帧接收完成 (T3.5超时)
  - CRC验证结果
  - 功能码解析结果
```

## 2. 帧结构多维分析

### 2.1 帧结构决策树

```
Modbus RTU 帧解析决策树
│
├── 接收字节流
│   │
│   ├── T1.5超时触发?
│   │   ├── YES → 帧结束检查 → T3.5超时?
│   │   │                  ├── YES → 完整帧 → CRC校验
│   │   │                  └── NO  → 继续接收
│   │   └── NO  → 继续接收
│   │
│   └── 接收完成
│       │
│       ├── 长度 ≥ 4字节?
│       │   ├── NO → 丢弃帧 (ERROR: 帧过短)
│       │   └── YES → 继续
│       │
│       ├── CRC校验通过?
│       │   ├── NO → 丢弃帧 (ERROR: CRC错误)
│       │   └── YES → 继续
│       │
│       ├── 地址匹配?
│       │   ├── NO → 忽略帧 (非本机地址)
│       │   └── YES → 继续
│       │
│       └── 功能码解析
│           │
│           ├── 功能码支持?
│           │   ├── NO → 异常响应 (0x80 | function_code)
│           │   └── YES → 执行功能
│           │
│           └── 数据合法性检查
│               ├── NO → 异常响应 (ILLEGAL_DATA_VALUE)
│               └── YES → 正常响应
```

### 3.2 时序验证状态机

```c
/* 时序状态机形式化定义 */
typedef enum {
    TIMING_IDLE = 0,        /* 空闲状态 */
    TIMING_RECEIVING = 1,   /* 接收中 */
    TIMING_T1_5_WAIT = 2,   /* 等待T1.5超时 */
    TIMING_T3_5_WAIT = 3,   /* 等待T3.5超时 */
    TIMING_FRAME_READY = 4  /* 帧就绪 */
} TimingState;

/* 状态转换函数 */
TimingState timing_transition(TimingState current, Event event) {
    switch (current) {
        case TIMING_IDLE:
            if (event == CHAR_RECEIVED)
                return TIMING_RECEIVING;
            break;
            
        case TIMING_RECEIVING:
            if (event == CHAR_RECEIVED)
                return TIMING_RECEIVING;  /* 保持 */
            else if (event == T1_5_TIMEOUT)
                return TIMING_T1_5_WAIT;
            break;
            
        case TIMING_T1_5_WAIT:
            if (event == CHAR_RECEIVED)
                return TIMING_RECEIVING;  /* 继续接收 */
            else if (event == T3_5_TIMEOUT)
                return TIMING_FRAME_READY;
            break;
            
        case TIMING_FRAME_READY:
            if (event == FRAME_PROCESSED)
                return TIMING_IDLE;
            break;
            
        default:
            return TIMING_IDLE;
    }
    return current;
}
```

## 4. CRC16算法递归分析

### 4.1 CRC16数学定义

**定义 4.1 (Modbus CRC16)**
```
Modbus使用CRC-16-IBM (多项式: x¹⁶ + x¹⁵ + x² + 1)

生成多项式: G(x) = x¹⁶ + x¹⁵ + x² + 1
多项式系数: 0x8005 (1000 0000 0000 0101)
初始值: 0xFFFF
输入反射: 是 (LSB first)
输出反射: 是
结果异或: 0x0000
```

**定理 4.1 (CRC检测能力)**
```
CRC16可以检测:
1. 所有奇数位错误
2. 所有双位错误
3. 所有小于16位的突发错误
4. 大多数16位以上的突发错误 (概率 1 - 2⁻¹⁶)
```

### 4.2 CRC计算递归算法

```c
/* 递归定义 */
uint16_t crc16_recursive(const uint8_t *data, size_t len, uint16_t crc) {
    /* 基例 */
    if (len == 0) {
        return crc;
    }
    
    /* 递归步: 处理第一个字节 */
    uint8_t byte = *data;
    crc ^= (uint16_t)byte;  /* XOR输入字节 */
    
    /* 对当前字节进行8次移位处理 */
    for (int i = 0; i < 8; i++) {
        if (crc & 0x0001) {
            crc = (crc >> 1) ^ 0xA001;  /* 0x8005反射后 */
        } else {
            crc = crc >> 1;
        }
    }
    
    /* 递归处理剩余数据 */
    return crc16_recursive(data + 1, len - 1, crc);
}

/* 迭代优化版本 (实际使用) */
uint16_t crc16_modbus(const uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    
    for (size_t i = 0; i < len; i++) {
        crc ^= (uint16_t)data[i];
        
        for (int j = 0; j < 8; j++) {
            crc = (crc & 0x0001) ? ((crc >> 1) ^ 0xA001) : (crc >> 1);
        }
    }
    
    return crc;  /* 注意: Modbus要求低字节先传 */
}
```

### 4.3 CRC查找表优化

```c
/* 预计算查找表 (256 entries) */
static const uint16_t crc16_table[256] = {
    0x0000, 0xC0C1, 0xC181, 0x0140, 0xC301, 0x03C0, 0x0280, 0xC241,
    0xC601, 0x06C0, 0x0780, 0xC741, 0x0500, 0xC5C1, 0xC481, 0x0440,
    /* ... 完整表格 ... */
    0x8201, 0x42C0, 0x4380, 0x8341, 0x4100, 0x81C1, 0x8081, 0x4040
};

/* 优化后的CRC计算 */
uint16_t crc16_table_lookup(const uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    
    while (len--) {
        uint8_t idx = (crc ^ *data++) & 0xFF;
        crc = (crc >> 8) ^ crc16_table[idx];
    }
    
    return crc;
}

/* 性能对比:
 * 逐位计算: 每字节8次循环迭代
 * 查表法: 每字节1次数组访问 + 1次XOR
 * 加速比: 约8倍
 */
```

### 5.2 异常响应形式化定义

**定义 5.1 (异常响应)**
```
异常响应 ::= ExceptionFunctionCode × ExceptionCode

ExceptionFunctionCode = RequestFunctionCode | 0x80

ExceptionCode ∈ {
    0x01: ILLEGAL_FUNCTION      (非法功能码)
    0x02: ILLEGAL_DATA_ADDRESS   (非法数据地址)
    0x03: ILLEGAL_DATA_VALUE     (非法数据值)
    0x04: SLAVE_DEVICE_FAILURE   (从站设备故障)
    0x05: ACKNOWLEDGE           (已接受，需轮询)
    0x06: SLAVE_DEVICE_BUSY      (从站忙)
    0x08: MEMORY_PARITY_ERROR    (存储器校验错误)
    0x0A: GATEWAY_PATH_UNAVAILABLE (网关路径不可用)
    0x0B: GATEWAY_TARGET_DEVICE_FAILED (网关目标设备失败)
}
```

**异常响应决策树**
```
异常响应判定树
│
├── 功能码有效性检查
│   ├── 功能码 > 0x7F? 
│   │   └── YES → ERROR: ILLEGAL_FUNCTION (0x01)
│   │
│   └── 功能码在支持列表?
│       └── NO → ERROR: ILLEGAL_FUNCTION (0x01)
│
├── 数据地址检查
│   └── 地址超出范围?
│       └── YES → ERROR: ILLEGAL_DATA_ADDRESS (0x02)
│
├── 数据值检查
│   ├── 寄存器数量 > 最大限制?
│   │   └── YES → ERROR: ILLEGAL_DATA_VALUE (0x03)
│   │
│   └── 输出值不在允许集合?
│       └── YES → ERROR: ILLEGAL_DATA_VALUE (0x03)
│
├── 设备状态检查
│   ├── 设备故障?
│   │   └── YES → ERROR: SLAVE_DEVICE_FAILURE (0x04)
│   │
│   └── 设备忙?
│       └── YES → ERROR: SLAVE_DEVICE_BUSY (0x06)
│
└── 通过所有检查 → 正常响应
```

### 6.2 数据编码规则

| 数据类型 | 大小 | C语言映射 | 字节序 | 示例 |
|----------|------|-----------|--------|------|
| **线圈** | 1位 | `uint8_t` (位掩码) | MSB first | 0x05 = 线圈0、2为ON |
| **离散输入** | 1位 | `uint8_t` (位掩码) | MSB first | 同上 |
| **寄存器** | 16位 | `uint16_t` | **Big-Endian** | 0x1234 → 传输: [0x12, 0x34] |
| **双寄存器** | 32位 | `uint32_t` | Big-Endian | 0x12345678 → [0x12, 0x34, 0x56, 0x78] |
| **浮点数** | 32位 | `float` (IEEE 754) | Big-Endian | 依赖于平台实现 |
| **字符串** | n×8位 | `char[]` | Big-Endian | 每个字符1字节 |

**字节序转换定理**
```
定理: Modbus使用大端序(Big-Endian)，与小端主机需要转换

设主机为小端序，寄存器值为 V = 0x1234

主机内存布局: [0x34] [0x12] (低地址→高地址)
Modbus传输:    [0x12] [0x34] (先传高字节)

转换函数:
    htons(x) = ((x << 8) & 0xFF00) | ((x >> 8) & 0x00FF)

证明: 转换后字节序满足网络传输要求
```

## 7. 实现架构决策树

```
Modbus RTU 从站实现架构决策
│
├── 接收方式选择
│   ├── 中断驱动?
│   │   ├── YES → 每字节中断
│   │   │         ├── 优点: 实时性好
│   │   │         └── 缺点: 中断频率高
│   │   └── NO → DMA+空闲中断
│   │             ├── 优点: 低CPU占用
│   │             └── 缺点: 需要DMA支持
│   │
│   └── 轮询方式?
│       └── 仅适用于简单应用
│
├── 状态机实现
│   ├── 查表法 (Table-driven)
│   │   ├── 适用: 功能码多且复杂
│   │   └── 结构: {function_code, handler_ptr}
│   │
│   └── switch-case
│       └── 适用: 功能码较少
│
├── 缓冲区策略
│   ├── 单缓冲区 (简单)
│   ├── 双缓冲区 (乒乓)
│   └── 环形缓冲区 (连续接收)
│
└── 错误处理
    ├── 静默丢弃 (安全)
    └── 异常响应 (符合规范)
```
