# ISO 26262汽车功能安全与Ada

## 1. ISO 26262概述

### 1.1 标准背景

ISO 26262是汽车电子电气系统的功能安全国际标准，基于IEC 61508改编：

| 属性 | 说明 |
|------|------|
| **全称** | Road vehicles — Functional safety |
| **适用范围** | 量产道路车辆（乘用车，≤3500kg） |
| **发布历史** | 2011年（第1版）、2018年（第2版）、2023年（第3版） |
| **核心目标** | 避免因电气/电子系统故障导致的不合理风险 |

### 1.2 汽车安全完整性等级（ASIL）

```
ASIL等级金字塔:

           ┌─────────┐
           │ ASIL D  │ ← 最高安全等级（灾难性故障）
          ┌┴─────────┴┐
          │  ASIL C   │
         ┌┴───────────┴┐
         │   ASIL B    │
        ┌┴─────────────┴┐
        │    ASIL A     │ ← 最低安全等级
       ┌┴───────────────┴┐
       │     QM          │ ← 质量管理（非安全相关）
       └─────────────────┘
```

**ASIL确定因素**（Severity × Exposure × Controllability）：

| 严重度(S) | 暴露度(E) | 可控性(C) | 典型ASIL |
|-----------|-----------|-----------|----------|
| 危及生命(S3) | 高概率(E4) | 难控制(C3) | D |
| 严重伤害(S2) | 中等概率(E3) | 可控(C2) | B |
| 轻伤(S1) | 低概率(E2) | 可控(C2) | A |

## 2. Ada/SPARK在ISO 26262中的地位

### 2.1 语言选择与ASIL关系

```
ISO 26262-6: 产品软件开发
├── 软件架构设计
│   └── 语言选择考虑:
│       ├── ASIL D/C: 强推荐Ada/SPARK
│       ├── ASIL B: 推荐Ada或MISRA C
│       └── ASIL A/QM: 无限制
│
└── 软件单元设计与实现
    └── 语言使用约束:
        ├── 使用语言子集
        ├── 强制类型检查
        └── 防御性编程
```

### 2.2 Ada vs MISRA C在汽车领域对比

| 特性 | MISRA C | Ada/SPARK | ISO 26262适用性 |
|------|---------|-----------|-----------------|
| **类型安全** | ⚠️ 编码规范约束 | ✅ 编译器强制 | ASIL D需强类型 |
| **数组边界** | ⚠️ 静态分析检查 | ✅ 运行时检查 | ASIL C+必需 |
| **整数溢出** | ⚠️ 手动检查 | ✅ 自动检查 | ASIL B+重要 |
| **形式化验证** | ❌ 有限 | ✅ SPARK支持 | ASIL D推荐 |
| **工具成熟度** | ✅ 高 | ✅ 高 | 两者皆可 |
| **人才可得性** | ✅ 高 | ⚠️ 中等 | 培训成本考虑 |

### 2.3 ISO 26262-6语言子集要求

```ada
-- ISO 26262推荐的Ada子集（ASIL D）

-- ✅ 推荐特性:
-- 强类型和子类型
subtype Percentage is Integer range 0 .. 100;
subtype Speed_Kmh is Float range 0.0 .. 300.0;

-- 范围检查
Wheel_Speed : Speed_Kmh := 0.0;
-- Wheel_Speed := 350.0;  -- 运行时错误，自动检测

-- 数组边界检查
Sensor_Data : array (1 .. 4) of Float;
-- Sensor_Data (5) := 1.0;  -- 运行时错误

-- 合约编程
procedure Brake_Control (Pressure : Brake_Pressure)
   with Pre => Pressure in Min_Pressure .. Max_Pressure;

-- ⚠️ ASIL D需谨慎使用的特性:
-- 访问类型（指针）- 需SPARK证明安全
-- 任务 - 需使用Ravenscar Profile
-- 异常 - 需有处理程序
-- 动态分配 - 避免在运行时
```

## 3. ASIL等级与SPARK验证等级映射

### 3.1 映射矩阵

| ASIL等级 | SPARK验证等级 | 验证目标 | 典型应用 |
|----------|--------------|----------|----------|
| **QM** | Stone | 编码规范合规 | 信息娱乐系统 |
| **ASIL A** | Bronze | 无运行时错误 | 座椅控制、灯光 |
| **ASIL B** | Silver | 功能合约验证 | 空调控制、雨刷 |
| **ASIL C** | Silver+ | 关键功能证明 | 转向辅助、巡航控制 |
| **ASIL D** | Gold | 完全形式化验证 | 制动系统、转向系统 |

### 3.2 详细映射说明

#### ASIL A → SPARK Bronze

```ada
-- 目标：消除运行时错误
-- 方法：gnatprove基础证明

package Engine_Sensors with SPARK_Mode is
   Max_Temp : constant := 150.0;

   subtype Temperature is Float range -40.0 .. Max_Temp;

   procedure Read_Temperature (Sensor_Id : Integer; Value : out Temperature)
      with
         Pre => Sensor_Id in 1 .. 4;  -- 有效传感器ID

   -- Bronze证明:
   -- ✓ 数组不越界（Sensor_Id范围约束）
   -- ✓ 浮点运算无溢出
   -- ✓ 无除零错误
end Engine_Sensors;
```

#### ASIL B/C → SPARK Silver

```ada
-- 目标：功能正确性验证
-- 方法：合约验证

package ABS_Controller with SPARK_Mode is
   type Wheel_Speed is delta 0.1 range 0.0 .. 300.0;

   procedure Calculate_Slip (
      Wheel_Spd : Wheel_Speed;
      Vehicle_Spd : Wheel_Speed;
      Slip_Ratio : out Float)
   with
      Pre  => Vehicle_Spd > 0.0,
      Post => Slip_Ratio in 0.0 .. 1.0;
   -- Silver证明:
   -- ✓ 后置条件满足
   -- ✓ 滑移率计算正确
   -- ✓ 无运行时错误
end ABS_Controller;
```

#### ASIL D → SPARK Gold

```ada
-- 目标：数学级正确性保证
-- 方法：完整形式化验证 + 手工辅助

package Brake_By_Wire with SPARK_Mode is
   Max_Brake_Force : constant := 20000.0;  -- 牛顿

   subtype Brake_Force is Float range 0.0 .. Max_Brake_Force;

   procedure Apply_Braking (
      Pedal_Input : Percentage;
      Wheel_Forces : out Wheel_Force_Array)
   with
      Pre  => Pedal_Input in 0 .. 100,
      Post =>
         -- 制动力与踏板输入成正比
         (for all W in Wheel_ID =>
            Wheel_Forces (W) <= Max_Brake_Force * Float (Pedal_Input) / 100.0)
         and then
         -- 左右轮平衡
         abs (Wheel_Forces (Left_Front) - Wheel_Forces (Right_Front)) < 100.0;
   -- Gold证明:
   -- ✓ 所有路径满足合约
   -- ✓ 复杂量化表达式验证
   -- ✓ 可能需要辅助引理
end Brake_By_Wire;
```

```ada
-- ISO 26262 ASIL D 编码标准示例

-- 1. 所有函数必须有前置条件
procedure Set_Throttle_Position (Position : Percentage)
   with Pre => Engine_State = Running;

-- 2. 所有输出参数必须初始化
procedure Read_Sensor (Value : out Sensor_Reading)
   with Post => Value.Valid;

-- 3. 禁止未初始化变量
declare
   Speed : Velocity;  -- 编译器检查初始化
begin
   Speed := Get_Velocity;  -- 必须在使用前赋值
end;

-- 4. 循环必须有明确的终止条件
while Fuel_Level > Reserve_Level loop
   pragma Loop_Variant (Decreases => Fuel_Level);
   Consume_Fuel;
end loop;

-- 5. 异常必须有处理程序
begin
   Critical_Operation;
exception
   when others =>
      Enter_Safe_State;
      Log_Fault;
      raise;  -- 重新抛出供上层处理
end;
```

## 6. 安全机制实现

### 6.1 E2E（End-to-End）保护

```ada
package E2E_Protection with SPARK_Mode is
   -- CRC校验实现
   subtype CRC_Type is Interfaces.Unsigned_16;

   function Calculate_CRC (Data : Byte_Array) return CRC_Type
      with Post => Calculate_CRC'Result /= 0;  -- CRC永不为零

   -- 序列计数器检测丢失消息
   type Sequence_Counter is mod 16;

   procedure Verify_Message (
      Data : Byte_Array;
      Expected_Seq : Sequence_Counter;
      CRC : CRC_Type;
      Valid : out Boolean)
   with
      Post => Valid = (Calculate_CRC (Data) = CRC);
end E2E_Protection;
```

### 6.2 安全监控器

```ada
package Safety_Monitor with SPARK_Mode is
   type System_State is (Normal, Degraded, Safe);

   procedure Monitor_Task with
      Global => (Input => Sensor_Readings, Output => Control_Commands),
      Depends => (Control_Commands => (Sensor_Readings, Control_Commands));

   -- SPARK证明：监控器不会引入新的数据依赖
   -- 证明：输出仅依赖于当前输入和之前状态
end Safety_Monitor;
```

project Brake_Control is
   for Source_Dirs use ("src");
   for Object_Dir use "obj";
   for Exec_Dir use "bin";

   -- 编译器选项（ISO 26262合规）
   package Compiler is

## 9. 总结与建议

### 9.1 何时选择Ada/SPARK

```
选择Ada/SPARK的场景:
├── ASIL D/C 安全关键功能
│   └── 制动、转向、动力控制
├── 复杂算法需要形式化验证
│   └── 传感器融合、控制算法
├── 需要高可信度证据
│   └── 首次认证、新供应商
└── 长期维护项目
    └── 10+年生命周期

选择MISRA C的场景:
├── ASIL A/B 功能
├── 现有C代码库扩展
├── 快速原型开发
└── 团队C经验丰富
```

### 9.2 迁移路径

```
从C到Ada/SPARK的渐进式迁移:

阶段1: 新模块用Ada (3-6个月)
├── 非安全关键功能
├── 学习曲线适应
└── 工具链建立

阶段2: ASIL模块用SPARK (6-12个月)
├── 重写关键算法
├── 形式化验证
└── 证据收集

阶段3: 遗留接口封装 (3-6个月)
├── C代码Ada绑定
├── 混合编译
└── 完整集成
```
