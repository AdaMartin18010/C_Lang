# 数据链路层 (Data Link Layer)

> **描述**: 工业通信数据链路层协议实现，包括Modbus RTU/TCP、CANopen、EtherCAT、PROFIBUS等工业现场总线协议的帧格式、错误检测、流量控制和主从通信机制

## 概述

数据链路层是OSI模型的第二层，负责节点间的可靠数据传输。在工业通信中，数据链路层需要满足实时性、确定性和可靠性的严格要求。本目录涵盖主流工业通信协议的数据链路层实现。

工业数据链路层协议通常采用主从架构或多主架构，支持广播和点对点通信，具备完善的错误检测机制(CRC校验)和故障恢复能力。理解这些协议的实现原理，对于开发工业自动化系统至关重要。

## 核心内容

- **Modbus协议栈**：支持Modbus RTU(串行)和Modbus TCP(以太网)两种模式，掌握功能码定义和异常响应机制
- **CANopen通信**：理解CAN总线基础上构建的CANopen协议，包括PDO/SDO通信对象、对象字典结构
- **EtherCAT协议**：掌握EtherCAT的过程数据接口、分布式时钟和状态机管理
- **PROFIBUS通信**：理解DP-V0、DP-V1的通信周期和诊断机制
- **数据帧结构**：深入分析各协议的帧格式、寻址方式和优先级机制
- **错误处理**：CRC校验、超时重传、错误帧处理等可靠性机制

## 学习建议

1. **协议对比学习**：对比不同协议的帧结构特点，理解其设计取舍
2. **抓包分析**：使用Wireshark等工具分析实际工业通信数据包
3. **故障模拟**：故意制造通信错误，观察协议的错误恢复机制
4. **实时性分析**：测量各协议的响应延迟，理解确定性保证机制
