## 📖 技术概述

本目录涵盖**内存数据库**核心技术——B+树索引、LSM树、LRU缓存和RESP协议。
这些技术是Redis、LevelDB、RocksDB等高性能存储引擎的基础，对于理解现代数据系统的内部工作原理至关重要。

### 核心知识点

| 主题 | 描述 | 应用场景 |
|:-----|:-----|:---------|
| B+树索引 | 自平衡树结构，支持高效的范围查询 | MySQL InnoDB、SQLite、PostgreSQL |
| LSM树 | 写优化的分层结构，顺序写入优势 | RocksDB、Cassandra、LevelDB |
| LRU缓存 | 最近最少使用淘汰策略，O(1)操作 | Redis、Memcached、Caffeine |
| RESP协议 | Redis序列化协议，简单高效的通信 | Redis客户端、中间件开发 |

### 技术对比

| 特性 | B+树 | LSM树 | LRU缓存 |
|:-----|:-----|:------|:--------|
| 读写比 | 平衡 | 写优化 | 读优化 |
| 范围查询 | 优秀 | 一般 | 不支持 |
| 内存使用 | 中等 | 较高（多版本）| 固定 |
| 适用场景 | OLTP、索引 | 日志、时序数据 | 热点缓存 |

### 01_B_Tree_Index.md 详细内容

- **B+树核心结构**：内部节点(仅键)与叶子节点(键+数据)分离设计
- **节点结构设计**：4KB页对齐、变长键前缀压缩、版本号乐观锁
- **完整CRUD实现**：
  - 搜索：二分查找、搜索路径记录
  - 插入：节点分裂、级联分裂、根节点分裂
  - 删除：借用、合并、级联合并
  - 更新：原地更新策略
- **范围查询**：迭代器模式、双向链表遍历
- **并发控制**：读写锁、乐观锁版本号
- **内存管理**：节点池预分配、页分配器、空闲列表

### 01_RESP_Protocol.md 详细内容

- **RESP数据类型**：Simple String、Error、Integer、Bulk String、Array
- **类型前缀**：`+`、`-`、`:`、`$`、`*`的语义
- **增量式解析器**：状态机实现、支持流水线处理
- **零拷贝优化**：Bulk String直接引用缓冲区
- **序列化器**：完整的RESP格式输出
- **命令构建API**：数组命令构造、便捷宏定义
- **内存管理**：RESP对象生命周期、嵌套数组释放

### 02_LRU_Cache.md 详细内容

- **核心结构**：哈希表(快速查找) + 双向链表(访问顺序)
- **操作复杂度**：Get/Put均为O(1)
- **节点池管理**：预分配策略、空闲链表、避免内存碎片
- **PIN标记**：防止关键数据被淘汰
- **回调函数**：自定义哈希、相等比较、释放函数
- **统计信息**：命中率、miss率、当前大小/容量
- **遍历接口**：MRU到LRU顺序访问
- **动态调整**：运行时修改容量、自动淘汰

### 02_LSM_Tree.md 详细内容

- **核心架构**：MemTable → Immutable → Level 0 → Level 1+ SSTable
- **写入流程**：WAL追加 → MemTable插入 → 后台刷盘 → Compaction
- **读取流程**：MemTable → Immutable → L0(时间序) → L1+(范围查询)
- **布隆过滤器**：快速判断key不存在，减少读放大
- **三大放大问题**：
  - 写放大(Write Amplification)：10-30x
  - 读放大(Read Amplification)：多层查询
  - 空间放大(Space Amplification)：多版本数据
- **压缩策略**：分层压缩 vs 通用压缩
- **完整测试程序**：简化版LSM树模拟、写放大计算

## ⚠️ 学习建议

### 常见陷阱

1. **忽视并发安全**：单线程正确的代码在多线程下可能崩溃
2. **内存泄漏**：复杂数据结构需要仔细的释放顺序
3. **过度优化**：先保证正确性，再考虑性能优化
4. **忽视边界条件**：空树、满节点、分裂/合并等边界需特殊处理

### 实践建议

- 编写完整的单元测试，覆盖各种边界条件
- 使用valgrind检测内存泄漏
- 用sysbench或自定义工具进行性能基准测试
- 对比不同实现的性能差异，理解设计权衡
