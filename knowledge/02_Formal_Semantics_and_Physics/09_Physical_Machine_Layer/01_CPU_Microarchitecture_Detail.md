## 1. 处理器设计范式

### 1.1 CISC vs RISC演进

```
处理器架构演进
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第一代 (1970s-1980s):
  CISC: x86, 68000
  - 复杂变长指令
  - 微码实现
  - 内存操作数

RISC革命 (1980s-1990s):
  MIPS, SPARC, ARM
  - 精简定长指令
  - 单周期执行
  - Load/Store架构

现代融合 (2000s-现在):
  x86: 内部译码为微操作(RISC化)
  ARM: 复杂指令增强
  界限模糊，取长补短
```

**现代CISC内部RISC化:**

```
x86指令 → 前端译码 → 微操作(μops) → 乱序执行

示例:
  add rax, [rbx+8]  ; x86指令

译码为:
  load  tmp, [rbx+8]  ; μop 1
  add   rax, rax, tmp ; μop 2
```

## 2. 流水线架构

### 2.1 经典5级流水线

```
时钟:    1      2      3      4      5      6      7
        │      │      │      │      │      │      │
Instr1: IF →  ID →  EX →  MEM →  WB
Instr2:       IF →  ID →  EX →  MEM →  WB
Instr3:             IF →  ID →  EX →  MEM →  WB
Instr4:                   IF →  ID →  EX →  MEM →  WB

IF:  Instruction Fetch (取指)
ID:  Instruction Decode (译码)
EX:  Execute (执行)
MEM: Memory Access (访存)
WB:  Write Back (写回)
```

### 2.2 现代超标量流水线

```
Intel Golden Cove 流水线 (20+级)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

前端 (Front-end):
  ┌──────────────────────────────────────────────────────┐
  │ 分支预测 → 取指(ITLB/I$) → 预译码 → 指令队列(IQ)     │
  │    1-2      3-4          5-6       7-8               │
  └──────────────────────────────────────────────────────┘
                           ↓
分配/重命名 (Allocate/Rename):
  ┌──────────────────────────────────────────────────────┐
  │ 译码(4→8) → μop队列 → 重排序缓冲区(ROB)分配          │
  │   9-12      13-14        15-16                       │
  └──────────────────────────────────────────────────────┘
                           ↓
执行 (Execute):
  ┌──────────────────────────────────────────────────────┐
  │ 调度/分派 → 执行单元(ALU/AGU/FP/Vec) → 完成          │
  │   17-18                               19-20          │
  └──────────────────────────────────────────────────────┘
                           ↓
提交 (Commit):
  ┌──────────────────────────────────────────────────────┐
  │ ROB按序提交 → 寄存器文件更新 → 状态可见化            │
  └──────────────────────────────────────────────────────┘
```

### 3.2 寄存器重命名

```c
// 寄存器重命名示例
// 架构寄存器动态映射到物理寄存器，消除WAR/WAW冲突

// 原始代码 (x86-64)
mov rax, 1      // RAX = 1
mov rax, 2      // RAX = 2 (WAW: Write-After-Write)
mov rbx, rax    // RBX = RAX (RAW: Read-After-Write)

// 重命名后
// 假设: 物理寄存器池 P0-Pn
mov P0, 1       // 分配P0给RAX
mov P1, 2       // 分配P1给RAX, P0可回收
mov P2, P1      // 分配P2给RBX

// RAT (寄存器别名表) 状态:
// RAX → P1
// RBX → P2

// 重命名消除WAW依赖，允许并行执行
```

  ┌─────────────────────────────────────────┐
  │ BTB条目: 分支PC → 目标PC + 预测状态    │

  └─────────────────────────────────────────┘

第三级: 条件方向预测器

### 4.3 返回地址预测

```
返回地址栈 (Return Address Stack)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

结构:
  ┌─────────────────────────────────┐
  │ RAS: 16-32条目循环缓冲区        │
  │                                 │
  │  call指令: 压入返回地址         │
  │  ret指令:  弹出预测地址         │
  └─────────────────────────────────┘

深度递归问题:
  // RAS溢出导致预测失败
  void deep_recursive(int n) {
      if (n <= 0) return;
      deep_recursive(n - 1);  // 每次call压栈
  }  // ret弹出，深度>32时失败

缓解:
  • 更大RAS (32-64条目)
  • RSB填充 (retpoline)
```

### 5.2 TLB层次

```
TLB结构 (Intel Golden Cove)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

L1指令TLB:
  • 128条目, 4-way, 4周期

L1数据TLB:
  • 96条目, 可覆盖4KB/2MB页

L2 TLB (STLB):
  • 2048条目, 12-way
  • 覆盖4KB/2MB/1GB页
  • 15周期延迟

TLB Miss处理:
  • 硬件页表遍历 (HW walker)
  • 4级页表: 4×内存访问
  • 总计: ~200周期

优化:
  • 大页减少TLB压力
  • 预取TLB条目
```
