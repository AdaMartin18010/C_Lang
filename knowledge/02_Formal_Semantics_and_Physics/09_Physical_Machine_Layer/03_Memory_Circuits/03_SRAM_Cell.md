# SRAM存储单元 (Static Random Access Memory)

## 1. 概述

SRAM(静态随机存取存储器)是一种使用双稳态电路存储数据的半导体存储器。与DRAM不同，SRAM不需要周期性刷新，具有更快的访问速度和更低的功耗，但集成度较低、成本较高。

```
┌─────────────────────────────────────────────────────────────────┐
│                    存储器层次结构对比                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  存储器类型    单元结构      速度      密度      成本    应用     │
│  ─────────────────────────────────────────────────────────────  │
│  Register     D触发器       最快      最低      最高   CPU内部   │
│  SRAM         6T单元        快        低        高     Cache    │
│  DRAM         1T1C          中等      高        低     主存     │
│  Flash        浮栅晶体管    慢        很高      很低   SSD/U盘  │
│  磁盘         磁性存储      最慢      最高      最低   大容量   │
│                                                                 │
│  典型延迟对比：                                                  │
│  Register:  ~0.1 ns                                            │
│  L1 Cache:  ~1 ns   (SRAM)                                     │
│  L2 Cache:  ~4 ns   (SRAM)                                     │
│  L3 Cache:  ~10 ns  (SRAM)                                     │
│  DRAM:      ~100 ns                                            │
│  SSD:       ~10 μs                                             │
│  磁盘:      ~10 ms                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 读写操作详解

### 3.1 读操作流程

```
读操作步骤：

初始状态（假设存储'1'）：
    Q  = 1 (Vdd)
    Q̄ = 0 (GND)
    BL = BL̄ = Vdd (预充电)
    WL = 0

步骤1: 预充电 (Precharge)
    ┌─────────────────────────────────────────────────────────┐
    │  时序: 所有位线预充电到Vdd                                │
    │                                                         │
    │  BL ──────┐   ┌─────────────────────────                │
    │           │   │                                          │
    │  BL̄ ─────┤   ├─────────────────────────                │
    │           │   │                                          │
    │  WL ──────┘   └─────────────────────────                │
    │               ↑                                          │
    │            预充电完成                                    │
    └─────────────────────────────────────────────────────────┘

步骤2: 字线激活 (Word Line Enable)
    ┌─────────────────────────────────────────────────────────┐
    │  WL拉高，传输管导通                                       │
    │                                                         │
    │  Q(=1)通过PG1连接到BL                                    │
    │  Q̄(=0)通过PG2连接到BL̄                                   │
    │                                                         │
    │  BL ────────────────────────────────                    │
    │                                   (保持高，略有下降)      │
    │  BL̄ ───────────────┐                                    │
    │                    ↓                                    │
    │                 (放电下降)                               │
    │  WL ───────────────┐                                    │
    │                    │                                    │
    │                    └────────────────                    │
    │                                                         │
    │  电压差: ΔV = 50-200mV (小信号)                          │
    └─────────────────────────────────────────────────────────┘

步骤3: 灵敏放大 (Sense Amplification)
    ┌─────────────────────────────────────────────────────────┐
    │  灵敏放大器检测微小的BL/BL̄电压差                          │
    │                                                         │
    │  BL ────────────────────────────────                    │
    │                                   (被放大到全摆幅)        │
    │  BL̄ ───────────────────────────────                     │
    │                    (被放大到全摆幅)                       │
    │  SA_EN ────────────┐                                    │
    │                    │                                    │
    │                    └────────────────                    │
    │                                                         │
    │  输出: Data = 1 (BL高，BL̄低)                             │
    └─────────────────────────────────────────────────────────┘

步骤4: 字线关闭，预充电恢复

完整时序图：

信号:    CLK     PRECHARGE       WL        SA_EN       DATA_OUT
         │          │            │           │           │
         │    ┌─────┐            │           │           │
PRECHARGE├────┤     ├────────────┤           │           │
         │    └─────┘            │           │           │
         │                       │           │           │
         │            ┌──────────┐           │           │
WL       ├────────────┤          ├───────────┤           │
         │            └──────────┘           │           │
         │                                   │           │
         │                        ┌──────────┐          │
SA_EN    ├────────────────────────┤          ├──────────┤
         │                        └──────────┘          │
         │                                               │
         │                                    ┌──────────┐
DATA_OUT ├────────────────────────────────────┤          │
         │                                    └──────────┘
         │                                               │
时序标注: t_precharge    t_access      t_sense    t_output
          (预充电)      (访问延迟)    (放大时间)  (输出稳定)
```

```
    │                                                         │
```

## 4. 位线预充电

### 4.1 预充电电路

```
位线预充电电路：

         Vdd
          │
     ┌────┴────┐
     │         │
    ┌┴┐       ┌┴┐
    │ │ PC0   │ │ PC1
    │ │ PMOS  │ │ PMOS
    └┬┘       └┬┘
     │         │
     ├────┬────┤
     │    │    │
     │   ┌┴┐   │
     │   │E│   │ 均衡管(Equalizer)
     │   │Q│   │
     │   └┬┘   │
     │    │    │
     BL   EQ   BL̄
     │         │
     │    ┌────┘
     │    │
    ┌┴┐  ┌┴┐
    │ │  │ │  列选择管
    │ │  │ │
    └┬┘  └┬┘
     │    │

    灵敏放大器/写入驱动

操作模式：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  PRECHARGE = 0 (低电平有效):                                    │
│  - PC0, PC1导通：BL和BL̄被充电到Vdd                              │
│  - EQ导通：BL和BL̄短路均衡                                       │
│  - 结果：BL = BL̄ = Vdd                                          │
│                                                                 │
│  PRECHARGE = 1:                                                 │
│  - 所有PMOS关闭                                                 │

│                                                                 │
│  预充电时序：                                                    │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                                                           │  │
│  │  CLK:   ___/‾‾‾‾\____/‾‾‾‾\____/‾‾‾‾\____                  │  │
│  │                                                           │  │
│  │  PRE:   ‾‾‾\____/‾‾‾‾‾‾‾‾‾‾‾‾\____/‾‾‾‾‾‾‾‾                │  │

│  │           预充电  读/写     预充电                          │  │
│  │           阶段    阶段      阶段                           │  │
│  │                                                           │  │
│  │  BL:    ═══╪══════╪═════════╪═══════                      │  │
│  │            Vdd   浮动/放电   Vdd                           │  │
│  │                                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
```

### 4.2 Verilog实现

```verilog
// ============================================================
// 位线预充电电路模型
// ============================================================

module bitline_precharge (
    input  wire        clk,
    input  wire        precharge_en,    // 预充电使能（高有效）
    inout  wire        bl,              // 位线
    inout  wire        blb              // 互补位线
);

    // 预充电模型：当precharge_en为高时，BL和BLB被驱动到Vdd
    assign bl  = precharge_en ? 1'b1 : 1'bz;
    assign blb = precharge_en ? 1'b1 : 1'bz;
    
    // 均衡功能
    // 当预充电时，BL和BLB应该相等（通过均衡管连接）

endmodule

// ============================================================
// 完整的SRAM列电路（包含预充电、单元阵列、灵敏放大器）
// ============================================================

module sram_column (
    input  wire        clk,
    input  wire        precharge_en,
    input  wire        wl,              // 字线
    input  wire        write_en,
    input  wire        write_data,
    output reg         read_data,
    output wire        bl, blb
);

    // SRAM单元存储节点（内部）
    reg  q, qb;
    wire bl_internal, blb_internal;
    
    // 预充电阶段模型
    reg  bl_precharged, blb_precharged;
    
    always @(*) begin
        if (precharge_en) begin
            bl_precharged = 1'b1;
            blb_precharged = 1'b1;
        end else begin
            // 读操作：单元驱动位线
            if (wl && !write_en) begin
                // 简单模型：Q=1时保持BL高，Q=0时拉低BL
                bl_precharged = q;
                blb_precharged = qb;
            end else begin
                bl_precharged = 1'bz;
                blb_precharged = 1'bz;
            end
        end
    end
    
    assign bl = bl_precharged;
    assign blb = blb_precharged;
    
    // SRAM单元行为模型
    always @(posedge clk) begin
        if (wl && write_en) begin
            // 写操作
            q <= write_data;
            qb <= ~write_data;
        end
    end
    
    // 初始值
    initial begin
        q = 1'b0;
        qb = 1'b1;
    end
    
    // 读数据输出（经过灵敏放大）
    always @(posedge clk) begin
        if (!precharge_en && wl && !write_en) begin
            // 读操作完成时采样
            read_data <= q;
        end
    end

endmodule
```

## 5. 灵敏放大器

### 5.1 差分灵敏放大器

```
差分灵敏放大器电路：

         Vdd
          │
      ┌───┴───┐
      │       │
     PC      PC
    (PMOS)  (PMOS)
      │       │
      ├───┬───┤
      │   │   │
     BL   │   BL̄
      │   │   │
     ┌┴┐  │  ┌┴┐
     │ │  │  │ │
     │N1│  │  │N2│
     │ │  │  │ │
     └┬┘  │  └┬┘
      ├───┴───┤
      │       │
      │  ┌─┐  │
      └──┤N3├──┘
         │ │  (尾电流管)
         └┬┘
          │
         GND

    SA_EN ────→ N3栅极

工作原理：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  阶段1: 预充电                                                 │
│  - SA_EN = 0, N3关闭                                           │
│  - BL = BL̄ = Vdd (由预充电电路设置)                             │
│  - N1, N2都导通但无电流（N3关闭）                               │
│                                                                 │
│  阶段2: 字线激活，小信号发展                                    │
│  - 字线拉高，SRAM单元驱动位线                                    │
│  - 产生小电压差: BL - BL̄ = ΔV (50-200mV)                       │
│  - SA_EN仍为0                                                   │
│                                                                 │
│  阶段3: 灵敏放大使能                                           │
│  - SA_EN = 1, N3导通                                           │
│  - 交叉耦合对开始正反馈                                         │
│  - 假设 BL > BL̄:                                               │
│    * N1比N2导通更强                                            │
│    * BL放电更快                                                │
│    * 进一步增大BL和BL̄的电压差                                   │
│    * 正反馈放大                                                │
│  - 最终: BL → 0, BL̄ → Vdd                                      │
│                                                                 │
│  放大时序：                                                     │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                                                         │    │
│  │  BL:    ═════╪════╪══════════╪═══════════════           │    │
│  │              Vdd  Vdd-ΔV      ↓                         │    │
│  │                             (快速下降到0)                │    │
│  │  BL̄:   ═════╪════╪══════════╪═══════════════           │    │
│  │              Vdd  Vdd          ↑                        │    │
│  │                             (保持高电平)                 │    │
│  │                                                         │    │
│  │  SA_EN: ______/‾‾‾‾‾‾‾‾‾‾‾‾\____                       │    │
│  │                     ↑                                  │    │
│  │                  放大开始                               │    │
```

```
        SAE = 1
        正反馈快速放大电压差到全摆幅
```

### 7.3 与C语言的关系

```c
// C语言代码与Cache SRAM的对应关系

// 示例1: 数组访问
int array[1024];  // 4KB数组，可能映射到多个Cache行

int sum = 0;
for (int i = 0; i < 1024; i++) {
    sum += array[i];  // 每次访问可能引起Cache命中或缺失
}

/*
 * Cache行为分析：
 * 
 * 假设：Cache行大小64字节，int=4字节
 * 
 * 数组占用: 1024 × 4B = 4KB
 * Cache行数: 4KB / 64B = 64行
 * 
 * 访问模式：
 * - 第0次访问 array[0]: 冷缺失 (Cold Miss)
 *   → 从内存加载64字节到Cache (array[0]-array[15])
 *   
 * - 第1-15次访问 array[1]-array[15]: 命中 (Hit)
 *   → 直接从Data SRAM读取
 *   
 * - 第16次访问 array[16]: 冷缺失
 *   → 从内存加载下一行
 *   
 * 总缺失数: 64次 (每行第一次访问)
 * 总命中数: 960次
 * 命中率: 960/1024 = 93.75%
 */

// 示例2: 结构体访问
struct Node {
    int key;        // 偏移0
    int value;      // 偏移4
    struct Node* next;  // 偏移8
};

struct Node* head = ...;
while (head != NULL) {
    printf("%d: %d\n", head->key, head->value);
    head = head->next;
}

/*
 * Cache行为分析：
 * 
 * 结构体大小: 12字节 (假设64位指针)
 * 每次访问结构体可能涉及：
 * 
 * 1. 访问 head->key:
 *    - 地址: head + 0
 *    - 可能命中或缺失
 * 
 * 2. 访问 head->value:
 *    - 地址: head + 4
 *    - 大概率命中（与key在同一Cache行）
 * 
 * 3. 访问 head->next:
 *    - 地址: head + 8
 *    - 大概率命中（与前两个字段在同一Cache行）
 * 
 * 空间局部性优化:
 * - 将小字段放在一起，减少Cache行访问
 * - 避免伪共享（False Sharing）
 */

// 示例3: 伪共享问题
struct {
    int counter1;   // CPU 0频繁修改
    int counter2;   // CPU 1频繁修改
} shared;  // 两个字段在同一Cache行！

/*
 * 问题：
 * - counter1和counter2在同一64字节Cache行
 * - CPU 0修改counter1 → 整个Cache行变为Modified
 * - CPU 1需要修改counter2 → 必须从CPU 0获取最新副本
 * - 频繁的Cache一致性流量，性能急剧下降
 * 
 * 解决方案：填充到不同Cache行
 */
struct {
    int counter1;
    char pad[60];   // 填充到64字节边界
    int counter2;
} separated;
```

### 8.1 6T SRAM单元

```verilog
// ============================================================
// 6T SRAM单元行为模型
// ============================================================

module sram_cell_6t (
    input  wire        wl,          // 字线
    inout  wire        bl,          // 位线
    inout  wire        blb          // 互补位线
);

    // 内部存储节点（双稳态）
    reg  q, qb;
    
    // 存储节点初始化
    initial begin
        q = 1'b0;
        qb = 1'b1;
    end
    
    // 读操作模型：单元驱动位线
    // 注意：实际电路是模拟行为，这里是简化模型
    assign bl = (wl && q) ? 1'b0 : 1'bz;   // Q=1时，BL略放电（简化）
    assign blb = (wl && qb) ? 1'b0 : 1'bz; // Q̄=1时，BL̄略放电
    
    // 写操作模型
    always @(bl or blb or wl) begin
        if (wl) begin
            // 字线激活时，根据位线状态写入
            // 实际电路：强驱动位线可以翻转单元状态
            if (bl === 1'b0 && blb === 1'b1) begin
                q <= 1'b0;
                qb <= 1'b1;
            end else if (bl === 1'b1 && blb === 1'b0) begin
                q <= 1'b1;
                qb <= 1'b0;
            end
        end
    end

endmodule
```

```verilog
module sram_array (
    input  wire        clk,
    input  wire        rst_n,
    
    // 控制信号
    input  wire        cs_n,        // 片选（低有效）
    input  wire        we_n,        // 写使能（低有效）
    input  wire        oe_n,        // 输出使能（低有效）
    
    // 地址和数据
    input  wire [9:0]  addr,        // 10位地址 (1024行)
    input  wire [31:0] data_in,     // 写入数据
    output wire [31:0] data_out     // 读出数据
);

    // 存储阵列: 1024 × 32位
    reg [31:0] memory [0:1023];
    
    // 内部信号
    wire       chip_selected;
    wire       write_enable;
    wire       read_enable;
    reg [31:0] data_out_reg;
    
    assign chip_selected = ~cs_n;
    assign write_enable = chip_selected & ~we_n;
    assign read_enable = chip_selected & we_n & ~oe_n;
    
    integer i;
    
    // 写操作（时序）
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            // 复位时清零（实际SRAM通常不复位内容）
            // for (i = 0; i < 1024; i = i + 1)
            //     memory[i] <= 32'b0;
        end else if (write_enable) begin
            memory[addr] <= data_in;
        end
    end
    
    // 读操作（组合，实际SRAM是时序的）
    always @(*) begin
        if (read_enable) begin
            data_out_reg = memory[addr];
        end else begin
            data_out_reg = 32'bz;
        end
    end
    
    assign data_out = data_out_reg;

endmodule
```
