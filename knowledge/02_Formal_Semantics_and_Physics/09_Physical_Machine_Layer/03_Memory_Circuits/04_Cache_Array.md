# 缓存阵列 (Cache Array)

## 1. 概述

缓存(Cache)是位于CPU和主存之间的高速存储层次，利用程序访问的局部性原理来减少平均内存访问时间。
理解Cache的硬件实现对于编写高性能C语言程序至关重要。

```
┌─────────────────────────────────────────────────────────────────┐
│                    现代处理器存储层次                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  层级      容量        延迟        技术        实现              │
│  ─────────────────────────────────────────────────────────────  │
│  寄存器    1KB        0.3ns      CMOS         CPU核心           │
│     ↓                                                           │
│  L1 Cache  32-64KB   1ns        6T-SRAM     CPU核心             │
│     ↓                                                           │
│  L2 Cache  256KB-1MB 4ns        6T-SRAM     CPU核心/簇          │
│     ↓                                                           │
│  L3 Cache  8-64MB    10ns       6T-SRAM     共享/片外           │
│     ↓                                                           │
│  DRAM      8-512GB   100ns      1T1C        内存条              │
│     ↓                                                           │
│  SSD/HDD   256GB-XXTB 10μs-10ms Flash/磁   存储设备             │
│                                                                 │
│  缓存设计目标：                                                  │
│  - 高命中率（利用时间/空间局部性）                               │
│  - 低访问延迟（快速Tag比较）                                     │
│  - 高带宽（并行访问、多端口）                                    │
│  - 低功耗（选择性激活）                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. Cache行结构

### 2.1 行内部组织

```
Cache行详细结构（64字节为例）：

│  │  元数据区 (Metadata)                                     │    │

│  │  │  Tag    │  Valid  │  Dirty  │  LRU    │  其他标志   │    │

│  │  └─────────┴─────────┴─────────┴─────────┴─────────────┘    │
│  │                      │                                      │    │
│  │                      ▼                                      │    │
│  │  数据区 (Data) - 64字节 (512位)                              │    │
│  │  ┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐     │    │
│  │  │Byte│Byte│Byte│Byte│    │    │    │    │    │Byte│     │    │
│  │  │ 0  │ 1  │ 2  │ 3  │... │... │... │... │... │ 63 │     │    │
│  │  └────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘     │    │
│  │                                                          │    │
│  │  字节偏移: 6位 (log₂64 = 6)                                 │    │
│  │  可寻址: Byte0 - Byte63                                     │    │
│  │                                                          │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  总位数计算：                                                    │
│  - Tag: 21位                                                    │
│  - Valid: 1位                                                   │
│  - Dirty: 1位                                                   │
│  - LRU: 2位 (4路组相联)                                         │

│  - 总行大小: ~541位 ≈ 68字节                                    │
```

### 3.2 Tag比较器

```
21位Tag比较器电路：

单比特比较器 (XNOR):

    Tag_A[i] ───┬───┐
                │   │
    Tag_B[i] ───┼───┤ XNOR ─── Match[i]
                │   │
               ┌┴┐  │
               │=│←─┘
               └┬┘

21位相等比较器：

    ┌─────────────────────────────────────────────────────────┐
    │                                                         │
│  Match[0] ───┐                                          │
│              │                                          │
│  Match[1] ───┼───┐                                      │
│              │   │                                      │
│  Match[2] ───┼───┼───┐                                  │
│              │   │   │                                  │
│     ...      │   │   │                                  │
│              │   │   │                                  │
│  Match[20] ──┼───┼───┼───┐                              │
│              │   │   │   │                              │
│              ▼   ▼   ▼   ▼                              │
│            ┌─────────────────┐                          │
│            │    21输入AND    │                          │
│            │                 │                          │
│            │  Match = ∧(Match[i])  │                    │
│            │                 │                          │
│            └────────┬────────┘                          │
│                     │                                   │
│                     ▼                                   │
│                  Match_All                              │
│                                                         │
│  与Valid信号组合：                                       │
│                                                         │
│  Valid ────┐                                            │
│            ├──→ AND ───→ Hit                            │
│  Match ────┘                                            │
│                                                         │
│  (只有当Tag匹配且Valid=1时才命中)                        │
│                                                         │
└─────────────────────────────────────────────────────────┘

4路并行比较：

    Tag[20:0] ───→ ┌─────┐
                   │     ├──→ Compare[0] ──┐
    Way0_Tag ───→ │ CMP │                    │
                   │     ├──→ Valid[0] ─────┼──→ AND ──→ Hit[0]
                   └─────┘                    │
                                              │
    Tag[20:0] ───→ ┌─────┐                    │
                   │     ├──→ Compare[1] ──┐  │
    Way1_Tag ───→ │ CMP │                   ├──→ AND ──→ Hit[1]
                   │     ├──→ Valid[1] ────┘  │
                   └─────┘                    │
                   ...                        │
                                              │
    Tag[20:0] ───→ ┌─────┐                    │
                   │     ├──→ Compare[3] ──┐  │
    Way3_Tag ───→ │ CMP │                   ├──→ AND ──→ Hit[3]
                   │     ├──→ Valid[3] ────┘
                   └─────┘

    Hit = Hit[0] | Hit[1] | Hit[2] | Hit[3]

    Way_Select = 优先级编码(Hit[3:0])
               = 00 (Way0), 01 (Way1), 10 (Way2), 11 (Way3)
```

### 3.3 Verilog实现

```verilog
// ============================================================
// Tag SRAM模块
// ============================================================

module tag_sram #(
    parameter TAG_WIDTH = 21,
    parameter INDEX_WIDTH = 7,      // 128组
    parameter NUM_WAYS = 4
)(
    input  wire                      clk,
    input  wire                      rst_n,

    // 读端口 (用于查找)
    input  wire [INDEX_WIDTH-1:0]    read_index,
    output wire [TAG_WIDTH-1:0]      read_tag [0:NUM_WAYS-1],
    output wire                      read_valid [0:NUM_WAYS-1],
    output wire                      read_dirty [0:NUM_WAYS-1],

    // 写端口 (用于填充/更新)
    input  wire                      write_en,
    input  wire [$clog2(NUM_WAYS)-1:0] write_way,
    input  wire [INDEX_WIDTH-1:0]    write_index,
    input  wire [TAG_WIDTH-1:0]      write_tag,
    input  wire                      write_valid,
    input  wire                      write_dirty,

    // 更新Dirty位 (独立，不需要更新Tag)
    input  wire                      update_dirty_en,
    input  wire [$clog2(NUM_WAYS)-1:0] update_dirty_way,
    input  wire [INDEX_WIDTH-1:0]    update_dirty_index,
    input  wire                      update_dirty_value
);

    // Tag存储阵列
    reg [TAG_WIDTH-1:0] tag_array [0:NUM_WAYS-1][0:(1<<INDEX_WIDTH)-1];
    reg                 valid_array [0:NUM_WAYS-1][0:(1<<INDEX_WIDTH)-1];
    reg                 dirty_array [0:NUM_WAYS-1][0:(1<<INDEX_WIDTH)-1];

    integer i, j;

    // 复位和写操作
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            // 复位时清除所有Valid位
            for (i = 0; i < NUM_WAYS; i = i + 1) begin
                for (j = 0; j < (1<<INDEX_WIDTH); j = j + 1) begin
                    valid_array[i][j] <= 1'b0;
                    dirty_array[i][j] <= 1'b0;
                end
            end
        end else begin
            // 写新Tag (Cache填充)
            if (write_en) begin
                tag_array[write_way][write_index] <= write_tag;
                valid_array[write_way][write_index] <= write_valid;
                dirty_array[write_way][write_index] <= write_dirty;
            end

            // 更新Dirty位 (写命中时设置)
            if (update_dirty_en) begin
                dirty_array[update_dirty_way][update_dirty_index] <= update_dirty_value;
            end
        end
    end

    // 读操作（组合逻辑，实际SRAM有延迟）
    genvar way;
    generate
        for (way = 0; way < NUM_WAYS; way = way + 1) begin : gen_read
            assign read_tag[way] = tag_array[way][read_index];
            assign read_valid[way] = valid_array[way][read_index];
            assign read_dirty[way] = dirty_array[way][read_index];
        end
    endgenerate

endmodule

// ============================================================
// Tag比较器模块
// ============================================================

module tag_comparator #(
    parameter TAG_WIDTH = 21,
    parameter NUM_WAYS = 4
)(
    input  wire [TAG_WIDTH-1:0]      request_tag,
    input  wire [TAG_WIDTH-1:0]      stored_tag [0:NUM_WAYS-1],
    input  wire                      valid [0:NUM_WAYS-1],
    output wire                      hit,
    output wire [$clog2(NUM_WAYS)-1:0] hit_way,
    output wire [NUM_WAYS-1:0]       way_hit
);

    // 每路的比较结果
    genvar i;
    generate
        for (i = 0; i < NUM_WAYS; i = i + 1) begin : gen_compare
            assign way_hit[i] = (request_tag == stored_tag[i]) && valid[i];
        end
    endgenerate

    // 总命中信号
    assign hit = |way_hit;

    // 命中路选择（优先级编码器）
    // Way0优先级最高，Way3最低
    assign hit_way = way_hit[0] ? 2'd0 :
                     way_hit[1] ? 2'd1 :
                     way_hit[2] ? 2'd2 :
                     way_hit[3] ? 2'd3 : 2'd0;

endmodule
```

### 4.2 字节选择逻辑

```
64字节到4字节的选择逻辑（32位访问）：

输入: 64字节数据 (Byte0 - Byte63)
      6位Offset (选择起始字节)

输出: 4字节数据 (32位)

选择逻辑：

Offset(5:0)    选择字节          有效?
─────────────────────────────────────
6'b000000      Byte0-Byte3       Yes (对齐访问)
6'b000001      Byte1-Byte4       No (非对齐)
6'b000010      Byte2-Byte5       No (非对齐)
6'b000011      Byte3-Byte6       No (非对齐)
6'b000100      Byte4-Byte7       Yes (对齐访问)
...            ...               ...
6'b111100      Byte60-Byte63     Yes (对齐访问)
6'b111101      Byte61-Byte0      No (跨越行边界)
...

8:1 MUX层级结构：

64B数据 ───→ ┌─────────────────────────────────────────┐
             │           第一层: 8个8:1 MUX              │
             │  每组选择8字节中的一个                     │
             │                                         │
             │  MUX0: Byte0/8/16/24/32/40/48/56 ──→    │
             │  MUX1: Byte1/9/17/25/33/41/49/57 ──→    │
             │  MUX2: Byte2/10/18/26/34/42/50/58 ──→   │
             │  MUX3: Byte3/11/19/27/35/43/51/59 ──→   │
             │  ...                                    │
             │  MUX7: Byte7/15/23/31/39/47/55/63 ──→   │
             └──────────────────┬──────────────────────┘
                                │
                                ▼
             ┌─────────────────────────────────────────┐
             │         第二层: 4个2:1 MUX              │
             │                                         │
             │  Offset[2:0]选择第一层的哪个MUX输出      │
             │                                         │
             │  最终输出: 4字节对齐数据                  │
             │                                         │
             └──────────────────┬──────────────────────┘
                                │
                                ▼
                          32位数据输出

非对齐访问处理：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  情况1: 非对齐在同一Cache行内                                    │
│  例: Offset=2, 读取Byte2-Byte5                                  │
│  - 使用移位器或额外的MUX选择                                    │
│  - 单周期完成                                                   │
│                                                                 │
│  情况2: 非对齐跨越Cache行边界                                    │
│  例: Offset=62, 读取Byte62-Byte65                               │
│  - Byte62-63来自当前行                                          │
│  - Byte64-65来自下一行（需要第二次访问）                         │
│  - 两周期完成                                                   │
│                                                                 │
│  硬件实现选择：                                                  │
│  - 简单设计：只支持对齐访问，非对齐产生异常                      │
│  - 复杂设计：硬件自动处理非对齐，可能需要多周期                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 Verilog实现

```verilog
// ============================================================
// Data SRAM模块
// ============================================================

module data_sram #(
    parameter DATA_WIDTH = 512,     // 64字节 = 512位
    parameter INDEX_WIDTH = 7,      // 128组
    parameter NUM_WAYS = 4
)(
    input  wire                      clk,

    // 读端口
    input  wire [INDEX_WIDTH-1:0]    read_index,
    input  wire [$clog2(NUM_WAYS)-1:0] read_way,
    output wire [DATA_WIDTH-1:0]     read_data,

    // 写端口 (支持字节写使能)
    input  wire                      write_en,
    input  wire [INDEX_WIDTH-1:0]    write_index,
    input  wire [$clog2(NUM_WAYS)-1:0] write_way,
    input  wire [DATA_WIDTH-1:0]     write_data,
    input  wire [DATA_WIDTH/8-1:0]   write_byte_en   // 64位字节写使能
);

    // 数据存储阵列
    reg [DATA_WIDTH-1:0] data_array [0:NUM_WAYS-1][0:(1<<INDEX_WIDTH)-1];

    integer i;

    // 读操作（组合逻辑）
    assign read_data = data_array[read_way][read_index];

    // 写操作（时序，支持字节写）
    always @(posedge clk) begin
        if (write_en) begin
            for (i = 0; i < DATA_WIDTH/8; i = i + 1) begin
                if (write_byte_en[i]) begin
                    data_array[write_way][write_index][i*8 +: 8] <=
                        write_data[i*8 +: 8];
                end
            end
        end
    end

endmodule

// ============================================================
// 数据选择器 (Way选择 + 字节选择)
// ============================================================

module data_selector #(
    parameter DATA_WIDTH = 512,     // 64字节
    parameter OUTPUT_WIDTH = 32,    // 4字节输出
    parameter NUM_WAYS = 4
)(
    input  wire [DATA_WIDTH-1:0]     way_data [0:NUM_WAYS-1],
    input  wire [$clog2(NUM_WAYS)-1:0] selected_way,
    input  wire [5:0]                offset,         // 字节偏移
    input  wire [1:0]                size,           // 00=1B, 01=2B, 10=4B, 11=8B
    output reg  [OUTPUT_WIDTH-1:0]   selected_data
);

    wire [DATA_WIDTH-1:0] way_selected;

    // 4:1 MUX选择路
    assign way_selected = (selected_way == 2'd0) ? way_data[0] :
                          (selected_way == 2'd1) ? way_data[1] :
                          (selected_way == 2'd2) ? way_data[2] : way_data[3];

    // 字节选择逻辑
    always @(*) begin
        case (size)
            2'b00: begin // 1字节
                selected_data = {24'b0, way_selected[offset*8 +: 8]};
            end
            2'b01: begin // 2字节
                selected_data = {16'b0, way_selected[offset*8 +: 16]};
            end
            2'b10: begin // 4字节
                selected_data = way_selected[offset*8 +: 32];
            end
            2'b11: begin // 8字节 (64位系统)
                selected_data = way_selected[offset*8 +: 32]; // 截断到32位输出
            end
        endcase
    end

endmodule
```

### 6.2 替换策略

```
LRU (Least Recently Used) 实现：

4路组相联的LRU需要3位：

│              │                                                   │
│        ┌─────┴─────┐                                             │
│        │           │                                             │
│    ┌───┴───┐   ┌───┴───┐                                         │
│    │ Bit 1 │   │ Bit 2 │                                         │
│    │       │   │       │                                         │
│    └───┬───┘   └───┬───┘                                         │
│        │           │                                             │
│    ┌───┴───┐   ┌───┴───┐                                         │
│    Way0   Way1 Way2  Way3                                        │
│                                                                 │
│  访问更新规则：                                                  │
│  - 访问Way0: Bit0=1, Bit1=1  (表示Way2-3最近未用，Way1最近未用)   │
│  - 访问Way1: Bit0=1, Bit1=0                                       │
│  - 访问Way2: Bit0=0, Bit2=1                                       │
│  - 访问Way3: Bit0=0, Bit2=0                                       │
│                                                                 │
│  替换选择：                                                      │
│  - 从根节点开始，每次选择LRU位指示的方向                          │
│  - 例如: Bit0=1 → 选择0分支 (Way0-1)                             │
│          Bit1=1 → 选择Way1                                        │
│                                                                 │
│  更新示例：                                                      │
│  初始: LRU = 3'b000 (Way0最近最少使用)                            │
│  访问Way2 → LRU更新为 3'b101 (Way3最近最少使用)                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

Verilog实现：

module plru_4way (
    input  wire        clk,
    input  wire        rst_n,
    input  wire        access,      // 有访问发生
    input  wire [1:0]  access_way,  // 访问的路 (0-3)
    output wire [1:0]  replace_way  // 建议替换的路
);

    reg [2:0] lru_bits;  // {bit0, bit1, bit2}

    // 计算替换路（根据当前LRU位）
    assign replace_way = lru_bits[0] ?
                            (lru_bits[1] ? 2'd1 : 2'd0) :
                            (lru_bits[2] ? 2'd3 : 2'd2);

    // 更新LRU位
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            lru_bits <= 3'b0;
        end else if (access) begin
            case (access_way)
                2'd0: lru_bits <= {1'b1, 1'b1, lru_bits[0]};  // 访问Way0
                2'd1: lru_bits <= {1'b1, 1'b0, lru_bits[0]};  // 访问Way1
                2'd2: lru_bits <= {1'b0, lru_bits[1], 1'b1};  // 访问Way2
                2'd3: lru_bits <= {1'b0, lru_bits[1], 1'b0};  // 访问Way3
            endcase
        end
    end

endmodule
```

### 7.2 C语言Cache优化

```c
// ============================================================
// Cache友好的C代码示例
// ============================================================

// 示例1: 数组访问顺序
#define N 1024
int A[N][N];

// Cache不友好的代码 (行优先 vs 列优先)
void bad_access() {
    // 列优先访问，Cache行利用率低
    for (int j = 0; j < N; j++) {
        for (int i = 0; i < N; i++) {
            A[i][j] = i + j;  // 每次访问间隔N×4字节
        }
    }
}

// Cache友好的代码
void good_access() {
    // 行优先访问，充分利用Cache行
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            A[i][j] = i + j;  // 连续访问，一次加载Cache行
        }
    }
}
/*
 * 性能差异：
 * - Cache行大小64字节，可存储16个int
 * - 行优先：每16次访问才产生1次Cache Miss
 * - 列优先：几乎每次访问都Miss
 * - 性能差距可达10-100倍
 */

// 示例2: 循环分块 (Loop Tiling)
void matrix_multiply_naive(int C[N][N], int A[N][N], int B[N][N]) {
    for (int i = 0; i < N; i++)
        for (int j = 0; j < N; j++)
            for (int k = 0; k < N; k++)
                C[i][j] += A[i][k] * B[k][j];
}

void matrix_multiply_tiled(int C[N][N], int A[N][N], int B[N][N]) {
    const int BLOCK = 64;  // 适应Cache大小
    for (int ii = 0; ii < N; ii += BLOCK)
        for (int jj = 0; jj < N; jj += BLOCK)
            for (int kk = 0; kk < N; kk += BLOCK)
                // 处理一个BLOCK×BLOCK的小块
                for (int i = ii; i < min(ii+BLOCK, N); i++)
                    for (int j = jj; j < min(jj+BLOCK, N); j++)
                        for (int k = kk; k < min(kk+BLOCK, N); k++)
                            C[i][j] += A[i][k] * B[k][j];
}
/*
 * 分块优势：
 * - 小块可以全部装入L1 Cache
 * - 减少Cache冲突
 * - 提高数据复用
 */

// 示例3: 数据结构对齐
struct bad_struct {
    char a;      // 1字节
    // 3字节填充
    int b;       // 4字节 (地址4对齐)
    char c;      // 1字节
    // 7字节填充
    double d;    // 8字节 (地址8对齐)
};  // 总大小: 24字节

struct good_struct {
    double d;    // 8字节
    int b;       // 4字节
    char a;      // 1字节
    char c;      // 1字节
    // 2字节填充
};  // 总大小: 16字节
/*
 * 优化效果：
 * - 减少内存占用
 * - 更好的Cache行对齐
 * - 可能的向量化优化
 */

// 示例4: 避免伪共享
struct alignas(64) padded_counter {  // C++17
    int value;
    char pad[60];  // 填充到64字节
};

padded_counter counters[4];  // 每个counter在不同Cache行

// 或者使用C语言方式
struct padded_counter_c {
    int value;
    char pad[60];
} __attribute__((aligned(64)));
/*
 * 伪共享消除：
 * - 确保多线程修改的不同变量在不同Cache行
 * - 避免不必要的Cache一致性流量
 */
```

```c
// MESI协议在多核编程中的体现

// 场景1: 只读共享 (E → S)
const int shared_data = 42;  // 所有核的Cache为E或S状态

void read_only_access() {

    // 第一次读取：E状态（独占）
    // 第二次读取：变为S状态（共享）
    int x = shared_data;
}

// 场景2: 写写冲突 (M → I)
int shared_counter = 0;

// 核0
void core0_increment() {
    shared_counter++;  // M状态，其他核的副本变为I
}

// 核1
void core1_increment() {
    shared_counter++;  // 需要先从核0获取最新值
}
/*
 * 性能问题：
 * - 频繁的M→I状态转换
 * - 大量Cache一致性流量
 * - 解决方案：使用线程本地变量或原子操作
 */

// 场景3: 使用原子操作优化
#include <stdatomic.h>

_Atomic int atomic_counter = 0;

void atomic_increment() {
    atomic_fetch_add(&atomic_counter, 1);
    // 硬件原子指令，配合Cache锁定
}

// 场景4: 内存屏障与Cache刷新
void producer_consumer() {
    int data = 0;
    volatile int flag = 0;

    // Producer (核0)
    data = 42;           // 写入数据
    __sync_synchronize(); // 内存屏障：确保写入顺序
    flag = 1;            // 设置标志

    // Consumer (核1)
    while (flag == 0);   // 等待标志
    __sync_synchronize(); // 内存屏障：确保读取顺序
    int value = data;    // 读取数据
}
/*
 * 内存屏障作用：
 * - 确保Store/Load顺序
 * - 触发必要的Cache刷新/失效
 * - 保证多核间可见性
```

## 8. 完整Verilog实现

### 8.1 完整Cache模块

```verilog
// ============================================================
// 4KB直接映射Cache (简化版，用于理解原理)
// ============================================================

module direct_mapped_cache #(
    parameter ADDR_WIDTH = 32,
    parameter DATA_WIDTH = 32,
    parameter CACHE_SIZE = 4096,        // 4KB
    parameter LINE_SIZE = 32,           // 32字节/行
    parameter NUM_LINES = CACHE_SIZE / LINE_SIZE  // 128行
)(
    input  wire                    clk,
    input  wire                    rst_n,

    // CPU接口
    input  wire                    cpu_req,
    input  wire                    cpu_we,
    input  wire [ADDR_WIDTH-1:0]   cpu_addr,
    input  wire [DATA_WIDTH-1:0]   cpu_wdata,
    output reg  [DATA_WIDTH-1:0]   cpu_rdata,
    output reg                     cpu_ready,

    // 内存接口
    output reg                     mem_req,
    output reg                     mem_we,
    output reg  [ADDR_WIDTH-1:0]   mem_addr,
    output reg  [LINE_SIZE*8-1:0]  mem_wdata,
    input  wire [LINE_SIZE*8-1:0]  mem_rdata,
    input  wire                    mem_ready
);

    // 地址分解
    localparam OFFSET_BITS = $clog2(LINE_SIZE);     // 5位 (32字节)
    localparam INDEX_BITS = $clog2(NUM_LINES);      // 7位 (128行)
    localparam TAG_BITS = ADDR_WIDTH - OFFSET_BITS - INDEX_BITS;  // 20位

    wire [TAG_BITS-1:0]     tag = cpu_addr[ADDR_WIDTH-1:ADDR_WIDTH-TAG_BITS];
    wire [INDEX_BITS-1:0]   index = cpu_addr[ADDR_WIDTH-TAG_BITS-1:OFFSET_BITS];
    wire [OFFSET_BITS-1:0]  offset = cpu_addr[OFFSET_BITS-1:0];

    // Cache存储
    reg [TAG_BITS-1:0]      tag_array [0:NUM_LINES-1];
    reg [LINE_SIZE*8-1:0]   data_array [0:NUM_LINES-1];
    reg                     valid_array [0:NUM_LINES-1];
    reg                     dirty_array [0:NUM_LINES-1];

    // 命中检测
    wire hit = valid_array[index] && (tag_array[index] == tag);

    // 状态机
    localparam IDLE = 2'b00;
    localparam WRITE_BACK = 2'b01;
    localparam ALLOCATE = 2'b10;
    reg [1:0] state;

    integer i;

    // 状态机和Cache控制
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state <= IDLE;
            cpu_ready <= 1'b0;
            mem_req <= 1'b0;
            for (i = 0; i < NUM_LINES; i = i + 1) begin
                valid_array[i] <= 1'b0;
                dirty_array[i] <= 1'b0;
            end
        end else begin
            case (state)
                IDLE: begin
                    cpu_ready <= 1'b0;
                    if (cpu_req) begin
                        if (hit) begin
                            // 命中处理
                            if (cpu_we) begin
                                // 写命中：更新数据，设置Dirty
                                data_array[index][offset*8 +: 32] <= cpu_wdata;
                                dirty_array[index] <= 1'b1;
                            end else begin
                                // 读命中：返回数据
                                cpu_rdata <= data_array[index][offset*8 +: 32];
                            end
                            cpu_ready <= 1'b1;
                        end else begin
                            // 未命中
                            if (valid_array[index] && dirty_array[index]) begin
                                // 需要写回
                                state <= WRITE_BACK;
                                mem_req <= 1'b1;
                                mem_we <= 1'b1;
                                mem_addr <= {tag_array[index], index, {OFFSET_BITS{1'b0}}};
                                mem_wdata <= data_array[index];
                            end else begin
                                // 直接分配
                                state <= ALLOCATE;
                                mem_req <= 1'b1;
                                mem_we <= 1'b0;
                                mem_addr <= {tag, index, {OFFSET_BITS{1'b0}}};
                            end
                        end
                    end
                end

                WRITE_BACK: begin
                    if (mem_ready) begin
                        mem_req <= 1'b0;
                        state <= ALLOCATE;
                        mem_req <= 1'b1;
                        mem_we <= 1'b0;
                        mem_addr <= {tag, index, {OFFSET_BITS{1'b0}}};
                    end
                end

                ALLOCATE: begin
                    if (mem_ready) begin
                        mem_req <= 1'b0;
                        // 填充新行
                        data_array[index] <= mem_rdata;
                        tag_array[index] <= tag;
                        valid_array[index] <= 1'b1;
                        dirty_array[index] <= cpu_we;  // 写分配时设置Dirty
                        state <= IDLE;
                        cpu_ready <= 1'b1;
                        if (!cpu_we) begin
                            cpu_rdata <= mem_rdata[offset*8 +: 32];
                        end
                    end
                end
            endcase
        end
    end

endmodule
```
