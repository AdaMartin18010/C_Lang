# Lab5: Cache控制器实现

> **实验**: 存储系统实验 - 实现一个可配置的Cache控制器，支持直接映射、组相联映射，掌握写回/写穿策略和LRU替换算法，理解存储层次结构的设计原理

## 概述

Cache是弥合CPU与主存速度差距的关键技术，对系统性能有着决定性影响。本实验通过实现一个完整的Cache控制器，深入理解存储层次结构的设计原理和优化方法。

学员将设计和实现Cache的核心功能，包括地址映射、标签比较、替换策略、写策略等。通过本实验，学员将理解为什么Cache能够提高系统性能，以及不同配置对性能的影响。

## 实验目标

- **理解Cache原理**：掌握局部性原理和Cache的工作机制
- **实现地址映射**：实现直接映射、组相联映射等地址映射方式
- **设计替换策略**：实现LRU(最近最少使用)等替换算法
- **实现写策略**：支持写回(Write-back)和写穿(Write-through)策略
- **处理一致性**：理解并处理Cache与主存之间的一致性问题
- **评估性能影响**：通过仿真评估不同Cache配置对性能的影响

## 实验步骤

1. **理论学习**：深入理解Cache的工作原理、局部性原理和性能指标
2. **参数定义**：确定Cache容量、块大小、相联度等关键参数
3. **架构设计**：设计Cache控制器的整体架构和接口定义
4. **直接映射实现**：先实现简单的直接映射Cache
5. **组相联扩展**：扩展到组相联结构，增加替换策略选择
6. **写策略实现**：实现写回和写穿两种写策略
7. **替换算法**：实现LRU替换算法(可使用近似LRU简化)
8. **功能验证**：设计测试用例验证各种访问场景
9. **性能评估**：使用内存访问轨迹评估命中率和缺失惩罚
10. **集成测试**：将Cache连接到CPU进行系统集成测试

## 预期成果

- 实现一个可配置(容量、相联度、块大小)的Cache控制器
- 支持直接映射和组相联两种映射方式
- 实现LRU替换算法和写回/写穿策略
- 通过测试验证Cache功能的正确性
- 能够分析并报告命中率、缺失率等性能指标
- 理解Cache设计中的各种权衡(容量vs延迟、相联度vs复杂度)
