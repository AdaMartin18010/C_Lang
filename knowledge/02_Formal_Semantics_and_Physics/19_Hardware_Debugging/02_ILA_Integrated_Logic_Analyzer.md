## 1. ILA 概述

### 1.1 什么是 ILA

**ILA（Integrated Logic Analyzer，集成逻辑分析仪）** 是 Xilinx Vivado 设计套件中提供的片上调试工具。
它允许工程师在 FPGA 运行时捕获和分析内部信号，无需外部测试设备。

## 2. ILA 核心概念

### 2.1 ILA 核组件

#### 2.1.1 探针端口（Probe Port）

```
┌─────────────────────────────────────────────────────────────────┐
│                   ILA 探针端口详解                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  探针端口类型：                                                  │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  1. 触发端口 (TRIG)                                       │ │
│  │     ┌───────────────────────────────────────────────┐    │ │
│  │     │ 用途: 设置触发条件                            │    │ │
│  │     │                                               │    │ │
│  │     │ 特性:                                         │    │ │
│  │     │ • 支持边缘检测 (上升/下降/双边)               │    │ │
│  │     │ • 支持数值比较 (=, ≠, <, >)                   │    │ │
│  │     │ • 支持布尔逻辑组合                            │    │ │
│  │     │                                               │    │ │
│  │     │ 使用建议:                                     │    │ │
│  │     │ • 连接状态信号                                │    │ │
│  │     │ • 连接控制信号                                │    │ │
│  │     │ • 连接错误标志                                │    │ │
│  │     └───────────────────────────────────────────────┘    │ │
│  │                                                           │ │
│  │  2. 数据端口 (DATA)                                       │ │
│  │     ┌───────────────────────────────────────────────┐    │ │
│  │     │ 用途: 捕获和记录信号值                        │    │ │
│  │     │                                               │    │ │
│  │     │ 特性:                                         │    │ │
│  │     │ • 只记录数据，不触发                          │    │ │
│  │     │ • 占用 BRAM 存储                              │    │ │
│  │     │ • 可以捕获总线数据                            │    │ │
│  │     │                                               │    │ │
│  │     │ 使用建议:                                     │    │ │
│  │     │ • 连接数据总线                                │    │ │
│  │     │ • 连接中间计算结果                            │    │ │
│  │     │ • 捕获不频繁但重要的信号                      │    │ │
│  │     └───────────────────────────────────────────────┘    │ │
│  │                                                           │ │
│  │  3. 触发/数据端口 (TRIG_DATA)                              │ │

│                                                                 │
```

#### 2.2.1 基本采集模式

```
┌─────────────────────────────────────────────────────────────────┐
│                   ILA 采集模式详解                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │

│  │                                                           │ │

│  │                                                           │ │

│  │              [B]             [F]             [J]        │ │
│  │                                                           │ │
│  │   特点: 仅在条件满足时采样                                │ │

│  │              [B] [C] [D] [E] [F] [G]                    │ │
│  │                                                           │ │
│  │   特点: 窗口期间连续采样                                  │ │
```

#### 2.2.2 高级采集特性

```
┌─────────────────────────────────────────────────────────────────┐
│                 ILA 高级采集特性                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 分段存储 (Segmented Storage)                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  传统模式:  连续存储，触发后覆盖                            │ │
│  │  ┌──────────────────────────────────────────────────┐    │ │
│  │  │ [S0][S1][S2]...[Sn-1]← 触发 →[Sn][Sn+1]...[Sm] │    │ │
│  │  └──────────────────────────────────────────────────┘    │ │
│  │                                                           │ │
│  │  分段模式:  多个独立缓冲区                                  │ │
│  │  ┌──────────┬──────────┬──────────┬──────────┐          │ │
│  │  │ Segment0 │ Segment1 │ Segment2 │ Segment3 │          │ │
│  │  │ [T0数据] │ [T1数据] │ [T2数据] │ [T3数据] │          │ │
│  │  └──────────┴──────────┴──────────┴──────────┘          │ │
│  │                                                           │ │
│  │  用途: 捕获多个触发事件，不丢失中间事件                   │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  2. 数据压缩 (Data Compression)                                  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  原始数据:  重复值占用大量存储                              │ │
│  │  [AA][AA][AA][AA][BB][BB][BB][CC][CC][CC][CC][CC]...     │ │
│  │                                                           │ │
│  │  压缩数据:  仅存储变化                                      │ │
│  │  [AA][cnt=3][BB][cnt=2][CC][cnt=4]...                     │ │
│  │                                                           │ │
│  │  优势: 相同数据场景下深度可增加 10-100 倍                  │ │
│  │  限制: 消耗额外逻辑资源，不适合高频变化信号               │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  3. 触发后采集 (Post-trigger Capture)                            │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  存储分配:                                                │ │
│  │  ┌──────────────────────────────────────────────────────┐ │ │
│  │  │  预触发   │   触发点   │   后触发                    │ │ │
│  │  │  (20%)   │    ↓      │   (80%)                     │ │ │
│  │  │←───────→│←────────→│←────────────────────────→│  │ │
│  │  │ 1024    │  事件     │    4096 个样本            │  │ │
│  │  │ 个样本  │          │                           │  │ │
│  │  └──────────────────────────────────────────────────────┘ │ │
│  │                                                           │ │
│  │  用途: 观察触发后系统响应                                 │ │
│  │  场景: 错误发生后查看后续处理                             │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 探针压缩与扩展

#### 4.3.1 位压缩技术

```verilog
// ============================================================
// 探针位压缩技术 - 减少资源使用
// ============================================================

// 场景: 需要监控 256-bit 数据总线，但资源有限

// 方法1: 只监控部分位
ila_compressed u_ila_comp (
    .clk(clk),
    // 只监控高、中、低各8位，共24位
    .probe0({data_bus[255:248], data_bus[127:120], data_bus[7:0]})
);

// 方法2: 使用校验和压缩
reg [31:0] data_checksum;
always @(posedge clk) begin
    if (data_valid) begin
        data_checksum <= data_checksum ^ data_bus[255:224]
                                     ^ data_bus[223:192]
                                     ^ data_bus[191:160]
                                     ^ data_bus[159:128]
                                     ^ data_bus[127:96]
                                     ^ data_bus[95:64]
                                     ^ data_bus[63:32]
                                     ^ data_bus[31:0];
    end
end

ila_checksum u_ila_checksum (
    .clk(clk),
    .probe0(data_checksum),     // 32-bit 代替 256-bit
    .probe1(data_valid),
    .probe2(expected_checksum)  // 期望校验值
);

// 方法3: 时分复用 (只适用于低速场景)
reg [2:0] mux_sel;
reg [31:0] muxed_data;
always @(*) begin
    case (mux_sel)
        3'd0: muxed_data = data_bus[31:0];
        3'd1: muxed_data = data_bus[63:32];
        3'd2: muxed_data = data_bus[95:64];
        3'd3: muxed_data = data_bus[127:96];
        3'd4: muxed_data = data_bus[159:128];
        3'd5: muxed_data = data_bus[191:160];
        3'd6: muxed_data = data_bus[223:192];
        3'd7: muxed_data = data_bus[255:224];
    endcase
end

ila_tdm u_ila_tdm (
    .clk(clk),
    .probe0(muxed_data),
    .probe1(mux_sel),       // 需要同时捕获选择信号
    .probe2(data_valid)
);
// 捕获后在软件端重新组合数据
```

### 7.2 长时间监控

#### 7.2.1 数据导出与分析

```tcl
# ============================================================
# ILA 数据导出 Tcl 脚本
# ============================================================

# 1. 导出波形数据到 CSV
proc export_ila_to_csv {ila_handle filename} {
    # 获取波形数据
    set waveform_data [get_waveform_data $ila_handle]

    # 打开文件
    set fp [open $filename w]

    # 写入表头
    puts $fp "Sample,Time,probe0,probe1,probe2,probe3"

    # 写入数据
    set sample 0
    foreach datapoint $waveform_data {
        set time [lindex $datapoint 0]
        set p0 [lindex $datapoint 1]
        set p1 [lindex $datapoint 2]
        set p2 [lindex $datapoint 3]
        set p3 [lindex $datapoint 4]

        puts $fp "$sample,$time,$p0,$p1,$p2,$p3"
        incr sample
    }

    close $fp
    puts "数据已导出到 $filename"
}

# 2. 自动连续采集
proc continuous_capture {ila_handle num_captures} {
    for {set i 0} {$i < $num_captures} {incr i} {
        # 设置触发
        set_property TRIGGER_CONDITION "probe0 = X\"05\"" $ila_handle

        # 启动触发
        run_hw_ila $ila_handle

        # 等待触发完成
        wait_on_hw_ila $ila_handle

        # 上传数据
        upload_hw_ila_data $ila_handle

        # 导出数据
        set filename "capture_[format %04d $i].csv"
        export_ila_to_csv $ila_handle $filename

        puts "完成采集 $i / $num_captures"
    }
}

# 使用示例
set ila [get_hw_ilas -of_objects [current_hw_device]]
continuous_capture $ila 100
```

```
│  │      // Probe2: 写响应通道                                │ │
│  │      .probe2({bvalid, bready, bresp, bid[3:0]}),         │ │
│  │                                                           │ │
│  │      // Probe3: 内部状态                                  │ │
│  │      .probe3({wr_state, fifo_cnt, timeout_cnt})          │ │
│  │  );                                                       │ │
│  │                                                           │ │
│  │  // 触发条件: bresp != 0 (OKAY)                           │ │
│  │  // 在 Vivado 中设置: probe2[3:2] != 0                    │ │
│  │                                                           │ │
│  │  endmodule                                                │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【问题定位】                                                    │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  波形分析发现:                                            │ │
│  │  1. 最后一个数据 beat 之后，WVALID 提前拉低                │ │
│  │  2. 但 WLAST 没有在正确时机拉高                            │ │
│  │  3. 导致从设备认为传输未完成，超时后返回错误                │ │
│  │                                                           │ │
│  │  根因: 写状态机在处理非连续地址时 WLAST 生成逻辑有bug       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 9. 性能优化

### 9.1 资源优化

```
┌─────────────────────────────────────────────────────────────────┐
│                   ILA 资源使用优化                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【优化策略1：合理选择采样深度】                                  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  深度选择原则:                                            │ │
│  │  • 调试初期: 大深度 (8192+)，捕获更多信息                 │ │
│  │  • 问题定位后: 减小深度，节省资源                         │ │
│  │  • 生产版本: 移除 ILA 或最小配置                          │ │
│  │                                                           │ │
│  │  深度 vs 资源 (128-bit 探针):                             │ │
│  │  ┌───────────┬──────────────┬─────────────────────────┐  │ │
│  │  │ 深度      │ BRAM 18K     │ 用途                    │  │ │
│  │  ├───────────┼──────────────┼─────────────────────────┤  │ │
│  │  │ 1024      │ 1            │ 简单问题                │  │ │
│  │  │ 4096      │ 4            │ 一般调试                │  │ │
│  │  │ 8192      │ 8            │ 复杂时序                │  │ │
│  │  │ 32768     │ 32           │ 长时间监控              │  │ │
│  │  └───────────┴──────────────┴─────────────────────────┘  │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【优化策略2：减少探针位宽】                                     │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  优化前: 直接连接完整总线                                  │ │
│  │  ila (.probe0(data_bus[255:0]));  // 256-bit              │ │
│  │                                                           │ │
│  │  优化后: 只监控关键位                                      │ │
│  │  ila (.probe0({                           // 40-bit        │ │
│  │      data_bus[255:240],  // 高字节 (重要标志)             │ │
│  │      data_bus[127:120],  // 中字节                        │ │
│  │      data_bus[7:0],      // 低字节                        │ │
│  │      valid,               // 有效标志                     │ │
│  │      error                // 错误标志                     │ │
│  │  }));                                                     │ │
│  │                                                           │ │
│  │  节省: 256-bit → 40-bit，节省 84% 资源                    │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【优化策略3：条件采集】                                         │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  启用 Capture Control，只在需要时采样                      │ │
│  │                                                           │ │
│  │  配置:                                                    │ │
│  │  • 勾选 "Enable Capture Control"                          │ │
│  │  • 设置 capture 条件: transaction_active                  │ │
│  │                                                           │ │
│  │  效果: 只捕获有效交易，相同存储深度下捕获更多有效信息       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 9.2 时序优化

```
┌─────────────────────────────────────────────────────────────────┐
│                   ILA 时序优化                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【流水线插入】                                                  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  问题: ILA 探针输入增加组合逻辑延迟                        │ │
│  │                                                           │ │
│  │  解决方案: 在探针前插入流水线寄存器                        │ │
│  │                                                           │ │
│  │  reg [31:0] data_bus_reg;                                 │ │
│  │  always @(posedge clk) begin                              │ │
│  │      data_bus_reg <= data_bus;  // 打一拍                 │ │
│  │  end                                                      │ │
│  │                                                           │ │
│  │  ila u_ila (                                              │ │
│  │      .clk(clk),                                           │ │
│  │      .probe0(data_bus_reg)  // 使用寄存后信号             │ │
│  │  );                                                       │ │
│  │                                                           │ │
│  │  注意: 波形会延迟一个周期，需要理解这一点                   │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【专用时钟域】                                                  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  问题: 高频时钟 (>300MHz) 上 ILA 可能无法满足时序          │ │
│  │                                                           │ │
│  │  解决方案: 使用分频时钟驱动 ILA                            │ │
│  │                                                           │ │
│  │  reg ila_clk;                                             │ │
│  │  always @(posedge high_speed_clk) begin                   │ │
│  │      ila_clk <= ~ila_clk;  // 2分频                       │ │
│  │  end                                                      │ │
│  │                                                           │ │
│  │  // 在时钟上升沿采样信号                                   │ │
│  │  reg [31:0] sampled_data;                                 │ │
│  │  always @(posedge high_speed_clk) begin                   │ │
│  │      if (ila_clk)  // 在ila_clk上升沿采样                 │ │
│  │          sampled_data <= high_speed_data;                 │ │
│  │  end                                                      │ │
│  │                                                           │ │
│  │  ila u_ila (                                              │ │
│  │      .clk(ila_clk),                                       │ │
│  │      .probe0(sampled_data)                                │ │
│  │  );                                                       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 10.1 连接与配置问题

```
┌─────────────────────────────────────────────────────────────────┐
│                   ILA 常见问题排查                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Q1: Hardware Manager 中看不到 ILA                              │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  可能原因及解决方案:                                      │ │
│  │                                                           │ │
│  │  1. 未正确生成 probes 文件 (.ltx)                         │ │

│  │                                                           │ │
│  │  2. 编程时未加载 probes 文件                              │ │
│  │     → Program Device 时确保选择了 .ltx 文件               │ │
│  │                                                           │ │
│  │  3. ILA 时钟未运行                                        │ │
│  │     → 检查 ILA 的 clk 输入是否正确连接且有时钟            │ │
│  │                                                           │ │
│  │  4. 复位信号问题                                          │ │
│  │     → 检查 ILA 是否处于复位状态                           │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q2: ILA 显示 "Waiting for Trigger" 但无法触发                  │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  排查步骤:                                                │ │
│  │                                                           │ │
│  │  1. 检查触发条件是否可达                                   │ │
│  │     → 尝试设置 Always 触发，看是否能采集数据              │ │
│  │                                                           │ │
│  │  2. 检查时钟域                                            │ │
│  │     → 确保探针信号和 ILA 使用相同时钟                     │ │
│  │                                                           │ │
│  │  3. 检查信号值                                            │ │
│  │     → 使用 Vivado 的 "Set Up Debug" 功能查看信号实时值    │ │
│  │                                                           │ │
│  │  4. 触发条件语法错误                                      │ │
│  │     → 检查数值格式是否正确 (X"...", D"...", B"...")       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q3: 波形显示数据全为 0 或全为 1                                │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  可能原因:                                                │ │
│  │                                                           │ │
│  │  1. 信号未连接或悬空                                       │ │
│  │     → 检查 RTL 中探针是否正确连接到信号                   │ │
│  │                                                           │ │
│  │  2. 信号被优化掉了                                        │ │
│  │     → 添加 (* DONT_TOUCH = "true" *) 属性保留信号         │ │
│  │     → 或使用 (* MARK_DEBUG = "true" *)                   │ │
│  │                                                           │ │
│  │  3. 时钟问题                                              │ │
│  │     → 确保被测信号和 ILA 使用相同时钟                     │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q4: 实现时出现时序违例 (Timing Violation)                      │

│  │                                                           │ │
│  │  解决方案:                                                │ │
│  │                                                           │ │
│  │  1. 降低 ILA 采样时钟频率                                  │ │
│  │                                                           │ │
│  │  2. 在探针信号前插入流水线                                 │ │
│  │                                                           │ │
│  │  3. 使用 "Input Pipeline Stages" 选项                      │ │
│  │     → 在 ILA IP 配置中增加流水线级数                      │ │
│  │                                                           │ │
│  │  4. 减少探针数量或位宽                                     │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
