## 1. SignalTap II 概述

### 1.1 什么是 SignalTap II

**SignalTap II** 是 Intel/Altera Quartus Prime 设计软件中集成的逻辑分析仪工具。作为业界广泛使用的 FPGA 片上调试解决方案，它允许设计者在系统全速运行时捕获和分析内部信号。

### 2.2 数据流与控制流

#### 2.2.1 数据捕获流程

```
┌─────────────────────────────────────────────────────────────────┐
│              SignalTap II 数据捕获流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  阶段1: 信号采样                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  待测信号 ───→ [触发比较器] ───→ 匹配结果                 │ │
│  │       │                                     │             │ │
│  │       └─────────────────────────────────────┘             │ │
│  │                   (同时采样数据)                           │ │
│  │                                                           │ │
│  │  采样时钟: 用户定义 (通常是系统时钟或分频时钟)              │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  阶段2: 触发检测                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  触发状态机:                                               │ │
│  │                                                           │ │
│  │  ┌────────┐    ┌────────┐    ┌────────┐                  │ │
│  │  │ Level0 │───→│ Level1 │───→│ Level2 │───→ ...          │ │
│  │  │ 等待   │    │ 条件1  │    │ 条件2  │                   │ │
│  │  └────────┘    └────────┘    └────────┘                  │ │
│  │       ↑                                     │              │ │
│  │       └─────────────────────────────────────┘              │ │
│  │              (条件不满足时重置)                             │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  阶段3: 数据存储                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  存储模式:                                                │ │
│  │                                                           │ │
│  │  连续模式:                                                │ │
│  │  ┌────────────────────────────────────────────────────┐  │ │
│  │  │ [数据0][数据1][数据2]...[触发][数据N]...[数据M]    │  │ │
│  │  │ ←───── 触发前 ──────→│←──────── 触发后 ────────→│  │ │
│  │  └────────────────────────────────────────────────────┘  │ │
│  │                                                           │ │
│  │  分段模式:                                                │ │
│  │  ┌────────────────────────────────────────────────────┐  │ │
│  │  │ [段0] [段1] [段2] ... [段N]                         │  │ │
│  │  │ 每段保存一次触发事件                                │  │ │
│  │  └────────────────────────────────────────────────────┘  │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  阶段4: 数据传输                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  JTAG 读取 ───→ 波形显示 ───→ 分析/导出                   │ │
│  │                                                           │ │
│  │  传输时间 ≈ 存储深度 × 探针数 / JTAG 带宽                 │ │
│  │  例: 8K 深度 × 32 探针 / 6 Mbps ≈ 43 秒                   │ │
│  │  (使用 USB-Blaster II 可大幅提速)                         │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. SignalTap II 配置与使用

### 3.1 创建 SignalTap II 文件

#### 3.1.1 创建步骤

```
┌─────────────────────────────────────────────────────────────────┐
│              SignalTap II 文件创建步骤                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【步骤1】打开 SignalTap II Logic Analyzer                      │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  Tools → SignalTap II Logic Analyzer                     │ │
│  │                                                           │ │
│  │  或点击工具栏图标: 🔍 SignalTap II                        │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【步骤2】创建新的 .stp 文件                                    │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  File → New → SignalTap II Logic Analyzer File           │ │
│  │                                                           │ │
│  │  或按 Ctrl+N 选择 "University Program VWF" 类型           │ │
│  │                                                           │ │
│  │  保存为: <project_name>.stp                               │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【步骤3】设置采样时钟                                          │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  在 "Signal Configuration" 面板:                          │ │
│  │                                                           │ │
│  │  ┌─────────────────────────────────────────────────────┐ │ │
│  │  │ Clock: [Browse...]                                  │ │ │
│  │  │       [  sys_clk  ]  ← 选择采样时钟                  │ │ │
│  │  │                                                     │ │ │
│  │  │ 点击 "Browse" 按钮，从 Node Finder 中选择            │ │ │
│  │  └─────────────────────────────────────────────────────┘ │ │
│  │                                                           │ │
│  │  采样时钟要求:                                            │ │
│  │  • 频率 ≥ 被测信号最高频率                                │ │
│  │  • 建议使用全局时钟网络                                   │ │
│  │  • 避免使用门控时钟                                       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  【步骤4】添加被测信号                                          │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  在 "Setup" 标签页:                                       │ │
│  │                                                           │ │
│  │  1. 双击空白处 或 右键 → Add Nodes                        │ │
│  │                                                           │ │

## 10. 常见问题

### 10.1 配置与连接问题

```
┌─────────────────────────────────────────────────────────────────┐
│              SignalTap II 常见问题排查                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Q1: 编译时提示 "SignalTap II instance not found"               │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  解决:                                                    │ │
│  │  1. 确保 .stp 文件已保存                                  │ │
│  │  2. 检查 Project Navigator 中 .stp 文件已添加到工程        │ │
│  │  3. Assignments → Settings → SignalTap II                │ │
│  │     确认 "Enable SignalTap II Logic Analyzer" 已勾选      │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q2: 下载后无法识别 SignalTap II 实例                           │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  排查:                                                    │ │
│  │  1. 检查编程电缆连接                                        │ │
│  │  2. 确保下载了正确的 .sof 文件                             │ │
│  │  3. 检查采样时钟是否运行 (时钟未运行则无法识别)            │ │
│  │  4. 尝试重新编译 (可能是信号被优化)                        │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q3: 波形显示信号全为0或全为1                                   │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  可能原因:                                                │ │
│  │  1. 信号被 Quartus 优化掉了                                │ │
│  │     → 添加 (* preserve *) 属性                           │ │
│  │                                                           │ │
│  │  2. 采样时钟与被测信号不在同一时钟域                       │ │
│  │     → 使用相同时钟域                                       │ │
│  │                                                           │ │
│  │  3. 信号未正确连接                                        │ │
│  │     → 检查 RTL 代码                                       │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q4: 触发条件无法达到                                          │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  排查:                                                    │ │
│  │  1. 检查触发条件设置是否正确                              │ │
│  │  2. 使用 "Auto Run" 连续捕获，观察信号实际值               │ │
│  │  3. 尝试简单的触发条件 (如 always true) 测试基本功能       │ │
│  │  4. 检查触发级数设置，多级触发需按顺序满足                 │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  Q5: 编译时间过长                                              │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │                                                           │ │
│  │  优化:                                                    │ │
│  │  1. 使用 Incremental Compilation (增量编译)               │ │
│  │     Processing → Start → Start Incremental Compilation   │ │
│  │                                                           │ │
│  │  2. 减少探针数量                                          │ │
│  │                                                           │ │
│  │  3. 减小采样深度 (调试完成后再增加)                        │ │
│  │                                                           │ │
│  │  4. 使用多个小实例代替一个大实例                           │ │
│  │                                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
