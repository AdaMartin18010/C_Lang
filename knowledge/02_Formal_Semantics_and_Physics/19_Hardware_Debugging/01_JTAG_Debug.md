## 1. JTAG 协议概述

### 1.1 JTAG 的起源与发展

**JTAG（Joint Test Action Group）** 是由 IEEE 1149.1 标准定义的一种硬件测试和调试接口。
它最初由联合测试行动小组于 1980 年代开发，旨在解决 PCB（印刷电路板）上集成电路的测试问题。

#### 1.1.1 JTAG 的历史背景

```
时间线：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1985年 - 联合测试行动小组（JTAG）成立
      ↓
1990年 - IEEE 1149.1-1990 标准发布
      ↓
1993年 - IEEE 1149.1a-1993 修订版
      ↓
1994年 - IEEE 1149.1b-1994 补充标准
      ↓
2001年 - IEEE 1149.1-2001 重大更新
      ↓
2013年 - IEEE 1149.1-2013 现代版本
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 1.1.2 JTAG 的核心优势

| 特性 | 传统测试方法 | JTAG 测试方法 |
|------|-------------|--------------|
| 物理探针 | 需要大量探针 | 仅需4-5个信号线 |
| PCB 可测试性 | 依赖测试点布局 | 芯片级边界扫描 |
| 调试能力 | 有限 | 完整的芯片内调试 |
| 成本 | 高（专用夹具） | 低（标准化接口） |
| 自动化程度 | 低 | 高（软件驱动） |

#### 1.2.1 标准 JTAG 引脚定义

#### 1.2.2 信号详细说明

**TCK（Test Clock）**

- 类型：输入
- 功能：JTAG 操作的时钟源
- 特性：
  - 上升沿采样 TMS 和 TDI
  - 下降沿驱动 TDO
  - 频率范围：通常 1-20 MHz
- 注意事项：
  - 时钟必须连续，不能间断
  - 建议使用时钟缓冲器增强驱动能力

**TMS（Test Mode Select）**

- 类型：输入
- 功能：控制 TAP 控制器状态机
- 特性：
  - 在 TCK 上升沿采样
  - 决定状态机转换方向
- 设计建议：
  - 上拉电阻 10kΩ（防止悬空）
  - 走线尽量短，减少干扰

**TDI（Test Data In）**

- 类型：输入
- 功能：串行数据输入
- 特性：
  - LSB 或 MSB 优先（取决于实现）
  - 在 TCK 上升沿采样
- 连接方式：
  - 菊花链连接多个芯片
  - 单个芯片直接连接调试器

**TDO（Test Data Out）**

- 类型：输出
- 功能：串行数据输出
- 特性：
  - 三态输出，高阻态当不活动
  - 在 TCK 下降沿驱动
- 注意事项：
  - 需要外部上拉（防止悬空）
  - 多芯片时连接到下一级 TDI

**TRST（Test Reset）- 可选**

- 类型：输入，低电平有效
- 功能：异步复位 TAP 控制器
- 建议：
  - 强烈建议使用
  - 上拉电阻确保默认高电平

### 1.3 JTAG 链路与菊花链

#### 1.3.1 单芯片 JTAG 连接

```
         调试器                                  目标芯片
    ┌──────────────┐                        ┌──────────────┐
    │              │        TCK ─────────→ │              │
    │   JTAG       │        TMS ─────────→ │    TAP       │
    │   接口       │        TDI ─────────→ │    控制器    │
    │              │←──────── TDO ─────────│              │
    │              │        TRST ─────────→│              │
    └──────────────┘                        └──────────────┘
```

#### 1.3.2 多芯片菊花链连接

```
         调试器
    ┌──────────────┐
    │              │        TCK
    │   JTAG       │──────────┬──────────┬──────────┐
    │   调试器     │          │          │          │
    │              │        TMS          │          │
    │              │──────────┼──────────┼──────────┤
    │              │          │          │          │
    │              │        TDI          │          │
    │              │────────→ │          │          │
    │              │          ↓          │          │
    │              │    ┌──────────┐     │          │
    │              │    │  芯片 A  │     │          │
    │              │    │  TAP     │     │          │
    │              │    └────┬─────┘     │          │
    │              │         │ TDO       │          │
    │              │         └──────→┐   │          │
    │              │                 ↓   ↓          │
    │              │            ┌──────────┐        │
    │              │            │  芯片 B  │        │
    │              │            │  TAP     │        │
    │              │            └────┬─────┘        │
    │              │                 │ TDO          │
    │              │                 └──────→┐      │
    │              │                         ↓      ↓
    │              │                    ┌──────────┐
    │              │                    │  芯片 C  │
    │              │                    │  TAP     │
    │              │                    └────┬─────┘
    │              │←────────────────────────┘ TDO
    └──────────────┘
```

### 2.2 状态详解

#### 2.2.1 复位相关状态

**Test-Logic-Reset（测试逻辑复位）**

- 状态编码：所有寄存器清零
- 进入方式：
  - TRST 引脚低电平（异步）
  - TMS 保持高电平至少 5 个 TCK 周期（同步）
- 功能：
  - 复位 TAP 控制器到已知状态
  - 选择 IDCODE 或 BYPASS 指令
- 退出：TMS=0 进入 Run-Test/Idle

```c
// TAP 复位伪代码
void tap_reset() {
    // 方法1：使用 TRST（如果可用）
    TRST_LOW();
    delay_us(100);
    TRST_HIGH();

    // 方法2：使用 TMS（通用方法）
    for (int i = 0; i < 5; i++) {
        TMS_HIGH();
        TCK_CYCLE();  // 产生一个TCK周期
    }
    TMS_LOW();
    TCK_CYCLE();  // 进入 Run-Test/Idle
}
```

#### 2.2.2 空闲状态

**Run-Test/Idle（运行测试/空闲）**

- 状态编码：1000
- 功能：
  - 指令执行状态（如 RUNBIST）
  - 空闲等待
- 特点：
  - 保持当前指令寄存器不变
  - 是大多数操作的起点和终点

#### 2.2.3 数据寄存器扫描路径

```
Select-DR-Scan → Capture-DR → Shift-DR → Exit1-DR → Update-DR
                      ↓           ↓          ↓
                   (捕获)      (移位)     (退出/暂停)
                      │           │          │
                      └───────────┴──────────┘
                              ↓
                    ┌─────────────────┐
                    │   Exit1-DR      │ TMS=1
                    │   [0011]        │────→ Exit2-DR
                    └─────────────────┘
```

**Capture-DR（捕获数据）**

- 功能：并行加载数据到选定的数据寄存器
- 时序：在 TCK 上升沿进入此状态时发生捕获
- 应用：
  - 边界扫描：捕获引脚状态
  - 旁路寄存器：加载固定值 0

**Shift-DR（移位数据）**

- 功能：串行移入/移出数据
- 数据流：
  - TDI → 寄存器 → TDO
  - 每个 TCK 周期移动一位
- TMS=0 保持在 Shift-DR
- TMS=1 进入 Exit1-DR

**Exit1-DR（退出1-DR）**

- 功能：决定是暂停还是完成扫描
- TMS=0 → Pause-DR（暂停）
- TMS=1 → Update-DR（更新）

**Pause-DR（暂停-DR）**

- 功能：暂时中断移位操作
- 应用：
  - 处理长时间操作
  - 与其他系统同步
- 恢复：TMS=1 → Exit2-DR

**Exit2-DR（退出2-DR）**

- 功能：决定是继续移位还是完成
- TMS=0 → Shift-DR（继续）
- TMS=1 → Update-DR（完成）

**Update-DR（更新-DR）**

- 功能：将移位寄存器内容锁存到并行输出
- 时序：在 TCK 下降沿发生更新
- 应用：
  - 边界扫描：更新引脚驱动值
  - 设备编程：应用编程数据

#### 2.2.4 指令寄存器扫描路径

```
Select-IR-Scan → Capture-IR → Shift-IR → Exit1-IR → Update-IR
                      ↓           ↓          ↓
                   (捕获)      (移位)     (退出/暂停)
```

指令寄存器扫描路径与数据寄存器类似，但操作对象是 IR（Instruction Register）。

#### 2.3.2 状态机控制代码示例

```c
// TAP 状态机控制实现

typedef enum {
    TEST_LOGIC_RESET = 0x0F,
    RUN_TEST_IDLE    = 0x00,
    SELECT_DR_SCAN   = 0x01,
    CAPTURE_DR       = 0x02,
    SHIFT_DR         = 0x03,
    EXIT1_DR         = 0x04,
    PAUSE_DR         = 0x05,
    EXIT2_DR         = 0x06,
    UPDATE_DR        = 0x07,
    SELECT_IR_SCAN   = 0x09,
    CAPTURE_IR       = 0x0A,
    SHIFT_IR         = 0x0B,
    EXIT1_IR         = 0x0C,
    PAUSE_IR         = 0x0D,
    EXIT2_IR         = 0x0E,
    UPDATE_IR        = 0x08
} tap_state_t;

// 状态转换表
static const uint8_t state_transition[16][2] = {
    // [当前状态][TMS=0/TMS=1] = 下一状态
    /* RUN_TEST_IDLE    [00] */ {RUN_TEST_IDLE,    SELECT_DR_SCAN},
    /* SELECT_DR_SCAN   [01] */ {CAPTURE_DR,       SELECT_IR_SCAN},
    /* CAPTURE_DR       [02] */ {SHIFT_DR,         EXIT1_DR},
    /* SHIFT_DR         [03] */ {SHIFT_DR,         EXIT1_DR},
    /* EXIT1_DR         [04] */ {PAUSE_DR,         UPDATE_DR},
    /* PAUSE_DR         [05] */ {PAUSE_DR,         EXIT2_DR},
    /* EXIT2_DR         [06] */ {SHIFT_DR,         UPDATE_DR},
    /* UPDATE_DR        [07] */ {RUN_TEST_IDLE,    SELECT_DR_SCAN},
    /* UPDATE_IR        [08] */ {RUN_TEST_IDLE,    SELECT_DR_SCAN},
    /* SELECT_IR_SCAN   [09] */ {CAPTURE_IR,       TEST_LOGIC_RESET},
    /* CAPTURE_IR       [0A] */ {SHIFT_IR,         EXIT1_IR},
    /* SHIFT_IR         [0B] */ {SHIFT_IR,         EXIT1_IR},
    /* EXIT1_IR         [0C] */ {PAUSE_IR,         UPDATE_IR},
    /* PAUSE_IR         [0D] */ {PAUSE_IR,         EXIT2_IR},
    /* EXIT2_IR         [0E] */ {SHIFT_IR,         UPDATE_IR},
    /* TEST_LOGIC_RESET [0F] */ {RUN_TEST_IDLE,    TEST_LOGIC_RESET}
};

// TAP 时钟周期
void tap_cycle(int tms, int *tdo) {
    set_tms(tms);
    set_tdi(*tdo);

    delay_ns(TCK_HALF_PERIOD);  // 半周期延迟
    set_tck(1);                 // TCK 上升沿
    delay_ns(TCK_HALF_PERIOD);

    *tdo = get_tdo();           // 采样 TDO

    set_tck(0);                 // TCK 下降沿
}

// 状态转移函数
void tap_goto_state(tap_state_t target_state) {
    static tap_state_t current_state = TEST_LOGIC_RESET;

    while (current_state != target_state) {
        // 计算需要的 TMS 值
        int tms = (state_transition[current_state][0] == target_state) ? 0 : 1;

        // 如果两个路径都不到达目标，选择更接近的路径
        if (state_transition[current_state][0] != target_state &&
            state_transition[current_state][1] != target_state) {
            // 使用启发式选择路径
            tms = calculate_best_path(current_state, target_state);
        }

        int dummy_tdo;
        tap_cycle(tms, &dummy_tdo);
        current_state = state_transition[current_state][tms];
    }
}
```

#### 4.2.2 互连测试算法

```c
/**
 * 互连测试算法 -  walking ones/zeros
 * 用于检测开路和短路故障
 */

// Walking Ones 测试向量生成
void generate_walking_ones_vectors(uint32_t *vectors, int num_nets) {
    for (int i = 0; i < num_nets; i++) {
        vectors[i] = (1U << i);  // 只有一个1，其余为0
    }
}

// Walking Zeros 测试向量生成
void generate_walking_zeros_vectors(uint32_t *vectors, int num_nets) {
    for (int i = 0; i < num_nets; i++) {
        vectors[i] = ~(1U << i);  // 只有一个0，其余为1
    }
}

// 完整互连测试函数
int interconnect_test(jtag_chain_t *chain,
                      netlist_t *netlist,
                      test_report_t *report) {
    int num_nets = netlist->num_nets;
    uint32_t *drive_vectors = malloc(num_nets * sizeof(uint32_t));
    uint32_t *expected = malloc(num_nets * sizeof(uint32_t));
    uint32_t *captured = malloc(num_nets * sizeof(uint32_t));

    // 生成测试向量
    generate_walking_ones_vectors(drive_vectors, num_nets);

    // 加载 EXTEST 指令到所有芯片
    jtag_ir_scan(chain, EXTEST_OPCODE, NULL);

    // 执行测试
    for (int test_num = 0; test_num < num_nets; test_num++) {
        // 预加载驱动值
        jtag_dr_scan(chain, drive_vectors[test_num], NULL);

        // 应用测试向量（Update-DR 自动完成）
        jtag_run_idle(chain, 1);

        // 捕获响应
        jtag_dr_scan(chain, 0, captured);

        // 计算期望响应
        expected[test_num] = calculate_expected_response(
            netlist, drive_vectors[test_num]
        );

        // 验证结果
        if (captured[test_num] != expected[test_num]) {
            report_fault(report, test_num,
                        drive_vectors[test_num],
                        expected[test_num],
                        captured[test_num]);
        }
    }

    free(drive_vectors);
    free(expected);
    free(captured);

    return report->num_faults;
}
```

```tcl
proc memory_test {addr size} {
    echo "测试内存区域: 0x[format %08X $addr] - 0x[format %08X [expr $addr + $size]]"

    # 写测试模式
    set patterns {0x55555555 0xAAAAAAAA 0xFFFFFFFF 0x00000000}

    foreach pattern $patterns {
        echo "写入模式: 0x[format %08X $pattern]"

        # 填充内存
        for {set i 0} {$i < $size} {incr i 4} {
            mww [expr $addr + $i] $pattern
        }

        # 验证
        set errors 0
        for {set i 0} {$i < $size} {incr i 4} {
            set value [expr [mdw [expr $addr + $i] 1] & 0xFFFFFFFF]
            if {$value != $pattern} {
                echo "错误在地址 0x[format %08X [expr $addr + $i]]: 期望 0x[format %08X $pattern], 读取 0x[format %08X $value]"
                incr errors
                if {$errors > 10} {
                    echo "错误太多，停止测试"
                    break
                }
            }
        }

        if {$errors == 0} {
            echo "模式 0x[format %08X $pattern] 测试通过"
        }
    }

    echo "内存测试完成"
}
```

```python
### debug_script.py - 自动化调试脚本

import gdb

class AutomatedDebug(gdb.Command):
    """自动化调试命令"""

    def __init__(self):
        super().__init__("auto-debug", gdb.COMMAND_USER)

    def invoke(self, arg, from_tty):
        # 1. 加载程序
        gdb.execute("file firmware.elf")

        # 2. 连接目标
        gdb.execute("target remote localhost:3333")

        # 3. 复位并暂停

        # 4. 下载程序

        # 5. 设置断点
        breakpoints = [
            "main",
            "Error_Handler",
            "HardFault_Handler"
        ]
        for bp in breakpoints:
```

#### 7.2.2 HardFault 处理脚本

```c
// hardfault_handler.c
// HardFault 处理与信息收集

typedef struct {
    uint32_t r0;
    uint32_t r1;
    uint32_t r2;
    uint32_t r3;
    uint32_t r12;
    uint32_t lr;
    uint32_t pc;
    uint32_t psr;
} HardFault_StackFrame_t;

typedef struct {
    uint32_t cfsr;
    uint32_t hfsr;
    uint32_t mmfar;
    uint32_t bfar;
    uint32_t afsr;
    HardFault_StackFrame_t stack_frame;
} HardFault_Info_t;

volatile HardFault_Info_t g_hardfault_info;

__attribute__((naked)) void HardFault_Handler(void) {
    __asm volatile (
        "tst lr, #4             \n"  // 检查 EXC_RETURN 位2
        "ite eq                 \n"
        "mrseq r0, msp          \n"  // 使用 MSP
        "mrsne r0, psp          \n"  // 使用 PSP
        "b HardFault_Handler_C  \n"  // 跳转到 C 处理函数
    );
}

void HardFault_Handler_C(uint32_t *stack_frame) {
    // 保存栈帧
    g_hardfault_info.stack_frame.r0  = stack_frame[0];
    g_hardfault_info.stack_frame.r1  = stack_frame[1];
    g_hardfault_info.stack_frame.r2  = stack_frame[2];
    g_hardfault_info.stack_frame.r3  = stack_frame[3];
    g_hardfault_info.stack_frame.r12 = stack_frame[4];
    g_hardfault_info.stack_frame.lr  = stack_frame[5];
    g_hardfault_info.stack_frame.pc  = stack_frame[6];
    g_hardfault_info.stack_frame.psr = stack_frame[7];

    // 保存故障状态寄存器
    g_hardfault_info.cfsr  = *(volatile uint32_t *)0xE000ED28;
    g_hardfault_info.hfsr  = *(volatile uint32_t *)0xE000ED2C;
    g_hardfault_info.mmfar = *(volatile uint32_t *)0xE000ED34;
    g_hardfault_info.bfar  = *(volatile uint32_t *)0xE000ED38;
    g_hardfault_info.afsr  = *(volatile uint32_t *)0xE000ED3C;

    // 这里可以添加断点或 LED 指示
    __asm volatile ("bkpt #0");

    // 无限循环等待调试
    while (1) {
        // 保持在此处便于调试器连接
    }
}
```

```c
// mpu_config.c
// 使用 MPU 检测内存越界

#include <stdint.h>

// MPU 区域配置
void MPU_Config(void) {
    // 禁用 MPU
    __asm volatile ("dsb");
    MPU->CTRL = 0;

    // 区域0: Flash (只读)
    MPU->RNR = 0;
    MPU->RBAR = 0x08000000 | 0x10;  // 地址 + 有效位
    MPU->RASR = (0x11 << 1) |       // 大小: 512KB (2^(17+1))
                (0x06 << 24) |      // 访问权限: 只读
                (1 << 0);           // 使能

    // 区域1: RAM (读写)
    MPU->RNR = 1;
    MPU->RBAR = 0x20000000 | 0x10;
    MPU->RASR = (0x0E << 1) |       // 大小: 128KB
                (0x03 << 24) |      // 访问权限: 读写
                (1 << 0);

    // 区域2: 栈保护区（红区）
    MPU->RNR = 2;
    MPU->RBAR = 0x2001F000 | 0x10;  // 栈底部
    MPU->RASR = (0x04 << 1) |       // 大小: 4KB
                (0x00 << 24) |      // 访问权限: 无
                (1 << 0);

    // 使能 MPU
    MPU->CTRL = 1 |                 // 使能 MPU
                (1 << 2);           // 使能 HardFault 对内存故障
    __asm volatile ("dsb");
    __asm volatile ("isb");
}
```
