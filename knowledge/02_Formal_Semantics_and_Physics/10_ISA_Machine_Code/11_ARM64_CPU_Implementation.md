## 1. ARM64架构特点回顾

### 1.1 架构演进历史

ARM64 (AArch64) 是ARM Holdings于2011年推出的64位指令集架构，首次应用于ARMv8-A处理器。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARM架构演进历程                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1985-1990: 早期ARM (ARMv1-v3)                                               │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  ARM1 (1985) - 首个ARM处理器                                         │  │
│   │  • 26位地址总线                                                      │  │
│   │  • 简单三级流水线                                                    │  │
│   │                                                                      │  │
│   │  ARM3 (1989) - 首次集成缓存                                          │  │
│   │  • 4KB统一缓存                                                       │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                              │                                              │
│                              ▼                                              │
│   1990-2000: 经典ARM (ARMv4-v5)                                               │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  ARM7TDMI (1994) - 最成功的嵌入式处理器                              │  │
│   │  • Thumb指令集 (16位压缩)                                            │  │
│   │  • 32位地址总线                                                      │  │
│   │                                                                      │  │
│   │  ARM9 (1998) - 五级流水线                                            │  │
│   │  • Harvard架构                                                       │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                              │                                              │
│                              ▼                                              │
│   2000-2011: 现代ARM32 (ARMv6-v7)                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  ARM11 (2002) - ARMv6架构                                            │  │
│   │  • SIMD指令 (NEON前身)                                               │  │
│   │  • TrustZone安全扩展                                                 │  │
│   │                                                                      │  │
│   │  Cortex-A8/A9 (2005-2007) - ARMv7架构                                │  │
│   │  • 超标量执行                                                        │  │
│   │  • NEON SIMD单元                                                     │  │
│   │  • 32位架构巅峰                                                      │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                              │                                              │
│                              ▼                                              │

│   │  • 64位通用寄存器 (X0-X30)                                           │  │
│   │  • 全新指令编码                                                      │  │
│   │  • 条件执行简化                                                      │  │
│   │                                                                      │  │
│   │  ARMv9-A (2021) - 新一代架构                                         │  │
```

### 1.2 与ARM32的关键差异

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARM64 vs ARM32 关键差异                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌────────────────────┬─────────────────────┬────────────────────────────┐ │
│  │     特性           │     ARM32           │        ARM64               │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 通用寄存器         │ 16个 (R0-R15)       │ 31个 (X0-X30) + SP         │ │
│  │                    │ R15=PC, R14=LR      │ 独立PC, 64位宽度           │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 条件执行           │ 几乎所有指令        │ 仅分支和条件选择           │ │
│  │                    │ (ARM状态)           │ (B.cond, CSEL等)           │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 指令编码           │ 32位 (ARM) /        │ 固定32位 (A64)             │ │
│  │                    │ 16位 (Thumb)        │                            │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 加载/存储多寄存器  │ LDM/STM (复杂)      │ LDP/STP (成对，简化)       │ │
│  │                    │                     │                            │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 立即数处理         │ 复杂桶形移位        │ 简洁立即数编码             │ │
│  │                    │                     │ (12位 + 2位移位)           │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 异常模型           │ 7种模式             │ 4个异常级别 (EL0-EL3)      │ │
│  │                    │ (USR, FIQ, IRQ...)  │ 更简洁、更安全             │ │
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 寻址模式           │ 多种复杂寻址        │ 简化但功能完整             │ │
│  │                    │ (回写、偏移等)      │ (基址+变址)                │ │
│  └────────────────────┴─────────────────────┴────────────────────────────┘ │
│                                                                             │
│  设计哲学转变:                                                               │
│  • ARM32: 极致代码密度 + 灵活性 (条件执行、复杂寻址)                         │
│  • ARM64: 简化硬件实现 + 高性能 (规则化指令、更大寄存器堆)                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │

│  ├────────────────────┼─────────────────────┼────────────────────────────┤ │
│  │ 生态成熟度         │ 非常成熟            │ 快速发展中                 │ │

│  └────────────────────┴─────────────────────┴────────────────────────────┘ │
│                                                                             │

│  • RISC-V: 学术研究出身，极简设计，模块化扩展，开放生态                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. 指令集架构详解

### 2.1 指令格式分类

ARM64 (A64) 指令集采用统一的32位定长编码，主要指令格式如下：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARM64 指令格式分类                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 数据处理 - 立即数 (Data Processing - Immediate)                           │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │ 31 30 29 28        24 23 22 21        10 9         5 4        0      │  │
│  │ ├─┼─┼─┼──────────┼─┼─┼─┼────────────┼──────────┼──────────┤        │  │
│  │ │s│f│1│0│0│0│1│0│  sh  │      imm12       │    Rn    │   Rd   │        │  │
│  │ └─┴─┴─┴──────────┴─┴─┴─┴──────────────────┴──────────┴──────────┘        │  │
│  │                                                                     │  │
│  │  s = 1: 更新标志位  f = 0:32位/1:64位  sh = 左移(0/12)               │  │
│  │  示例: ADDI X0, X1, #0x123     // X0 = X1 + 0x123                   │  │
│  │        ADDIS X0, X1, #0x1, LSL #12  // X0 = X1 + 0x1000, 更新标志   │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  2. 数据处理 - 寄存器 (Data Processing - Register)                            │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │ 31 30 29 28 27 26 25 24 21 20 16 15       10 9    5 4        0      │  │
│  │ ├─┼─┼─┼─┼─┼─┼─┼─┼────┼─┼────┼──────────┼─────┼──────┤        │  │
│  │ │s│f│0│1│0│1│0│ shift  │0│ Rm │  imm6   │  Rn  │  Rd  │        │  │
│  │ └─┴─┴─┴─┴─┴─┴─┴─┴────┴─┴────┴──────────┴─────┴──────┘        │  │
│  │                                                                     │  │
│  │  shift = 00:LSL 01:LSR 10:ASR 11:ROR                              │  │
│  │  示例: ADD X0, X1, X2, LSL #3   // X0 = X1 + (X2 << 3)            │  │
│  │        SUB X0, X1, X2, ASR #5   // X0 = X1 - (X2 >>> 5)           │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  3. 加载/存储 (Load/Store)                                                    │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │ 31 30 29 28        24 23 22 21        10 9         5 4        0      │  │
│  │ ├─┼─┼─┼──────────┼─┼─┼─┼────────────┼──────────┼──────────┤        │  │
│  │ │s│f│1│1│1│0│0│0│ 0│0│    imm9      │ 0│0│   Rn   │   Rt   │        │  │
│  │ └─┴─┴─┴──────────┴─┴─┴─┴────────────┴──────────┴──────────┘        │  │
│  │                                                                     │  │
│  │  示例: LDR X0, [X1, #8]         // X0 = MEM[X1 + 8]                 │  │
│  │        STR X0, [X1, #-4]!       // X1 = X1 - 4; MEM[X1] = X0        │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  4. 条件分支 (Conditional Branch)                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │ 31 30 29 28        24 23       5 4        0                          │  │
│  │ ├─┼─┼─┼──────────┼─┼──────────┼──────────┤                          │  │
│  │ │0│1│0│1│0│1│0│0│    cond    │    imm19   │   o0   │                │  │
│  │ └─┴─┴─┴──────────┴─┴──────────┴──────────┘                          │  │
│  │                                                                     │  │
│  │  cond: 0000-EQ 0001-NE 0010-CS/HS 0011-CC/LO ... 1110-AL 1111-NV   │  │
```

### 2.2 条件执行机制

ARM64简化了条件执行，仅保留分支和条件选择指令的条件执行能力：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARM64 条件执行详解                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  条件码 (cond字段，4位)                                                      │
│  ┌───────────┬───────────┬──────────────────────────────────────────────┐  │
│  │  编码     │  助记符   │  含义              │  检查的标志位            │  │
│  ├───────────┼───────────┼──────────────────┼──────────────────────────┤  │
│  │  0000     │  EQ       │  等于              │  Z == 1                  │  │
│  │  0001     │  NE       │  不等于            │  Z == 0                  │  │
│  │  0010     │  CS/HS    │  进位/无符号≥      │  C == 1                  │  │
│  │  0011     │  CC/LO    │  无进位/无符号<    │  C == 0                  │  │
│  │  0100     │  MI       │  负数              │  N == 1                  │  │
│  │  0101     │  PL       │  非负              │  N == 0                  │  │
│  │  0110     │  VS       │  溢出              │  V == 1                  │  │
│  │  0111     │  VC       │  无溢出            │  V == 0                  │  │
│  │  1000     │  HI       │  无符号>           │  C == 1 && Z == 0        │  │
│  │  1001     │  LS       │  无符号≤           │  C == 0 || Z == 1        │  │
│  │  1010     │  GE       │  有符号≥           │  N == V                  │  │
│  │  1011     │  LT       │  有符号<           │  N != V                  │  │
│  │  1100     │  GT       │  有符号>           │  Z == 0 && N == V        │  │
│  │  1101     │  LE       │  有符号≤           │  Z == 1 || N != V        │  │
│  │  1110     │  AL       │  总是              │  无条件                  │  │
│  │  1111     │  NV       │  从不 (保留)       │  -                       │  │
│  └───────────┴───────────┴──────────────────┴──────────────────────────┘  │
│                                                                             │
│  条件执行指令:                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  1. 条件分支: B.cond                                                  │  │
│  │     B.EQ label   // if (Z) goto label                                │  │
│  │     B.GT label   // if (!Z && N==V) goto label                       │  │
│  │                                                                      │  │
│  │  2. 条件选择: CSEL, CSINC, CSNEG, CSINV                              │  │
│  │     CSEL X0, X1, X2, EQ   // X0 = (Z) ? X1 : X2                      │  │
│  │     CINC X0, X1, GT       // X0 = (GT) ? X1+1 : X1                   │  │
│  │                                                                      │  │
│  │  3. 条件比较: CCMN, CCMP                                             │  │
│  │     CCMP X0, X1, #0, EQ   // if (EQ) 比较X0和X1，否则设置标志为0    │  │
│  │                                                                      │  │
│  │  4. 条件设置: CSET, CSETM                                            │  │
│  │     CSET X0, EQ           // X0 = (EQ) ? 1 : 0                       │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  与ARM32对比:                                                                │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 数据通路设计

### 4.1 64位数据通路概览

ARM64采用64位数据通路，所有寄存器和ALU操作都是64位宽度。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ARM64 64位数据通路详细设计                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  关键特性:                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  • 数据宽度: 64位 (支持32位操作，结果零扩展到64位)                    │  │
│  │  • 寄存器: 31个64位通用寄存器 (X0-X30)                               │  │
│  │  • 特殊寄存器: SP (堆栈指针), PC (程序计数器)                        │  │
│  │  • 零寄存器: XZR/WZR (读总为0，写被丢弃)                             │  │
```

```verilog
);
```

### 4.3 寄存器文件设计

ARM64有31个64位通用寄存器，X31根据上下文作为SP (堆栈指针) 或 XZR (零寄存器)。

```verilog
//============================================================================
// ARM64 寄存器文件 - 31个64位通用寄存器 + SP
//============================================================================
module register_file (
    input         clk,
    input         reset,

    // 读端口1 (用于Rn)
    input  [4:0]  read_addr1,
    output [63:0] read_data1,

    // 读端口2 (用于Rm/Rd读取)
    input  [4:0]  read_addr2,
    output [63:0] read_data2,

    // 写端口
    input  [4:0]  write_addr,
    input  [63:0] write_data,
    input         write_enable,

    // SP相关 (X31作为SP时的特殊处理)
    input         use_sp_as_x31,      // 1: X31是SP, 0: X31是XZR
    output [63:0] sp_current          // 当前SP值输出
);
    // 寄存器存储: X0-X30 (31个寄存器)
    // X31不存储，根据上下文返回0或SP
    reg [63:0] regs [0:30];
    reg [63:0] sp_reg;                // 专用SP寄存器

    integer i;

    // 初始化
    initial begin
        for (i = 0; i < 31; i = i + 1)
            regs[i] = 64'h0;
        sp_reg = 64'h0;
    end

    // 读端口1 (Rn)
    assign read_data1 = (read_addr1 == 5'd31) ?
                        (use_sp_as_x31 ? sp_reg : 64'h0) :
                        regs[read_addr1];

    // 读端口2 (Rm/Rd)
    assign read_data2 = (read_addr2 == 5'd31) ?
                        (use_sp_as_x31 ? sp_reg : 64'h0) :
                        regs[read_addr2];

    // SP输出
    assign sp_current = sp_reg;

    // 写操作 (时序逻辑)
    always @(posedge clk or posedge reset) begin
        if (reset) begin
            for (i = 0; i < 31; i = i + 1)
                regs[i] <= 64'h0;
            sp_reg <= 64'h0;
        end else if (write_enable) begin
            if (write_addr == 5'd31) begin
                // 写入X31
                if (use_sp_as_x31)
                    sp_reg <= write_data;  // 写入SP
                // 否则作为XZR，丢弃写入
            end else if (write_addr != 5'd31) begin
                // 写入普通寄存器
                regs[write_addr] <= write_data;
            end
        end
    end

endmodule
```

### 4.4 立即数扩展单元

ARM64有多种立即数编码格式，需要相应的扩展逻辑。

```verilog
//============================================================================
// ARM64 立即数扩展单元
//============================================================================
module immediate_encoder (
    input  [31:0] instruction,
    output [63:0] immediate_out,
    output        imm_valid
);
    // 指令字段提取
    wire [1:0]  op0 = instruction[28:27];  // 指令类别
    wire [8:0]  opcode = instruction[28:20];
    wire [11:0] imm12 = instruction[21:10];
    wire [15:0] imm16 = instruction[20:5];
    wire [1:0]  hw = instruction[22:21];   // MOVZ/MOVK的移位
    wire [18:0] imm19 = instruction[23:5]; // 条件分支
    wire [25:0] imm26 = instruction[25:0]; // 无条件分支

    reg [63:0] imm_result;
    reg        valid;

    // 立即数类型定义
    localparam IMM_TYPE_NONE = 3'd0;
    localparam IMM_TYPE_ARITH = 3'd1;   // 12位 + 移位
    localparam IMM_TYPE_MOVZ = 3'd2;    // 16位 + 移位
    localparam IMM_TYPE_BRANCH_COND = 3'd3;  // 19位有符号
    localparam IMM_TYPE_BRANCH_UNCOND = 3'd4; // 26位有符号

    reg [2:0] imm_type;

    // 解码立即数类型
    always @(*) begin
        // 默认
        imm_type = IMM_TYPE_NONE;

        // 数据处理立即数 (ADD/SUB等)
        if (instruction[28:24] == 5'b10001) begin
            imm_type = IMM_TYPE_ARITH;
        end
        // MOVZ/MOVN/MOVK
        else if (instruction[28:23] == 6'b100101) begin
            imm_type = IMM_TYPE_MOVZ;
        end
        // 条件分支
        else if (instruction[31:24] == 8'b01010100) begin
            imm_type = IMM_TYPE_BRANCH_COND;
        end
        // 无条件分支
        else if (instruction[31:26] == 6'b000101) begin
            imm_type = IMM_TYPE_BRANCH_UNCOND;
        end
    end

    // 立即数生成
    always @(*) begin
        case (imm_type)
            IMM_TYPE_ARITH: begin
                // 12位立即数，可选左移12位
                // instruction[22] = shift (0: 无移位, 1: 左移12位)
                if (instruction[22])
                    imm_result = {52'b0, imm12} << 12;
                else
                    imm_result = {52'b0, imm12};
                valid = 1'b1;
            end

            IMM_TYPE_MOVZ: begin
                // 16位立即数，移位0/16/32/48位
                case (hw)
                    2'b00: imm_result = {48'b0, imm16};
                    2'b01: imm_result = {32'b0, imm16, 16'b0};
                    2'b10: imm_result = {16'b0, imm16, 32'b0};
                    2'b11: imm_result = {imm16, 48'b0};
                endcase
                valid = 1'b1;
            end

            IMM_TYPE_BRANCH_COND: begin
                // 19位有符号立即数，左移2位 (4字节对齐)
                imm_result = {{45{imm19[18]}}, imm19, 2'b00};
                valid = 1'b1;
            end

            IMM_TYPE_BRANCH_UNCOND: begin
                // 26位有符号立即数，左移2位
                imm_result = {{38{imm26[25]}}, imm26, 2'b00};
                valid = 1'b1;
            end

            default: begin
                imm_result = 64'h0;
                valid = 1'b0;
            end
        endcase
    end

    assign immediate_out = imm_result;
    assign imm_valid = valid;

endmodule
```

### 4.5 64位ALU设计

ARM64的ALU支持64位和32位操作，32位操作结果零扩展到64位。

```verilog
//============================================================================
// ARM64 64位ALU
//============================================================================
module alu_arm64 (
    input  [63:0] operand_a,      // 第一操作数
    input  [63:0] operand_b,      // 第二操作数
    input  [4:0]  alu_control,    // ALU控制信号
    input         set_flags,      // 更新标志位
    input         is_32bit,       // 32位操作 (结果零扩展)

    output reg [63:0] result,
    output reg        n_flag,     // 负数
    output reg        z_flag,     // 零
    output reg        c_flag,     // 进位
    output reg        v_flag      // 溢出
);
    // ALU操作码定义
    localparam ALU_ADD  = 5'b00000;
    localparam ALU_ADDS = 5'b00001;  // 带标志更新
    localparam ALU_SUB  = 5'b00010;
    localparam ALU_SUBS = 5'b00011;  // 带标志更新 (CMP)
    localparam ALU_AND  = 5'b00100;
    localparam ALU_ANDS = 5'b00101;  // 带标志更新 (TST)
    localparam ALU_ORR  = 5'b00110;
    localparam ALU_EOR  = 5'b00111;
    localparam ALU_LSL  = 5'b01000;  // 逻辑左移
    localparam ALU_LSR  = 5'b01001;  // 逻辑右移
    localparam ALU_ASR  = 5'b01010;  // 算术右移
    localparam ALU_ROR  = 5'b01011;  // 循环右移
    localparam ALU_ADC  = 5'b01100;  // 带进位加
    localparam ALU_SBC  = 5'b01101;  // 带进位减
    localparam ALU_PASS = 5'b01110;  // 直通 (用于MOV)
    localparam ALU_NOT  = 5'b01111;  // 按位取反

    // 内部信号
    wire [63:0] a_32 = {32'b0, operand_a[31:0]};
    wire [63:0] b_32 = {32'b0, operand_b[31:0]};
    wire [63:0] a_val = is_32bit ? a_32 : operand_a;
    wire [63:0] b_val = is_32bit ? b_32 : operand_b;

    reg [64:0] add_result;  // 65位用于进位检测
    reg [63:0] raw_result;

    // 移位量 (限制在0-63)
    wire [5:0] shift_amt = operand_b[5:0];

    // ALU运算逻辑
    always @(*) begin
        case (alu_control)
            ALU_ADD, ALU_ADDS: begin
                add_result = {1'b0, a_val} + {1'b0, b_val};
                raw_result = add_result[63:0];
            end

            ALU_SUB, ALU_SUBS: begin
                add_result = {1'b0, a_val} + {1'b0, ~b_val} + 64'd1;
                raw_result = add_result[63:0];
            end

            ALU_AND, ALU_ANDS: begin
                raw_result = a_val & b_val;
                add_result = {1'b0, raw_result};
            end

            ALU_ORR: begin
                raw_result = a_val | b_val;
                add_result = 65'b0;
            end

            ALU_EOR: begin
                raw_result = a_val ^ b_val;
                add_result = 65'b0;
            end

            ALU_LSL: begin
                raw_result = a_val << shift_amt;
                add_result = 65'b0;
            end

            ALU_LSR: begin
                raw_result = a_val >> shift_amt;
                add_result = 65'b0;
            end

            ALU_ASR: begin
                if (is_32bit)
                    raw_result = {{32{a_val[31]}}, $signed(a_val[31:0]) >>> shift_amt};
                else
                    raw_result = $signed(a_val) >>> shift_amt;
                add_result = 65'b0;
            end

            ALU_ROR: begin
                raw_result = (a_val >> shift_amt) | (a_val << (64 - shift_amt));
                add_result = 65'b0;
            end

            ALU_ADC: begin
                // 需要外部传入c_flag
                raw_result = a_val + b_val + {63'b0, c_flag};
                add_result = 65'b0;
            end

            ALU_SBC: begin
                raw_result = a_val - b_val - {63'b0, ~c_flag};
                add_result = 65'b0;
            end

            ALU_PASS: begin
                raw_result = b_val;
                add_result = 65'b0;
            end

            ALU_NOT: begin
                raw_result = ~a_val;
                add_result = 65'b0;
            end

            default: begin
                raw_result = 64'h0;
                add_result = 65'b0;
            end
        endcase
    end

    // 结果处理 (32位操作零扩展)
    always @(*) begin
        if (is_32bit)
            result = {32'b0, raw_result[31:0]};
        else
            result = raw_result;
    end

    // 标志位更新
    always @(*) begin
        if (set_flags || alu_control == ALU_ADDS || alu_control == ALU_SUBS ||
            alu_control == ALU_ANDS) begin
            // N标志 - 结果最高位
            n_flag = is_32bit ? raw_result[31] : raw_result[63];

            // Z标志 - 结果全零
            z_flag = is_32bit ? (raw_result[31:0] == 32'b0) : (raw_result == 64'b0);

            // C标志 - 进位/借位
            if (alu_control == ALU_SUB || alu_control == ALU_SUBS)
                c_flag = ~add_result[64];  // 减法时进位取反
            else
                c_flag = add_result[64];

            // V标志 - 有符号溢出
            if (is_32bit) begin
                v_flag = (a_val[31] == b_val[31]) && (a_val[31] != raw_result[31]);
            end else begin
                v_flag = (a_val[63] == b_val[63]) && (a_val[63] != raw_result[63]);
            end
        end else begin
            // 保持原标志位
            n_flag = n_flag;
            z_flag = z_flag;
            c_flag = c_flag;
            v_flag = v_flag;
        end
    end

endmodule
```

### 4.6 数据存储器接口

ARM64支持多种数据大小和字节序的加载/存储操作。

```verilog
//============================================================================
// ARM64 数据存储器接口 - 支持字节/半字/字/双字访问
//============================================================================
module data_memory_arm64 (
    input         clk,
    input  [63:0] address,        // 字节地址
    input  [63:0] write_data,     // 写入数据
    input         mem_read,       // 读使能
    input         mem_write,      // 写使能
    input  [2:0]  size,           // 访问大小: 000=字节, 001=半字, 010=字, 011=双字
    input         sign_extend,    // 符号扩展 (加载时)
    input         is_pair,        // 寄存器对访问 (LDP/STP)

    output reg [63:0] read_data   // 读取数据
);
    // 64KB数据存储器
    reg [7:0] mem [0:65535];

    // 地址对齐处理
    wire [15:0] addr_aligned = address[15:0];
    wire [2:0]  byte_offset = address[2:0];

    // 读取逻辑 (组合逻辑)
    always @(*) begin
        if (mem_read) begin
            if (is_pair) begin
                // LDP - 加载两个64位值 (16字节)
                read_data[63:0] = {
                    mem[addr_aligned + 7],
                    mem[addr_aligned + 6],
                    mem[addr_aligned + 5],
                    mem[addr_aligned + 4],
                    mem[addr_aligned + 3],
                    mem[addr_aligned + 2],
                    mem[addr_aligned + 1],
                    mem[addr_aligned]
                };
            end else begin
                case (size)
                    3'b000: begin // 字节加载
                        if (sign_extend)
                            read_data = {{56{mem[addr_aligned][7]}}, mem[addr_aligned]};
                        else
                            read_data = {56'b0, mem[addr_aligned]};
                    end

                    3'b001: begin // 半字加载
                        if (sign_extend)
                            read_data = {{48{mem[addr_aligned + 1][7]}},
                                        mem[addr_aligned + 1], mem[addr_aligned]};
                        else
                            read_data = {48'b0, mem[addr_aligned + 1], mem[addr_aligned]};
                    end

                    3'b010: begin // 字加载
                        if (sign_extend)
                            read_data = {{32{mem[addr_aligned + 3][7]}},
                                        mem[addr_aligned + 3], mem[addr_aligned + 2],
                                        mem[addr_aligned + 1], mem[addr_aligned]};
                        else
                            read_data = {32'b0, mem[addr_aligned + 3], mem[addr_aligned + 2],
                                        mem[addr_aligned + 1], mem[addr_aligned]};
                    end

                    3'b011: begin // 双字加载
                        read_data = {
                            mem[addr_aligned + 7], mem[addr_aligned + 6],
                            mem[addr_aligned + 5], mem[addr_aligned + 4],
                            mem[addr_aligned + 3], mem[addr_aligned + 2],
                            mem[addr_aligned + 1], mem[addr_aligned]
                        };
                    end

                    default: read_data = 64'h0;
                endcase
            end
        end else begin
            read_data = 64'h0;
        end
    end

    // 写入逻辑 (时序逻辑)
    always @(posedge clk) begin
        if (mem_write) begin
            if (is_pair) begin
                // STP - 存储两个64位值
                {mem[addr_aligned + 7], mem[addr_aligned + 6],
                 mem[addr_aligned + 5], mem[addr_aligned + 4],
                 mem[addr_aligned + 3], mem[addr_aligned + 2],
                 mem[addr_aligned + 1], mem[addr_aligned]} <= write_data[63:0];
            end else begin
                case (size)
                    3'b000: begin // 字节存储
                        mem[addr_aligned] <= write_data[7:0];
                    end

                    3'b001: begin // 半字存储
                        mem[addr_aligned] <= write_data[7:0];
                        mem[addr_aligned + 1] <= write_data[15:8];
                    end

                    3'b010: begin // 字存储
                        mem[addr_aligned] <= write_data[7:0];
                        mem[addr_aligned + 1] <= write_data[15:8];
                        mem[addr_aligned + 2] <= write_data[23:16];
                        mem[addr_aligned + 3] <= write_data[31:24];
                    end

                    3'b011: begin // 双字存储
                        mem[addr_aligned] <= write_data[7:0];
                        mem[addr_aligned + 1] <= write_data[15:8];
                        mem[addr_aligned + 2] <= write_data[23:16];
                        mem[addr_aligned + 3] <= write_data[31:24];
                        mem[addr_aligned + 4] <= write_data[39:32];
                        mem[addr_aligned + 5] <= write_data[47:40];
                        mem[addr_aligned + 6] <= write_data[55:48];
                        mem[addr_aligned + 7] <= write_data[63:56];
                    end
                endcase
            end
        end
    end

endmodule
```

### 5.1 PSTATE寄存器

PSTATE (Process State) 是ARM64保存处理器状态的寄存器组，包含条件标志和执行状态。

```verilog
//============================================================================
// PSTATE 寄存器组实现
//============================================================================
module pstate_registers (
    input         clk,
    input         reset,
    input         update_flags,       // 更新标志位使能
    input         exception_entry,    // 异常入口
    input         exception_return,   // 异常返回
    input  [3:0]  flags_in,           // 新标志位 {N, Z, C, V}
    input  [3:0]  saved_flags,        // 保存的标志位 (异常返回用)
    input  [1:0]  current_el,         // 当前异常级别
    input         irq_masked,         // IRQ屏蔽
    input         fiq_masked,         // FIQ屏蔽
    input         debug_masked,       // 调试异常屏蔽

    output [3:0]  flags_out,          // 当前标志位 {N, Z, C, V}
    output [1:0]  el_out,             // 当前异常级别
    output [3:0]  daif_out            // 中断屏蔽位
);
    // 条件标志
    reg n_flag, z_flag, c_flag, v_flag;

    // 异常级别 (EL0-EL3)
    reg [1:0] exception_level;

    // 中断屏蔽位 (DAIF)
    reg d_mask, a_mask, i_mask, f_mask;

    // 复位值
    initial begin
        n_flag = 1'b0;
        z_flag = 1'b0;
        c_flag = 1'b0;
        v_flag = 1'b0;
        exception_level = 2'b11;  // EL3 (最高特权)
        d_mask = 1'b1;
        a_mask = 1'b1;
        i_mask = 1'b1;
        f_mask = 1'b1;
    end

    // 同步更新
    always @(posedge clk or posedge reset) begin
        if (reset) begin
            n_flag <= 1'b0;
            z_flag <= 1'b0;
            c_flag <= 1'b0;
            v_flag <= 1'b0;
            exception_level <= 2'b11;  // EL3
            d_mask <= 1'b1;
            a_mask <= 1'b1;
            i_mask <= 1'b1;
            f_mask <= 1'b1;
        end else begin
            // 异常入口处理
            if (exception_entry) begin
                // 保存当前状态，切换到更高异常级别
                if (exception_level > 2'b00)
                    exception_level <= exception_level - 2'b01;
                // 屏蔽中断
                i_mask <= 1'b1;
                f_mask <= 1'b1;
            end
            // 异常返回处理
            else if (exception_return) begin
                // 恢复状态，切换回更低异常级别
                if (exception_level < 2'b11)
                    exception_level <= exception_level + 2'b01;
                // 恢复标志位
                {n_flag, z_flag, c_flag, v_flag} <= saved_flags;
            end
            // 正常标志位更新
            else if (update_flags) begin
                {n_flag, z_flag, c_flag, v_flag} <= flags_in;
            end

            // DAIF更新
            if (irq_masked) i_mask <= 1'b1;
            if (fiq_masked) f_mask <= 1'b1;
            if (debug_masked) d_mask <= 1'b1;
        end
    end

    // 输出
    assign flags_out = {n_flag, z_flag, c_flag, v_flag};
    assign el_out = exception_level;
    assign daif_out = {d_mask, a_mask, i_mask, f_mask};

endmodule
```

```verilog
//============================================================================
// 条件分支判断单元
//============================================================================
module condition_check (
    input  [3:0]  cond,       // 条件码 (4位)
    input  [3:0]  flags,      // 标志位 {N, Z, C, V}
    output        cond_pass   // 条件满足
);
    wire n = flags[3];
    wire z = flags[2];
    wire c = flags[1];
    wire v = flags[0];

    reg result;

    always @(*) begin
        case (cond)
            4'b0000: result = z;                    // EQ - 等于
            4'b0001: result = ~z;                   // NE - 不等于
            4'b0010: result = c;                    // CS/HS - 进位/无符号≥
            4'b0011: result = ~c;                   // CC/LO - 无进位/无符号<
            4'b0100: result = n;                    // MI - 负数
            4'b0101: result = ~n;                   // PL - 非负
            4'b0110: result = v;                    // VS - 溢出
            4'b0111: result = ~v;                   // VC - 无溢出
            4'b1000: result = c & ~z;               // HI - 无符号>
            4'b1001: result = ~c | z;               // LS - 无符号≤
            4'b1010: result = (n == v);             // GE - 有符号≥
            4'b1011: result = (n != v);             // LT - 有符号<
            4'b1100: result = ~z & (n == v);        // GT - 有符号>
            4'b1101: result = z | (n != v);         // LE - 有符号≤
            4'b1110: result = 1'b1;                 // AL - 总是执行
            4'b1111: result = 1'b0;                 // NV - 保留 (永不执行)
            default: result = 1'b0;
        endcase
    end

    assign cond_pass = result;

endmodule
```

```verilog
//============================================================================
// 地址生成单元 (AGU) - 支持ARM64各种寻址模式
//============================================================================
module address_generation_unit (
    input  [63:0] base_reg,       // 基址寄存器 (Xn/SP)
    input  [63:0] offset_reg,     // 偏移寄存器 (Xm)
    input  [63:0] immediate,      // 立即数偏移
    input  [1:0]  shift_type,     // 偏移移位类型
    input  [2:0]  shift_amount,   // 移位数
    input  [2:0]  addressing_mode,// 寻址模式
    input         is_32bit,       // 32位操作 (影响偏移移位)

    output [63:0] address,        // 计算出的地址
    output [63:0] writeback_val,  // 回写值 (用于前变址)
    output        do_writeback    // 需要回写基址寄存器
);
    // 寻址模式编码
    localparam ADDR_MODE_OFFSET    = 3'b000;  // 基址+偏移
    localparam ADDR_MODE_PRE_IDX   = 3'b001;  // 前变址 [Xn, #off]!
    localparam ADDR_MODE_POST_IDX  = 3'b010;  // 后变址 [Xn], #off
    localparam ADDR_MODE_LITERAL   = 3'b011;  // 字面量 (PC相对)
    localparam ADDR_MODE_REG_OFF   = 3'b100;  // 寄存器偏移
    localparam ADDR_MODE_UNSCALED  = 3'b101;  // 无缩放立即数 (LDUR/STUR)

    // 计算偏移值
    reg [63:0] offset_value;

    always @(*) begin
        case (addressing_mode)
            ADDR_MODE_OFFSET, ADDR_MODE_PRE_IDX, ADDR_MODE_POST_IDX: begin
                // 立即数偏移 (缩放)
                if (is_32bit)
                    offset_value = immediate << 2;  // 32位访问: 偏移*4
                else
                    offset_value = immediate << 3;  // 64位访问: 偏移*8
            end

            ADDR_MODE_UNSCALED: begin
                // 9位无符号立即数，不缩放
                offset_value = {{55{immediate[8]}}, immediate[8:0]};
            end

            ADDR_MODE_REG_OFF: begin
                // 寄存器偏移，可选移位
                case (shift_type)
                    2'b00: offset_value = offset_reg;                           // LSL #0
                    2'b01: offset_value = offset_reg << (is_32bit ? 2 : 3);     // LSL #2/#3
                    2'b10: offset_value = $signed(offset_reg) >>> shift_amount; // ASR
                    2'b11: offset_value = {offset_reg[63], offset_reg[63:1]};   // SXTW/UXTW
                endcase
            end

            ADDR_MODE_LITERAL: begin
                // PC相对寻址，19位有符号立即数，左移2位
                offset_value = {{45{immediate[18]}}, immediate[18:0], 2'b00};
            end

            default: offset_value = 64'h0;
        endcase
    end

    // 地址计算
    assign address = base_reg + offset_value;

    // 回写处理
    assign writeback_val = address;  // 前变址时基址更新为计算后的地址
    assign do_writeback = (addressing_mode == ADDR_MODE_PRE_IDX) ||
                          (addressing_mode == ADDR_MODE_POST_IDX);

endmodule
```

```verilog
module exception_vector_table (
    input  [63:0] vbar_base,      // 向量基址寄存器 (VBAR_ELx)

    localparam OFFSET_IRQ_SP0    = 12'h080;  // 当前EL，SP0，IRQ
    localparam OFFSET_FIQ_SP0    = 12'h100;  // 当前EL，SP0，FIQ
    localparam OFFSET_SERR_SP0   = 12'h180;  // 当前EL，SP0，SError

    localparam OFFSET_SYNC_SPX   = 12'h200;  // 当前EL，SPx，同步异常
    localparam OFFSET_IRQ_SPX    = 12'h280;  // 当前EL，SPx，IRQ
    localparam OFFSET_FIQ_SPX    = 12'h300;  // 当前EL，SPx，FIQ
    localparam OFFSET_SERR_SPX   = 12'h380;  // 当前EL，SPx，SError

    localparam OFFSET_SYNC_LOWER = 12'h400;  // 更低EL，同步异常
    localparam OFFSET_IRQ_LOWER  = 12'h480;  // 更低EL，IRQ
    localparam OFFSET_FIQ_LOWER  = 12'h500;  // 更低EL，FIQ
    localparam OFFSET_SERR_LOWER = 12'h580;  // 更低EL，SError

    reg [11:0] offset;

    // 异常类型编码
    localparam EXC_SYNC  = 4'b0000;  // 同步异常
    localparam EXC_IRQ   = 4'b0001;  // IRQ
    localparam EXC_FIQ   = 4'b0010;  // FIQ
    localparam EXC_SERR  = 4'b0011;  // SError

    always @(*) begin
        // 判断是同一EL还是更低EL
        if (from_el == to_el) begin
            // 同一异常级别
            case (exception_type)
                EXC_SYNC: offset = sp_sel ? OFFSET_SYNC_SPX : OFFSET_SYNC_SP0;

                default:  offset = OFFSET_SYNC_SPX;
            endcase
        end else begin
            // 来自更低EL
            case (exception_type)
                EXC_SYNC: offset = OFFSET_SYNC_LOWER;
                EXC_IRQ:  offset = OFFSET_IRQ_LOWER;
                EXC_FIQ:  offset = OFFSET_FIQ_LOWER;
                EXC_SERR: offset = OFFSET_SERR_LOWER;
                default:  offset = OFFSET_SYNC_LOWER;
            endcase
        end
    end

    assign handler_addr = vbar_base + {52'b0, offset};
```

```verilog
                    {3'b000, 4'b1100, 4'b0000, 3'b000}: begin // VBAR_EL1
                        valid_access = (current_el >= 2'b01);
                        reg_defined = 1'b1;
                    end
                    {3'b000, 4'b0100, 4'b0000, 3'b000}: begin // CurrentEL
                        valid_access = 1'b1;  // 任何EL可读
                        reg_defined = 1'b1;
                    end
                    default: begin
                        valid_access = 1'b0;
                        reg_defined = 1'b0;
                    end
                endcase
            end
            default: begin
                valid_access = 1'b0;
                reg_defined = 1'b0;
            end
        endcase
    end

    // 读操作
    always @(*) begin
        if (mrs_en && valid_access) begin
            case ({op1, crn, crm, op2})
                {3'b000, 4'b0000, 4'b0000, 3'b000}: read_data = 64'h410FD034; // MIDR_EL1示例值
                {3'b000, 4'b0100, 4'b0000, 3'b000}: read_data = currentel;
                {3'b000, 4'b1100, 4'b0000, 3'b000}: read_data = vbar_el1;
                {3'b000, 4'b0101, 4'b0000, 3'b000}: read_data = elr_el1;
                {3'b000, 4'b0101, 4'b0010, 3'b000}: read_data = spsr_el1;
                default: read_data = 64'h0;
            endcase
        end else begin
            read_data = 64'h0;
        end
    end

    // 写操作
    always @(posedge clk or posedge reset) begin
        if (reset) begin
            vbar_el1 <= 64'h0;
            sctlr_el1 <= 64'h0;
            elr_el1 <= 64'h0;
            spsr_el1 <= 64'h0;
        end else if (msr_en && valid_access) begin
            case ({op1, crn, crm, op2})
                {3'b000, 4'b1100, 4'b0000, 3'b000}: vbar_el1 <= write_data;
                {3'b000, 4'b0001, 4'b0000, 3'b000}: sctlr_el1 <= write_data;
                {3'b000, 4'b0101, 4'b0000, 3'b000}: elr_el1 <= write_data;
                {3'b000, 4'b0101, 4'b0010, 3'b000}: spsr_el1 <= write_data;
            endcase
        end
    end

    assign access_ok = valid_access;
    assign undefined = !reg_defined;

endmodule
```

### 8.2 控制单元模块

```verilog
//============================================================================
// ARM64 控制单元
//============================================================================

module control_unit_arm64 (
    input  [31:0] instruction,
    input  [3:0]  flags,
    input  [1:0]  current_el,

    output reg        reg_write,
    output reg        mem_read,
    output reg        mem_write,
    output reg        mem_to_reg,
    output reg        alu_src,
    output reg        branch,
    output reg        cond_branch,
    output reg [4:0]  alu_control,
    output reg        set_flags,
    output reg        is_32bit,
    output reg        use_sp,
    output reg        is_exception,
    output reg        is_eret
);
    // 指令字段
    wire [4:0]  op0 = instruction[28:24];
    wire [8:0]  opcode = instruction[28:20];
    wire [1:0]  op1 = instruction[25:24];
    wire        sf = instruction[31];      // 0=32位, 1=64位

    // 指令类别解码
    wire is_dp_imm = (op0 == 5'b10001);    // 数据处理立即数
    wire is_dp_reg = (op0 == 5'b01010);    // 数据处理寄存器
    wire is_ldr_imm = (instruction[27:24] == 4'b1110);  // 加载立即数
    wire is_str_imm = (instruction[27:24] == 4'b1110);  // 存储立即数
    wire is_b_uncond = (instruction[31:26] == 6'b000101);  // 无条件分支
    wire is_b_cond = (instruction[31:24] == 8'b01010100);  // 条件分支
    wire is_svc = (instruction[31:24] == 8'b11010100) && (instruction[20:0] == 21'd1);
    wire is_eret = (instruction[31:24] == 8'b11010110) && (instruction[20:0] == 21'b100111111000000000000);

    always @(*) begin
        // 默认值
        reg_write = 1'b0;
        mem_read = 1'b0;
        mem_write = 1'b0;
        mem_to_reg = 1'b0;
        alu_src = 1'b0;
        branch = 1'b0;
        cond_branch = 1'b0;
        alu_control = 5'b0;
        set_flags = 1'b0;
        is_32bit = ~sf;
        use_sp = 1'b0;
        is_exception = 1'b0;
        is_eret = 1'b0;

        casez (instruction[31:24])
            // 数据处理立即数 (ADD/SUB等)
            8'b1??10001: begin
                reg_write = 1'b1;
                alu_src = 1'b1;
                use_sp = instruction[29];  // SP作为基址

                if (instruction[30]) begin // SUB
                    alu_control = instruction[29] ? 5'b00011 : 5'b00010;  // SUBS/SUB
                    set_flags = instruction[29];
                end else begin // ADD
                    alu_control = instruction[29] ? 5'b00001 : 5'b00000;  // ADDS/ADD
                    set_flags = instruction[29];
                end
            end

            // 数据处理寄存器
            8'b0?01010: begin
                reg_write = 1'b1;
                alu_src = 1'b0;

                case (instruction[30:29])
                    2'b00: alu_control = instruction[21] ? 5'b00011 : 5'b00010;  // SUB/SUBS
                    2'b01: alu_control = 5'b00110;  // ADD
                    2'b10: alu_control = 5'b00100;  // AND
                    2'b11: alu_control = 5'b00111;  // ORR
                endcase

                set_flags = instruction[29];
            end

            // 加载/存储立即数
            8'b111?1000, 8'b111?1001: begin
                alu_src = 1'b1;  // 使用立即数偏移
                alu_control = 5'b00000;  // ADD用于地址计算

                if (instruction[22]) begin // 加载
                    reg_write = 1'b1;
                    mem_read = 1'b1;
                    mem_to_reg = 1'b1;
                end else begin // 存储
                    mem_write = 1'b1;
                end
            end

            // 无条件分支
            8'b000101??: begin
                branch = 1'b1;
                reg_write = instruction[31];  // BL需要写LR (X30)
            end

            // 条件分支
            8'b01010100: begin
                cond_branch = 1'b1;
            end

            // SVC
            8'b11010100: begin
                is_exception = 1'b1;
            end

            // ERET
            8'b11010110: begin
                is_eret = 1'b1;
            end

            default: begin
                // NOP或未知指令
            end
        endcase
    end

endmodule
```

```
│  ├─────────────────────────┼───────────────────┼──────────────────────┤  │
│  │ 地址生成单元           │ 复杂              │ 简单                 │  │
│  │  - 寻址模式            │  9种模式          │  2-3种模式           │  │
│  │  - 移位逻辑            │  多种移位方式     │  简单偏移            │  │
│  ├─────────────────────────┼───────────────────┼──────────────────────┤  │
│  │ 分支单元               │ 中等              │ 简单                 │  │
│  │  - 条件分支            │  16种条件码       │  比较+无条件分支     │  │
│  │  - 间接分支            │  支持             │  支持                │  │
│  ├─────────────────────────┼───────────────────┼──────────────────────┤  │
│  │ 异常处理               │ 复杂              │ 简单                 │  │
│  │  - 异常级别            │  EL0-EL3 (4级)    │  U/S/M (3模式)       │  │
│  │  - 向量表              │  复杂向量表结构   │  简单入口点          │  │
│  │  - 系统寄存器          │  大量系统寄存器   │  CSR (数量有限)      │  │
│  ├─────────────────────────┼───────────────────┼──────────────────────┤  │
│  │ 功耗/面积              │ 略高              │ 略低                 │  │
│  │  - 标志位更新          │  动态功耗         │  静态设计            │  │
│  │  - 复杂控制逻辑        │  更多组合逻辑     │  规则化设计          │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│  关键差异总结:                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  1. ARM64的条件执行需要额外的标志位寄存器和条件判断逻辑             │  │
│  │  2. ARM64的寻址模式需要更复杂的地址生成单元                         │  │
│  │  3. RISC-V的简化设计使得流水线控制和数据转发更简单                  │  │
│  │  4. ARM64的异常模型硬件开销更大但功能更完整                         │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
