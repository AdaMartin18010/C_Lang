## 📖 技术概述

本目录深入探讨**指令集架构(Instruction Set Architecture, ISA)**——软硬件之间的契约接口。
ISA定义了软件可见的处理器状态、指令编码格式、执行语义和调用约定，是理解编译器输出、操作系统内核和处理器实现的关键基础。

| 主题 | 描述 | 重要性 |

| ISA形式化定义 | 状态空间、指令集、解码函数、执行语义 | ⭐⭐⭐⭐⭐ |
| x86-64编码 | REX前缀、ModR/M、SIB字节、变长指令格式 | ⭐⭐⭐⭐⭐ |

| 特权级别 | Ring 0-3、系统调用、特权指令 | ⭐⭐⭐⭐ |

### 01_Instruction_Set_Architecture.md 详细内容

- **ISA形式化定义**：抽象机模型 `ISA = (State, Instructions, Decode, Execute)`
- **x86-64状态空间**：RIP、16个通用寄存器、RFLAGS、XMM向量寄存器建模
- **x86-64指令编码**：Prefixes-REX-Opcode-ModR/M-SIB-Displacement-Immediate格式解析
- **指令解码器实现**：REX前缀解析(W/R/X/B位)、ModR/M字节解码、SIB寻址
- **指令执行语义**：MOV/ADD/JMP/JE等指令的状态转换函数实现
- **完整执行循环**：取指-译码-执行-更新的解释器框架
- **System V AMD64 ABI**：参数寄存器(RDI/RSI/RDX/RCX/R8/R9)、调用者/被调用者保存约定
- **栈帧布局**：返回地址、保存的RBP、局部变量、栈对齐要求的可视化
- **x86特权级别**：Ring 0-3架构、特权指令列表、系统调用实现
- **常见陷阱分析**：未定义行为、部分寄存器写入、内存对齐要求

## 🔬 深入主题

### ISA设计哲学对比

| 特性 | CISC (x86) | RISC (ARM) | VLIW (Itanium) |
|:-----|:-----------|:-----------|:---------------|
| 指令长度 | 变长(1-15字节) | 定长(4字节) | 定长(bundle) |
| 寄存器数量 | 16 (x86-64) | 31 (ARM64) | 128 |
| 寻址模式 | 复杂 | 简单(load/store) | 显式并行 |
| 硬件复杂度 | 高(解码复杂) | 低 | 低(依赖编译器) |
| 功耗效率 | 一般 | 高 | 高 |

### 新兴ISA扩展

- **AVX-512**：512位向量运算，用于HPC/AI
- **SVE/SVE2**：ARM可扩展向量扩展，向量长度可变
- **RISC-V Vector扩展**：模块化向量架构设计

## ⚠️ 学习建议

### 常见陷阱

1. **混淆ISA与微架构**：ISA是规范，微架构是实现，同一ISA可有不同实现
2. **忽视字节序**：x86-64使用小端序，网络协议使用大端序，混合时易出错
3. **误解未定义行为**：某些指令在特定条件下的行为未定义，依赖它会导致bug
4. **栈对齐忽视**：x86-64要求16字节栈对齐，违反会导致SSE指令崩溃

### 实践建议

- 使用`objdump -d`反汇编二进制文件，对照手册验证理解
- 编写内联汇编代码，测试ABI边界情况
- 阅读Linux内核系统调用入口代码(`entry_64.S`)
- 用GDB单步跟踪程序，观察寄存器变化
