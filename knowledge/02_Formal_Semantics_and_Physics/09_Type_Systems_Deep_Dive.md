```
│              类型系统的形式化框架                            │

│                                                              │
```

## 2. Lambda演算与类型

### 2.1 无类型Lambda演算

```
┌─────────────────────────────────────────────────────────────┐
│              无类型Lambda演算 (λ演算)                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  语法：                                                       │
│  e ::= x           (变量)                                    │
│      | λx. e      (抽象)                                     │
│      | e₁ e₂      (应用)                                     │
│                                                              │
│  归约规则：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  α-转换 (Alpha): 变量重命名                         │   │
│  │  λx. e = λy. e[y/x]   (y不在e中自由出现)           │   │
│  │                                                     │   │
│  │  β-归约 (Beta): 函数应用                            │   │
│  │  (λx. e₁) e₂ → e₁[e₂/x]  (将e₁中的x替换为e₂)       │   │
│  │                                                     │   │
│  │  η-转换 (Eta): 扩展性                               │   │
│  │  λx. (f x) = f   (x不在f中自由出现)                │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  示例：                                                       │
│  (λx. x) y → y               (恒等函数)                     │
│  (λx. λy. x) a b → a         (K组合子)                      │
│  (λf. λx. f (f x)) (λy. y+1) 2                              │
│      → (λx. (λy. y+1) ((λy. y+1) x)) 2                      │
│      → (λy. y+1) ((λy. y+1) 2)                              │
│      → (λy. y+1) 3 → 4                                      │
│                                                              │
│  图灵完备性：                                                 │
│  Lambda演算可编码任意可计算函数，是图灵完备的。               │
│  数据结构的编码：                                             │
│  • 布尔值: true = λt. λf. t, false = λt. λf. f              │
│  • 自然数 (Church数): 0 = λf. λx. x, 1 = λf. λx. f x        │
│  • 数对: pair = λx. λy. λf. f x y                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 简单类型Lambda演算

```
┌─────────────────────────────────────────────────────────────┐
│              简单类型Lambda演算 (λ→)                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  语法扩展：                                                   │
│  e ::= x | λx:T. e | e₁ e₂ | c                             │
│  T ::= B | T₁ → T₂    (B: 基本类型，如Int, Bool)            │
│                                                              │
│  类型规则：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  (T-Var)   ───────────                             │   │
│  │            Γ, x:T ⊢ x : T                          │   │
│  │                                                     │   │
│  │  (T-Abs)   Γ, x:T₁ ⊢ e : T₂                        │   │
│  │            ─────────────────                       │   │
│  │            Γ ⊢ λx:T₁.e : T₁ → T₂                   │   │
│  │                                                     │   │
│  │  (T-App)   Γ ⊢ e₁ : T₁ → T₂   Γ ⊢ e₂ : T₁          │   │
│  │            ────────────────────────────────        │   │
│  │            Γ ⊢ e₁ e₂ : T₂                          │   │
│  │                                                     │   │
│  │  (T-Const) ───────────                             │   │
│  │            Γ ⊢ c : type(c)                         │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  示例推导：                                                   │
│  ⊢ λx:Int. x : Int → Int                                    │
│  ─────────────────────────                                  │
│  1. x:Int ⊢ x : Int        (T-Var)                          │
│  2. ─────────────────        (T-Abs, 1)                     │
│     ⊢ λx:Int. x : Int → Int                                 │
│                                                              │
│  关键性质：                                                   │
│  • 类型安全 (Type Safety): 良类型的程序不会卡住             │
│  • 规范化 (Normalization): 良类型的程序必然终止             │
│  • 类型检查可判定                                             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 3. 多态类型系统

### 3.1 参数多态 (System F)

```
┌─────────────────────────────────────────────────────────────┐
│              System F: 多态Lambda演算                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  语法扩展：                                                   │
│  e ::= ... | Λα. e | e[T]    (类型抽象与类型应用)            │
│  T ::= ... | α | ∀α. T       (类型变量与全称类型)            │
│                                                              │
│  类型规则：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  (T-TypeAbs)  Γ ⊢ e : T    α ∉ FTV(Γ)              │   │
│  │               ────────────────────────              │   │
│  │               Γ ⊢ Λα. e : ∀α. T                     │   │
│  │                                                     │   │
│  │  (T-TypeApp)  Γ ⊢ e : ∀α. T                         │   │
│  │               ────────────────────────              │   │
│  │               Γ ⊢ e[T'] : T[T'/α]                   │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  示例：多态恒等函数                                           │
│  id = Λα. λx:α. x  : ∀α. α → α                              │
│                                                              │
│  使用：                                                       │
│  id[Int] 2 → 2        (实例化为 Int → Int)                  │
│  id[Bool] true → true  (实例化为 Bool → Bool)               │
│                                                              │
│  表达能力的代价：                                             │
│  • 类型推断不可判定 (需要有类型标注)                         │
│  • 与ML的多态限制 (let多态) 不同                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Hindley-Milner类型推断

```
┌─────────────────────────────────────────────────────────────┐
│              Hindley-Milner类型推断 (ML风格)                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  核心算法：W算法 (Damas-Milner)                               │
│                                                              │
│  算法步骤：                                                   │
│  1. 为每个表达式分配一个类型变量                             │
│  2. 根据语法结构生成约束方程                                 │
│  3. 使用统一 (Unification) 求解约束                          │
│  4. 得到最一般的类型 (Principal Type)                        │
│                                                              │
│  统一算法：                                                   │
│  unify(t₁, t₂) =                                            │
│    • 如果 t₁ = t₂: 成功                                      │
│    • 如果 t₁ 是变量 α 且 α ∉ FTV(t₂): α ↦ t₂                 │
│    • 如果 t₂ 是变量 α 且 α ∉ FTV(t₁): α ↦ t₁                 │
│    • 如果 t₁ = f(a₁,...,an), t₂ = f(b₁,...,bn):              │
│      统一每对参数 unify(aᵢ, bᵢ)                              │
│    • 否则: 失败 (类型错误)                                   │
│                                                              │
│  示例推导：                                                   │
│  λf. λx. f (f x)                                            │
│                                                              │
│  1. 分配变量: f:α, x:β, (f x):γ, f (f x):δ, 结果:ε          │
│  2. 生成约束:                                                │
│     • 从 f x: α = β → γ                                      │
│     • 从 f (f x): α = γ → δ                                  │
│  3. 统一:                                                   │
│     • α = β → γ                                              │
│     • β → γ = γ → δ  ⇒ β = γ = δ                            │
│  4. 结果: α = β → β, ε = (β → β) → β → β                    │
│                                                              │
│  即: ∀β. (β → β) → β → β                                    │
│                                                              │
│  实际系统: OCaml, Haskell, Standard ML, Rust(部分)          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 4. 高级类型特性

### 4.1 代数数据类型 (ADT)

```c
/* ============================================================
 * 代数数据类型 (ADT)
 * ============================================================ */

// 在C语言中模拟ADT

// 积类型 (Product Type): 结构体
struct point {
    int x;      // 有x AND y
    int y;      // 大小 = sizeof(int) + sizeof(int)
};

// 和类型 (Sum Type): 带标签的联合体
// 类似于 Rust 的 enum 或 Haskell 的 data
enum shape_tag { CIRCLE, RECTANGLE };

struct shape {
    enum shape_tag tag;  // 标签指示变体
    union {
        struct { int radius; } circle;
        struct { int width, height; } rectangle;
    } data;
};

// 使用和类型 (模式匹配的手动实现)
double area(const struct shape* s) {
    switch (s->tag) {
        case CIRCLE:
            return 3.14159 * s->data.circle.radius
                        * s->data.circle.radius;
        case RECTANGLE:
            return s->data.rectangle.width
                 * s->data.rectangle.height;
    }
    return 0;  // 不可达，但C需要
}

/* 代数关系：
 * 积类型 T₁ × T₂:  |T₁ × T₂| = |T₁| × |T₂|
 * 和类型 T₁ + T₂:  |T₁ + T₂| = |T₁| + |T₂|
 * 函数类型 T₁ → T₂: |T₁ → T₂| = |T₂|^|T₁|
 *
 * 示例:
 * Bool × Bool = {(T,T), (T,F), (F,T), (F,F)}  (4 = 2×2)
 * Bool + Bool = {Left T, Left F, Right T, Right F} (4 = 2+2)
 */
```

### 4.2 线性类型与资源管理

```
┌─────────────────────────────────────────────────────────────┐
│              线性类型系统                                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  核心思想：                                                   │
│  每个资源必须且只能使用一次。                                 │
│  "Use it or lose it"                                        │
│                                                              │
│  类型规则：                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  线性箭头 T₁ ⊸ T₂: 消耗 T₁ 产生 T₂                  │   │
│  │                                                     │   │
│  │  Γ₁ ⊢ e₁ : T₁ ⊸ T₂    Γ₂ ⊢ e₂ : T₁                │   │
│  │  ────────────────────────────────────────          │   │
│  │  Γ₁ ∘ Γ₂ ⊢ e₁ e₂ : T₂    (Γ₁, Γ₂ 不相交)          │   │
│  │                                                     │   │
│  │  Γ, x:T₁ ⊢ e : T₂                                 │   │
│  │  ──────────────────────────────── (x在线性出现)    │   │
│  │  Γ ⊢ λx.e : T₁ ⊸ T₂                               │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  应用：Rust的所有权系统                                       │
│                                                              │
│  伪代码示例：                                                 │
│  fn read_file(path: String) -> File ⊸ String {               │
│      let file = open(path);    // 消耗path                  │
│      let content = read(file); // 消耗file                  │
│      content                   // 返回content               │
│  } // path和file不再可用                                     │
│                                                              │
│  fn copy_file(src: String, dst: String) {                    │
│      let content = read_file(src);                           │
│      // 错误: src已被消耗，不能再用                          │
│      // let content2 = read_file(src);                      │
│      write_file(dst, content);                               │
│  }                                                           │
│                                                              │
│  优势：                                                       │
│  • 编译时防止use-after-free                                  │
│  • 编译时防止data race                                       │
│  • 无GC的内存安全                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 C11 _Generic 使用示例

```c
/* ============================================================
 * C11 _Generic: 类型安全的泛型编程
 * ============================================================ */

#include <stdio.h>
#include <math.h>

/* _Generic 表达式根据类型选择不同的实现 */

#define print_value(x) _Generic((x), \
    int: print_int,                   \
    double: print_double,             \
    char*: print_string,              \
    default: print_unknown            \
)(x)

void print_int(int x) { printf("int: %d\n", x); }
void print_double(double x) { printf("double: %f\n", x); }
void print_string(char* x) { printf("string: %s\n", x); }
void print_unknown(void* x) { printf("unknown\n"); }

/* 泛型数学函数示例 */
#define generic_sqrt(x) _Generic((x), \
    float: sqrtf,                     \
    double: sqrt,                     \
    long double: sqrtl                \
)(x)

/* 泛型容器 */
#define DECLARE_STACK(type, name)               \
    typedef struct {                           \
        type* data;                             \
        size_t size;                            \
        size_t capacity;                        \
    } Stack_##name;                             \
                                                \
    void stack_##name##_push(Stack_##name* s, type val) { \
        if (s->size >= s->capacity) {           \
            s->capacity *= 2;                   \
            s->data = realloc(s->data, s->capacity * sizeof(type)); \
        }                                       \
        s->data[s->size++] = val;               \
    }                                           \
    type stack_##name##_pop(Stack_##name* s) {  \
        return s->data[--s->size];              \
    }

DECLARE_STACK(int, int)
DECLARE_STACK(double, double)

int main() {
    print_value(42);        // int: 42
    print_value(3.14);      // double: 3.140000
    print_value("hello");   // string: hello

    float f = 16.0f;
    double d = 16.0;
    printf("%f\n", generic_sqrt(f));   // 使用sqrtf
    printf("%f\n", generic_sqrt(d));   // 使用sqrt

    return 0;
}
```
