## 【科学·所有权的形式化】

### 1. 所有权类型系统

**定义**: 所有权是资源管理的类型级编码。

**类型判断**:

```
Γ ⊢ t : own(T)    -- t拥有T的所有权
Γ ⊢ t : &T        -- t借用T
Γ ⊢ t : &mut T    -- t可变借用T
```

### 2. XOR规则

**定理**: 在任何时刻，对于值v，以下互斥：

- 存在多个 &v 借用
- 存在唯一的 &mut v 借用
- 不存在借用，v可移动

**形式化**:

```
borrow_state(v) ::= Shared(n) | Unique | Owned

其中:
- Shared(n): n个不可变借用 (n ≥ 1)
- Unique: 1个可变借用
- Owned: 无借用，所有权可转移
```

### 3. 生命周期的偏序

**定义**: 生命周期是代码区域的抽象。

```
'a : Lifetime  -- 区域标记
```

**子生命周期关系**:

```
'b ⊆ 'a  表示 \'b 不超出 \'a
```

**约束求解**:

- 借用约束生成
- 约束图构建
- 最小生命周期推断

## 【工程·Rust到C编译】

### 1. 所有权擦除

**定理**: Rust所有权在运行时擦除。

**证明**: 所有权检查在编译期完成，生成的C代码不包含所有权信息。

### 2. 借用检查的实现

**编译期**: 借用检查器验证XOR规则

**运行时**: 生成的C代码：

```c
// Rust: fn foo(x: &i32)
void foo(int32_t* x) {
  // 无运行时检查！
}
```

### 3. Drop的实现

**Rust**:

```rust
drop(x);  // 显式释放
// 或隐式作用域结束
```

**生成的C**:

```c
void drop_T(T* x) {
  // 析构逻辑
  free_resources(x);
}

// RAII模式
void scope() {
  T x = init_T();
  // ...
  drop_T(&x);  // 自动插入
}
```

## 【定理推理树·借用安全】

```
借用安全性证明:
────────────────────────────────────────
目标: 借用规则防止数据竞争和UAF

引理1: XOR规则防止数据竞争
  情况1: 多个&借用
    - 所有借用只读
    - 无写操作，无竞争

  情况2: 单个&mut借用
    - 独占写访问
    - 无其他访问，无竞争

  情况3: 所有权移动
    - 原所有者无效
    - 新所有者独占
────────────────────────────────────────
引理2: 生命周期防止UAF
  - 借用生命周期 ⊆ 被借用者生命周期
  - 被借用者不能在借用期间被释放
  - 编译器静态验证
────────────────────────────────────────
定理: Rust借用系统内存安全
  由引理1和引理2，无数据竞争，无UAF ∎
```
