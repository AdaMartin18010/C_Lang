## CompCert 概述

CompCert 是一个经过**形式化验证**的 C 编译器，使用 Coq 证明助手对所有优化 pass 进行数学正确性证明。
它是首个适用于安全关键系统生产的经过完全验证的编译器。

### 核心特性

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                          CompCert 核心特性                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│  ✅ 数学正确性证明     - 所有编译 pass 都经过 Coq 形式化验证                 │
│  ✅ 语义保持定理       - 证明生成的汇编代码与源代码语义等价                   │
│  ✅ 零误编译风险       - 排除编译器引入的错误                                │
│  ✅ 工业级应用         - 用于航空、核能、汽车等安全关键领域                   │
│  ✅ 合理性能           - 生成代码性能约为 GCC -O1 的 90%                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 支持的语言和平台

| 特性 | 支持情况 |
|:-----|:---------|
| C 标准 | **C99 (完整)**, C11/C17 (部分), **C23 (不支持)** |
| 目标架构 | PowerPC, ARM, x86, x86-64, RISC-V |
| 操作系统 | Linux, FreeBSD, NetBSD, OpenBSD |
| ABI | System-V AMD64 ABI |

## 形式化验证原理

### 语义保持定理

CompCert 的核心是证明以下定理：

```coq
Theorem transf_c_program_correct:
  forall prog tprog behavior,
  transf_c_program prog = Some tprog ->
  Csem.exec_program prog behavior ->
  Asm.exec_program tprog behavior.
```

**定理含义**: 如果源程序 S 产生行为 B，那么编译后的程序 C 也产生行为 B。

### 可观察行为定义

可观察行为包括：

1. **终止**: 有限的 I/O 事件跟踪 + 主函数返回的整数（退出码）
2. **发散**: 有限或无限的 I/O 事件跟踪
3. **错误**: 未定义行为（如除以零）

```text
语义保持的含义:
────────────────────────────────────────────────────────
如果源程序 S 满足规范 Σ（所有可观察行为都在 Σ 中）
且编译器将 S 编译为 C
则编译后的程序 C 也满足规范 Σ
────────────────────────────────────────────────────────
```

### 验证方法论

```
源代码 S
    │
    ▼
[Clight 中间语言] ──► Coq 证明: 语义保持
    │
    ▼
[Cminor 中间语言] ──► Coq 证明: 语义保持
    │
    ▼
[RTL 寄存器传输] ──► Coq 证明: 语义保持
    │
    ▼
[LTL 位置传输] ──► Coq 证明: 语义保持
    │
    ▼
[线性化代码] ──► Coq 证明: 语义保持
    │
    ▼
[汇编代码]
```

## 编译器架构

### 三阶段架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  阶段 1: 解析、类型检查和预简化 (未验证)                                      │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 将 C 源代码转换为 CompCert C AST                                         │
│  • 展开块作用域变量、简化非结构化 switch                                    │
│  • 拒绝变长数组等不支持特性                                                 │
│  • 可使用 -dc 标志输出生成的 CompCert C 代码进行检查                        │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  阶段 2: CompCert C 到汇编 AST (✅ 经过 Coq 验证)                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 16 个编译 pass，10 种中间语言                                            │
│  • 每种中间语言都有形式化语义                                               │
│  • 每个 pass 都有语义保持证明                                               │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  阶段 3: 汇编和链接 (未验证)                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 生成汇编代码并调用系统汇编器/链接器                                       │
│  • 可使用 Valex 工具进行事后验证                                            │
│  • 与 GCC 编译的目标文件可链接                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

| 优化 | 描述 | 验证状态 |
|:-----|:-----|:--------:|

| 指令选择 | 利用目标架构组合指令 | ✅ 已验证 |

| 公共子表达式消除 | 避免冗余计算 | ✅ 已验证 |
| 死代码消除 | 移除无用操作 | ✅ 已验证 |
| 函数内联 | 消除小函数调用开销 | ✅ 已验证 |

## 验证工作流

### 安装和使用

```bash
# 安装 CompCert (需要 Coq 和 OCaml)
./configure powerpc-linux  # 或其他目标平台
make
sudo make install

# 基本编译
ccomp -o program program.c

# 生成中间表示进行检查
ccomp -dc -o program.compcert.c program.c  # CompCert C 形式
ccomp -dclight -o program.clight program.c # Clight 形式
ccomp -S -o program.s program.c            # 汇编代码

# 使用参考解释器验证
ccomp -interp program.c  # 解释执行并验证行为
```

## 实践案例

### 内存安全证明

```c
// 使用 ACSL (ANSI/ISO C Specification Language) 注释
/*@
  requires \valid(p) && \valid(q);
  requires \separated(p, q);
  assigns *p, *q;
  ensures *p == \old(*q);
  ensures *q == \old(*p);
*/
void swap(int *p, int *q) {
    int tmp = *p;
    *p = *q;
    *q = tmp;
}
```

### 函数契约示例

```c
// 安全的数组访问
/*@
  requires 0 <= index < length;
  requires \valid_read(array + (0 .. length-1));
  assigns \nothing;
*/
int safe_access(int *array, int length, int index) {
    return array[index];
}
```
