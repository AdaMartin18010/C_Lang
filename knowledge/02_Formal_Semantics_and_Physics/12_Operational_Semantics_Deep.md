## 一、操作语义基础

### 1.1 什么是操作语义

```
操作语义 (Operational Semantics):
用抽象机器的行为来定义程序语言的意义

与其他语义对比：
┌─────────────────────────────────────────────────────────────┐
│  操作语义 (Operational)                                      │
│  • "如何执行"                                                │
│  • 基于状态转换                                               │
│  • 适合实现和验证                                             │
├─────────────────────────────────────────────────────────────┤
│  指称语义 (Denotational)                                     │
│  • "计算什么"                                                │
│  • 数学对象（函数、域）                                       │
│  • 适合等价性证明                                             │
├─────────────────────────────────────────────────────────────┤
│  公理语义 (Axiomatic)                                        │
│  • "满足什么性质"                                            │
│  • 逻辑公式（Hoare三元组）                                    │
│  • 适合程序验证                                               │
├─────────────────────────────────────────────────────────────┤
│  关系：                                                      │
│  操作语义 ⊇ 实现细节                                          │
│  指称语义 = 抽象函数                                          │
│  公理语义 = 逻辑约束                                          │
└─────────────────────────────────────────────────────────────┘

操作语义的形式：
  配置 (Configuration) → 配置

  记法：
  ⟨程序, 状态⟩ ⟶ ⟨程序', 状态'⟩

  或简化为：
  C ⟶ C'
```

### 1.2 大步语义 vs 小步语义

```
大步语义 (Big-step / Natural Semantics):
直接给出最终结果

小步语义 (Small-step / Structural Semantics):
一步一步展示执行过程

对比：
┌─────────────────────────────────────────────────────────────┐
│  大步语义                                                   │
│  ─────────────────────                                      │
│  ⟨e₁, σ⟩ ⇓ n₁    ⟨e₂, σ⟩ ⇓ n₂                             │
│  ─────────────────────────────                              │
│  ⟨e₁ + e₂, σ⟩ ⇓ n₁ + n₂                                    │
│                                                             │
│  优点：简洁，适合类型安全证明                               │
│  缺点：不表达交错、不表达非终止                             │
├─────────────────────────────────────────────────────────────┤
│  小步语义                                                   │
│  ─────────────────────                                      │
│  ⟨e₁, σ⟩ ⟶ ⟨e₁', σ⟩                                        │
│  ──────────────────────────                                 │
│  ⟨e₁ + e₂, σ⟩ ⟶ ⟨e₁' + e₂, σ⟩                              │
│                                                             │
│  优点：表达并发交错、表达无限循环                           │
│  缺点：更多细节，证明更复杂                                 │
└─────────────────────────────────────────────────────────────┘

示例：表达式 (3 + 4) * 5

大步：
  ⟨(3 + 4) * 5, σ⟩ ⇓ 35

小步：
  ⟨(3 + 4) * 5, σ⟩
  ⟶ ⟨7 * 5, σ⟩
  ⟶ ⟨35, σ⟩
```

## 二、表达式语义

### 2.1 算术表达式

```
语法：
  e ::= n | x | e₁ + e₂ | e₁ - e₂ | e₁ * e₂

大步语义规则：

────────────────── (NUM)
⟨n, σ⟩ ⇓ n

σ(x) = n
────────────────── (VAR)
⟨x, σ⟩ ⇓ n

⟨e₁, σ⟩ ⇓ n₁    ⟨e₂, σ⟩ ⇓ n₂
────────────────────────────── (ADD)
⟨e₁ + e₂, σ⟩ ⇓ n₁ + n₂

⟨e₁, σ⟩ ⇓ n₁    ⟨e₂, σ⟩ ⇓ n₂    n₁ ≥ n₂
─────────────────────────────────────── (SUB, guarded)
⟨e₁ - e₂, σ⟩ ⇓ n₁ - n₂

⟨e₁, σ⟩ ⇓ n₁    ⟨e₂, σ⟩ ⇓ n₂
────────────────────────────── (MUL)
⟨e₁ * e₂, σ⟩ ⇓ n₁ * n₂
```

### 2.2 C语言表达式语义（简化）

```
C表达式复杂性：
• 副作用 (i++)
• 序列点
• 未定义行为

小步语义处理副作用：

配置：⟨e, σ, ρ⟩
  e: 表达式
  σ: 存储 (地址 → 值)
  ρ: 环境 (变量 → 地址)

副作用规则：

⟨i, σ, ρ⟩ ⟶ ⟨n, σ, ρ⟩   其中 σ(ρ(i)) = n

⟨i++, σ, ρ⟩ ⟶ ⟨n, σ', ρ⟩
  其中 σ(ρ(i)) = n
        σ' = σ[ρ(i) ↦ n+1]

序列点规则：
在序列点处，所有副作用必须完成

⟨e₁, σ, ρ⟩ ⟶* ⟨v₁, σ₁, ρ⟩    (无副作用或序列点前)
⟨e₂, σ₁, ρ⟩ ⟶* ⟨v₂, σ₂, ρ⟩   (序列点后)
────────────────────────────────────────────
⟨e₁, e₂, σ, ρ⟩ at sequence point
  ⟶ ⟨v₁, v₂, σ₂, ρ⟩
```

## 三、命令式语言语义

### 3.1 简单命令语言

```
语法：
  c ::= skip
      | x := e
      | c₁ ; c₂
      | if b then c₁ else c₂
      | while b do c

大步语义：

────────────────── (SKIP)
⟨skip, σ⟩ ⇓ σ

⟨e, σ⟩ ⇓ n
──────────────────────── (ASSIGN)
⟨x := e, σ⟩ ⇓ σ[x ↦ n]

⟨c₁, σ⟩ ⇓ σ₁    ⟨c₂, σ₁⟩ ⇓ σ₂
──────────────────────────────── (SEQ)
⟨c₁ ; c₂, σ⟩ ⇓ σ₂

⟨b, σ⟩ ⇓ true    ⟨c₁, σ⟩ ⇓ σ'
──────────────────────────────────── (IF-TRUE)
⟨if b then c₁ else c₂, σ⟩ ⇓ σ'

⟨b, σ⟩ ⇓ false   ⟨c₂, σ⟩ ⇓ σ'
──────────────────────────────────── (IF-FALSE)
⟨if b then c₁ else c₂, σ⟩ ⇓ σ'

⟨b, σ⟩ ⇓ false
────────────────── (WHILE-FALSE)
⟨while b do c, σ⟩ ⇓ σ

⟨b, σ⟩ ⇓ true    ⟨c, σ⟩ ⇓ σ₁    ⟨while b do c, σ₁⟩ ⇓ σ₂
──────────────────────────────────────────────────────── (WHILE-TRUE)
⟨while b do c, σ⟩ ⇓ σ₂
```

### 3.2 循环的推导语义

```
循环展开为递归：

while b do c  ≡  if b then (c ; while b do c) else skip

不动点视角：

W = λσ. if ⟦b⟧σ then W(⟦c⟧σ) else σ

其中 ⟦-⟧ 表示语义函数

这个不动点的存在性由Tarski定理保证：
• 在完全格上
• 单调函数有最小不动点

具体计算（迭代）：
  W₀(σ) = ⊥  (未定义/发散)
  Wₙ₊₁(σ) = if ⟦b⟧σ then Wₙ(⟦c⟧σ) else σ
  W = ⊔ₙ Wₙ

语义域的选择：
• 全函数：难以处理非终止
• 部分函数：W(σ)可能未定义
• 提升格 (Lifted domain)：⊥ 表示发散
```

## 四、并发操作语义

### 4.1 交错语义

```
并行命令：c₁ || c₂

小步语义（交错）：

⟨c₁, σ⟩ ⟶ ⟨c₁', σ'⟩
──────────────────────────── (PAR-1)
⟨c₁ || c₂, σ⟩ ⟶ ⟨c₁' || c₂, σ'⟩

⟨c₂, σ⟩ ⟶ ⟨c₂', σ'⟩
──────────────────────────── (PAR-2)
⟨c₁ || c₂, σ⟩ ⟶ ⟨c₁ || c₂', σ'⟩

⟨c₁, σ⟩ ⟶ ⟨skip, σ'⟩
─────────────────────── (PAR-SKIP-1)
⟨c₁ || c₂, σ⟩ ⟶ ⟨c₂, σ'⟩

关键问题：这种语义允许所有可能的交错
实际执行只是其中一条路径

所有可能执行：
  σ₀ ⟶ σ₁ ⟶ σ₂ ... (路径1)
  σ₀ ⟶ σ₁' ⟶ σ₂' ... (路径2)
  ...

不确定性 (Nondeterminism)：
从同一配置可能有多个转换
```

### 4.2 原子操作与同步

```
引入原子块：atomic c

语义规则：

⟨c, σ⟩ ⟶* ⟨skip, σ'⟩   (c无内部交错)
───────────────────────────── (ATOMIC)
⟨atomic c, σ⟩ ⟶ ⟨skip, σ'⟩

互斥锁实现：
  lock(m) = while !test_and_set(m) do skip
  unlock(m) = m := false

信号量：
  P(s): atomic { if s > 0 then s-- else block }
  V(s): atomic { s++ ; wakeup one if any }
```

## 五、存储模型与指针

### 5.1 存储模型

```
存储组件：

┌─────────────────────────────────────────────────────────────┐
│  环境 (Environment) ρ: Var → Loc                            │
│  • 变量到位置的映射                                          │
│  • 支持作用域（嵌套环境）                                     │
│                                                             │
│  存储 (Store) σ: Loc → Val                                  │
│  • 位置到值的映射                                            │
│  • 支持未初始化（部分函数）                                   │
│                                                             │
│  分配 (Allocation) alloc: () → Loc                          │
│  • 生成新位置                                                │
│  • 支持堆/栈区分                                             │
└─────────────────────────────────────────────────────────────┘

malloc语义：

⟨e, σ, ρ⟩ ⇓ n          (计算大小)
l = alloc(n)          (分配n个连续位置)
─────────────────────────────────────
⟨malloc(e), σ, ρ⟩ ⇓ l

语义条件：
• l, l+1, ..., l+n-1 都未在σ中定义
• alloc永不返回已使用位置

free语义：

⟨e, σ, ρ⟩ ⇓ l
───────────────────────────────────────────
⟨free(e), σ, ρ⟩ ⇓ ⟨skip, σ \\ {l, l+1, ...}, ρ⟩
```

### 5.2 别名与指针分析

```
别名问题在语义中的体现：

if p and q alias then:
  [*p := 5] ; [x := *q]  ⇒  x = 5
else:
  [*p := 5] ; [x := *q]  ⇒  x = σ(q)

操作语义不追踪别名关系，需要额外分析：
• 流不敏感：可能别名集合
• 流敏感：程序点特定的别名
• 上下文敏感：调用上下文特定的别名

类型系统可以限制别名：
• 线性类型：唯一引用
• 区域类型：作用域限制
• 所有权：转移语义
```

## 六、与公理语义的关系

### 6.1 语义一致性

```
操作语义 vs Hoare逻辑：

操作语义定义：
  C ⇓ σ'  或  C ⟶* C'

Hoare逻辑公理：
  {P} C {Q}

可靠性 (Soundness)：
  如果 {P} C {Q} 可证，
  那么对于所有满足P的σ，
  如果 ⟨C, σ⟩ ⇓ σ'，则σ'满足Q

完备性 (Completeness)：
  如果对于所有满足P的σ，
  ⟨C, σ⟩ ⇓ σ' 蕴含 σ'满足Q，
  那么 {P} C {Q} 可证

注意：
• 对于图灵完备语言，完备性不成立
• 相对完备性（相对于断言语言）可成立
```

### 6.2 从操作语义提取验证条件

```
符号执行作为操作语义的实例：

符号配置：⟨C, σ, φ⟩
  C: 待执行代码
  σ: 符号状态 (变量 → 符号表达式)
  φ: 路径约束

符号执行规则：

⟨x := e, σ, φ⟩ ⟶ ⟨skip, σ[x ↦ σ(e)], φ⟩

⟨if b then c₁ else c₂, σ, φ⟩
  ⟶ ⟨c₁, σ, φ ∧ σ(b)⟩   (真分支)
  ⊔ ⟨c₂, σ, φ ∧ ¬σ(b)⟩  (假分支)

验证条件生成：
  对于路径 φ ⇒ ψ
  检查 SMT(φ ∧ ¬ψ) 不可满足
```

## 七、实际应用

### 7.1 编译器验证

```
CompCert编译器验证：
• 源语言：Clight (C子集) 的操作语义
• 目标语言：汇编的操作语义
• 证明：每个优化保持语义等价

证明结构：
  ⟦prog⟧_source = ⟦compile(prog)⟧_target

使用模拟关系 (Simulation)：
  源程序的每一步对应目标程序的若干步
  状态间保持特定不变式
```

### 7.2 程序分析工具

```
抽象解释基于操作语义：

具体语义：⟨C, σ⟩ ⟶ ⟨C', σ'⟩

抽象语义：⟨C, σ^#⟩ ⟶ ⟨C', σ'^#⟩
  其中 σ^# ∈ AbstractDomain

伽罗华连接：
  α: Concrete → Abstract
  γ: Abstract → Concrete

正确性条件：
  σ ∈ γ(σ^#) ∧ ⟨C, σ⟩ ⟶ ⟨C', σ'⟩
  ⇒ σ' ∈ γ(σ'^#)

这意味着抽象执行是安全的近似
```
