## 与Lean 4的关联

### 1.1 算法正确性证明

**定理1.1 (二分查找正确性)**:
```
给定有序数组A和值x，
binary_search(A, x) 返回x在A中的索引或None
```

**Lean形式化**:
```lean
theorem binary_search_correct {n : Nat} (A : Vector Nat n) 
    (sorted : ∀ i j, i < j → A[i] ≤ A[j]) (x : Nat) :
    binary_search A x = some i ↔ A[i] = x
```

### 1.2 复杂度分析的形式化

**定理1.2 (主定理)**:
```
T(n) = aT(n/b) + f(n)
其中 a ≥ 1, b > 1
```

**三种情况**:
1. f(n) = O(n^(log_b(a)-ε)): T(n) = Θ(n^(log_b(a)))
2. f(n) = Θ(n^(log_b(a))): T(n) = Θ(n^(log_b(a)) log n)
3. f(n) = Ω(n^(log_b(a)+ε)): T(n) = Θ(f(n))

## 动态规划的形式化

### 2.1 最优子结构

**定理2.1 (最优子结构性质)**:
```
若S是问题P的最优解，
则S的子解是子问题的最优解
```

**关联**: Lean中的递归定义与证明

### 2.2 最长公共子序列

**定理2.2 (LCS最优性)**:
```
LCS(X, Y) = max { |Z| | Z是X和Y的公共子序列 }
```

**递推关系**:
```
LCS[i,j] = if X[i] = Y[j] then LCS[i-1,j-1] + 1
           else max(LCS[i-1,j], LCS[i,j-1])
```

## 图算法与形式化

### 3.1 Dijkstra算法正确性

**定理3.1 (Dijkstra正确性)**:
```
对于带权有向图G=(V,E)，权重w: E → ℝ⁺，
Dijkstra(s) 计算从s到所有v∈V的最短路径
```

**证明要点**:
- 不变式: 已确定距离的顶点集S，满足∀u∈S, d(u) = δ(s,u)
- 每次加入最小距离的顶点
- 归纳证明保持最短性

### 3.2 最大流最小割

**定理3.2 (最大流最小割定理)**:
```
|f*| = c(S, T)
其中f*是最大流，(S,T)是最小割
```

**关联**: 网络流问题的线性规划对偶

## 摊还分析

### 4.1 势能方法

**定理4.1 (势能方法)**:
```
摊还成本 ĉ_i = 实际成本 c_i + Φ(D_i) - Φ(D_{i-1})
```

**总成本界**:
```
Σ c_i ≤ Σ ĉ_i + Φ(D_0) - Φ(D_n)
```

**应用**: 动态表、并查集

## 计算复杂性

### 5.1 P与NP

**定义5.1 (复杂度类)**:
```
P = { L | 存在多项式时间DTM判定L }
NP = { L | 存在多项式时间NTM判定L }
  = { L | 存在多项式时间验证器 }
```

### 5.2 NP完全性

**定理5.1 (Cook-Levin)**:
```
SAT是NP完全的
```

**归约链**:
```
SAT → 3-SAT → Vertex Cover → Independent Set
```

## 与编译的关联

### 6.1 算法选择影响编译

| 算法 | 编译优化 | Lean 4实现 |
|:-----|:---------|:-----------|
| 哈希表 | 内联缓存 | `HashMap` |
| 平衡树 | 尾递归优化 | `RBTree` |
| 图遍历 | 闭包转换 | `Graph` |

### 6.2 复杂度保证的编译

**定理6.1 (编译保持复杂度)**:
```
若M在Lean中有O(f(n))复杂度，
则编译后的C代码保持O(f(n))复杂度
```
