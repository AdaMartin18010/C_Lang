## 一、非终止程序 (Divergence)

### 1.1 经典例子：Ω组合子

**定义**:

```
ω = λx. x x
Ω = ω ω = (λx. x x)(λx. x x)
```

**操作语义**:

```
Ω → (x x)[x↦ω] = ω ω = Ω → Ω → Ω → ...
```

Ω无限归约到自身，不终止。

**指称语义**:

```
⟦Ω⟧ = ⊥  (底元素，表示未定义/发散)
```

**关键问题**:

```
⟦Ω⟧ = ⟦⊥⟧  但  Ω →* ⊥  不成立！
```

**对语义等价定理的影响**:

原始定理（错误版本）:

```
M →* V ⟺ ⟦M⟧ = ⟦V⟧
```

**反例**: `M = Ω`, `V = ⊥`

- `⟦Ω⟧ = ⟦⊥⟧` 成立
- 但 `Ω →* ⊥` 不成立（Ω不归约到任何值）

**修正后的定理**:

```
M →* V ⟺ ⟦M⟧ = ⟦V⟧ 且 ⟦M⟧ ≠ ⊥
```

### 1.2 类型化Ω

在简单类型λ演算中，Ω是**不可类型化**的：

```
λx. x x 需要 x : A → A 且 x : A
这要求 A = A → A，无解
```

**结论**: 类型系统排除了部分非终止程序。

但在**递归类型**中，可以构造类型化的发散项。

### 1.3 在Lean 4中的处理

**Lean策略**:

- 内核归约器设置**最大归约深度**（默认无穷，但可配置）
- 使用`partial`关键字标记可能不终止的函数
- `unsafe`代码可以包含非终止计算

**代码示例**:

```lean
-- 明确非终止
partial def loop : Nat → Nat
| n => loop n  -- 无限循环

-- 尝试求值会挂起或超时
-- #eval loop 0  -- 不终止
```

## 二、运行时错误

### 2.1 头空列表错误

**例子**:

```
head([])
```

**不同语义选择**:

**选择A: 与发散等同** (Haskell风格)

```
⟦head([])⟧ = ⊥
```

问题：无法区分错误和真正的无限循环

**选择B: 显式错误值**

```
⟦head([])⟧ = error
```

需要扩展值域包含错误

**选择C: 类型系统排除** (Lean风格)

```lean
-- head要求非空列表
def head (xs : List α) (h : xs ≠ []) : α := ...

-- 编译时保证不调用head []
```

### 2.2 除零错误

**例子**:

```
1 / 0
```

**Lean 4处理**:

```lean
-- 选项1: 类型系统保证
def div (n d : Nat) (h : d ≠ 0) : Nat := n / d

-- 选项2: Option类型
def safeDiv (n d : Nat) : Option Nat :=
  if d = 0 then none else some (n / d)

-- 选项3: 证明携带代码
def verifiedDiv (n d : Nat) : d ≠ 0 → Nat := ...
```

### 2.3 数组越界

**理论问题**:

```
a[i]  其中 i ≥ length(a)
```

**Lean 4的Fin类型解决方案**:

```lean
-- 索引类型自带边界证明
def get (a : Array α) (i : Fin a.size) : α := a[i]

-- Fin i 表示 {n : Nat // n < i}
-- 编译时保证 i < a.size
```

## 三、经典vs构造语义差异

### 3.1 排中律 (Law of Excluded Middle)

**经典逻辑**:

```
∀A. A ∨ ¬A
```

**构造逻辑**:
上述不成立。必须有A的证明或¬A的证明。

**语义差异**:

| 性质 | 经典语义 | 构造语义 |
|:-----|:---------|:---------|
| 真值 | 二值 {T, F} | 证据 {证明, 反证明, 未知} |
| A ∨ ¬A | 总是真 | 需要证据 |
| ¬¬A → A | 成立 | 不成立 |

### 3.2 双重否定消除

**经典**:

```
¬¬A → A
```

**构造**: 不成立。
从"A不可证"不能推出"A可证"。

**反例**:

```
A = "存在连续统假设的反例"
¬¬A = "假设CH可证不会导致矛盾"
但 ¬¬A → A 不成立，因为我们不知道具体反例
```

**在Lean中**:

```lean
-- 经典逻辑 (可选)
open Classical

theorem dne (A : Prop) : ¬¬A → A :=
  by apply Classical.byContradiction

-- 构造逻辑 (默认)
-- ¬¬A → A 不可证
```

### 3.3 选择公理

**经典** (选择公理AC):

```
∀S : Set(Set α). (∀s ∈ S. s ≠ ∅) → ∃f : S → α. ∀s ∈ S. f(s) ∈ s
```

**构造**:
AC不可接受，因为它提供"非构造性"存在证明。

**弱选择** (可数选择):
构造主义者通常接受可数选择。

**在Lean中**:

```lean
-- 选择公理作为公理提供
axiom choice {α : Sort u} {β : α → Sort v} :
  (∀x, Nonempty (β x)) → Nonempty (∀x, β x)

-- 但证明提取时会跳过（不能计算）
```

## 四、上下文等价 vs 指称等价

### 4.1 定义

**指称等价**:

```
M ≅_den N ⟺ ⟦M⟧ = ⟦N⟧
```

**上下文等价**:

```
M ≅_ctx N ⟺ ∀C[·]. C[M] ⇓ V ⟺ C[N] ⇓ V
```

### 4.2 反例：指称等价但上下文不等价

**例子** (需要高阶类型):

```
M = λx. x + x
N = λx. 2 * x
```

**指称语义** (若考虑外延):

```
⟦M⟧ = ⟦N⟧ = f 其中 f(n) = 2n
```

**但操作语义不同**:

```
M 3 → 3 + 3 → 6
N 3 → 2 * 3 → 6
```

虽然结果相同，但归约路径不同。

**在纯函数式语言中**，这通常不影响等价性。

### 4.3 有副作用时的差异

**例子** (带状态):

```
M = (x := 1; x := 2)
N = (x := 2)
```

**若只看最终值**: 等价
**若看中间状态**: 不等价

**Lean 4处理**: 使用Monad显式跟踪状态

```lean
-- State Monad显式化副作用
def M : State Nat Unit := do
  set 1
  set 2

def N : State Nat Unit := do
  set 2
```

## 五、类型系统边界

### 5.1 类型重建不可判定

**问题**:

```
给定无类型项 M，是否存在类型 τ 使得 ⊢ M : τ？
```

**对于System F**: **不可判定**。

**影响**:

- 类型推断算法可能不终止（实际中很少见）
- Lean 4使用约束求解，可能有复杂情况

### 5.2 统一化失败

**例子**:

```
λx. x x  -- 需要 x : A 且 x : A → B
```

统一化 `A = A → B` **无解**（ occur check 失败）。

**Lean错误**:

```
error: type mismatch
  x
has type
  A
but is expected to have type
  A → B
```

### 5.3 无限类型

**例子**:

```
let rec x = x + 1  -- 若允许，x : ?
```

在简单类型中不允许。
在**同构递归类型**中可以有 `A ≅ A → B`。

## 六、并发与不确定性

### 6.1 交错语义的不确定性

**例子**:

```
并行执行:
  线程1: x := 1
  线程2: x := 2
```

**可能结果**: x = 1 或 x = 2（取决于交错）

**指称语义挑战**:
需要表示**幂域** (Powerdomain):

```
⟦C⟧ : State → P(State)
```

### 6.2 数据竞争

**例子**:

```
线程1读x，线程2写x
```

**未定义行为** (C/C++中)
**在Lean中**: 通过线性类型禁止数据竞争

## 七、证明论边界

### 7.1 Gödel不完备性

**定理**: 任何足够强的形式系统，如果一致，则存在不可判定的命题。

**对Lean的影响**:

- Lean可以表达这些命题
- 但不能证明或证伪所有命题
- 需要添加新公理（如选择公理）来扩展证明能力

### 7.2 停机问题

**定理**: 不存在算法可以判定任意程序是否停机。

**在Lean中**:

```lean
-- 无法定义
-- def halts (M : Program) : Bool := ...
```

**部分解决方案**:

- 只允许**全函数**（可证明终止）
- 使用`partial`标记已知可能不终止的函数

## 八、资源限制

### 8.1 栈溢出

**例子**:

```
非尾递归的递归函数，深度太大
```

**Lean处理**:

- 优化尾调用
- 增加栈大小限制
- 建议程序员使用尾递归

### 8.2 内存限制

**例子**:

```
构造巨大的数据结构
```

**语义不处理**，但在实际执行中会失败。

## 九、数学基础问题

### 9.1 集合论悖论

**Russell悖论**:

```
R = {x | x ∉ x}
R ∈ R ⟺ R ∉ R
```

**解决方案**:

- **类型论**: 分层类型，禁止 x : A 且 x : Set A
- **Lean**: 使用Type Universe层次结构

### 9.2 类型宇宙的一致性

**问题**: `Type : Type` 会导致矛盾。

**Lean解决方案**:

```
Type 0 : Type 1 : Type 2 : ...
```

**边界**: 不允许 `Type n : Type n`。

## 十、最佳实践建议

### 10.1 避免边界情况

1. **使用类型系统排除错误**:
   - `Fin n` 代替裸索引
   - 证明携带的函数

2. **显式处理可能失败**:
   - `Option α` 或 `Except ε α`
   - 不要静默失败

3. **证明终止性**:
   - 使用结构递归
   - 提供良基关系

### 10.2 调试边界情况

**技巧**:

```lean
-- 检查归约过程
set_option trace.Meta.whnf true
set_option trace.Meta.debug true

-- 检查类型推导
set_option trace.Elab true

-- 设置超时防止无限循环
set_option maxHeartbeats 1000000
```

## 十一、结论

边界情况揭示了语义理论的深刻限制：

1. **非终止**: 指称语义使用 ⊥，但操作语义无法到达 ⊥
2. **错误**: 类型系统可以排除部分，但不能全部
3. **经典vs构造**: 深刻的哲学差异影响可计算性
4. **资源限制**: 理论忽略实际限制

**对Lean用户的建议**:

- 利用类型系统排除错误
- 理解`partial`和`unsafe`的风险
- 知道哪些命题不可判定
