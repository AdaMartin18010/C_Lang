```

        │
        │ + 多态
        ↓

        │

        ↓
System Fω (λω)
        │
```

## 二、MLTT到CoC的关联

### 2.1 忠实嵌入定理

**定理2.1.1 (MLTT ↪ CoC忠实)**:

```
存在翻译 ⟦−⟧ : MLTT → CoC 满足：
1. 类型保持: Γ ⊢_MLTT M : A ⟹ ⟦Γ⟧ ⊢_CoC ⟦M⟧ : ⟦A⟧
2. 规约保持: M →_β N ⟹ ⟦M⟧ →_β ⟦N⟧
3. 忠实性: ⟦M⟧ ≡_β ⟦N⟧ ⟹ M ≡_β N
```

**证明构造**:

**翻译定义**:

```
⟦Prop⟧ = Prop                    (证明类型)
⟦Type_i⟧ = Type_i                (数据宇宙)
⟦Πx:A.B⟧ = Πx:⟦A⟧.⟦B⟧            (依赖函数)
⟦Σx:A.B⟧ = Σx:⟦A⟧.⟦B⟧            (依赖对)
⟦W A B⟧ = 归纳定义                (良基树)
```

**验证**:

- MLTT的推理规则在CoC中可模拟
- CoC的Prop提供证明无关性

**结论**: MLTT忠实嵌入CoC ∎

---

### 2.2 保守扩展定理

**定理2.1.2 (CoC保守扩展MLTT)**:

```
对纯MLTT命题φ：
CoC ⊢ ⟦φ⟧ ⟹ MLTT ⊢ φ
```

**证明概要**:

- CoC对MLTT片段的额外证明能力来自Prop
- 但MLTT命题不涉及Prop的额外能力
- 因此不产生新定理

**结论**: 保守扩展 ∎

---

### 2.3 证明提取关联

**定理2.1.3 (提取算法兼容性)**:

```
CoC的提取与MLTT的提取结果一致
```

**机制**:

- MLTT: 擦除Set到Prop的项
- CoC: 擦除Prop类型的项
- 翻译保持可提取性

## 三、CoC到HoTT的关联

### 3.1 基础扩展

**扩展内容**:

```
CoC + {
  Path A a b : Type,           -- 路径类型
  refl a : Path A a a,         -- 自反路径
  J : (P : (b:A) → (p:a=b) → Type) →  -- 路径归纳
      P a refl → (b:A) → (p:a=b) → P b p
}
```

### 3.2 Univalence公理的地位

**定理3.2.1 (UA的强度)**:

```
HoTT = CoC + J + UA
UA ⊢ funext + propext
```

**关系**:

- UA不是CoC的定理（独立性）
- UA蕴含函数外延性（强化）
- UA使等价≃相等（革命性）

---

### 3.3 类型层级的同伦解释

**定理3.3.1 (类型层级对应)**:

```
类型层级    同伦层级        逻辑解释
─────────────────────────────────────────
Contractible (-2)-类型    可证真理
Subsingleton (-1)-类型    命题/真值
Set         0-类型        集合
Groupoid    1-类型        群胚
...         ...           ...
∞-Groupoid  ∞-类型        一般类型
```

**证明思路**:

- 归纳定义n-类型
- 证明层级严格递增
- 展示同伦论对应

## 四、MLTT到HoTT的直接关联

### 4.1 路径类型vs归纳相等

**MLTT相等**:

```
data Eq (A:Type) (a:A) : A → Type where
  refl : Eq A a a
```

**HoTT路径**:

```
Path A a b := I → A   -- 区间映射
```

**关联定理**:

**定理4.1.1 (相等与路径等价)**:

```
Eq A a b ≃ Path A a b
```

**证明**: 两种构造满足相同的归纳原理

---

### 4.2 集合类型的保持

**定理4.1.2 (Set类型在HoTT中)**:

```
MLTT的Set对应HoTT的hSet = Σ(A:Type). isSet(A)
```

**验证**:

- 包含标准数学集合
- 排除高阶结构（如圆）
- 保持经典逻辑推理

## 七、Lean 4中的统一

### 7.1 Lean 4的设计

Lean 4实现了三者的统一：

```
Lean 4 = MLTT核心 + CoC宇宙 + HoTT可选扩展
```

**具体体现**:

- 核心类型论: 基于MLTT
- 宇宙层次: CoC风格
- 可选: HoTT库提供路径类型

### 7.2 实际使用模式

| 场景 | 使用理论 | 关键特性 |
|:-----|:---------|:---------|
| 通用编程 | MLTT | 依赖类型、模式匹配 |
| 证明开发 | CoC | Prop类型、tactics |
| 数学形式化 | HoTT | 同伦推理、univalence |

## 八、未来方向

1. **立方类型论**: 更高效的同伦推理
2. **二维类型论**: 显式同伦层次
3. **线性类型论**: 资源敏感的组合

### 8.2 实践融合

- **Gradual Typing**: 动态与静态的统一
- **Effect Systems**: 计算效果的类型化
- **Refinement Types**: 精化与自动推理
