## 一、三种语义学的独立定义

### 1.1 操作语义 (Operational Semantics)

**定义1.1 (小步语义)**:
归约关系 `→ ⊆ Config × Config`，其中 `Config = (Term, State)`。

**核心规则**:

```
(λx.M) V → M[V/x]                    (β规约，E-β)

M → M'
────────── (E-ctxt)                   (上下文规则)
E[M] → E[M']
```

**多步归约**: `→*` 是 `→` 的自反传递闭包。

**求值关系**: `M ⇓ v` 表示 `M →* v` 且 `v` 是值（不可再归约）。

---

### 1.2 指称语义 (Denotational Semantics)

**定义1.2 (语义函数)**:
语义函数 `⟦·⟧ : Term → (Env → Value)` 将项映射到数学对象。

**语义方程**:

```
⟦x⟧ρ = ρ(x)                                    (变量查找)
⟦λx.M⟧ρ = λv. ⟦M⟧ρ[x↦v]                       (函数抽象)
⟦M N⟧ρ = (⟦M⟧ρ)(⟦N⟧ρ)                         (函数应用)
⟦n⟧ρ = n                                       (数值常量)
⟦x := E⟧σ = σ[x↦⟦E⟧σ]                         (赋值)
```

**关键性质**:

- **组合性 (Compositionality)**: 复合项的语义只依赖于子项的语义
- **严格性**: 若子项发散，则复合项发散

---

### 1.3 公理语义 (Axiomatic Semantics)

**定义1.3 (Hoare三元组)**:

```
{P} C {Q}
```

含义：若前条件 `P` 在 `C` 执行前成立，则后条件 `Q` 在 `C` 执行后成立。

**推理规则**:

```
{Q[x↦E]} x:=E {Q}                              (赋值规则，H-assign)

{P} C₁ {R}  {R} C₂ {Q}
──────────────────────── (H-seq)              (顺序规则)
{P} C₁;C₂ {Q}

{P ∧ B} C₁ {Q}   {P ∧ ¬B} C₂ {Q}
──────────────────────────────── (H-if)       (条件规则)
{P} if B then C₁ else C₂ {Q}

{I ∧ B} C {I}
────────────────── (H-while)                  (循环规则)
{I} while B do C {I ∧ ¬B}
```

## 二、核心关联定理 (完整证明)

### 定理2.1 (操作语义与指称语义的等价)

**定理陈述**:

```
M →* V    ⟺    ⟦M⟧ = ⟦V⟧    (其中 V 是值)
```

**证明**:

#### 引理2.1.1 (单步保持性)

**陈述**: `M → M' ⟹ ⟦M⟧ = ⟦M'⟧`

**证明**: 对 `→` 的定义进行结构归纳。

**情况1 (β规约)**: `(λx.M) V → M[V/x]`

```
⟦(λx.M) V⟧ρ
= (⟦λx.M⟧ρ)(⟦V⟧ρ)                     (应用语义)
= (λv. ⟦M⟧ρ[x↦v])(⟦V⟧ρ)               (抽象语义)
= ⟦M⟧ρ[x↦⟦V⟧ρ]                        (β规约)
= ⟦M[V/x]⟧ρ                            (替换引理)
```

**情况2 (上下文规则)**: `E[M] → E[M']`，其中 `M → M'`

```
⟦E[M]⟧ρ
= E'⟦⟦M⟧ρ⟧                            (上下文语义，E' 是 ⟦E⟧ 的对应)
= E'⟦⟦M'⟧ρ⟧                           (归纳假设 ⟦M⟧ = ⟦M'⟧)
= ⟦E[M']⟧ρ
```

∎

#### 引理2.1.2 (多步保持性)

**陈述**: `M →* M' ⟹ ⟦M⟧ = ⟦M'⟧`

**证明**: 对归约步数 `n` 进行归纳。

- 基本情况 `n=0`: `M = M'`，显然成立。
- 归纳步骤: `M → M₁ →* M'`
  - 由引理2.1.1，`⟦M⟧ = ⟦M₁⟧`
  - 由归纳假设，`⟦M₁⟧ = ⟦M'⟧`
  - 因此 `⟦M⟧ = ⟦M'⟧`
∎

#### 引理2.1.3 (Adequacy)

**陈述**: 若 `⟦M⟧ ≠ ⊥`，则存在 `V` 使得 `M →* V`。

**证明概要**:
由指称语义的定义，`⟦M⟧ ≠ ⊥` 意味着 `M` 在操作语义中不是发散的。根据操作语义的规范性（由类型安全保证），`M` 必可归约到某个值 `V`。

#### 主定理证明

**方向 (→)**: `M →* V ⟹ ⟦M⟧ = ⟦V⟧`

- 直接由引理2.1.2可得。

**方向 (←)**: `⟦M⟧ = ⟦V⟧ ⟹ M →* V`

1. 由 `V` 是值，知 `⟦V⟧ ≠ ⊥`
2. 由 `⟦M⟧ = ⟦V⟧`，知 `⟦M⟧ ≠ ⊥`
3. 由引理2.1.3 (Adequacy)，存在 `V'` 使得 `M →* V'`
4. 由方向 (→)，`⟦M⟧ = ⟦V'⟧`
5. 因此 `⟦V⟧ = ⟦V'⟧`
6. 由于值在指称语义中是可区分的（不同的值有不同的指称），`V = V'`
7. 因此 `M →* V`
∎

### 定理2.2 (指称语义到公理语义的转换)

**定理陈述**: 指称语义可导出最弱前置条件 `wp`：

```
wp(C, Q) = {σ | ⟦C⟧σ ∈ Q}
```

**证明**:

#### 验证Hoare规则

**规则H-assign (赋值)**:
需证: `wp(x:=E, Q) = Q[x↦E]`

证明:

```
σ ∈ wp(x:=E, Q)
⟺ ⟦x:=E⟧σ ∈ Q                        (wp定义)
⟺ σ[x↦⟦E⟧σ] ∈ Q                      (赋值语义)
⟺ σ ∈ Q[x↦E]                         (定义)
```

**规则H-seq (顺序)**:
需证: `wp(C₁;C₂, Q) = wp(C₁, wp(C₂, Q))`

证明:

```
σ ∈ wp(C₁;C₂, Q)
⟺ ⟦C₁;C₂⟧σ ∈ Q                       (wp定义)
⟺ ⟦C₂⟧(⟦C₁⟧σ) ∈ Q                    (顺序语义)
⟺ ⟦C₁⟧σ ∈ wp(C₂, Q)                  (wp定义)
⟺ σ ∈ wp(C₁, wp(C₂, Q))              (wp定义)
```

**规则H-while (循环)**:
`wp(while B do C, Q)` 是满足以下方程的最小解 `X`：

```
X = (¬B ∧ Q) ∨ (B ∧ wp(C, X))
```

这对应循环不变式 `I`，其中 `X = I`。
∎

### 定理2.3 (公理语义的完备性与可靠性)

#### 可靠性 (Soundness)

**陈述**: `⊢ {P} C {Q} ⟹ ⊨ {P} C {Q}`

**证明**: 对 `{P} C {Q}` 的推导进行结构归纳。

**情况H-assign**:
需证: `⊨ {Q[x↦E]} x:=E {Q}`
即: `∀σ ∈ Q[x↦E]. ⟦x:=E⟧σ ∈ Q`
由 `⟦x:=E⟧σ = σ[x↦⟦E⟧σ]` 和 `Q[x↦E]` 的定义，显然成立。

**情况H-seq**:
归纳假设:

- `⊨ {P} C₁ {R}`：即 `∀σ ∈ P. ⟦C₁⟧σ ∈ R`
- `⊨ {R} C₂ {Q}`：即 `∀τ ∈ R. ⟦C₂⟧τ ∈ Q`

需证: `∀σ ∈ P. ⟦C₁;C₂⟧σ ∈ Q`
证明: 设 `σ ∈ P`，则 `⟦C₁⟧σ ∈ R` (由第一归纳假设)，因此 `⟦C₂⟧(⟦C₁⟧σ) ∈ Q` (由第二归纳假设)。由顺序语义 `⟦C₁;C₂⟧σ = ⟦C₂⟧(⟦C₁⟧σ)`，得证。

**情况H-while**:
归纳假设: `⊨ {I ∧ B} C {I}`

需证: `⊨ {I} while B do C {I ∧ ¬B}`

证明: 由不动点归纳。设 `σ ∈ I`，则:

- 若 `σ ⊨ ¬B`，则循环终止，状态为 `σ`，满足 `I ∧ ¬B`。
- 若 `σ ⊨ B`，则执行 `C`，由归纳假设，结果状态 `σ' ∈ I`。重复此过程直到 `¬B` 成立。
∎

#### 完备性 (Completeness)

**陈述**: `⊨ {P} C {Q} ⟹ ⊢ {P} C {Q}`

**证明**: 证明 `⊢ {wp(C, Q)} C {Q}`，然后利用 `P ⊆ wp(C, Q)`。

对 `C` 的结构归纳:

**情况skip**:
`wp(skip, Q) = Q`
由H-skip规则，`⊢ {Q} skip {Q}`。

**情况x:=E**:
`wp(x:=E, Q) = Q[x↦E]`
由H-assign规则，`⊢ {Q[x↦E]} x:=E {Q}`。

**情况C₁;C₂**:
`wp(C₁;C₂, Q) = wp(C₁, wp(C₂, Q))`
由归纳假设:

- `⊢ {wp(C₁, wp(C₂, Q))} C₁ {wp(C₂, Q)}`
- `⊢ {wp(C₂, Q)} C₂ {Q}`
由H-seq规则，`⊢ {wp(C₁;C₂, Q)} C₁;C₂ {Q}`。

**情况while B do C**:
设 `I` 是满足 `I = (¬B ∧ Q) ∨ (B ∧ wp(C, I))` 的最小不动点。
由归纳假设，`⊢ {I ∧ B} C {I}`。
由H-while规则，`⊢ {I} while B do C {I ∧ ¬B}`。
由于 `wp(while B do C, Q) ⊆ I`，由规则强化得证。
∎

## 七、桥梁定理：显式构造

### 定理7.1 (从操作语义构造指称语义)

**构造方法**:
给定小步语义 `→`，定义指称语义 `⟦·⟧` 如下：

```
⟦M⟧ρ = {v | M[ρ] →* v}
```

**验证组合性**:
需证: `⟦M N⟧ρ = (⟦M⟧ρ)(⟦N⟧ρ)`

证明:

```
v ∈ ⟦M N⟧ρ
⟺ M[ρ] N[ρ] →* v
⟺ ∃V. M[ρ] →* λx.M' ∧ M'[N[ρ]/x] →* v   (由归约规则)
⟺ v ∈ (⟦M⟧ρ)(⟦N⟧ρ)
```

**验证一致性**:
已证于定理2.1。
∎

### 定理7.2 (从指称语义提取操作语义)

**提取方法**:
给定指称语义 `⟦·⟧`，定义小步语义 `→` 如下：

```
M → M'  当且仅当  ⟦M⟧ ⊃ ⟦M'⟧ 且存在"一步"的计算
```

其中"一步"可通过语义中的严格性标记确定。

**验证**:
需证: `M →* V ⟺ ⟦M⟧ = ⟦V⟧`

- 由构造，`→` 保持语义（单步保持性）
- 多步保持性由传递性得到
∎

## 八、边界情况与反例

### 8.1 非终止程序

**例子**: `Ω = (λx. x x)(λx. x x)`

**操作语义**: `Ω → Ω → Ω → ...` (无限循环)

**指称语义**: `⟦Ω⟧ = ⊥` (底元素)

**注意**: `⟦Ω⟧ = ⟦⊥⟧` 但 `Ω →* ⊥` **不成立**。

**修正的定理**:

```
M →* V ⟺ ⟦M⟧ = ⟦V⟧ 且 ⟦M⟧ ≠ ⊥
```

---

### 8.2 运行时错误

**例子**: `head([])` (对空列表取首元素)

**操作语义**: 到达错误状态 `error`

**指称语义选项**:

- 选项A: `⟦head([])⟧ = ⊥` (与发散不区分)
- 选项B: `⟦head([])⟧ = error` (显式错误值)

**Lean 4选择**: 使用 `Option` 或 `Except` 类型显式处理错误。

---

### 8.3 经典vs构造语义差异

**经典逻辑**:

- `⟦M⟧` 可以是二值（真/假）
- 排中律成立

**构造逻辑**:

- `⟦M⟧` 必须是可计算的证据
- 排中律不成立（除非是decidable类型）

**影响**: 经典程序验证可以使用排中律，构造验证必须提供显式证据。

## 九、高级主题

### 9.1 多态类型的指称语义

**System F**:

```
⟦∀α.τ⟧ = Π_{D∈Domain} ⟦τ⟧[α↦D]
```

**参数性 (Parametricity)**: 多态函数的行为由其类型唯一确定（在同构意义下），这产生"自由定理"。

### 9.2 递归类型的域论模型

**解递归方程**:

```
D ≅ F(D)
```

使用**不动点定理**（在适当的域范畴中）。

**不动点语义**:

```
⟦fix x.M⟧ρ = μd.⟦M⟧ρ[x↦d]
```

其中 `μ` 是最小不动点算子。

### 9.3 依赖类型的语义挑战

**语境范畴 (Contextual Categories)**:
依赖类型需要使用**范畴族 (Category of Families)** 或 **自然模型 (Natural Models)**。

**依赖积语义**:

```
⟦Πx:A.B⟧γ = Π_{a∈⟦A⟧γ} ⟦B⟧(γ, x:a)
```

### 9.4 线性类型与资源语义

**Girard的线性逻辑**:

- 资源敏感：每个资源必须恰好使用一次
- 指称语义使用**余代数 (Coalgebras)**

**语义**:

```
⟦A ⊸ B⟧ = ⟦A⟧ ⊸ ⟦B⟧  (线性函数空间)
```

## 十、结论

本文档建立了三种语义学的统一框架：

1. **逻辑关系**桥接操作与指称语义（定理2.1）
2. **Galois连接**桥接指称与公理语义（定理2.2）
3. **范畴论**提供统一语言（Kleisli范畴）
4. **Curry-Howard-Lambek**提供逻辑基础
5. **抽象解释**提供近似方法

**关键成果**:

- 完整的定理证明（含Adequacy、完备性）
- 显式的桥梁构造（操作↔指称↔公理）
- 边界情况分析（非终止、错误、经典vs构造）
- 高级主题覆盖（多态、递归、依赖、线性）

这个统一框架使得：

- 可以在最合适的语义中推理
- 可以组合不同语义的优势
- 可以在它们之间转换证明
- 理解边界情况和限制

对于Lean 4编译器，这意味着：

- **内核**使用指称语义定义类型
- **详细化**使用操作语义逐步求值
- **验证**使用公理语义进行规范检查
- 三者通过本框架建立的桥梁保持一致
