# 编译阶段关联：端到端正确性论证

> **目标**: 建立Lean 4编译各阶段的完整关联与保持性证明
> **方法**: 阶段局部保持性 + 传递组合

---

## 一、编译阶段全景图

### 1.1 阶段流程

```
Lean Source Code (L1/L2)
    │
    ▼
[Parser] ───→ Concrete Syntax Tree
    │
    ▼
[Elaborator] ───→ Lean Core (Expr) (L3)
    │              │
    │              ▼
    │         [Kernel] ───→ Verified Core
    │              │
    ▼              ▼
[IR Generator] ───→ Lean IR (L4)
    │
    ├─→ [Optimizer] ───→ Optimized IR
    │
    ▼
[C Codegen] ───→ C Code (L5)
    │
    ▼
[C Compiler] ───→ Machine Code
    │
    ▼
[Linker] ───→ Executable
```

### 1.2 阶段间数据流

| 阶段 | 输入表示 | 输出表示 | 主要转换 |
|:-----|:---------|:---------|:---------|
| 解析 | 文本 | CST | 语法分析 |
| 详细化 | CST | Expr | 类型推断+检查 |
| 核验证 | Expr | Verified Expr | 证明检查 |
| IR生成 | Expr | IR | 中间表示 |
| 优化 | IR | IR | 代码优化 |
| 代码生成 | IR | C | 目标代码 |

## 二、阶段间保持性定理

### 2.1 解析阶段 (L1→L2)

**定理2.1.1 (解析正确性)**:

```
∀s ∈ Source. parse(unparse(parse(s))) = parse(s)
```

**关联论证**:

- 语法树的唯一性保证
- 与表面语义的对应

---

### 2.2 详细化阶段 (L2→L3)

**定理2.2.1 (详细化正确性)**:

```
Γ ⊢_surface M : A    ⟹    ⟦Γ⟧ ⊢_core ⟦M⟧ : ⟦A⟧
```

**关联论证**:

- 表面语法糖到核心演算
- 隐式参数显式化
- 类型推断的完备性

**定理2.2.2 (详细化完备性)**:

```
若存在核心项M\'使得 ⟦M\'⟧ = M，
则详细化器能找到这样的M\'
```

---

### 2.3 核验证阶段 (L3内部)

**定理2.3.1 (核类型检查正确性)**:

```
kernel_check(M) = true  ⟹  M良类型
```

**关联论证**:

- 小核设计保证可靠性
- 与详细化的验证分离
- 证明的完整检查

---

### 2.4 IR生成阶段 (L3→L4)

**定理2.4.1 (IR生成语义保持)**:

```
∀M ∈ Core. ⟦to_IR(M)⟧_IR = ⟦M⟧_core
```

**关联论证**:

- 核心演算到SSA的映射
- 闭包转换的语义保持
- 控制流的显式化

**转换详情**:

```
核心Expr              IR
─────────────────────────────────
λx.M          ↦  function(x) { M\' }
let x = A in B ↦  x = A; B
f a           ↦  call(f, a)
match x with  ↦  switch(x, cases)
```

---

### 2.5 优化阶段 (L4内部)

**定理2.5.1 (优化正确性)**:

```
∀opt ∈ Optimizations. ⟦opt(IR)⟧ = ⟦IR⟧
```

**关联论证**:

- 每个优化的局部保持性
- 优化组合的全局保持性
- 优化顺序的依赖关系

**优化依赖图**:

```
常量折叠 ──┐
           ├──→ 死代码消除
内联 ──────┘      │
           ├──→ 闭包转换 ──→ 尾调用优化
β规约 ─────┘
```

---

### 2.6 代码生成阶段 (L4→L5)

**定理2.6.1 (C代码生成正确性)**:

```
∀IR. exec(codegen(IR)) = eval_IR(IR)
```

**关联论证**:

- IR到C的语义映射
- 内存模型对应
- 调用约定兼容

**关键映射**:

```
IR构造          C构造
─────────────────────────────────
lean_object*    统一对象表示
unboxed types   C原生类型
closure         函数指针+环境
apply           闭包调用协议
```

## 三、端到端正确性

### 3.1 组合定理

**定理3.1.1 (端到端语义保持)**:

```
⟦compile(M)⟧_C = ⟦M⟧_Lean
```

**证明**: 通过各阶段保持性的传递组合

```
⟦M⟧_Lean
   = ⟦⟦M⟧_elaborated⟧_core              (详细化正确性)
   = ⟦⟦⟦M⟧_elaborated⟧_IR⟧_IR           (IR生成正确性)
   = ⟦⟦⟦⟦M⟧_elaborated⟧_IR⟧_opt⟧_IR    (优化正确性)
   = ⟦codegen(⟦⟦⟦M⟧_elaborated⟧_IR⟧_opt)⟧_C  (代码生成正确性)
   = ⟦compile(M)⟧_C
```

**结论**: 端到端语义保持 ∎

---

### 3.2 可靠性推论

**推论3.2.1 (类型安全保持)**:

```
若 ⊢_Lean M : A，则编译后的C代码类型安全
```

**推论3.2.2 (部分正确性)**:

```
若M在Lean中终止，则编译后的代码终止于相同值
```

## 五、横向关联：优化交互

### 5.1 优化组合理论

**定理5.1.1 (优化交换性)**:

```
某些优化可交换顺序：
O₁ ∘ O₂ = O₂ ∘ O₁
```

**定理5.1.2 (优化幂等性)**:

```
某些优化是幂等的：
O ∘ O = O
```

## 六、与类型论的关联

### 6.1 编译即证明

**Curry-Howard视角**:

```
编译过程 : 源程序 → 目标程序
证明转换 : 源证明  → 目标证明
```

**编译正确性 = 证明转换正确性**

### 6.2 类型保持编译

**定理6.2.1 (类型保持)**:

```
Γ ⊢_Lean M : A  ⟹  ⟦Γ⟧ ⊢_C ⟦M⟧ : ⟦A⟧
```

**关联类型**:

- Lean的Π类型 ↦ C的函数指针
- Lean的归纳类型 ↦ C的 tagged union
- Lean的Prop ↦ 擦除

## 七、验证策略

### 7.1 验证层次

```
L6 (元理论)
    │ 证明编译器正确性
    ▼
L5 (目标代码)
    │ 测试执行正确性
    ▼
L4 (IR)
    │ 验证优化等价性
    ▼
L3 (核心)
    │ 类型检查验证
    ▼
L2/L1 (表面)
    │ 解析测试
```

### 7.2 验证方法

| 方法 | 适用阶段 | 保证级别 |
|:-----|:---------|:---------|
| 形式化证明 | L3-L6 | 数学确定性 |
| 模型检测 | L4-L5 | 穷尽状态空间 |
| 测试 | L1-L5 | 统计置信 |
| 类型检查 | L2-L3 | 语法保证 |

## 八、未来扩展方向

### 8.1 形式化缺口

1. **详细化**: 当前缺乏完整的形式化
2. **优化**: 复杂优化的验证困难
3. **代码生成**: C语义的形式化挑战

### 8.2 改进方向

- **Certified Compilation**: 完全验证的编译器
- **Translation Validation**: 每次编译的验证
- **Proof-Carrying Code**: 携带证明的目标代码
