## 三、关键定理详细依赖

### 3.1 替换引理依赖

**定理**: Substitution Lemma

```
Γ, x:A, Δ ⊢ M : B    Γ ⊢ N : A
────────────────────────────────
Γ, Δ[N/x] ⊢ M[N/x] : B[N/x]
```

**依赖**:

- 语法定义 (变量、替换)
- 类型规则的归纳结构
- 变量绑定理论

**被依赖**:

- 类型保持性 (必需)
- β规约语义
- 编译中的内联优化

---

### 3.2 类型保持性依赖

**定理**: Preservation

```
M : A  且  M → N  ⟹  N : A
```

**直接依赖**:

- 替换引理 (Substitution)
- 弱化引理 (Weakening)
- 类型规则的逆推

**间接依赖**:

- 上下文良形性
- 类型唯一性

**启用**:

- 进展性证明中的类型维持
- 多步规约的类型保持

---

### 3.3 强规范化依赖

**定理**: Strong Normalization

```
所有良类型项都有有限规约序列
```

**证明方法依赖**:

- Tait可计算性方法 或
- Girard可约性候选

**依赖概念**:

- 逻辑关系 (Logical Relations)
- 可计算性谓词
- 类型层次的归纳

**启用**:

- 一致性证明
- 类型检查终止性
- 相等判定算法

---

### 3.4 编译保持性依赖

**定理**: Compilation Preservation

```
⟦compile(M)⟧ = ⟦M⟧
```

**依赖链**:

```
语义保持性
    │
    ├──→ 闭包转换语义
    │       ├──→ 环境捕获正确性
    │       └──→ 调用协议正确性
    │
    ├──→ 尾调用优化语义
    │       ├──→ 栈帧复用正确性
    │       └──→ 参数传递保持
    │
    └──→ 内存管理语义
            ├──→ 线性类型保证
            └──→ RC正确性
```

## 六、关键路径分析

### 6.1 到类型安全的最短路径

```
起点: 语法定义
路径: 语法 → 替换引理 → 类型保持 → 进展 → 类型安全
长度: 5步
关键: 替换引理的正确性
```

### 6.2 到编译正确性的最短路径

```
起点: 源语义
路径: 源语义 → 详细化正确性 → IR保持性 → 代码生成正确性
长度: 4步
关键: 阶段间语义对应
```

### 6.3 到一致性的最短路径

```
起点: 类型规则
路径: 类型规则 → SN → 一致性
长度: 3步
关键: 强规范化证明
```

## 七、定理模块化组织

### 7.1 模块依赖图

```
基础模块 (Syntax)
    │
    ├──→ 类型模块 (Typing)
    │       │
    │       ├──→ 计算模块 (Reduction)
    │       │       │
    │       │       └──→ 元性质模块 (Meta)
    │       │
    │       └──→ 语义模块 (Semantics)
    │
    └──→ 证明模块 (Proofs)
            │
            └──→ 编译模块 (Compilation)
```

### 7.2 接口契约

| 模块 | 提供接口 | 依赖接口 |
|:-----|:---------|:---------|
| Syntax | AST, 替换 | 无 |
| Typing | 类型规则 | Syntax |
| Reduction | 规约关系 | Syntax, Typing |
| Meta | SN, Preservation | Typing, Reduction |
| Semantics | 指称 | Syntax |
| Compilation | 编译函数 | 全部 |

## 八、未证明假设与风险

### 8.1 当前形式化缺口

| 假设 | 风险等级 | 缓解策略 |
|:-----|:---------|:---------|
| C编译器正确性 | 高 | 使用CompCert或验证 |
| 详细化完备性 | 中 | 测试覆盖 |
| 运行时系统安全 | 中 | 代码审查 |

### 8.2 依赖循环风险

**潜在循环**:

```
类型保持 → 进展 → 类型安全 → (回指?) → 类型保持
```

**解决**: 确保严格偏序，无循环依赖
