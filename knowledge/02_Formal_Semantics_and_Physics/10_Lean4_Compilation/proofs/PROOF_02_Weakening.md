# 弱化引理 (Weakening): 完整形式化证明

## 1. 定理陈述

### 1.1 正式陈述

**定理 (弱化引理 / Weakening)**:

```
∀Γ, Δ, M, A.
  Γ ⊢ M : A ⟹
  Γ ⊆ Δ ⟹
  ⊢ Δ ⟹
  Δ ⊢ M : A
```

**变体 (插入弱化)**:

```
∀Γ, Δ, M, A, B.
  Γ, Δ ⊢ M : A ⟹
  Γ ⊢ B : Typeᵢ ⟹
  Γ, B, Δ ⊢ M : A
```

**依赖**:

- **引理**: 上下文良形性传递 (Context Well-formedness Transfer)
- **引理**: 变量查找弱化 (Variable Lookup Weakening)
- **定义**: 上下文包含关系 `⊆`
- **定义**: 上下文良形性 `⊢ Γ`

### 1.2 通俗解释

弱化引理陈述了一个直观上显而易见的性质：如果某个项 `M` 在上下文 `Γ` 中具有类型 `A`，那么在我们向 `Γ` 中添加额外的假设（即扩展为更大的上下文 `Δ`）后，`M` 仍然具有相同的类型 `A`。

换句话说，额外的假设不会"破坏"已有的类型推导。这与替换引理形成互补：替换引理告诉我们如何用具体的项替换变量，而弱化引理告诉我们如何安全地扩展假设集合。

在编程语言的语境中，弱化对应着这样一个事实：如果一段代码在某个作用域中类型正确，那么在更大的作用域（包含更多变量声明）中它仍然类型正确。这是模块化编程和作用域嵌套的理论基础。

### 1.3 历史背景

- **起源**: 弱化引理作为结构规则的正式研究可追溯至 Gerhard Gentzen 在 1934-1935 年关于自然演绎和相继式演算的开创性工作。在 Gentzen 的系统中，弱化是 LK 和 LJ 演算的结构规则之一。

- **类型论中的引入**: Haskell Curry 和 Robert Feys 在 1958 年的著作 *Combinatory Logic* 中明确讨论了在类型系统中弱化规则的适用性。

- **现代形式**: Per Martin-Löf 在 1970 年代发展直觉主义类型论时，将弱化作为上下文管理的基本操作。在 Martin-Löf 的表述中，弱化与替换、收缩（contraction）和交换（exchange）一起构成了上下文操作的四大结构规则。

- **线性逻辑的视角**: Jean-Yves Girard 在 1987 年引入线性逻辑，其中弱化不再是普遍适用的规则，而是与特定类型（如 `!A`）相关联。这揭示了弱化与资源管理之间的深刻联系。

- **证明助手机械化**: 在 Coq、Agda 和 Lean 等现代证明助手中，弱化引理是类型检查器核心的一部分，通常作为上下文管理的内部操作实现。

## 2. 证明策略

### 2.1 核心思想

弱化引理的证明基于一个关键观察：**类型推导是局部的**——一个项的类型只依赖于其自由变量在上下文中的声明，而不依赖于上下文中的其他声明。

这意味着当我们向上下文中添加新的假设时，不会影响已有项的类型推导，只要这些新假设不会与已有声明冲突（即保持上下文的良形性）。

证明的核心是展示类型推导规则的**单调性**：如果推导的输入在更大的上下文中有效，则输出也在该上下文中有效。

### 2.2 证明方法选择

**为什么选择结构归纳**:

- 与替换引理类似，类型推导的归纳结构是最自然的证明路径
- 每个类型规则都可以独立分析其单调性

**为什么需要上下文良形性前提**:

- 单纯的 `Γ ⊆ Δ` 不能保证 `Δ` 是良形上下文
- 例如，`Δ` 可能包含矛盾的声明
- 需要 `⊢ Δ` 确保扩展后的上下文有效

**直接证明 vs 弱化-替换组合**:

- 某些表述中，弱化被视为替换的特例（用任意项替换未使用的变量）
- 我们采用独立证明，因为这更符合实际应用

### 2.3 证明草图

```
1. 对类型推导 Γ ⊢ M : A 进行结构归纳
2. 基础情形:
   - 变量: 若 x : A ∈ Γ 且 Γ ⊆ Δ，则 x : A ∈ Δ
   - 常量: 不依赖上下文，直接成立
3. 归纳步骤:
   - λ 抽象: 需证明 Γ, x:A ⊢ M : B 弱化到 Δ, x:A ⊢ M : B
     * 关键: 若 Γ ⊆ Δ 则 Γ, x:A ⊆ Δ, x:A
   - 函数应用: 分别弱化函数和参数的类型推导
   - 类型构造: 类似处理
4. 每种情形验证类型保持不变
5. 组合所有情形
```

### 2.4 与相关证明的比较

**与替换引理的对比**:

- **弱化**: 添加假设，项不变
- **替换**: 替换假设，项变化
- 两者是正交的操作，经常一起使用

**与收缩引理 (Contraction) 的关系**:

- **弱化**: Γ ⊢ M : A ⟹ Γ, x:B ⊢ M : A
- **收缩**: Γ, x:B, y:B ⊢ M : A ⟹ Γ, z:B ⊢ M[z/x, z/y] : A
- 两者是结构规则的对偶

**与交换引理 (Exchange) 的关系**:

- **交换**: 改变上下文中假设的顺序
- 在无序上下文（如集合表示）中，交换是平凡的

## 3. 形式化证明

### 3.1 前置引理

**引理 3.1 (变量查找弱化)**:

**陈述**:

```
∀Γ, Δ, x, A.
  lookup(Γ, x) = Some(A) ⟹
  Γ ⊆ Δ ⟹
  lookup(Δ, x) = Some(A)
```

**证明**:
对 `Γ ⊆ Δ` 的定义进行归纳。上下文包含关系保持变量声明，因此若 `x` 在 `Γ` 中声明为 `A`，则在任何包含 `Γ` 的 `Δ` 中也如此声明。
∎

**引理 3.2 (上下文扩展保持良形性)**:

**陈述**:

```
∀Γ, A, i.
  ⊢ Γ ⟹
  Γ ⊢ A : Typeᵢ ⟹
  ⊢ Γ, x : A
```

**证明**:
由上下文良形性的定义直接可得。良形上下文的扩展规则正是如此定义的。
∎

**引理 3.3 (弱化保持自由变量)**:

**陈述**:

```
∀Γ, M, A.
  Γ ⊢ M : A ⟹
  FV(M) ⊆ dom(Γ)
```

**证明**:
对类型推导进行结构归纳。每个类型规则都保持该性质。
∎

### 3.2 主要证明

**证明** (对类型推导 `Γ ⊢ M : A` 进行结构归纳):

我们证明：对于所有 `Γ, Δ, M, A`，如果 `Γ ⊢ M : A`，`Γ ⊆ Δ`，且 `⊢ Δ`，则 `Δ ⊢ M : A`。

---

**情形 1: 变量规则**

假设推导以变量规则结束:

```
x : A ∈ Γ
─────────── (var)
Γ ⊢ x : A
```

- 由前提 `Γ ⊆ Δ` 和引理 3.1，`x : A ∈ Δ`
- 因此可以应用变量规则得到 `Δ ⊢ x : A`

---

**情形 2: λ 抽象规则**

假设推导以 λ 规则结束:

```
Γ ⊢ A : Typeᵢ    Γ, x : A ⊢ M : B
────────────────────────────────── (λ)
Γ ⊢ λx:A. M : Πx:A. B
```

- 由归纳假设于 `Γ ⊢ A : Typeᵢ`：`Δ ⊢ A : Typeᵢ`
- 对于 `Γ, x : A ⊢ M : B`：
  - 我们需要证明 `(Γ, x : A) ⊆ (Δ, x : A)`
  - 由 `Γ ⊆ Δ`，若 `y : C ∈ Γ, x : A`：
    - 若 `y = x` 且 `C = A`，则 `y : C ∈ Δ, x : A`
    - 若 `y : C ∈ Γ`，则由 `Γ ⊆ Δ` 有 `y : C ∈ Δ`，故 `y : C ∈ Δ, x : A`
  - 因此 `(Γ, x : A) ⊆ (Δ, x : A)`
  - 还需验证 `⊢ (Δ, x : A)`：
    - 由 `⊢ Δ` 和 `Δ ⊢ A : Typeᵢ`（归纳假设结果），根据引理 3.2，`⊢ (Δ, x : A)`
  - 应用归纳假设：`Δ, x : A ⊢ M : B`
- 综合结果，应用 λ 规则：`Δ ⊢ λx:A. M : Πx:A. B`

---

**情形 3: 函数应用规则**

假设推导以应用规则结束:

```
Γ ⊢ M : Πx:A. B    Γ ⊢ N : A
────────────────────────────── (app)
Γ ⊢ M N : B[N/x]
```

- 由归纳假设于 `Γ ⊢ M : Πx:A. B`：`Δ ⊢ M : Πx:A. B`
- 由归纳假设于 `Γ ⊢ N : A`：`Δ ⊢ N : A`
- 应用应用规则：`Δ ⊢ M N : B[N/x]`
- 注意类型 `B[N/x]` 保持不变，因为我们没有替换上下文

---

**情形 4: Π 类型构造规则**

假设推导以 Π-formation 规则结束:

```
Γ ⊢ A : Typeᵢ    Γ, x : A ⊢ B : Typeⱼ
────────────────────────────────────── (Π)
Γ ⊢ Πx:A. B : Typeₖ
```

其中 `k = max(i, j) + 1` 或其他适当的宇宙规则。

- 由归纳假设于 `Γ ⊢ A : Typeᵢ`：`Δ ⊢ A : Typeᵢ`
- 对于 `Γ, x : A ⊢ B : Typeⱼ`，如情形 2 所示：
  - `(Γ, x : A) ⊆ (Δ, x : A)`
  - `⊢ (Δ, x : A)`
  - 因此 `Δ, x : A ⊢ B : Typeⱼ`
- 应用 Π-formation 规则：`Δ ⊢ Πx:A. B : Typeₖ`

---

**情形 5: 宇宙规则**

假设推导以宇宙规则结束:

```
──────────── (sort)
Γ ⊢ Typeᵢ : Typeᵢ₊₁
```

- 宇宙规则不依赖上下文
- 直接应用：`Δ ⊢ Typeᵢ : Typeᵢ₊₁`

---

**情形 6-15: 其他类型构造**

（积类型、和类型、归纳类型等的处理类似，都遵循相同模式：对子推导应用归纳假设，然后重新构造规则）

每种情形下，关键观察是：类型规则的结论类型只依赖于前提中的类型，而这些类型在弱化后保持不变（或按预期变化）。

**边界情况处理**:

- **空上下文**: 若 `Γ = ∅`，则 `∅ ⊆ Δ` 总是成立。若 `⊢ M : A` 在空上下文中成立，则 `M` 是封闭项（无自由变量），在任何上下文中都保持类型 `A`。

- **变量重名**: 若 `Δ` 中包含与 `Γ` 中冲突的声明（如 `x : B` 在 `Δ` 中但 `x : A` 在 `Γ` 中且 `A ≠ B`），这与 `Γ ⊆ Δ` 矛盾，因此不会出现。

- **依赖类型中的上下文依赖**: 在 `Γ ⊢ A : Type` 且 `A` 依赖于 `Γ` 中变量的情形，弱化后 `Δ ⊢ A : Type` 仍然成立，因为 `A` 本身不变。

**综上**, 弱化引理对所有类型推导成立。∎

## 4. 证明分析

### 4.1 证明复杂度

- **证明长度**: 约 80-120 步详细推导
- **归纳深度**: 等于推导树高度
- **情况分析数**: 与类型系统的规则数相同，通常 10-20 种

与替换引理相比，弱化引理的证明更简单，因为没有涉及项的替换操作，只需要处理上下文的扩展。

### 4.2 关键步骤分析

**关键步骤 1: λ 抽象情形中的上下文扩展**

这是证明中最需要仔细处理的部分。当我们从 `Γ, x : A ⊢ M : B` 过渡到 `Δ, x : A ⊢ M : B` 时，需要验证：

1. 扩展后的上下文包含关系成立
2. 扩展后的上下文是良形的

这要求我们对上下文包含关系有精确定义，通常定义为：

```
Γ ⊆ Δ ↔ ∀x A. lookup(Γ, x) = Some(A) → lookup(Δ, x) = Some(A)
```

**关键步骤 2: 依赖类型中的类型保持**

在依赖函数类型 `Πx:A. B` 中，`B` 可能依赖于 `x`。弱化后，`B` 本身不变，因此我们不需要对 `B` 进行任何替换。这与替换引理形成对比。

### 4.3 替代证明路径

**路径 A: 通过替换引理**
假设我们有一个"新鲜"变量 `y` 不在 `M` 中出现，则：

- `Γ, y:B ⊢ M : A`（弱化）
- 等价于 `Γ ⊢ M[y/y] : A[y/y]` = `Γ ⊢ M : A`

这不是一个真正的替代证明，但展示了弱化与替换的关系。

**路径 B: 语义方法**
使用指称语义，弱化对应于投影函数：若 `⟦M⟧` 在环境 `ρ ∈ ⟦Γ⟧` 上定义，则对 `ρ' ∈ ⟦Δ⟧` 其中 `ρ'` 投影到 `ρ`，`⟦M⟧(ρ) = ⟦M⟧(ρ')`。

### 4.4 证明的可机械化

**可在 Lean 4 中形式化吗?**: 是，弱化引理是类型论形式化的标准组件。

**已有形式化版本**:

- Coq: 在 `MetaCoq` 项目中作为上下文管理的一部分
- Agda: 在标准库的 `Context` 模块
- Lean 4: 在 `Mathlib4/TypeTheory` 开发中

**机械化难点**:

1. **上下文表示**: 列表、映射或命名元组的选择影响证明复杂度
2. **包含关系**: 需要形式化 `⊆` 并证明其传递性等性质
3. **良形性传播**: 需要确保弱化操作保持上下文良形性

## 5. 应用与推论

### 5.1 直接推论

**推论 5.1 (空上下文弱化)**:

```
∀Δ, M, A.
  ⊢ M : A (即 ∅ ⊢ M : A) ⟹
  ⊢ Δ ⟹
  Δ ⊢ M : A
```

封闭项在任何良形上下文中保持其类型。

**推论 5.2 (局部弱化)**:

```
∀Γ, M, A, B.
  Γ ⊢ M : A ⟹
  Γ ⊢ B : Typeᵢ ⟹
  Γ, x : B ⊢ M : A
```

在推导中途插入新假设。

### 5.2 实际应用

**在编译器中的使用**:

- **作用域分析**: 验证嵌套作用域中的类型保持
- **模块系统**: 验证导入模块后本地代码的类型保持
- **宏展开**: 宏展开后的代码在新上下文中保持类型

**在类型检查器中的使用**:

- **增量类型检查**: 当上下文扩展时，重用之前的类型检查结果
- **错误恢复**: 在部分类型信息缺失时，基于弱化的假设继续检查

**在程序验证中的使用**:

- **前置条件弱化**: 若 `P ⊢ Q`，则满足 `P` 的状态也满足 `Q`
- **不变式扩展**: 向循环不变式添加额外条件

### 5.3 与其他定理的关系

```
弱化引理 ←──→ 替换引理 (对偶操作)
    │
    ├──→ 上下文管理基础
    │
    ├──→ 用于证明类型保持性
    │
    └──→ 与收缩引理组合构成结构规则

弱化引理 ───→ 单调性推理
    │
    └──→ 子类型推导 (Subtyping)
```

## 6. 常见错误与陷阱

### 6.1 证明中的常见错误

**错误 1: 忽略上下文良形性前提**
**描述**: 仅假设 `Γ ⊆ Δ` 而忽略 `⊢ Δ`。

**为什么错**: `Γ ⊆ Δ` 只保证 `Δ` 包含 `Γ` 的所有声明，不保证 `Δ` 本身良形。例如，`Δ` 可能包含矛盾的声明如 `x : A` 和 `x : B` 其中 `A ≠ B`。

**如何修正**: 始终将 `⊢ Δ` 作为显式前提，或证明 `Γ ⊆ Δ` 且 `⊢ Γ` 蕴含 `⊢ Δ`（这不总是成立，取决于具体定义）。

**错误 2: 在依赖类型中错误应用弱化**
**描述**: 尝试弱化 `Γ ⊢ A : Type` 到 `Γ, x : B ⊢ A : Type` 而不验证 `B` 的良形性。

**为什么错**: 在依赖类型中，`Γ, x : B` 的良形性要求 `Γ ⊢ B : Type`。

**如何修正**: 确保在扩展上下文时验证新假设的类型。

### 6.2 应用中的误用

**误用 1: 在资源敏感类型系统中应用弱化**
在线性逻辑或资源类型系统中，弱化不是普遍有效的规则。错误地应用会导致资源泄漏的验证通过。

**误用 2: 混淆弱化与类型转换**
弱化不改变类型，只是扩展上下文。若看到类型变化，可能是其他操作（如子类型转换）。

## 7. 扩展与变体

### 7.1 不同设置下的版本

**简单类型 λ 演算 (STLC)**:
弱化引理形式相同，但证明更简单，因为没有依赖类型。

**线性逻辑**:

```
Γ ⊢ M : A    (x 在 Γ 中未使用)
──────────────────────────────
Γ, x : B ⊢ M : A
```

弱化仅在变量未使用时适用。

**子类型系统**:

```
Γ ⊢ M : A    Γ ⊆ Δ    ∀x:A ∈ Γ, ∃B. x:B ∈ Δ ∧ A <: B
─────────────────────────────────────────────────────
            Δ ⊢ M : A'
```

需要考虑子类型关系。

### 7.2 强化版本

**定理 7.1 (弱化保持推导结构)**:
若 `D` 是 `Γ ⊢ M : A` 的推导，则存在 `D'` 是 `Δ ⊢ M : A` 的推导，且 `D'` 具有与 `D` 相同的结构。

**区别**: 不仅证明类型存在，还证明推导的结构保持。

### 7.3 弱化与收缩的组合

**结构规则完备性**:

```
Γ ⊢ M : A    σ : Γ → Δ (变量重命名)
────────────────────────────────────
Δ ⊢ M[σ] : A[σ]
```

其中 `σ` 可以包含弱化、收缩和交换。

## 8. Lean 4 形式化代码

```lean
import Mathlib.Tactic

namespace WeakeningLemma

-- 简化的项定义
inductive Term : Type
  | var (n : Nat)
  | abs (ty body : Term)
  | app (fn arg : Term)
  | sort (i : Nat)
  deriving DecidableEq

-- 上下文作为类型列表
def Context := List Term

-- 变量查找
def lookup (Γ : Context) (n : Nat) : Option Term :=
  Γ.get? n

-- 上下文包含: Γ ⊆ Δ 当且仅当 ∀n, lookup Γ n = some A → lookup Δ n = some A
def ctxSubset (Γ Δ : Context) : Prop :=
  ∀ n A, lookup Γ n = some A → lookup Δ n = some A

-- 简化的类型推导 (省略良形性检查以简化)
inductive Derives : Context → Term → Term → Prop
  | var {Γ n A} :
      lookup Γ n = some A →
      Derives Γ (var n) A

  | abs {Γ A B t} :
      Derives (A :: Γ) t B →
      Derives Γ (abs A t) (abs A B)  -- 简化的函数类型表示

  | app {Γ f a A B} :
      Derives Γ f (abs A B) →
      Derives Γ a A →
      Derives Γ (app f a) B

  | sort {Γ i} :
      Derives Γ (sort i) (sort (i+1))

-- 弱化引理
theorem weakening {Γ Δ : Context} {M A : Term}
    (h1 : Derives Γ M A)
    (h2 : ctxSubset Γ Δ) :
    Derives Δ M A := by
  induction h1 with
  | var h_lookup =>
    -- 变量情形：使用包含关系
    apply Derives.var
    exact h2 _ _ h_lookup

  | abs h_t ih =>
    -- λ 抽象情形
    apply Derives.abs
    -- 需要证明 (A :: Γ) ⊆ (A :: Δ)
    have h_extend : ctxSubset (A :: Γ) (A :: Δ) := by
      intro n B h
      cases n with
      | zero =>
        simp [lookup] at h ⊢
        exact h
      | succ n =>
        simp [lookup] at h ⊢
        apply h2
        exact h
    exact ih h_extend

  | app h_f h_a ih_f ih_a =>
    -- 函数应用情形
    apply Derives.app
    · exact ih_f h2
    · exact ih_a h2

  | sort =>
    -- 宇宙规则
    apply Derives.sort

-- 插入弱化的变体
theorem weakening_insert {Γ : Context} {M A B : Term}
    (h1 : Derives Γ M A)
    (h2 : Derives Γ B (sort 0)) :
    Derives (B :: Γ) M A := by
  apply weakening
  · exact h1
  · -- 证明 Γ ⊆ B :: Γ
    intro n A' h
    simp [lookup]
    right
    exact h

end WeakeningLemma
```

**解释**:

- `ctxSubset` 定义了上下文包含关系
- `weakening` 是主要定理，对类型推导进行归纳
- 关键技巧是在 `abs` 情形中证明扩展后的包含关系
- `weakening_insert` 展示了更实用的插入弱化形式

## 9. 教学建议

### 9.1 如何教这个证明

**建议课时**: 60 分钟

**教学顺序**:

1. **动机** (10 分钟): 从作用域嵌套引入
2. **陈述** (10 分钟): 明确前提条件和结论
3. **证明草图** (15 分钟): 强调归纳结构
4. **详细证明** (20 分钟): 重点分析 λ 抽象情形
5. **对比** (5 分钟): 与替换引理对比

**学生常见问题**:

- "为什么需要 `⊢ Δ` 前提?" —— 用包含矛盾声明的反例解释
- "弱化与替换有什么关系?" —— 解释对偶性

### 9.2 练习建议

**练习 1**: 在 STLC 中证明弱化引理

- 目标: 理解基础证明结构
- 难度: 容易

**练习 2**: 证明弱化引理的逆不成立

- 构造反例: `Γ, x:B ⊢ M : A` 但 `Γ` 不包含 `M` 所需的所有声明
- 难度: 中等

**练习 3**: 在线性逻辑片段中分析弱化的适用性

- 目标: 理解资源敏感类型系统
- 难度: 困难
