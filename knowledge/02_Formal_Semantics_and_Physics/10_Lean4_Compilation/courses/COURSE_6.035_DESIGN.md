4. 掌握中间代码表示和优化技术
5. 理解代码生成和运行时系统

1. 能够独立设计和实现一个完整的编译器
2. 能够分析和优化程序性能
3. 能够阅读和评估编译器相关研究论文
4. 能够设计和实现小型领域特定语言(DSL)

#### 素养目标

1. 培养系统级思维和工程能力
2. 培养对程序执行机制的深入理解
3. 培养严谨的形式化思维
4. 培养团队协作和项目管理能力

### 1.4 教学理念

#### 理论 vs 实践平衡

- **理论占比**: 30%
  - 形式语言与自动机理论
  - 类型系统理论
  - 数据流分析框架
  - 程序优化理论

- **实践占比**: 70%
  - 完整的编译器实现项目
  - 5个渐进式编程作业
  - 代码评审和迭代改进

#### 讲授 vs 自学比例

- **课堂讲授**: 每周3小时 × 14周 = 42小时
- **实验/讨论**: 每周2小时 × 12周 = 24小时
- **预期自学时间**: 每周8-12小时
- **比例**: 讲授:实验:自学 ≈ 1:0.6:3

#### 考核哲学

- **强调过程**: 项目里程碑持续评估
- **强调质量**: 代码通过严格的自动化测试
- **强调理解**: 考试考察深层理解而非记忆
- **允许合作**: 鼓励讨论但要求独立实现

## 3. 作业设计详解

### 3.1 理论作业

#### 作业1: 词法分析与正则表达式

- **发布时间**: 第1周周三
- **截止时间**: 第2周周三 11:59 PM
- **预期用时**: 6-8小时
- **设计目的**: 巩固词法分析理论基础，为正则表达式到DFA的转换建立数学直觉
- **知识点覆盖**:
  - 正则表达式运算
  - NFA构造
  - NFA到DFA子集构造法
  - DFA最小化
- **难度分布**: 基础40% / 进阶40% / 挑战20%

**典型题目**:

**题目1: NFA构造 (20分)**

- 为Decaf标识符规则构造NFA
- 要求: 能识别字母开头的字母数字串，区分关键字
- 考察点: NFA设计、ε转移使用

**题目2: 子集构造法 (25分)**

- 给定正则表达式 `(a|b)*abb`，构造等价的DFA
- 要求: 展示完整的状态转移表
- 考察点: 算法执行、状态等价类计算

**题目3: DFA最小化 (25分)**

- 对给定的DFA进行状态最小化
- 要求: 使用Hopcroft算法或表格法
- 考察点: 可区分状态识别、等价类合并

**题目4: 词法冲突解决 (挑战题, 15分)**

- 分析"最长匹配"和"优先级规则"在Decaf中的应用
- 给出消除二义性的策略
- 考察点: 实际编译器设计决策

**常见错误**:

1. NFA构造时遗漏ε转移
2. DFA最小化时状态划分不正确
3. 混淆正则表达式运算符优先级
4. 最长匹配规则理解错误

**评分标准**:

- 完全正确: 100%
- 小错误(如漏标状态): -5%
- 算法步骤缺失: -10%
- 概念错误: -20%

**参考答案要点**:

- 完整的NFA状态转移图
- 正确的子集构造过程表格
- 等价类划分的迭代过程
- 最小化DFA的状态转移表

### 3.2 编程作业

#### Lab 1: Scanner实现

- **环境要求**:
  - OS: Linux/macOS/Windows with WSL
  - Java 11+ 或 OCaml 4.12+
  - JFlex/Camllex 词法分析器生成器
  - 提供的Decaf语言规范文档

- **任务描述**:
  使用词法分析器生成器实现Decaf语言的scanner，识别所有token类型：
  - 关键字 (class, int, boolean, if, else, etc.)
  - 标识符
  - 常量 (整数、布尔值、字符串)
  - 运算符
  - 分隔符

- **输入输出**:
  - 输入: `.df` 源文件
  - 输出: token序列，包含位置信息
  - 错误: 非法字符报告，包含行列号

- **测试用例**:
  - 公开测试 (10个): 基本语法结构
  - 隐藏测试 (15个): 边界情况、错误恢复

- **性能要求**:
  - 时间: 处理1000行代码 < 1秒
  - 内存: < 10MB

- **提交方式**:
  - Git仓库提交
  - 包含源码和Makefile
  - 自动化测试脚本

- **评分细则**:
  - 正确性 (60%): 通过所有测试
  - 错误处理 (20%): 准确的错误报告
  - 代码质量 (10%): 可读性和注释
  - 文档 (10%): README和设计说明

## 7. 与Lean 4的整合

### 7.1 可用Lean形式化的内容

本课程有大量内容适合用Lean 4进行形式化验证：

1. **词法分析的形式化**
   - 正则表达式的形式化语义
   - NFA/DFA等价性证明
   - 子集构造法的正确性

2. **语法分析的形式化**
   - CFG推导的形式化
   - LR分析器的正确性
   - 解析结果与文法的关系

3. **类型系统的形式化**
   - 类型规则的判定过程
   - 类型安全性定理(Progress + Preservation)
   - 子类型关系的传递性

4. **优化的形式化**
   - 数据流方程的解的存在性和唯一性
   - 优化转换的语义保持性
   - SSA形式的正确性

**示例代码**:

```lean4
-- 简单的类型系统形式化
inductive Ty
  | int
  | bool
  | arrow : Ty → Ty → Ty

inductive Expr
  | const : Int → Expr
  | var : String → Expr
  | lam : String → Ty → Expr → Expr
  | app : Expr → Expr → Expr

-- 类型环境
def Context := String → Option Ty

-- 类型推导关系
inductive HasType : Context → Expr → Ty → Prop
  | const {Γ n} : HasType Γ (Expr.const n) Ty.int
  | var {Γ x τ} : Γ x = some τ → HasType Γ (Expr.var x) τ
  | lam {Γ x τ1 τ2 e} :
      HasType (λ y => if y = x then some τ1 else Γ y) e τ2 →
      HasType Γ (Expr.lam x τ1 e) (Ty.arrow τ1 τ2)
  | app {Γ e1 e2 τ1 τ2} :
      HasType Γ e1 (Ty.arrow τ1 τ2) →
      HasType Γ e2 τ1 →
      HasType Γ (Expr.app e1 e2) τ2

-- 类型安全性定理
theorem progress {e τ} :
  HasType (λ _ => none) e τ →
  Value e ∨ ∃ e', Step e e'
```

**证明练习**:
