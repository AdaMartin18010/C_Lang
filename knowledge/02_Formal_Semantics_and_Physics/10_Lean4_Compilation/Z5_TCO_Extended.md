## 定理1：尾调用的栈空间界

### 定理1.1 (栈空间O(1))

**陈述**: 尾调用优化的递归使用O(1)栈空间。

**证明**:
**无优化**: 递归调用创建新栈帧

```
Stack(n) = n × frame_size
```

**有优化**:
复用当前栈帧，不增长栈

```
Stack(n) = frame_size (常数)
```

**形式化**:
对任意递归深度n，栈使用与n无关。

结论：TCO保证O(1)栈空间。∎

## 定理2：尾位置的形式化定义

### 定理2.1 (尾位置归纳定义)

**陈述**: 尾位置可归纳定义为：

**定义**:

```
TailPos(λx.M) = {M}
TailPos(let x = A in B) = {A} ∪ TailPos(B)
TailPos(f a₁...aₙ) = 若f a₁...aₙ是返回值，则为尾位置
TailPos(if C then T else E) = TailPos(T) ∪ TailPos(E)
TailPos(match x with ...) = ∪ᵢ TailPos(caseᵢ)
```

**验证**:
此定义保证尾位置的求值结果直接是父表达式的结果。

结论：定义正确。∎

## 定理3：CPS转换与TCO等价性

### 定理3.1 (CPS ⇔ TCO)

**陈述**: CPS转换后的程序自然支持尾调用优化。

**证明**:
**CPS形式**:

```
M : (A → answer) → answer
```

**尾调用**:
继续传递本身就是尾调用：

```
k (f x)  -- 尾调用
```

**等价性**:

- CPS中所有调用都是尾调用
- CPS转换 = 显式化继续
- TCO = 隐式复用继续

结论：CPS与TCO等价。∎

## 定理4：相互递归的TCO

### 定理4.1 (相互递归优化)

**陈述**: 相互递归函数可被TCO优化。

**证明**:
**场景**:

```
let rec even n = if n=0 then true else odd (n-1)
    and odd n = if n=0 then false else even (n-1)
```

**优化**:

- even调用odd是尾调用
- odd调用even是尾调用
- 复用同一栈帧，只是跳转到不同代码

结论：相互递归TCO正确。∎

## 定理5：尾调用与异常安全

### 定理5.1 (异常安全)

**陈述**: TCO与异常处理兼容。

**证明**:
**问题**: 尾调用可能跳过finally块

**解决方案**:

```c
// 尾调用前执行清理
void f() {
    try {
        return g();  // 尾调用
    } finally {
        cleanup();   // 在跳转前执行
    }
}
```

编译器保证：

- 清理代码在跳转前执行
- 异常表正确处理跳转

结论：TCO异常安全。∎

## 定理6：调试信息与TCO

### 定理6.1 (调试可追溯性)

**陈述**: TCO可与调试信息共存。

**证明**:
**挑战**: TCO消除栈帧，破坏调用链

**方案**:

- 可选的调试模式：禁用TCO保留栈帧
- 或使用尾调用计数器记录跳过节点的信息

**权衡**:

- 调试模式：完整栈追踪，无TCO
- 发布模式：TCO启用，可能丢失部分信息

结论：可通过模式选择平衡。∎

## 定理7：尾递归模构造子

### 定理7.1 (TRMC正确性)

**陈述**: 尾递归模构造子优化保持语义。

**证明**:
**模式**:

```
map f [] = []
map f (x::xs) = f x :: map f xs
```

**观察**: 递归调用在构造子参数位置。

**优化**:
转换为尾递归累加器形式：

```
map_acc f acc [] = reverse acc
map_acc f acc (x::xs) = map_acc f (f x :: acc) xs
```

**验证**:
map_acc是严格尾递归，可被TCO。

结论：TRMC优化正确。∎

## 定理8：TCO与GC交互

### 定理8.1 (GC安全性)

**陈述**: TCO不破坏GC安全。

**证明**:
**潜在问题**:

- 尾调用复用栈帧，可能过早释放局部变量
- 但局部变量若不再使用，释放是安全的

**GC根**:

- 存活变量：参数（新帧）或提升的变量
- 死变量：旧帧中的局部变量

结论：TCO与GC兼容。∎

---

## 定理9：尾调用约定

### 定理9.1 (调用约定兼容性)

**陈述**: 特定调用约定支持尾调用。

**证明**:
**System V AMD64 ABI**:

- 要求caller清理栈
- 支持尾调用：jmp而非call

**问题调用约定**:

- Callee清理（如stdcall）
- 难以支持跨函数的尾调用

**解决方案**:

- 使用统一的调用约定
- 或限制TCO在同调用约定函数间

结论：调用约定决定TCO能力。∎

## 定理10：堆栈溢出防止

### 定理10.1 (TCO防止溢出)

**陈述**: 正确标记的尾递归不会导致栈溢出。

**证明**:
**假设**:

- 递归函数f在尾位置调用自身
- 编译器正确识别并应用TCO

**执行**:

- 每次递归：复用栈帧
- 栈深度：常数（初始调用）

**结论**:
对任意递归深度n，栈使用不增长。
无栈溢出风险。∎

## 尾调用识别决策树

```
调用位置
├── 返回值位置
│   ├── 直接递归调用
│   │   └── 优化: 尾递归优化 (直接跳转)
│   │
│   ├── 相互递归调用
│   │   └── 优化: 相互递归优化 (共享栈帧)
│   │
│   └── 其他函数调用
│       └── 优化: 尾调用优化 (jmp替代call)
│
└── 非返回值位置
    ├── 构造子参数 (TRMC)
    │   └── 优化: TRMC转换 + 尾递归
    │
    └── 其他位置
        └── 无优化 (保持call)
```
