## 定理1：闭包表示的正确性

### 定理1.1 (闭包语义保持)

**陈述**: 闭包转换保持程序语义。

**证明**:
**原语言**: λx.M，其中FV(λx.M)在环境中

**闭包表示**:

```c
typedef struct {
    void (*code)(void *env, value arg, value *result);
    void *env;
} closure_t;
```

**语义对应**:

- 创建闭包: λx.M ↦ (code_ptr, env_ptr)
- 应用: f a ↦ f.code(f.env, a, &result)

**证明**:
对项的结构归纳，验证每次转换保持求值结果。

结论：语义保持。∎

## 定理2：环境布局的最优性

### 定理2.1 (扁平环境最优)

**陈述**: 扁平环境表示在访问时间上最优。

**证明**:
**表示对比**:

- 嵌套环境: 访问深度d变量需d次间接
- 扁平环境: 所有访问为O(1)

**复杂度**:

- 嵌套: O(嵌套深度)
- 扁平: O(1)

结论：扁平环境访问最优。∎

## 定理3：变量逃逸分析

### 定理3.1 (逃逸分析正确性)

**陈述**: 逃逸分析正确识别需要堆分配的变量。

**证明**:
**定义**: 变量x逃逸当它：

- 被存储在堆分配的数据结构中
- 被返回或传递给未知函数

**分析**:

- 局部分析：检查变量的使用模式
- 全局分析：检查函数调用图

**验证**:
若分析判断x不逃逸，则x的分配可安全移至栈上。

结论：逃逸分析正确。∎

## 定理4：闭包内联的判定条件

### 定理4.1 (内联安全性)

**陈述**: 无逃逸闭包可安全内联。

**证明**:
**条件**:

1. 闭包只在创建点使用
2. 环境不逃逸
3. 函数是已知的（非高阶）

**安全性**:
内联后，环境变为局部变量，无堆分配。

结论：满足条件时内联安全。∎

## 定理5：函数提升的完备性

### 定理5.1 (lambda提升完备)

**陈述**: 所有局部函数可被提升至顶层。

**证明**:
**转换**:

```
λx. let f = λy. x + y in f 5
```

转换为：

```c
int f_lifted(int x, int y) { return x + y; }
int main() { return f_lifted(x, 5); }
```

**完备性**:
任何局部函数都可转换为带额外参数的顶层函数。

结论：lambda提升完备。∎

## 定理6：环境共享优化

### 定理6.1 (共享安全性)

**陈述**: 相同作用域的闭包可共享环境。

**证明**:
**条件**:

- 闭包在同一作用域创建
- 捕获相同变量集

**安全性**:
环境是只读的（函数参数不可变）。
共享不会引入别名问题。

结论：共享安全。∎

## 定理7：闭包GC安全性

### 定理7.1 (闭包GC安全)

**陈述**: 闭包正确参与垃圾回收。

**证明**:
**GC可见性**:

- 闭包本身是GC对象
- 环境包含的指针是GC根

**可达性**:
闭包存活当且仅当：

- 闭包本身可达
- 环境中的变量可达

结论：GC正确处理闭包。∎

## 定理8：部分应用的正确性

### 定理8.1 (部分应用语义)

**陈述**: 部分应用保持柯里化语义。

**证明**:
**柯里化**: f : A → B → C等价于f : A → (B → C)

**部分应用**:

- f a : B → C，其中f : A → B → C，a : A

**闭包表示**:

```c
closure_t *part_apply(closure_t *f, value a) {
    return make_closure(f->code, cons(a, f->env));
}
```

**验证**:
(part_apply f a) b = f a b，对所有b成立。

结论：部分应用正确。∎

## 定理9：尾调用与闭包优化

### 定理9.1 (尾调用闭包优化)

**陈述**: 尾位置应用可避免闭包分配。

**证明**:
**尾调用**:

```
let rec f x = ... g y ...
```

若g y在尾位置，则不需要为g创建闭包。

**优化**:
直接调用g.code，复用当前栈帧。

结论：尾调用优化闭包。∎

## 定理10：环境压缩

### 定理10.1 (环境压缩正确)

**陈述**: 未使用变量可从环境中移除。

**证明**:
**活跃分析**:
识别闭包体中实际使用的自由变量。

**压缩**:
仅保留活跃变量在环境中。

**正确性**:
移除未使用变量不影响闭包行为。

结论：压缩正确。∎

## 定理11：一级函数到C的映射

### 定理11.1 (函数指针等价性)

**陈述**: 无自由变量的函数可映射为C函数指针。

**证明**:
**情况**:

- 全局函数: 直接映射为函数指针
- 闭包但无捕获: 静态闭包，code指针即函数指针

**验证**:
调用约定匹配：

- Lean: lean_object* (*)(lean_object*)
- C: lean_object* (*)(lean_object*)

结论：映射保持语义。∎

## 优化策略决策树

```
函数类型
├── 无自由变量
│   └── 优化为: 普通函数指针
│
├── 只在创建点使用
│   └── 优化为: 内联 + β规约
│
├── 静态已知调用点
│   └── 优化为: 直接调用 (避免间接)
│
└── 高阶/逃逸
    ├── 环境小 (≤3变量)
    │   └── 优化为: 扁平闭包 (值内联)
    │
    └── 环境大
        └── 优化为: 堆分配环境 + 引用计数
```
