## 二、桥梁定理 BT-1：操作 → 指称

### 2.1 定理陈述

**给定**: 小步操作语义 `→ ⊆ Config × Config`

**构造**: 指称语义 `⟦·⟧ : Term → (Env → Value)`

**满足**:

1. 组合性: `⟦M N⟧ρ = (⟦M⟧ρ)(⟦N⟧ρ)`
2. 一致性: `M →* V ⟺ ⟦M⟧ = ⟦V⟧`

### 2.2 构造方法

**步骤1: 定义值域**

```
Value = ℕ + Bool + (Value → Value) + ...
```

值域必须包含所有可能的计算结果。

**步骤2: 定义语义函数**

```
⟦M⟧ρ = λσ. lim_{n→∞} eval_n(M, ρ, σ)
```

其中 `eval_n` 是受限的求值函数：

```
eval_0(M, ρ, σ) = ⊥                          (0步不计算任何内容)
eval_{n+1}(M, ρ, σ) =
  case M of
    x       → ρ(x)
    λx.M'   → λv. eval_n(M', ρ[x↦v], σ)
    M₁ M₂   → let f = eval_n(M₁, ρ, σ)
                  v = eval_n(M₂, ρ, σ)
              in f(v)  (若f是函数，否则⊥)
    ...
```

**步骤3: 证明组合性**

**引理**: `⟦M N⟧ρ = (⟦M⟧ρ)(⟦N⟧ρ)`

**证明**:

```
⟦M N⟧ρ
= λσ. lim_{n} eval_{n}(M N, ρ, σ)
= λσ. lim_{n} let f = eval_{n-1}(M, ρ, σ)
                  v = eval_{n-1}(N, ρ, σ)
              in f(v)
= λσ. (lim_{n} eval_{n}(M, ρ, σ))(lim_{n} eval_{n}(N, ρ, σ))
= (⟦M⟧ρ)(⟦N⟧ρ)
```

**步骤4: 证明一致性**

**方向 (→)**: `M →* V ⟹ ⟦M⟧ = ⟦V⟧`

- 由 `eval_n` 的定义，归约保持 `eval_n` 结果
- 极限也保持相等

**方向 (←)**: `⟦M⟧ = ⟦V⟧ ⟹ M →* V`

- 需要操作语义的**正规化定理**
- 由 Church-Rosser 和 Adequacy 保证

∎

### 2.3 构造性示例

**例子**: 构造简单算术表达式的指称语义

**操作语义**:

```
n → n                                  (数值已经是值)
E₁ + E₂ → n₁ + n₂  若 E₁ → n₁, E₂ → n₂
```

**构造的指称语义**:

```
⟦n⟧ = n
⟦E₁ + E₂⟧ = ⟦E₁⟧ + ⟦E₂⟧
```

**验证**:

- 组合性: `⟦E₁ + E₂⟧` 只依赖于 `⟦E₁⟧` 和 `⟦E₂⟧` ✓
- 一致性: `E →* n ⟺ ⟦E⟧ = n` ✓

## 三、桥梁定理 BT-2：指称 → 操作

### 3.1 定理陈述

**给定**: 指称语义 `⟦·⟧ : Term → (Env → Value)`

**构造**: 小步操作语义 `→`

**满足**:

1. 单步保持: `M → M' ⟹ ⟦M⟧ = ⟦M'⟧`
2. 完备性: 若 `⟦M⟧` 是确定的，则 `M` 可规约

### 3.2 构造方法

**步骤1: 定义"一步"**
从语义中提取严格性信息：

```
M → M'  当且仅当
  ⟦M⟧ 包含一个可约的子计算
  且 M' 是执行该归约后的结果
```

**形式化定义**:

```
M → M' ⟺
  ∃E, R, R'. M = E[R] ∧ M' = E[R'] ∧
  R 是 redex (可归约表达式) ∧
  R' 是 R 的 contractum (归约结果) ∧
  ⟦R⟧ ⊃ ⟦R'⟧ (语义包含关系)
```

**步骤2: 定义 redex 集合**

```
Redex = {(λx.M)V | V 是值} ∪     (β-redex)
        {n₁ + n₂ | n₁, n₂ 是数值} ∪  (δ-redex)
        ...
```

**步骤3: 定义 contractum**

```
contractum((λx.M)V) = M[V/x]
contractum(n₁ + n₂) = n₁ + n₂  (算术求和)
...
```

**步骤4: 验证单步保持**
由构造，`→` 只在保持语义的情况下允许。

∎

### 3.3 与BT-1的互逆性

**定理**: BT-1 和 BT-2 互逆（在同构意义下）

**证明概要**:

- 从操作语义 `→` 构造 `⟦·⟧`，再提取 `→'`，则 `→ = →'`
- 从指称语义 `⟦·⟧` 提取 `→`，再构造 `⟦·⟧'`，则 `⟦·⟧ = ⟦·⟧'`

∎

## 四、桥梁定理 BT-3：指称 → 公理

### 4.1 定理陈述

**给定**: 指称语义 `⟦·⟧ : Command → (State → State)`

**构造**: 最弱前置条件 `wp(C, Q) = {σ | ⟦C⟧σ ∈ Q}`

**满足**: Hoare规则在 `wp` 下成立

### 4.2 构造方法

**步骤1: 定义 wp**

```
wp(C, Q) = {σ ∈ State | ⟦C⟧σ ∈ Q}
```

**步骤2: 验证 Hoare 规则**

**H-assign**:

```
wp(x:=E, Q) = {σ | σ[x↦⟦E⟧σ] ∈ Q} = Q[x↦E]
```

**H-seq**:

```
wp(C₁;C₂, Q)
= {σ | ⟦C₁;C₂⟧σ ∈ Q}
= {σ | ⟦C₂⟧(⟦C₁⟧σ) ∈ Q}
= {σ | ⟦C₁⟧σ ∈ wp(C₂, Q)}
= wp(C₁, wp(C₂, Q))
```

**H-if**:

```
wp(if B then C₁ else C₂, Q)
= {σ | if ⟦B⟧σ then ⟦C₁⟧σ ∈ Q else ⟦C₂⟧σ ∈ Q}
= (B ∧ wp(C₁, Q)) ∨ (¬B ∧ wp(C₂, Q))
```

**H-while**:
`wp(while B do C, Q)` 是以下方程的最小不动点：

```
X = (¬B ∧ Q) ∨ (B ∧ wp(C, X))
```

**步骤3: 构造性求解**
使用 Kleene 不动点定理：

```
wp(while B do C, Q) = ⨆_{n} X_n

其中:
  X_0 = ⊥
  X_{n+1} = (¬B ∧ Q) ∨ (B ∧ wp(C, X_n))
```

∎

### 4.3 最强后置条件 sp

**对偶构造**:

```
sp(C, P) = {⟦C⟧σ | σ ∈ P}
```

**与 wp 的关系**:

```
sp(C, P) ⊆ Q  ⟺  P ⊆ wp(C, Q)
```

这正是 Galois 连接的定义。

## 五、桥梁定理 BT-4：公理 → 指称

### 5.1 定理陈述

**给定**: 公理语义（以 wp 或 Hoare 规则给出）

**构造**: 指称语义 `⟦·⟧ : Command → (State → State)`

**满足**: `⟦C⟧σ` 是满足所有后置条件的最小状态

### 5.2 构造方法

**步骤1: 从 wp 提取语义**

```
⟦C⟧σ = 满足 σ ∈ wp(C, Q) 的所有 Q 的交集
```

**步骤2: 验证正确性**
需证: `σ ∈ wp(C, Q) ⟺ ⟦C⟧σ ∈ Q`

**证明**:

- (→): 若 `σ ∈ wp(C, Q)`，则 `⟦C⟧σ` 满足 `Q` 的定义
- (←): 若 `⟦C⟧σ ∈ Q`，则 `σ` 满足 `wp(C, Q)` 的定义

∎

### 5.3 从 Hoare 规则构造

若只有 Hoare 规则而没有显式 wp：

**步骤1: 定义关系 `R_C`**

```
R_C = {(σ, τ) | ∀Q. ⊢ {P} C {Q} 且 σ ∈ P ⟹ τ ∈ Q}
```

**步骤2: 证明 `R_C` 是函数**
需证: 若 `(σ, τ₁) ∈ R_C` 且 `(σ, τ₂) ∈ R_C`，则 `τ₁ = τ₂`

这要求公理语义是**确定的**。

**步骤3: 定义 `⟦C⟧`**

```
⟦C⟧σ = 唯一的 τ 使得 (σ, τ) ∈ R_C
```

∎

## 六、桥梁定理 BT-5：操作 ↔ 公理（直接关系）

### 6.1 定理陈述

**操作 → 公理**:

```
{P} C {Q} 成立当且仅当
∀σ ∈ P. ∀τ. (⟨C,σ⟩ →* ⟨skip,τ⟩ ⟹ τ ∈ Q)
```

**公理 → 操作**:
从 Hoare 证明提取归约序列。

### 6.2 操作 → 公理的构造

**定义**:

```
P = {σ | ⟨C,σ⟩ 可归约}
Q = {τ | ∃σ ∈ P. ⟨C,σ⟩ →* ⟨skip,τ⟩}
```

则 `{P} C {Q}` 成立。

### 6.3 公理 → 操作的提取

**定理**: 若 `⊢ {P} C {Q}`，则可构造 `C` 的求值器。

**方法**:

1. 对每个 Hoare 规则，构造对应的解释器子程序
2. 组合子程序得到完整解释器
3. 可靠性保证解释器正确

## 八、在Lean 4中的应用

### 8.1 内核中的桥梁

**操作语义**: `Kernel.reduce` 实现小步归约
**指称语义**: `Typechecker.whnf` 计算弱头正规形
**关系**: `whnf` 是 `reduce` 的指称语义抽象

### 8.2 Tactic系统中的桥梁

**公理语义**: `MetaM` 的 goals 是前后条件
**指称语义**: 每个 tactic 计算新状态（新 goals）
**操作语义**: tactic 执行是逐步的状态转换

## 九、结论

本文档建立了三种语义之间的显式桥梁：

1. **BT-1**: 操作 → 指称（构造语义函数）
2. **BT-2**: 指称 → 操作（提取归约关系）
3. **BT-3**: 指称 → 公理（提取wp）
4. **BT-4**: 公理 → 指称（构造语义函数）
5. **BT-5**: 操作 ↔ 公理（直接关系）

这些桥梁使得：

- 可以在不同语义框架之间自由转换
- 选择最适合问题的语义视角
- 组合不同语义的优势
- 理解它们之间的精确关系

**关键洞察**:

- 指称语义是三种语义的"中心"
- 操作和公理语义都可以从指称语义提取
- Galois 连接是统一的主题
