## 2. 操作语义 (10题)

### 2.1 入门 ★

#### 习题 2.1.1 [★]

**题目**: 给定简单算术表达式语言：

```
E ::= n | E + E | E × E
```

定义大步操作语义，并求值表达式 `(3 + 4) × 5`。

<details>
<summary>解答</summary>

**语义规则**:

```
───── (NUM)
⟨n,σ⟩ ⇓ n

⟨E₁,σ⟩ ⇓ n₁    ⟨E₂,σ⟩ ⇓ n₂    n = n₁ + n₂
────────────────────────────────────────── (ADD)
⟨E₁ + E₂,σ⟩ ⇓ n

⟨E₁,σ⟩ ⇓ n₁    ⟨E₂,σ⟩ ⇓ n₂    n = n₁ × n₂
────────────────────────────────────────── (MUL)
⟨E₁ × E₂,σ⟩ ⇓ n
```

**求值过程**:

```
⟨3,σ⟩ ⇓ 3      ⟨4,σ⟩ ⇓ 4      7 = 3 + 4
──────────────────────────────────────── (ADD)
⟨3 + 4,σ⟩ ⇓ 7               ⟨5,σ⟩ ⇓ 5      35 = 7 × 5
────────────────────────────────────────────────────── (MUL)
⟨(3 + 4) × 5,σ⟩ ⇓ 35
```

</details>

#### 习题 2.1.2 [★]

**题目**: 解释大步语义与小步语义的优缺点。

<details>
<summary>解答</summary>

**大步语义优点**:

- 简洁，直接给出输入输出关系
- 适合证明程序等价性
- 自然对应函数式理解

**大步语义缺点**:

- 难以表达非终止
- 不适合描述并发交错
- 调试信息少

**小步语义优点**:

- 可表达非终止(无限归约序列)
- 适合描述并发交错执行
- 便于实现解释器(单步执行)

**小步语义缺点**:

- 描述冗长
- 证明复杂度高

</details>

### 2.2 中级 ★★

#### 习题 2.2.1 [★★]

**题目**: 证明 β-归约满足菱形性质：若 `M →β N₁` 且 `M →β N₂`，则存在 `P` 使得 `N₁ →*β P` 且 `N₂ →*β P`。

<details>
<summary>解答</summary>

**证明**: 对 `M` 的结构进行归纳。

**基本情况**:

- `M = (λx.M') V` (β-redex)
  - 情况1: 两个归约都在 `M'` 中
  - 情况2: 两个归约都在 `V` 中
  - 情况3: 一个在 `M'`，一个在 `V`
  由归纳假设可得结论。

- `M = x` (变量): 无归约，平凡成立。

**归纳步骤**:

- `M = M₁ M₂`:
  - 若两个归约在不同子项，独立进行，取 `P` 为同时归约后的项。
  - 若两个归约在同一子项，应用归纳假设。

**关键观察**: β-归约是局部合流的，且由于无类型λ演算的规范性，合流性成立。

</details>

#### 习题 2.2.2 [★★]

**题目**: 为以下命令式语言定义小步语义：

```
C ::= skip | x := E | C₁; C₂ | if B then C₁ else C₂ | while B do C
```

<details>
<summary>解答</summary>

**语义规则**:

```
⟨E,σ⟩ ⇓ n
───────────────── (ASSIGN)
⟨x := E, σ⟩ → skip, σ[x ↦ n]

⟨C₁, σ⟩ → C₁', σ'
────────────────────────── (SEQ_STEP)
⟨C₁; C₂, σ⟩ → C₁'; C₂, σ'

────────────────────────── (SEQ_SKIP)
⟨skip; C₂, σ⟩ → C₂, σ

⟨B,σ⟩ ⇓ true
────────────────────────────────── (IF_TRUE)
⟨if B then C₁ else C₂, σ⟩ → C₁, σ

⟨B,σ⟩ ⇓ false
────────────────────────────────── (IF_FALSE)
⟨if B then C₁ else C₂, σ⟩ → C₂, σ

⟨B,σ⟩ ⇓ true
──────────────────────────────────────────────── (WHILE_TRUE)
⟨while B do C, σ⟩ → C; while B do C, σ

⟨B,σ⟩ ⇓ false
────────────────────────────────── (WHILE_FALSE)
⟨while B do C, σ⟩ → skip, σ
```

</details>

### 2.3 高级 ★★★

#### 习题 2.3.1 [★★★]

**题目**: 证明简单类型λ演算(STLC)的**进展性(Progress)**：若 `∅ ⊢ M : T`，则 `M` 是值或存在 `M'` 使得 `M → M'`。

<details>
<summary>解答</summary>

**证明**: 对推导 `∅ ⊢ M : T` 进行归纳。

**基本情况**:

- **T-VAR**: 不可能，因为上下文为空。
- **T-TRUE/T-FALSE**: `M` 是值。
- **T-ZERO**: `M` 是值。

**归纳步骤**:

- **T-ABS**: `M = λx:T₁.M₂`，是值。
- **T-APP**: `M = M₁ M₂`
  - 由归纳假设，`M₁` 是值或可规约。
  - 若 `M₁` 可规约，则 `M` 可规约。
  - 若 `M₁` 是值，由正则性引理，`M₁ = λx:T₁₁.M₁₂`。
  - 由归纳假设，`M₂` 是值或可规约。
  - 若 `M₂` 可规约，则 `M` 可规约。
  - 若 `M₂` 是值，则 `M → M₁₂[x:=M₂]`。

**关键引理**: 正则性(Canonical Forms) - 若 `∅ ⊢ V : T₁ → T₂` 且 `V` 是值，则 `V = λx:T₁.M`。

</details>

## 3. 指称语义 (8题)

### 3.1 入门 ★

#### 习题 3.1.1 [★]

**题目**: 解释为什么 `⟦Ω⟧ = ⊥`，其中 `Ω = (λx.x x)(λx.x x)`。

<details>
<summary>解答</summary>

**解释**:

`Ω` 是发散项，它无限归约：`Ω → Ω → Ω → ...`

在指称语义中：

- 项的指称是其在域中的元素
- 发散项的指称是底部元素 `⊥`
- 这是因为 `⊥` 表示"无信息"或"未定义"

**形式化**:

```
⟦Ω⟧ = fix(λd.d) = ⨆ᵢ ⊥ = ⊥
```

其中 `fix` 是最小不动点算子，发散项对应于最小不动点 `⊥`。

</details>

#### 习题 3.1.2 [★]

**题目**: 给定指称 `⟦n⟧ = n` (自然数) 和 `⟦E₁ + E₂⟧ = ⟦E₁⟧ + ⟦E₂⟧`，计算 `⟦(2 + 3) + 4⟧`。

<details>
<summary>解答</summary>

**计算过程**:

```
⟦(2 + 3) + 4⟧
= ⟦2 + 3⟧ + ⟦4⟧       (加法指称)
= (⟦2⟧ + ⟦3⟧) + 4     (加法指称)
= (2 + 3) + 4         (数字指称)
= 5 + 4
= 9
```

</details>

### 3.2 中级 ★★

#### 习题 3.2.1 [★★]

**题目**: 证明指称语义中的 `⟦if true then M else N⟧ = ⟦M⟧`。

<details>
<summary>解答</summary>

**证明**:

指称语义定义：

```
⟦if B then M else N⟧ρ =
  if ⟦B⟧ρ then ⟦M⟧ρ else ⟦N⟧ρ
```

当 `B = true` 时：

```
⟦if true then M else N⟧ρ
= if ⟦true⟧ρ then ⟦M⟧ρ else ⟦N⟧ρ
= if true then ⟦M⟧ρ else ⟦N⟧ρ
= ⟦M⟧ρ
```

因此 `⟦if true then M else N⟧ = ⟦M⟧`。

</details>

#### 习题 3.2.2 [★★]

**题目**: 解释什么是指称语义的**组合性(Compositionality)**？

<details>
<summary>解答</summary>

**组合性**: 复合项的指称仅依赖于其子项的指称，而不依赖于子项的语法形式。

**形式化**: 对于每个语法构造子 `op`，存在语义函数 `f_op` 使得：

```
⟦op(M₁, ..., Mₙ)⟧ = f_op(⟦M₁⟧, ..., ⟦Mₙ⟧)
```

**例子**:

- `⟦M N⟧ = ⟦M⟧(⟦N⟧)` (函数应用)
- `⟦M + N⟧ = ⟦M⟧ + ⟦N⟧` (加法)

**重要性**:

- 允许模块化推理
- 支持抽象解释
- 便于验证程序等价性

</details>

### 3.3 高级 ★★★

#### 习题 3.3.1 [★★★]

**题目**: 证明 Adequacy 定理的一个方向：若 `M →* V` (操作语义)，则 `⟦M⟧ ≠ ⊥` (指称语义)。

<details>
<summary>解答</summary>

**证明**: 对归约步数进行归纳。

**基本情况** (0步): `M = V` 已是值。

- `⟦V⟧` 是域中的具体元素，不是 `⊥`。

**归纳步骤**: 假设对 `k` 步成立，证明对 `k+1` 步成立。

- `M → M' →* V`，其中 `M'` 经过 `k` 步到达 `V`。
- 由归纳假设，`⟦M'⟧ ≠ ⊥`。
- 需要证明：若 `M → M'` 则 `⟦M⟧ = ⟦M'⟧`。

**关键引理**: 归约保持指称 (`M → M' ⟹ ⟦M⟧ = ⟦M'⟧`)

- 对归约规则进行案例分析。
- β-归约：`⟦(λx.M) V⟧ = ⟦M⟧[x:=⟦V⟧] = ⟦M[x:=V]⟧`

因此 `⟦M⟧ = ⟦M'⟧ ≠ ⊥`。

</details>

## 4. 公理语义 (8题)

### 4.1 入门 ★

#### 习题 4.1.1 [★]

**题目**: 使用霍尔逻辑证明：`{x = n} x := x + 1 {x = n + 1}`

<details>
<summary>解答</summary>

**证明**:

1. **赋值公理**:

   ```
   {x + 1 = n + 1} x := x + 1 {x = n + 1}
   ```

2. **前置条件强化**:

   ```
   x = n ⟹ x + 1 = n + 1
   ```

3. **结论**:

   ```
   {x = n} x := x + 1 {x = n + 1}
   ```

</details>

#### 习题 4.1.2 [★]

**题目**: 解释最弱前置条件 `wp(C, Q)` 的含义。

<details>
<summary>解答</summary>

**定义**: `wp(C, Q)` 是使命令 `C` 执行后满足 `Q` 的最弱（最一般）前置条件。

**性质**:

1. **有效性**: `{wp(C, Q)} C {Q}` 总是有效
2. **最弱性**: 若 `{P} C {Q}` 有效，则 `P ⟹ wp(C, Q)`

**计算方法**:

- `wp(skip, Q) = Q`
- `wp(x := E, Q) = Q[x:=E]`
- `wp(C₁; C₂, Q) = wp(C₁, wp(C₂, Q))`
- `wp(if B then C₁ else C₂, Q) = (B ∧ wp(C₁, Q)) ∨ (¬B ∧ wp(C₂, Q))`

</details>

### 4.2 中级 ★★

#### 习题 4.2.1 [★★]

**题目**: 为 while 循环定义最弱前置条件 `wp(while B do C, Q)`。

<details>
<summary>解答</summary>

**定义**: `wp(while B do C, Q)` 是不动点方程的解：

```
W = (B ∧ wp(C, W)) ∨ (¬B ∧ Q)
```

**解释**:

- 若 `B` 为真，执行 `C` 后仍需满足 `W` (循环不变量)
- 若 `B` 为假，满足 `Q` (后置条件)

**计算方法**: 寻找满足以下条件的不变量 `I`:

1. `I ∧ B ⟹ wp(C, I)` (保持性)
2. `I ∧ ¬B ⟹ Q` (充分性)

则 `wp(while B do C, Q) = I`。

</details>

#### 习题 4.2.2 [★★]

**题目**: 证明霍尔逻辑赋值规则的可靠性。

<details>
<summary>解答</summary>

**规则**: `{Q[x:=E]} x := E {Q}`

**可靠性证明**:

需证：若 `σ ⊨ Q[x:=E]`，则执行 `x := E` 后 `σ' ⊨ Q`，其中 `σ' = σ[x:=⟦E⟧σ]`。

**证明**:

- `σ ⊨ Q[x:=E]` 意味着 `Q` 在 `x` 替换为 `E` 的值后在 `σ` 中成立。
- 执行 `x := E` 后，`x` 的值变为 `⟦E⟧σ`。
- 因此 `Q` 在新状态 `σ'` 中成立。

形式化：

```
σ ⊨ Q[x:=E]
⟺ σ[x:=⟦E⟧σ] ⊨ Q    (替换引理)
⟺ σ' ⊨ Q            (定义 σ')
```

</details>

### 4.3 高级 ★★★

#### 习题 4.3.1 [★★★]

**题目**: 证明霍尔逻辑的相对完备性：若 `{P} C {Q}` 有效，则可证明。

<details>
<summary>解答</summary>

**证明概要**:

**定理**: 若 `{P} C {Q}` 在指称语义中有效，则霍尔逻辑中存在证明。

**策略**: 使用最弱前置条件 `wp(C, Q)` 作为不变量。

**步骤**:

1. **前置条件强化**: 证明 `P ⟹ wp(C, Q)`
   - 由 `{P} C {Q}` 的有效性，这是成立的。

2. **构造证明**: 对 `C` 的结构归纳
   - **skip**: `wp(skip, Q) = Q`，直接应用 skip 规则。
   - **赋值**: `wp(x:=E, Q) = Q[x:=E]`，直接应用赋值规则。
   - **顺序**: 由归纳假设，`⊢ {wp(C₁, wp(C₂, Q))} C₁ {wp(C₂, Q)}` 和 `⊢ {wp(C₂, Q)} C₂ {Q}`，应用顺序规则。
   - **条件**: 分情况讨论。
   - **循环**: 使用 `wp(while B do C, Q)` 作为不变量，应用 while 规则。

**关键**: `wp(C, Q)` 是连接指称语义与公理语义的桥梁。

</details>

## 5. 类型理论 (10题)

### 5.1 入门 ★

#### 习题 5.1.1 [★]

**题目**: 推导 `λx:Bool. x` 的类型。

<details>
<summary>解答</summary>

**推导**:

```
x : Bool ⊢ x : Bool
──────────────────── (→I)
⊢ λx:Bool. x : Bool → Bool
```

**解释**:

- 变量 `x` 在上下文 `x:Bool` 中有类型 `Bool`
- 通过 →引入规则，抽象 `λx:Bool.x` 有类型 `Bool → Bool`

</details>

#### 习题 5.1.2 [★]

**题目**: 什么是类型保持性(Type Preservation)？

<details>
<summary>解答</summary>

**定义**: 若 `Γ ⊢ M : T` 且 `M → M'`，则 `Γ ⊢ M' : T`。

**意义**: 归约不会破坏类型正确性。

**例子**:

- `M = (λx:Nat. x + 1) 5`
- `Γ ⊢ M : Nat`
- `M → 5 + 1 → 6`
- `Γ ⊢ 6 : Nat` ✓

</details>

### 5.2 中级 ★★

#### 习题 5.2.1 [★★]

**题目**: 证明 System F 中 `Λα.λx:α.x` 的类型是 `∀α.α → α`。

<details>
<summary>解答</summary>

**推导**:

```
x : α ⊢ x : α
──────────────────── (→I)
⊢ λx:α. x : α → α
──────────────────── (∀I)
⊢ Λα.λx:α.x : ∀α.α → α
```

**说明**:

- 内部：`λx:α.x` 有类型 `α → α` (多态 identity)
- 外部：`Λα` 引入全称量词，得到 `∀α.α → α`

这是 System F 中的多态 identity 函数。

</details>

#### 习题 5.2.2 [★★]

**题目**: 解释 let-多态与全称类型多态的区别。

<details>
<summary>解答</summary>

**let-多态 (ML风格)**:

```ocaml
let id = λx.x in
(id 1, id true)  (* 合法：id在不同位置有不同实例 *)
```

- `id` 被实例化为 `Nat → Nat` 和 `Bool → Bool`
- 通过 let 绑定实现多态

**全称类型多态 (System F)**:

```
Λα.λx:α.x : ∀α.α → α
```

- 显式类型抽象 `Λα`
- 显式类型应用 `id[Nat]`
- 更强大但类型标注更多

**区别**:

- let-多态是隐式的，在 let 绑定处泛化
- 全称多态是显式的，需要类型抽象/应用

</details>

### 5.3 高级 ★★★

#### 习题 5.3.1 [★★★]

**题目**: 证明强规范化定理：若 `∅ ⊢ M : T` 在 STLC 中，则 `M` 不存在无限归约序列。

<details>
<summary>解答</summary>

**证明概要** (逻辑关系法):

定义类型解释 `V(T)` - "正规化的项集合"。

**基本思想**:

- 证明良类型项属于相应的逻辑关系
- 证明逻辑关系中的项都正规化

**步骤**:

1. 定义 `V(T)` 归纳于 `T`
2. 证明 **基本引理**: 若 `M ∈ V(T)`，则 `M` 正规化
3. 证明 **基本引理**: 若 `M` 是值且 `M ∈ V(T)`，则...
4. 证明 **基本定理**: 若 `Γ ⊢ M : T` 且 `σ ⊨ Γ`，则 `M[σ] ∈ V(T)`

**结论**: 良类型项都正规化。

</details>

## 6. 范畴语义 (6题)

### 6.1 入门 ★

#### 习题 6.1.1 [★]

**题目**: 解释为什么笛卡尔闭范畴(CCC)适合作为λ演算的语义模型。

<details>
<summary>解答</summary>

**CCC 的结构**:

1. **积 (×)**: 对应配对 `(M, N)` 和投影 `fst/snd`
2. **指数 (⇒)**: 对应函数类型 `A → B`
3. **求值 (eval)**: 对应函数应用
4. **currying**: 对应 λ-抽象

**对应关系**:

| λ演算 | CCC |
|-------|-----|
| 类型 `A → B` | 指数对象 `B^A` |
| 应用 `M N` | `eval ∘ ⟨M, N⟩` |
| 抽象 `λx.M` | `curry(M)` |

因此，CCC 提供了 λ演算的数学模型。

</details>

#### 习题 6.1.2 [★]

**题目**: 定义单子 (Monad) 并给出一个例子。

<details>
<summary>解答</summary>

**定义**: 范畴 **C** 上的单子是三元组 `(T, η, μ)`，其中：

- `T: C → C` 是 endofunctor
- `η: Id ⇒ T` 是单位 (return)
- `μ: T² ⇒ T` 是乘法 (join)

满足结合律和单位律。

**例子 (Maybe monad)**:

```haskell
data Maybe a = Nothing | Just a

return :: a -> Maybe a
return = Just

join :: Maybe (Maybe a) -> Maybe a
join Nothing = Nothing
join (Just Nothing) = Nothing
join (Just (Just x)) = Just x
```

**用途**: 建模部分函数、异常、非确定性等计算效应。

</details>

### 6.2 中级 ★★

#### 习题 6.2.1 [★★]

**题目**: 证明伴随 `F ⊣ G` 诱导单子 `(GF, η, GεF)`。

<details>
<summary>解答</summary>

**给定**: 伴随 `F: C → D`, `G: D → C`，单位 `η: Id_C ⇒ GF`，余单位 `ε: FG ⇒ Id_D`

**构造**:

- `T = GF: C → C`
- `η: Id ⇒ T` (单位)
- `μ = GεF: T² ⇒ T`

**验证结合律**:
需要证明 `μ ∘ Tμ = μ ∘ μT`。

**验证**:

```
GεF ∘ GFGεF = G(ε ∘ F G ε)F = G(ε ∘ ε F G)F = GεF ∘ GεFGF
```

由伴随的三角等式，这成立。

**结论**: 每个伴随都诱导一个单子。

</details>

### 6.3 高级 ★★★

#### 习题 6.3.1 [★★★]

**题目**: 证明 CCC 中的积在同构意义下唯一。

<details>
<summary>解答</summary>

**定理**: 若 `(A × B, π₁, π₂)` 和 `(A ⊗ B, p₁, p₂)` 都是 `A` 和 `B` 的积，则 `A × B ≅ A ⊗ B`。

**证明**:

1. 由积的泛性质，存在唯一的 `f: A × B → A ⊗ B` 使得 `p₁ ∘ f = π₁` 且 `p₂ ∘ f = π₂`。

2. 同理，存在唯一的 `g: A ⊗ B → A × B` 使得 `π₁ ∘ g = p₁` 且 `π₂ ∘ g = p₂`。

3. 证明 `g ∘ f = id_{A×B}`:
   - `π₁ ∘ (g ∘ f) = p₁ ∘ f = π₁ = π₁ ∘ id`
   - `π₂ ∘ (g ∘ f) = p₂ ∘ f = π₂ = π₂ ∘ id`
   - 由唯一性，`g ∘ f = id`

4. 同理，`f ∘ g = id_{A⊗B}`

因此 `A × B ≅ A ⊗ B`。

</details>

## 7. 并发理论 (8题)

### 7.1 入门 ★

#### 习题 7.1.1 [★]

**题目**: 定义强互模拟 (Strong Bisimulation)。

<details>
<summary>解答</summary>

**定义**: 关系 `R` 是强互模拟，如果：

若 `P R Q` 且 `P →α P'`，则存在 `Q'` 使得 `Q →α Q'` 且 `P' R Q'`。

反之亦然（对称性）。

**强互模拟等价** `∼` 是最大的强互模拟：

```
P ∼ Q ⟺ 存在强互模拟 R 使得 P R Q
```

**意义**: 两个进程在所有可能动作上都匹配。

</details>

#### 习题 7.1.2 [★]

**题目**: 解释为什么弱互模拟在 `+` (选择) 算子下不是同余。

<details>
<summary>解答</summary>

**反例**:

令 `τ.P` 表示执行内部动作后成为 `P`。

`τ.P ≈ P` (弱等价，因为内部τ可隐藏)

但 `τ.P + Q ≉ P + Q`:

- `τ.P + Q` 可以先执行 `τ` 变成 `P`，然后...
- `P + Q` 必须立即选择 `P` 或 `Q`

若环境在等待某个动作，而 `τ` 改变了进程状态，则行为不同。

**解决**: 使用同余闭包或观察同余 `≈^c`。

</details>

### 7.2 中级 ★★

#### 习题 7.2.1 [★★]

**题目**: 证明 `P | Q ∼ Q | P` (并行交换律)。

<details>
<summary>解答</summary>

**证明**: 构造互模拟关系 `R = {(P|Q, Q|P) | P, Q 是进程}`。

验证 `R` 是强互模拟：

假设 `P|Q →α R`，需找到 `Q|P →α R'` 使得 `R R'`。

**情况分析**:

- `P|Q` 中 `P` 执行 `α` 变成 `P'`：
  - `P|Q →α P'|Q`
  - `Q|P →α Q|P'` (对称规则)
  - `(P'|Q, Q|P') ∈ R` ✓

- `P|Q` 中 `Q` 执行 `α` 变成 `Q'`：
  - 类似分析

- `P|Q` 中 `P` 和 `Q` 通信：
  - `P →a P'`, `Q →ā Q'`
  - `P|Q →τ P'|Q'`
  - `Q|P →τ Q'|P'` (对称)
  - `(P'|Q', Q'|P') ∈ R` ✓

因此 `P|Q ∼ Q|P`。

</details>

### 7.3 高级 ★★★

#### 习题 7.3.1 [★★★]

**题目**: 陈述并证明 Hennessy-Milner 定理。

<details>
<summary>解答</summary>

**定理**: 对于有限分支的进程，

```
P ∼ Q  ⟺  P 和 Q 满足相同的 HM 公式
```

**HM 逻辑**:

```
φ ::= tt | ff | φ₁ ∧ φ₂ | φ₁ ∨ φ₂ | ⟨a⟩φ | [a]φ
```

**证明概要**:

**(→) 若 P ∼ Q，则 P 和 Q 满足相同公式**:

- 对公式结构归纳证明。
- 模态算子情况使用互模拟定义。

**(←) 若 P 和 Q 满足相同公式，则 P ∼ Q**:

- 定义关系 `R = {(P,Q) | P 和 Q 满足相同公式}`。
- 证明 `R` 是强互模拟。
- 反证法：假设不是互模拟，构造区分公式。

**结论**: 模态逻辑刻画了互模拟等价。

</details>

## 8. 综合挑战 (10题)

### 8.1 中级 ★★

#### 习题 8.1.1 [★★]

**题目**: 为带有引用(refs)的简单语言定义操作语义。

<details>
<summary>解答</summary>

**扩展语法**:

```
M ::= ... | ref M | !M | M := M | l
v ::= ... | l
```

**存储**: `σ: Loc → Val` (部分函数)

**语义规则**:

```
⟨M,σ⟩ ⇓ ⟨v,σ'⟩    l ∉ dom(σ')
──────────────────────────────── (REF)
⟨ref M,σ⟩ ⇓ ⟨l,σ'[l↦v]⟩

⟨M,σ⟩ ⇓ ⟨l,σ'⟩    σ'(l) = v
──────────────────────────── (DEREF)
⟨!M,σ⟩ ⇓ ⟨v,σ'⟩

⟨M₁,σ⟩ ⇓ ⟨l,σ'⟩    ⟨M₂,σ'⟩ ⇓ ⟨v,σ''⟩
──────────────────────────────────────── (ASSIGN)
⟨M₁ := M₂,σ⟩ ⇓ ⟨unit,σ''[l↦v]⟩
```

**注意**: 引用需要处理存储和位置。

</details>

#### 习题 8.1.2 [★★]

**题目**: 证明 de Bruijn 索引版本的 λ演算避免α转换。

<details>
<summary>解答</summary>

**de Bruijn 表示**:

- 变量不用名字，用数字表示绑定距离
- `λ. λ. 1` (外层绑定)
- `λ. λ. 0` (内层绑定)

**例子**:

- `λx.λy.x y` → `λ.λ.1 0`
- `λy.λx.y x` → `λ.λ.1 0` (相同！)

**优点**:

- `λx.x` 和 `λy.y` 都表示为 `λ.0`
- 无需α转换比较项
- 替换变为数值移位操作

**实现**:

```
shift c d (Var k) = if k < c then Var k else Var (k+d)
shift c d (Abs M) = Abs (shift (c+1) d M)
shift c d (App M N) = App (shift c d M) (shift c d N)
```

</details>

### 8.2 高级 ★★★

#### 习题 8.2.1 [★★★]

**题目**: 使用逻辑关系证明 System F 的正规化。

<details>
<summary>解答</summary>

**证明概要**:

**逻辑谓词** `V(T)` 归纳定义于类型 `T`：

**基本类型**:

- `V(Bool) = {M | M ⟹* true 或 M ⟹* false}`

**函数类型**:

- `V(T₁ → T₂) = {M | ∀N ∈ V(T₁). M N ∈ V(T₂)}`

**全称类型**:

- `V(∀α.T) = {M | ∀S. M[S] ∈ V(T[α:=S])}`

**基本定理**: 若 `∅ ⊢ M : T`，则 `M ∈ V(T)`。

**证明**: 对类型推导进行归纳，使用替换引理。

**结论**: `V(T)` 中的项都可正规化。

</details>
