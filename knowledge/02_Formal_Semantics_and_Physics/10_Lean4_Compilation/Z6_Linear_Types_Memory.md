## 【科学·线性类型形式化】

### 1. 线性逻辑连接词

**语法**:

```
A, B ::= A ⊗ B | A ⊸ B | !A | A & B | A ⊕ B | 1 | 0 | ⊤ | ⊥
```

**语义**:

| 连接词 | 名称 | 直觉 |
|:-------|:-----|:-----|
| A ⊗ B | 多plicative合取 | 同时拥有A和B |
| A ⊸ B | 线性蕴含 | 消耗A得到B |
| !A | 指数模态 | 可复制的A |
| A & B | additive合取 | 选择A或B |
| A ⊕ B | additive析取 | A或B |

### 2. 线性类型规则

**⊗-引入**:

```
Γ ⊢ t : A    Δ ⊢ u : B
───────────────────────
Γ, Δ ⊢ (t, u) : A ⊗ B
```

**⊸-引入**:

```
Γ, x : A ⊢ t : B
─────────────────
Γ ⊢ λx.t : A ⊸ B
```

**⊸-消去**:

```
Γ ⊢ t : A ⊸ B    Δ ⊢ u : A
───────────────────────────
Γ, Δ ⊢ t u : B
```

**关键**: 上下文Γ和Δ是分离的，资源不能复制。

### 3. 指数模态 (!)

**!-引入** (弱化 + 收缩):

```
!Γ ⊢ t : A
───────────
!Γ ⊢ !t : !A
```

**!-消去**:

```
Γ ⊢ t : !A    Δ, x : A ⊢ u : B
───────────────────────────────
Γ, Δ ⊢ let !x = t in u : B
```

**解释**: !A表示可复制的资源（无限制使用）。

## 【工程·Lean 4实现】

### 1. 借用检查的形式化

**定义**: 借用 @&T 创建只读引用

**规则**:

```
Γ ⊢ t : T
─────────────
Γ ⊢ &t : @&T
```

**线性类型解释**: @&T 是 T 的线性借用，不消耗所有权。

### 2. 引用计数的线性逻辑基础

**定理**: 引用计数实现线性类型的擦除语义。

**证明**:

- RC=1 ↔ 线性资源（必须恰好使用一次）
- RC=0 ↔ 资源已释放
- RC>1 ↔ !A（可复制资源）

### 3. 代码示例

**Lean 4**:

```lean
def useLinear (x : @&Nat) : Nat := x + 1
-- x的RC不变
```

**提取到C**:

```c
// 无RC操作！
uint64_t use_linear(uint64_t x) {
  return x + 1;
}
```

**对比（非借用）**:

```c
// 需要inc/dec
lean_object* use_copy(lean_object* x) {
  lean_inc(x);  // +1
  lean_object* r = lean_nat_add(x, lean_nat_one());
  lean_dec(x);  // -1
  return r;
}
```

## 【定理推理树·借用安全】

```
借用安全性:
────────────────────────────────────────
目标: 借用期间被借用对象保持有效

步骤1: 借用创建
  t : T
  ───────────
  &t : @&T
  借用开始，T的生命周期必须≥借用期
────────────────────────────────────────
步骤2: 借用使用
  x : @&T ⊢ use(x) : U
  使用x不消耗T的所有权
────────────────────────────────────────
步骤3: 借用结束
  借用期结束，T可正常释放
  无悬挂引用（由生命周期保证）
────────────────────────────────────────
结论: 借用安全 ∎
```
