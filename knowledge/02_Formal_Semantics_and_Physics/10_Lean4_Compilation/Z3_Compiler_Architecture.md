     ▼

     ▼

     │

## 【阶段1·解析与宏展开】

### 解析器理论

**算法**: 递归下降 + 优先级攀爬

**复杂度**: O(n)时间，O(1)空间（流式解析）

**输出**: Syntax树（类似S-表达式）

### 宏系统

**卫生宏**: 避免变量捕获

```
macro "my_if" c:term "then" t:term "else" e:term : term =>
  `(if $c then $t else $e)
```

**相位分离**: 宏在编译期展开，不影响运行时。

## 【阶段2·精化与类型检查】

### 双向类型检查

**判断**:

- 检查模式: Γ ⊢ e ⇐ T（检查e是否有类型T）
- 推断模式: Γ ⊢ e ⇒ T（推断e的类型为T）

**规则**:

```
检查模式:
Γ ⊢ e ⇒ T'    T' ≡ T
───────────────────
Γ ⊢ e ⇐ T

推断模式:
Γ(x) = T
───────────
Γ ⊢ x ⇒ T

Γ, x:A ⊢ e ⇐ B
───────────────────
Γ ⊢ λx.e ⇒ Π(x:A).B
```

### 统一算法

**问题**: 找到替换σ使得σ(A) = σ(B)

**算法**: 高阶统一（针对依赖类型扩展）

**复杂度**: 不可判定（一般情况），但对受限情况可判定。

## 【阶段3·IR与代码生成】

### IR设计

**A-normal form** (ANF):

- 每个中间结果绑定到变量
- 便于优化和分析

**示例**:

```
let x₁ = f a
let x₂ = g b
h x₁ x₂
```

### 优化pass

1. **内联展开**: 小函数直接展开
2. **尾调用优化**: 递归转循环
3. **引用计数优化**: 移除冗余inc/dec
4. **闭包优化**: 扁平化环境

### C代码生成策略

| Lean IR | C代码 |
|:--------|:------|
| 函数调用 | 直接调用（已知函数）或闭包调用 |
| 构造子 | 分配lean_object，设置tag |
| 模式匹配 | switch语句或if-else链 |
| 递归 | while循环（尾递归）或递归调用 |

## 【定理·编译正确性】

**定理**: 语义保持

如果e →*v在Lean中，则C(e) →* C(v)在C中。

**证明概要**:

1. 解析保持语义（语法正确性）
2. 精化保持语义（类型安全）
3. 擦除保持语义（证明无关性）
4. IR转换保持语义（操作语义等价）
5. C生成保持语义（操作语义映射）
