## 1. 课程概述

### 1.1 课程目标

#### 知识目标

| 目标层级 | 具体描述 | 达成标准 |
|---------|---------|---------|
| **记忆** | 掌握程序语义学的核心概念与术语 | 能准确表述各类语义定义 |
| **理解** | 理解不同语义方法的数学基础 | 能解释语义规则的设计原理 |
| **应用** | 能为简单语言定义形式语义 | 能独立完成语言语义规范 |
| **分析** | 能分析语义定义的等价性与性质 | 能证明语义相关的定理 |
| **综合** | 能设计与实现类型检查器和解释器 | 完成课程项目 |
| **评价** | 能评价不同语义方法的适用场景 | 在项目中合理选择方法 |

#### 能力目标

1. **形式化建模能力**: 将自然语言描述转化为精确的数学定义
2. **逻辑推理能力**: 运用归纳法、共归纳法进行严格证明
3. **工程实现能力**: 使用Lean 4实现语义工具
4. **批判性思维**: 评估形式化方法的优劣与局限

#### 素质目标

- 培养严谨的科学态度
- 建立数学与计算之间的桥梁
- 形成对编程语言本质的深刻理解

### 1.2 先修要求

#### 必备知识

| 领域 | 具体内容 | 自测问题 |
|-----|---------|---------|
| **离散数学** | 集合论、关系、函数、归纳法 | 能否证明简单的数学归纳命题？ |
| **数理逻辑** | 命题逻辑、一阶逻辑、证明系统 | 能否构造自然演绎证明？ |
| **数据结构** | 树、图、递归结构 | 能否熟练操作递归数据类型？ |
| **编程基础** | 函数式编程经验(Haskell/OCaml/Lean) | 能否编写高阶函数？ |

#### 建议背景

- 完成过编译原理课程（非必须）
- 接触过类型系统概念
- 具备基本的数学证明经验

### Week 2-3: 操作语义

#### 学习目标

- 掌握大步语义与小步语义的定义方法
- 理解结构化操作语义(SOS)的原理
- 能够证明语义性质（如规范性、合流性）

#### Week 2 授课内容

**第3讲: 大步操作语义 (2学时)**

```
核心概念:
├── 表达式语言: E ::= n | E + E | E × E
├── 求值关系: ⟨E, σ⟩ ⇓ n
├── 归纳规则设计
└── 证明技术: 结构归纳法
```

**讲解要点:**

1. **语法定义** (20min)
   - BNF文法表示
   - 抽象语法树(AST)
   - 具体语法vs抽象语法

2. **语义规则设计** (40min)

   ```
   ───── (NUM)
   ⟨n,σ⟩ ⇓ n

   ⟨E₁,σ⟩ ⇓ n₁    ⟨E₂,σ⟩ ⇓ n₂    n = n₁ + n₂
   ────────────────────────────────────────── (ADD)
   ⟨E₁ + E₂,σ⟩ ⇓ n
   ```

3. **求值树与证明** (30min)
   - 推导树的概念
   - 唯一性证明
   - 确定性定理

4. **扩展: 命令式语言** (30min)
   - 存储(Store)的概念
   - 命令序列语义
   - 条件与循环

**第4讲: 小步操作语义 (2学时)**

```
核心概念:
├── 归约关系: ⟨E, σ⟩ → ⟨E', σ'⟩
├── 求值上下文
├── 归约策略
└── 非终止与发散
```

1. **小步vs大步** (30min)
   - 比较与对比
   - 适用场景分析
   - 调试与追踪

2. **求值上下文** (30min)

   ```
   E ::= [] | E + e | n + E | ...
   ```

   - 上下文填充
   - 局部归约规则

3. **归约策略** (30min)
   - 最左最外 vs 最左最内
   - 应用序 vs 正规序
   - Church-Rosser定理

4. **λ演算语义** (30min)
   - α-转换、β-归约
   - 菱形性质
   - 合流性证明

#### Week 3 实验课

**实验1: 实现解释器 - 准备与启动 (4学时)**

见 [实验设计章节](#实验设计) 实验1。

### Week 4-5: 指称语义

#### 学习目标

- 理解指称语义的基本思想
- 掌握完全偏序(CPO)与连续函数
- 能够定义简单语言的指称语义
- 理解 adequacy 定理的意义

#### Week 4 授课内容

**第5讲: 数学基础 (2学时)**

```
前置数学:
├── 偏序集 (Poset)
├── 完全偏序 (CPO)
├── 连续函数
├── 最小不动点定理
└── 提升 (Lifting)
```

1. **偏序与完全偏序** (40min)
   - 偏序关系定义
   - 链与完全性
   - 底部元素 ⊥
   - 例子: 离散CPO、平坦CPO

2. **连续函数** (40min)
   - 单调性
   - 连续性定义
   - 函数空间的CPO结构
   - 点式序

3. **不动点定理** (40min)

   ```
   fix : (D → D) → D
   fix(f) = ⨆ᵢ fⁱ(⊥)
   ```

   - Kleene不动点定理
   - 最小不动点的存在性
   - 应用: while循环语义

**第6讲: 指称语义定义 (2学时)**

1. **表达式指称** (30min)

   ```
   ⟦n⟧ρ = n
   ⟦E₁ + E₂⟧ρ = ⟦E₁⟧ρ + ⟦E₂⟧ρ
   ```

2. **命令指称** (40min)

   ```
   ⟦C⟧ : Store → Store⊥
   ⟦skip⟧σ = σ
   ⟦x:=E⟧σ = σ[x ↦ ⟦E⟧σ]
   ⟦C₁;C₂⟧ = ⟦C₂⟧ ∘ ⟦C₁⟧
   ⟦while B do C⟧ = fix(λw.λσ. if ⟦B⟧σ then w(⟦C⟧σ) else σ)
   ```

3. **组合性** (20min)
   - 语义的模块化
   - 抽象解释的联系

4. **adequacy与full abstraction** (30min)
   - 操作语义与指称语义的对应
   - adequacy定理陈述
   - full abstraction的挑战

#### Week 5 授课与实验

**第7讲: 高阶类型与域论 (2学时)**

1. **类型的指称** (40min)
   - 基本类型: ⟦nat⟧ = ℕ⊥
   - 函数类型: ⟦τ₁ → τ₂⟧ = ⟦τ₁⟧ → ⟦τ₂⟧
   - 递归类型: 解域方程

```
  ×
 / \
```

*讲解要点*: 强调语法vs语义的区别

**Slide 3: 语义规则设计** (5分钟)

```
大步语义: ⟨E, σ⟩ ⇓ n

规则 (NUM):
─────
⟨n, σ⟩ ⇓ n

规则 (ADD):
⟨E₁, σ⟩ ⇓ n₁    ⟨E₂, σ⟩ ⇓ n₂    n = n₁ + n₂
──────────────────────────────────────────
⟨E₁ + E₂, σ⟩ ⇓ n

```

*讲解要点*: 解释推理规则格式

**Slide 4: 推导树示例** (5分钟)

```
求值: (3 + 4) × 5

推导树:
⟨3,σ⟩ ⇓ 3      ⟨4,σ⟩ ⇓ 4      7 = 3 + 4
────────────────────────────────────────
⟨3 + 4,σ⟩ ⇓ 7               ⟨5,σ⟩ ⇓ 5      35 = 7 × 5
──────────────────────────────────────────────────────
⟨(3 + 4) × 5,σ⟩ ⇓ 35

关键观察:
1. 叶子节点是公理(NUM)
2. 内部节点应用规则
3. 自底向上构造证明
```

*讲解要点*: 强调推导树是"证明"

**Slide 5: 确定性定理** (4分钟)

```
定理 (确定性):
若 ⟨E, σ⟩ ⇓ n₁ 且 ⟨E, σ⟩ ⇓ n₂，则 n₁ = n₂

证明思路 (结构归纳):
基础: E = n
  只能应用NUM规则，结果唯一
归纳: E = E₁ + E₂
  假设对E₁, E₂成立
  由归纳假设，子表达式值唯一
  因此和唯一
```

*讲解要点*: 介绍结构归纳法

```
- 保证正确性
```

**Slide 2: 霍尔三元组** (4分钟)

```
形式: {P} C {Q}

读法: "在P成立的前提下执行C，
      若C终止则Q成立"

例子:
{x = 5} x := x + 1 {x = 6}
{x = n} x := x + 1 {x = n + 1}
{x = n} x := x * 2 {x = 2n}
```

*互动*: 让学生构造例子

**Slide 3: 推理规则** (6分钟)

```
赋值规则:
{Q[x:=E]} x := E {Q}

例子:
{x + 1 = 6} x := x + 1 {x = 6}
即: {x = 5} x := x + 1 {x = 6}

顺序规则:
{P} C₁ {R}    {R} C₂ {Q}
─────────────────────────
{P} C₁;C₂ {Q}

条件规则:
{P∧B} C₁ {Q}    {P∧¬B} C₂ {Q}
───────────────────────────────
{P} if B then C₁ else C₂ {Q}
```

**Slide 4: 循环规则详解** (6分钟)

```
循环规则:
{I∧B} C {I}
─────────────────────────
{I} while B do C {I∧¬B}

关键: 循环不变量I

什么是循环不变量?
- 每次迭代前后都成立的性质
- 进入循环时成立
- 执行体保持它
- 退出时结合¬B推出Q

寻找不变量的技巧:
1. 观察循环目标
2. 概括循环变量关系
3. 验证保持性
```

*讲解要点*: 循环不变量是最难的部分，多给例子

**Slide 5: 实例证明** (6分钟)

```
证明阶乘计算正确:

{y = 1 ∧ i = 0}
while i < n do
  i := i + 1;
  y := y * i
{y = n!}

不变量: I = (y = i! ∧ i ≤ n)

验证:
1. 初始: y=1, i=0 → y=0!=1 ✓
2. 保持: 假设y=i!, i<n
   执行后: i'=i+1, y'=y*(i+1)=i!*(i+1)=(i+1)!=i'! ✓
3. 退出: i≥n ∧ i≤n → i=n → y=n! ✓
```

### 5.4 考试设计

#### 试卷结构

**期末考试 (100分, 120分钟)**

```
第一部分: 选择题 (30分, 30分钟)
├── 10道题
├── 每题3分
├── 考察概念理解
└── 单选/多选混合

第二部分: 填空题 (20分, 20分钟)
├── 5道大题，每题4个空
├── 每空1分
├── 考察定义与规则
└── 强调精确记忆

第三部分: 简答题 (20分, 30分钟)
├── 2道题，每题10分
├── 需要解释概念
├── 比较不同方法
└── 举例说明

第四部分: 综合题 (30分, 40分钟)
├── 2道题，每题15分
├── 设计/证明题
├── 需要综合运用
└── 展示推理能力
```

#### 样题示例

**选择题示例**:

```
以下哪个是霍尔逻辑赋值规则的正确形式?

A. {P} x:=E {P[x:=E]}
B. {P[x:=E]} x:=E {P}
C. {P} x:=E {P[x]}
D. {P} x:=E {E}

答案: B
考察: 赋值规则的记忆
```

**简答题示例**:

```
题目: 比较大步语义与小步语义的优缺点 (10分)

评分要点:
- 大步语义优点 (2分): 简洁、易证明
- 大步语义缺点 (2分): 无法表达非终止
- 小步语义优点 (2分): 可表达交错、非终止
- 小步语义缺点 (2分): 描述冗长
- 使用场景建议 (2分)
```

**综合题示例**:

```
题目: 为以下语言定义操作语义并证明确定性 (15分)

给定语言:
E ::= n | E + E | if E then E else E

要求:
1. 定义大步语义规则 (5分)
2. 定义小步语义规则 (5分)
3. 证明大步语义的确定性 (5分)

评分要点:
- 规则完整性
- 符号使用规范
- 证明结构清晰
- 归纳法使用正确
```
