## 1. 形式化方法概述

### 1.1 为什么需要形式化方法

```
传统软件开发的验证方式：
┌──────────────┬──────────────────────┬─────────────────────┐
│   方法       │      优点            │      缺点           │
├──────────────┼──────────────────────┼─────────────────────┤
│ 测试         │ 实际、易实施         │ 无法证明无缺陷      │
│              │ 发现已知问题         │ 覆盖率有限          │
├──────────────┼──────────────────────┼─────────────────────┤
│ 代码审查     │ 发现设计问题         │ 依赖人工经验        │
│              │ 知识共享             │ 无法穷尽所有情况    │
├──────────────┼──────────────────────┼─────────────────────┤
│ 静态分析     │ 自动化、快速         │ 误报率高            │
│              │ 发现常见错误         │ 无法验证功能正确性  │
├──────────────┼──────────────────────┼─────────────────────┤
│ 形式化验证   │ 数学保证             │ 需要专业知识        │
│              │ 证明所有情况         │ 状态空间爆炸        │
│              │ 发现深层缺陷         │ 验证本身可能出错    │
└──────────────┴──────────────────────┴─────────────────────┘

形式化方法适用场景：
• 安全关键系统（航空航天、医疗设备）
• 密码学协议
• 并发算法（锁、无锁数据结构）
• 操作系统内核
• 硬件设计验证
```

## 2. Hoare逻辑

### 2.1 霍尔三元组

```
霍尔三元组：{P} C {Q}

P: 前置条件 (Precondition)
C: 程序命令 (Command)
Q: 后置条件 (Postcondition)

含义：如果在执行C前P成立，且C终止，则执行后Q成立。

示例：
{x > 0} x := x + 1 {x > 1}

证明：
• 前置条件: x > 0
• 执行: x变为x+1
• 因此: x+1 > 1，即x > 0（原值）→ 新值x > 1 ✓

最强后置条件(Strongest Postcondition):
SP(x := E, P) = ∃v. x = E[v/x] ∧ P[v/x]

最弱前置条件(Weakest Precondition):
WP(x := E, Q) = Q[E/x]  (将Q中的x替换为E)
```

### 2.2 推理规则

```
赋值规则 (Assignment):
───────────────
{Q[E/x]} x := E {Q}

示例：
{x + 1 > 5} x := x + 1 {x > 5}

顺序组合 (Sequence):
{P} C1 {R}    {R} C2 {Q}
───────────────────────────
{P} C1; C2 {Q}

条件语句 (Conditional):
{P ∧ B} C1 {Q}    {P ∧ ¬B} C2 {Q}
───────────────────────────────────
{P} if B then C1 else C2 {Q}

循环 (While Loop):
{I ∧ B} C {I}
───────────────────────
{I} while B do C {I ∧ ¬B}

其中I是循环不变式(Loop Invariant)
```

### 2.3 循环不变式推导

```
问题：证明 while i < n do i := i + 1 终止时 i = n

循环不变式候选：I = (i ≤ n)

证明：
1. 初始化：进入循环前i=0，假设n≥0，则I成立
2. 保持：假设I ∧ B成立，即i ≤ n ∧ i < n
   执行i := i + 1后，i' = i + 1 ≤ n（因为i < n）
   所以I仍然成立
3. 终止：循环结束时I ∧ ¬B = (i ≤ n) ∧ (i ≥ n) = (i = n)

因此后置条件为i = n ✓

复杂示例：数组求和
{ n ≥ 0 }
i := 0;
s := 0;
while i < n do
    s := s + a[i];
    i := i + 1;
{ s = Σ(j=0 to n-1) a[j] }

循环不变式：
I = (0 ≤ i ≤ n) ∧ (s = Σ(j=0 to i-1) a[j])

证明：
• 初始化：i=0, s=0，Σ(j=0 to -1) = 0，成立
• 保持：假设I成立，循环体后
  s' = s + a[i] = Σ(j=0 to i-1) a[j] + a[i] = Σ(j=0 to i) a[j]
  i' = i + 1
  因此I仍然成立
• 终止：i = n时，s = Σ(j=0 to n-1) a[j] ✓
```

### 2.4 C语言中的应用

```c
// 使用ACSL (ANSI/ISO C Specification Language)
// Frama-C工具支持

/*@
    requires n >= 0;
    requires \valid_read(a + (0..n-1));
    assigns \nothing;
    ensures \result == \sum(0, n-1, \lambda integer k; a[k]);
*/
int array_sum(int *a, int n) {
    int i;
    int s = 0;

    /*@
        loop invariant 0 <= i <= n;
        loop invariant s == \sum(0, i-1, \lambda integer k; a[k]);
        loop assigns i, s;
        loop variant n - i;
    */
    for (i = 0; i < n; i++) {
        s += a[i];
    }

    return s;
}

// 使用方式：
// frama-c -wp sum.c -then -report
```
