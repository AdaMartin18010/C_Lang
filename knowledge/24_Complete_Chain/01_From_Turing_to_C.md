## 2. 每层核心概念详解

### 2.1 Layer 1: 计算理论

```
核心问题：什么是可计算的？

图灵机模型:
┌────────────────────────────────────────────────────┐
│  有限状态控制器          无限磁带（双向）            │
│  ┌──────┐          ┌───┬───┬───┬───┬───┬───┐       │
│  │ 状态 │─────────►│ 0 │ 1 │ 0 │ □ │ 0 │ 1 │...    │
│  │  q   │          └───┴───┴───┴───┴───┴───┘       │
│  └──────┘               ▲                         │
│                         │ 读/写/移动              │
└────────────────────────────────────────────────────┘

关键概念:
• 状态：机器的"心智状态"
• 转移函数：δ(当前状态, 读入符号) → (新状态, 写入符号, 移动方向)
• 可计算性：能被图灵机计算的问题集合
• 丘奇-图灵论题：任何可计算的函数都可被图灵机计算

与下层的连接:
图灵机的"状态"对应CPU的程序计数器和寄存器状态。
图灵机的"磁带"对应计算机的内存。
```

## 5. 学习检查点

### 5.1 自测问题

**基础理解：**

1. 图灵机模型与物理计算机之间的关键对应关系是什么？
2. 为什么需要汇编语言这一层？它解决了什么问题？
3. 虚拟机的价值是什么？为什么不直接使用物理机器？

**进阶分析：**
4. 描述从C源代码到物理执行经历的完整转换过程。
5. 如果要在某两层之间插入新的抽象层，你会选择哪里？为什么？
6. 对比不同ISA（RISC-V、x86、ARM）如何影响上层软件的编写。

**动手实践：**
7. 选择一个简单算法（如阶乘），写出它在每一层的表示形式。
8. 分析一个真实程序的编译输出，追踪从C到机器码的映射。

### 5.2 答案要点

<details>
<summary>点击查看答案</summary>

**问题1：**
图灵机的状态对应CPU的PC和寄存器状态；磁带对应内存；读写头对应内存地址总线。

**问题2：**
汇编语言提供人类可读的机器指令表示，解决二进制机器码难以编写和理解的问题，同时保持对硬件的精确控制。

**问题3：**
虚拟机提供硬件无关的执行环境，实现"一次编译，到处运行"，简化语言实现，支持动态特性。

**问题4：**
C源代码 → 预处理 → 编译 → 汇编代码 → 汇编 → 目标文件 → 链接 → 可执行文件 → 加载 → 执行

</details>
