## 1. 理论到物理的精确映射

### 1.1 图灵机状态 ↔ CPU状态

```
映射关系:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

图灵机                    物理实现
─────────────────────────────────────────────────
有限状态集 Q              CPU程序计数器(PC) + 状态寄存器
                          • PC决定下一条指令
                          • 条件码(NZCV等)表示分支状态

磁带字母表 Γ              内存地址空间
                          • 每个地址存储一个字节/字
                          • 可读写，类似于图灵机的磁带

磁头位置                  内存地址总线
                          • MAR(内存地址寄存器)
                          • 读写操作的目标地址

转移函数 δ                指令译码和执行单元
                          • 取指-译码-执行-访存-写回
                          • 控制单元产生控制信号

配置 (状态, 磁带, 磁头)   CPU上下文(寄存器+内存状态)
                          • 完整描述系统状态
                          • 上下文切换时保存/恢复

形式化描述:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

图灵机配置: (q, tape, head)
CPU状态: (PC, R[0..n-1], Memory[0..M-1])

转移:
  δ(q, tape[head]) = (q', symbol, direction)
  ↓
  Instruction[PC] = opcode
  Execute(opcode, operands) → 更新PC, 寄存器, 内存
```

### 1.2 λ演算 ↔ 函数调用

```
映射关系:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

λ演算                     C语言函数
─────────────────────────────────────────────────
λx.E                      int func(int x) { return E; }
变量 x                    参数/局部变量
应用 (E1 E2)              函数调用 func(arg)
β归约                     函数调用+参数替换

环境(上下文)              栈帧 + 寄存器
自由变量                  全局变量/闭包捕获

形式化描述:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

λ项: λx.x+1
↓ 编译
C函数: int f(int x) { return x + 1; }
↓ 汇编
addi a0, a0, 1    # a0 = x + 1
ret

闭包实现:
λx.λy.x+y  →  返回一个携带x值的函数
              →  分配闭包对象(函数指针+环境)
              →  调用时通过隐藏参数传递环境
```

### 2.1 数据类型映射

```
C类型 → 汇编/机器表示:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C类型          机器表示              汇编操作
─────────────────────────────────────────────────
char           8位有符号整数         lb/sb (符号扩展)
unsigned char  8位无符号整数         lbu/sb (零扩展)
short          16位有符号            lh/sh
int            32位有符号            lw/sw
long           32/64位(平台相关)     lw/sw 或 ld/sd
指针           32/64位地址           与long相同
float          IEEE 754单精度        flw/fsw
double         IEEE 754双精度        fld/fsd

数组           连续内存块            基地址+索引×元素大小
结构体         连续内存块(可能填充)   基地址+字段偏移
联合体         共享内存(最大成员大小)  同一地址，不同解释

类型解释示例:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

解释为有符号char:  -1  (补码)
解释为无符号char: 255
解释为有符号int:   -1  (符号扩展: 0xFFFFFFFF)

汇编层面只有位模式，没有类型！
类型是编译时概念，通过指令选择实现。
```

### 2.2 控制流映射

```
C控制结构 → 汇编实现:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C结构              汇编实现
─────────────────────────────────────────────────
if (cond)          比较 + 条件跳转
    stmt1;         (cond为假时跳过)
else
    stmt2;

while (cond)       loop标签:
    body;              比较 + 条件跳转到end
                       执行body
                       无条件跳转到loop
                   end标签:

for (init;cond;inc) init
    body;           loop标签:
                       比较 + 条件跳转到end
                       执行body
                   inc标签:
                       执行inc
                       无条件跳转到loop
                   end标签:

switch (x) {        加载x
    case 0: ...     如果稀疏: 一系列比较和跳转
    case 1: ...     如果密集: 跳转表
    ...             jmp [table + x * 4]
}

break              跳转到当前循环结束标签
continue           跳转到当前循环增量标签
return             恢复栈帧 + 跳转到返回地址

跳转表实现细节:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

switch(x) with cases 0,1,2,3:

    la   t0, jump_table
    slli t1, x, 2           # x * 4 (地址对齐)
    add  t0, t0, t1
    lw   t0, 0(t0)          # 加载目标地址
    jr   t0                 # 跳转

jump_table:
    .word case_0
    .word case_1
    .word case_2
    .word case_3
```

### 2.3 函数调用映射

```
C函数调用 → 汇编实现:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C代码:
int add(int a, int b) { return a + b; }
int result = add(3, 4);

RISC-V汇编:
# 调用者代码
    li   a0, 3              # 参数1
    li   a1, 4              # 参数2
    call add                # jal ra, add
    mv   t0, a0             # 保存返回值

# 被调者代码 (add函数)
add:
    # 如果函数复杂，需要保存ra和s寄存器
    addi sp, sp, -16        # 分配栈帧
    sd   ra, 8(sp)          # 保存返回地址

    add  a0, a0, a1         # a0 = a + b (返回值)

    ld   ra, 8(sp)          # 恢复返回地址
    addi sp, sp, 16         # 释放栈帧
    ret                     # jalr x0, ra, 0

调用约定细节:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

参数传递:
• 整数/指针: a0-a7 (x10-x17)
• 浮点: fa0-fa7
• 超过8个参数: 栈

返回值:
• a0 (和a1用于128位返回)

调用者保存: t0-t6, a0-a7 (临时)
被调者保存: s0-s11, ra (跨调用保持)

栈帧布局:
高地址
┌─────────────────┐ ← 调用者sp
│ 参数溢出        │
├─────────────────┤
│ 返回地址 (ra)   │ ← 保存的ra
├─────────────────┤
│ 保存的s0-s11    │
├─────────────────┤
│ 局部变量        │ ← 当前sp
├─────────────────┤
│ 参数构建区      │
└─────────────────┘
低地址

尾调用优化:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C代码:
int factorial(int n, int acc) {
    if (n <= 1) return acc;
    return factorial(n - 1, n * acc);  // 尾递归！
}

优化后的汇编:
factorial:
    li   t0, 1
    ble  n, t0, done        # if n <= 1
    mul  acc, n, acc        # acc = n * acc
    addi n, n, -1           # n = n - 1
    j    factorial          # 直接跳转，不是call！
done:
    mv   a0, acc
    ret

关键：复用当前栈帧，不保存ra
```

## 3. 内存模型映射

### 3.1 C内存模型到物理内存

```
C内存布局 → 物理内存:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

虚拟地址空间:
高地址
┌───────────────────┐ 0xFFFFFFFF (32位)
│    内核空间       │ ← 操作系统使用
├───────────────────┤ 0xC0000000
│                   │
│    栈 (向下增长)   │ ← 局部变量、函数调用
│         ↓         │
│                   │
├───────────────────┤
│                   │
│    堆 (向上增长)   │ ← malloc/free
│         ↑         │
│                   │
├───────────────────┤
│    BSS段          │ ← 未初始化的全局变量
├───────────────────┤
│    数据段         │ ← 已初始化的全局变量
├───────────────────┤
│    代码段         │ ← 程序指令
├───────────────────┤
│    保留区域       │ ← 包括NULL指针区域
└───────────────────┘ 0x00000000
低地址

内存分配映射:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C代码                汇编/机器实现
─────────────────────────────────────────────────
int x;               # BSS段
int y = 10;          # 数据段，初始化值
static int z;        # BSS段

void func() {
    int a;           # 栈: sp - offset
    int *p = malloc(100);
                     # 堆: 调用malloc
}

char *s = "hello";   # s在数据段，字符串在代码/只读段

MMU映射:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

虚拟地址 → MMU → 物理地址
              ↓
         页表查找
         TLB缓存
         页错误处理

保护:
• 代码段: 读+执行
• 数据段: 读+写
• 栈: 读+写
• NULL页: 无权限（捕获空指针解引用）
```

### 3.2 指针语义映射

```
C指针操作 → 汇编实现:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C操作               汇编实现
─────────────────────────────────────────────────
int x;              # 栈偏移 -4(s0)
int *p = &x;        # addi p, s0, -4

int y = *p;         # lw   y, 0(p)
*p = 10;            # li   t0, 10
                    # sw   t0, 0(p)

p++;                # addi p, p, 4   (int*)
p++;                # addi p, p, 1   (char*)

p[i]                # slli t0, i, 2  (i*4)
                    # add  t0, p, t0
                    # lw   y, 0(t0)

*(p + i)            # 同上

多级指针:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

int x = 10;
int *p = &x;
int **pp = &p;
int y = **pp;

汇编:
    lw   t0, x          # t0 = x
    la   t1, p          # t1 = &p
    ld   t2, 0(t1)      # t2 = p = &x
    lw   t3, 0(t2)      # t3 = *p = x
    # t3 = 10

函数指针:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

int (*fp)(int, int) = add;
int result = fp(3, 4);

汇编:
    la   t0, add        # t0 = 函数地址
    li   a0, 3
    li   a1, 4
    jalr ra, t0, 0      # 调用t0指向的函数
```

```
危险转换:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C代码:
int x = 1000;
char c = (char)x;    # 截断！c = -24 (0xE8)

    li   t0, 1000       # t0 = 0x3E8

# 读取时符号扩展:
    lb   t1, c          # t1 = 0xFFFFFFFFFFFFFFE8 = -24
    lbu  t2, c          # t2 = 0xE8 = 232
```

## 5. 形式化描述

### 5.1 操作语义表示

```
表达式求值的形式化规则:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌────────────────────────────────────────────────┐
│ 配置: ⟨表达式, 环境⟩ → ⟨值, 新环境⟩           │
└────────────────────────────────────────────────┘

常量:
───────────────────
⟨n, σ⟩ ⇒ ⟨n, σ⟩

变量查找:
───────────────────  其中 σ(x) = v
⟨x, σ⟩ ⇒ ⟨v, σ⟩

加法:
⟨e₁, σ⟩ ⇒ ⟨v₁, σ₁⟩   ⟨e₂, σ₁⟩ ⇒ ⟨v₂, σ₂⟩
──────────────────────────────────────────
⟨e₁ + e₂, σ⟩ ⇒ ⟨v₁ + v₂, σ₂⟩

赋值:
⟨e, σ⟩ ⇒ ⟨v, σ'⟩
──────────────────────────
⟨x = e, σ⟩ ⇒ ⟨v, σ'[x↦v]⟩

对应汇编:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

环境σ → 内存+寄存器状态
求值 → 指令执行序列
值 → 寄存器或内存中的位模式
```

### 5.2 编译的形式化视角

```
编译作为语义保持的转换:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

高阶语言 L ──[编译器]──► 低阶语言 L'

正确性条件:
对于所有程序 P ∈ L:
  如果 P 在 L 的语义下输出 O
  则 compile(P) 在 L' 的语义下也输出 O

形式化:
⟦P⟧_L = ⟦compile(P)⟧_L'

层次间的正确性:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

C语义 ⟹ 汇编语义 ⟹ 机器码语义 ⟹ 硬件执行

每一层都保持上层语义，
只是用更低级的原语表达。

示例:
C: x + y
汇编: lw t0, x; lw t1, y; add t2, t0, t1
机器码: [二进制编码]
硬件: ALU执行加法，更新寄存器
```

## 6. 学习检查点

### 6.1 自测问题

**基础理解：**

1. 图灵机的"状态"如何映射到CPU的物理组件？
2. C语言的"类型"在汇编/机器层面还存在吗？以什么形式？
3. 解释为什么尾递归优化可以节省栈空间。

**进阶分析：**
4. 详细描述一个C结构体在内存中的布局，包括填充字节。
5. 函数指针在汇编层面是如何实现的？
6. 静态类型语言和动态类型语言在内存表示上有什么区别？

**动手实践：**
7. 为一个简单的C程序手工编写对应的汇编代码。
8. 使用调试器观察变量在内存中的实际布局。

### 6.2 答案要点

<details>
<summary>点击查看答案</summary>

**问题1：**
图灵机的状态对应CPU的程序计数器（决定执行哪条指令）和条件码寄存器（决定分支走向）。

**问题2：**
C类型在运行时基本消失。仅存的是位模式和大小信息。类型转换通过选择不同的指令实现（符号扩展vs零扩展）。

**问题3：**
尾递归是函数的最后一个操作，不需要保存当前栈帧状态（局部变量不再需要），因此可以复用当前栈帧。

**问题4：**
结构体布局遵循对齐规则。例如`struct { char a; int b; }`在大多数系统上：a在偏移0，b在偏移4（4字节对齐），总大小8字节。

</details>
