### 编译流程数据流

```mermaid
flowchart TB
    subgraph Source["源代码"]
        S1[.zig 源文件]
        S2[@import 依赖]
    end

    subgraph Frontend["前端"]
        F1[词法分析]
        F2[语法分析]
        F3[AST生成]
    end

    subgraph Analysis["分析阶段"]
        A1[语义分析]
        A2[类型推断]
        A3[comptime求值]
    end

    subgraph IR["中间表示"]
        I1[ZIR]
        I2[AIR]
    end

    subgraph Backend["后端"]
        B1[代码生成]
        B2[优化]
        B3[链接]
    end

    subgraph Output["输出"]
        O1[可执行文件]
        O2[库文件]
    end

    S1 --> F1
    S2 --> F1
    F1 --> F2 --> F3
    F3 --> A1 --> A2 --> A3
    A3 --> I1 --> I2
    I2 --> B1 --> B2 --> B3
    B3 --> O1
    B3 --> O2
```

### Zig vs 其他语言特性对比

```text
内存管理模型对比

C:      完全手动 ───────────────────────────────────────→ 自由但危险
        malloc/free

Zig:    手动+安全 ──────────────────────────────────────→ 平衡
        分配器模式 + defer/errdefer

Rust:   编译器检查 ─────────────────────────────────────→ 安全但严格
        所有权系统

Go:     自动GC ─────────────────────────────────────────→ 简单但有开销
        垃圾回收

────────────────────────────────────────────────────────────────────▶
控制程度                                                          安全性
高                                                              高
```
