## 1. 概念解释练习 (30题)

### 1.1 基础概念解释

#### 问题 1-10

```
Q1: 解释"声明(declaration)"与"定义(definition)"的区别。
   要求: 用代码示例说明，并解释为什么头文件中通常只放声明。

参考答案要点:
- 声明告诉编译器标识符的类型和名称
- 定义分配实际存储空间
- 变量声明: extern int x; (声明) vs int x = 5; (定义)
- 函数声明: int func(int); (声明) vs int func(int x) { ... } (定义)
- 头文件多次包含不会导致重复定义错误
```

```
Q2: 解释"值传递(call by value)"在C语言中的含义。
   要求: 说明为什么不能通过值传递修改实参，并给出解决方案。

参考答案要点:
- C语言只有值传递，没有引用传递
- 函数参数是实参的副本
- 修改形参不影响实参
- 解决方案: 传递指针(void swap(int *a, int *b))
- 指针传递也是值传递，传递的是地址值
```

```
Q3: 解释"指针的类型决定了指针运算的步长"。
   要求: 用图示说明char*和int*的+1运算差异。

参考答案要点:
- 指针算术考虑指向类型的大小
- char* + 1 = 地址 + 1字节
- int* + 1 = 地址 + sizeof(int)字节
- 原因: 编译器根据类型计算偏移量
- 图示: [内存布局图显示不同步长]
```

```
Q4: 解释"数组名在大多数表达式中会退化为指向首元素的指针"。
   要求: 列出数组名不退化的例外情况。

参考答案要点:
- 退化规则: int arr[5]; int *p = arr; // arr退化为&arr[0]
- 例外情况:
  1. sizeof(arr) - 返回整个数组大小
  2. &arr - 返回数组指针，类型int(*)[5]
  3. 字符串初始化: char s[] = "abc";
  4. 数组初始化: int arr[] = {1,2,3};
- 退化后的类型: int* (对于int数组)
```

```
Q5: 解释"const int *p"与"int *const p"的区别。
   要求: 说明如何记忆这两种声明。

参考答案要点:
- const int *p: 指向常量的指针，p可变，*p不可变
- int *const p: 常量指针，p不可变，*p可变
- const int *const p: 都不可变
- 记忆法: const在*左边修饰值，在*右边修饰指针
- 口诀: "左定值，右定针"
```

```
Q6: 解释"结构体的内存对齐"。
   要求: 说明为什么要对齐，以及#pragma pack的作用。

参考答案要点:
- 对齐原因: 硬件访问效率，某些架构要求对齐
- 对齐规则: 成员按自身大小对齐，结构体按最大成员对齐
- 填充字节: 编译器自动插入
- #pragma pack(n): 设置对齐系数为n
- __attribute__((packed)): GCC取消对齐
- 内存布局图示: [展示填充位置]
```

```
Q7: 解释"extern "C""的作用。
   要求: 说明为什么需要它，以及C代码调用C++库时的用法。

参考答案要点:
- C++支持函数重载，需要名称修饰(mangling)
- C没有名称修饰，使用原始函数名
- extern "C"禁止C++编译器进行名称修饰
- 混合编程:
  #ifdef __cplusplus
  extern "C" {
  #endif
  // C接口声明
  #ifdef __cplusplus
  }
  #endif
```

```
Q8: 解释"未定义行为(Undefined Behavior, UB)"、
      "未指定行为(Unspecified Behavior)"和
      "实现定义行为(Implementation-defined Behavior)"的区别。

参考答案要点:
- UB: 标准不规定任何行为，程序可能崩溃、产生错误结果
  示例: 数组越界、空指针解引用、有符号整数溢出
- 未指定行为: 标准规定几种可能，由实现选择
  示例: 函数参数求值顺序
- 实现定义行为: 标准规定行为由实现文档说明
  示例: int的大小(16/32/64位)，有符号数的表示方式
```

```
Q9: 解释"volatile"关键字的作用。
   要求: 给出必须使用volatile的三个场景。

参考答案要点:
- 作用: 告诉编译器该变量可能被外部因素改变
- 效果: 禁止相关优化，每次访问都从内存读取
- 场景1: 硬件寄存器映射
- 场景2: 多线程共享变量(需配合同步原语)
- 场景3: 信号处理程序中的变量
- 注意: 不能保证原子性，不是同步机制
```

```
Q10: 解释"restrict"关键字(C99)的作用。
    要求: 说明什么情况下可以使用restrict。

参考答案要点:
- 作用: 承诺指针是访问对象的唯一且排他方式
- 效果: 编译器可以进行更激进的优化
- 使用条件: 程序员保证指针不重叠
- 示例: memcpy可用restrict, memmove不可用
- 违规后果: 未定义行为
- 类比: 承诺没有别名(aliasing)
```

### 1.2 进阶概念解释

#### 问题 11-20

```
Q11: 解释"内存序(Memory Order)"在C11原子操作中的作用。
    要求: 比较memory_order_relaxed和memory_order_seq_cst。

参考答案要点:
- 内存序控制原子操作的同步和排序语义
- relaxed: 无同步保证，仅保证原子性
- seq_cst: 最强保证，全局顺序一致性
- acquire/release: 单向同步，形成happens-before关系
- 性能: relaxed > acquire/release > seq_cst
```

```
Q12: 解释"严格别名规则(Strict Aliasing Rule)"。
    要求: 说明什么是别名，以及如何合法地进行类型双关。

参考答案要点:
- 规则: 不同类型的指针不能指向同一内存
- 目的: 允许编译器优化，假设指针不重叠
- 合法类型双关方法:
  1. 使用union (C99起合法)
  2. 使用memcpy
  3. 使用char*访问任何对象(特殊豁免)
- 违规: float*指向int变量
- 后果: 未定义行为
```

```
Q13: 解释"柔性数组成员(Flexible Array Member, FAM)"。
    要求: 说明使用FAM的结构体分配方法。

参考答案要点:
- C99特性: struct { int n; char data[]; };
- 特点: data不占用结构体大小(sizeof忽略它)
- 分配: malloc(sizeof(struct) + data_size)
- 优势: 一次分配，内存连续，缓存友好
- 替代: data[1]或data[0]的传统写法
- 注意: 只能作为结构体最后一个成员
```

```
Q14: 解释"VLA(变长数组, Variable Length Array)"的原理。
    要求: 说明VLA的内存分配方式。

参考答案要点:
- C99引入，C11可选，C23可能移除
- 语法: int arr[n]; // n是运行时变量
- 分配: 通常在栈上分配
- 实现: 编译器生成动态栈帧调整代码
- sizeof: 运行时计算
- 限制: 不能有静态存储期，不能初始化
- 风险: 栈溢出
```

```
Q15: 解释"内联函数(inline)"的工作原理。
    要求: 比较inline函数和宏的区别。

参考答案要点:
- 建议编译器在调用点展开函数体
- 优点: 消除调用开销，同时有类型检查
- C99内联规则: inline定义、extern inline定义、外部定义
- vs 宏:
  - 宏: 文本替换，无类型检查，可多次求参
  - inline: 真实函数，有类型检查，参数求值一次
- 注意: inline只是建议，编译器可忽略
```

```
Q16: 解释"原子操作(Atomic Operation)"与"锁(Lock)"的区别。
    要求: 说明为什么原子操作通常比锁快。

参考答案要点:
- 原子操作: 不可中断的单一操作，硬件支持
- 锁: 互斥机制，可能涉及上下文切换
- 性能差异:
  - 原子操作: 通常几个时钟周期
  - 锁: 无竞争时~100周期，有竞争时可能切换上下文(数千周期)
- 使用场景:
  - 原子操作: 简单计数、标志位
  - 锁: 复杂数据结构保护
- C11: _Atomic类型和stdatomic.h
```

```
Q17: 解释"函数指针"的声明语法。
    要求: 解释"void (*signal(int sig, void (*func)(int)))(int)"的含义。

参考答案要点:
- 基本语法: 返回类型 (*指针名)(参数列表)
- signal函数解析:
  - signal: 函数名
  - int sig: 第一个参数
  - void (*func)(int): 第二个参数，函数指针
  - void (*...)(int): 返回类型，函数指针
- typedef简化: typedef void (*sighandler_t)(int);
- 使用: sighandler_t signal(int sig, sighandler_t func);
```

```
Q18: 解释"线程局部存储(Thread Local Storage, TLS)"。
    要求: 说明TLS变量的生命周期和可见性。

参考答案要点:
- C11关键字: _Thread_local (C23: thread_local)
- 语义: 每个线程有独立实例
- 生命周期: 从线程创建到线程结束
- 可见性: 与普通变量相同，由声明位置决定
- 存储类: 可以与static/extern组合
- 实现: 通常通过线程特定数据段或寄存器
```

```
Q19: 解释"对齐(Alignment)"与"填充(Padding)"的关系。
    要求: 计算struct { char c; double d; }的大小。

参考答案要点:
- 对齐: 数据按特定边界存放(地址模对齐值=0)
- 填充: 编译器插入的未使用字节
- 关系: 为满足对齐要求而添加填充
- 示例计算:
  struct {
      char c;      // 偏移0，大小1
      // 填充7字节
      double d;    // 偏移8，大小8
  };               // 总大小16
- 对齐值: 通常等于类型大小(对基本类型)
- _Alignas: 可指定更大对齐
```

```
Q20: 解释"栈溢出(Stack Overflow)"的原因和检测方法。
    要求: 说明哪些代码模式容易导致栈溢出。

参考答案要点:
- 原因: 栈空间耗尽，通常1-8MB
- 危险模式:
  1. 大数组作为局部变量: int arr[1000000];
  2. 无限递归或深度递归
  3. VLA分配过大: int arr[n]; // n很大
  4. 大量alloca调用
- 检测:
  - 编译期: 静态分析工具
  - 运行期: 栈保护页、金丝雀值
  - 调试: 栈使用监控
- 防护: 大数组用malloc，递归加深度限制
```

### 1.3 高级概念解释

#### 问题 21-30

```
Q21: 解释"编译单元(Translation Unit)"的概念。
    要求: 说明头文件包含与编译单元的关系。

参考答案要点:
- 定义: 源文件经预处理后得到的代码
- 组成: 源文件 + 所有包含的头文件
- 预处理: #include展开，宏替换，条件编译
- 关系: 每个.c文件通常对应一个编译单元
- 链接: 多个编译单元的目标文件链接成可执行文件
- 内部链接vs外部链接: static决定
```

```
Q22: 解释"链接(Linking)"的静态链接与动态链接。
    要求: 比较两种链接方式的优缺点。

参考答案要点:
- 静态链接: 库代码复制到可执行文件
  - 优点: 独立运行，加载快
  - 缺点: 体积大，更新需重新编译
- 动态链接: 运行时加载共享库
  - 优点: 体积小，共享内存，可更新
  - 缺点: 依赖环境，加载稍慢
- 位置无关代码(PIC): 动态库需要
- LD_PRELOAD: 动态链接的灵活应用
```

```
Q23: 解释"位置无关代码(PIC)"的原理。
    要求: 说明为什么共享库需要PIC。

参考答案要点:
- 问题: 共享库可能被加载到不同地址
- 方案: 代码与绝对地址无关
- 实现:
  - 相对寻址: IP相对跳转
  - GOT: 全局偏移表，间接访问全局数据
  - PLT: 过程链接表，延迟绑定
- 性能: 略有开销(额外的间接访问)
- 编译: gcc -fPIC -shared
```

```
Q24: 解释"虚表(Vtable)"与C的关系。
    要求: 说明如何在C中模拟虚函数。

参考答案要点:
- C没有虚表，C++类才有
- C模拟虚函数:
  struct Base {
      struct VTable *vptr;
      // 数据成员
  };
  struct VTable {
      void (*method1)(struct Base *);
      void (*method2)(struct Base *);
  };
- 初始化: 构造函数设置vptr指向静态VTable
- 调用: obj->vptr->method1(obj);
- 完整实现: 需要手动实现继承、多态
```

```
Q25: 解释"尾调用优化(Tail Call Optimization, TCO)"。
    要求: 写出可被TCO优化的递归函数。

参考答案要点:
- 条件: 函数的最后操作是调用另一个函数
- 优化: 复用当前栈帧，不增加栈深度
- 可优化:
  int factorial(int n, int acc) {
      if (n <= 1) return acc;
      return factorial(n - 1, n * acc); // 尾调用
  }
- 不可优化:
  int factorial(int n) {
      if (n <= 1) return 1;
      return n * factorial(n - 1); // 非尾调用
  }
- 注意: C标准不保证TCO，GCC -O2通常支持
```

```
Q26: 解释"内存屏障(Memory Barrier/Fence)"。
    要求: 说明为什么需要内存屏障。

参考答案要点:
- 问题: 编译器和CPU可能重排内存操作
- 作用: 阻止重排，保证内存操作顺序
- 类型:
  - 编译器屏障: 阻止编译器重排
  - 内存屏障: 阻止CPU重排
- C11: atomic_thread_fence(memory_order_xxx)
- 场景: 无锁编程、驱动程序、双重检查锁定
```

```
Q27: 解释"竞态条件(Race Condition)"与"数据竞争(Data Race)"。
    要求: 说明二者的区别。

参考答案要点:
- 竞态条件: 程序行为依赖于事件/操作的相对时间
  - 可能导致错误结果
  - 即使原子操作也可能有竞态条件
- 数据竞争: 多个线程并发访问同一内存，至少一个写入
  - 属于特定的竞态条件
  - 导致未定义行为(C11)
- 关系: 所有数据竞争都是竞态条件，反之不成立
- 示例: 检查-然后-行动(非原子)是竞态条件
```

```
Q28: 解释"零开销抽象(Zero-Cost Abstraction)"。
    要求: 说明C语言如何支持零开销抽象。

参考答案要点:
- 定义: 抽象不带来运行时开销
- C的零开销抽象:
  - 宏: 编译期文本替换
  - inline函数: 建议内联展开
  - 类型定义: typedef无运行时开销
  - 泛型选择(_Generic): 编译期选择
- 对比: C++模板也是零开销，虚函数不是
- 原则: "你不用的，你不付费"
```

```
Q29: 解释"对象布局(Object Layout)"与"ABI"。
    要求: 说明为什么不同编译器的结构体可能不兼容。

参考答案要点:
- 对象布局: 数据在内存中的排列方式
- ABI(应用程序二进制接口): 包括
  - 类型大小和对齐
  - 函数调用约定
  - 结构体布局
  - 名称修饰规则
- 不兼容原因:
  - 不同对齐策略
  - 位域布局差异
  - 虚表布局(C++)
- 解决: 使用packed结构体，显式指定布局
```

```
Q30: 解释"缓冲区溢出"的防御机制。
    要求: 说明编译器和操作系统提供的保护。

参考答案要点:
- 编译时保护:
  - 栈金丝雀(Stack Canary): 检测栈溢出
  - FORTIFY_SOURCE: 检查已知函数边界
  - 安全函数: strncpy_s, snprintf等
- 运行时保护:
  - ASLR: 地址空间布局随机化
  - DEP/NX: 数据段不可执行
  - StackGuard: 栈保护
- 编程实践:
  - 使用安全函数
  - 边界检查
  - 避免不安全的gets, strcpy
```

| 7 | `break` | `continue` | 跳出循环 vs 跳过后续代码 |
| 8 | `++i` | `i++` | 先增后使用 vs 先使用后增 |

## 4. 自我测试 (20题)

### 4.1 快速自测

请在不查资料的情况下回答：

```
S1: 以下代码输出什么？为什么？
    int a[5] = {1, 2, 3, 4, 5};
    printf("%d\n", *(a + 2));
    printf("%d\n", a[2]);
    printf("%d\n", 2[a]);

答案: 都输出3。数组索引是语法糖，a[i]等价于*(a+i)等价于i[a]。
```

```
S2: 以下声明的含义是什么？
    int *(*(*fp)(int))(int, int);

答案: fp是一个指针，指向一个函数(参数int)，
     该函数返回一个指针，指向一个函数(参数int,int)，
     该函数返回int*。
```

```
S3: 以下代码是否有问题？
    char *p = malloc(10);
    free(p);
    if (p) { *p = 'a'; }

答案: 有。free后p成为悬空指针，尽管检查p非NULL，
     解引用仍是未定义行为。
```

```
S4: sizeof("Hello")的值是多少？

答案: 6。字符串字面量包含结尾的'\0'。
```

```
S5: 以下代码中，func的返回值类型是什么？
    int (*func(void))[10];

答案: 指向包含10个int的数组的指针。
```

```
S6: 以下代码的输出是什么？
    printf("%d\n", sizeof('A'));

答案: 在C中是sizeof(int)，通常是4。
     注意: C++中sizeof('A')是1。
```

```
S7: 以下代码的输出是什么？
    int x = 5;
    printf("%d %d %d\n", x++, x++, x++);

答案: 未指定行为。函数参数求值顺序未指定。
```

```
S8: static int x; 的默认初始值是什么？

答案: 0。静态存储期的变量默认初始化为0。
```

```
S9: 以下代码中，y的值是多少？
    int x = 5;
    int y = x++ + ++x;

答案: 未定义行为。在同一序列点中多次修改x。
```

```
S10: sizeof(void)的值是多少？

答案: C99前是1(GCC扩展)，C99起不完整类型，非法。
     注意: GCC作为扩展允许，值为1。
```

### 4.2 深度自测

```
S11: 解释为什么以下代码是未定义行为：
     int *p = (int*)0x12345678;
     int x = *p;

答案: 0x12345678可能不是有效的可访问内存地址。
     即使地址有效，也可能未对齐、无读取权限。
```

```
S12: 为什么以下代码可能导致内存泄漏？
     void func(void) {
         char *p = malloc(100);
         return;
     }

答案: p是局部变量，函数返回后丢失。
     分配的内存未释放，且无法再访问。
```

```
S13: 以下代码的潜在问题是什么？
     char *get_str(void) {
         char buf[] = "Hello";
         return buf;
     }

答案: 返回局部数组的地址。
     函数返回后buf已销毁，返回悬空指针。
```

```
S14: 解释为什么malloc(0)的行为是实现定义的。

答案: C标准允许返回NULL或唯一可释放指针。
     不同实现行为不同，应始终检查返回值。
```

```
S15: 以下代码是否有数据竞争？
     int x = 0;
     // Thread 1: x = 1;
     // Thread 2: if (x == 1) { ... }

答案: 是。非原子变量的并发读写是数据竞争，
     即使看起来"安全"也是未定义行为。
```

```
S16: 为什么strncpy不是strcpy的安全替代品？

答案: strncpy在源长于n时不保证'\0'结尾。
     需要手动添加'\0'或使用snprintf。
```

```
S17: 以下代码的输出在不同平台可能不同，为什么？
     int x = -1;
     unsigned int y = 1;
     if (x < y) printf("x < y\n");
     else printf("x >= y\n");

答案: 有符号与无符号比较时，有符号转为无符号。
     -1转为无符号是很大的数，不小于1。
```

```
S18: 以下代码的问题是什么？
     for (int i = 0; i < strlen(s); i++) { ... }

答案: strlen在每次循环都重新计算，O(n^2)复杂度。
     应改为: size_t len = strlen(s); for (...) {}
```

```
S19: 以下代码在64位系统上可能有什么问题？
     int *p = malloc(1000000 * sizeof(int));

答案: 乘法溢出。1000000 * sizeof(int)可能溢出32位。
     应使用: malloc(1000000 * sizeof *p);
     或使用calloc: calloc(1000000, sizeof(int));
```

```
S20: 解释以下代码的输出：
     int a[2][3] = {{1,2,3},{4,5,6}};
     printf("%p %p %p\n", a, a[0], &a[0][0]);
     printf("%p %p\n", a+1, a[0]+1);

答案: 前三个地址值相同(不同指针类型)。
     a+1移动6个int(第二行)，a[0]+1移动1个int。
```
