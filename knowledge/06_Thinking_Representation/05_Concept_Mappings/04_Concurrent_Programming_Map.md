## 一、并发vs并行

| 维度 | 并发(Concurrency) | 并行(Parallelism) |
|:-----|:------------------|:------------------|
| 定义 | 多个任务交替执行 | 多个任务同时执行 |
| 目标 | 提高资源利用率 | 提高计算吞吐量 |
| 需求 | 单核即可 | 需要多核/多处理器 |
| 关注点 | 任务分解与调度 | 数据分解与分配 |

```
并发 (单核交替):        并行 (多核同时):
[任务A][任务B][任务A]   CPU0: [A][A][A]
                        CPU1: [B][B][B]
```

## 三、同步原语选择

**决策树**:

```
需要同步保护?
├── 数据只读 → 无需同步 ✅
├── 单生产者单消费者 → 无锁环形队列
├── 多生产者多消费者
│   ├── 高竞争 → 互斥锁+条件变量
│   └── 读多写少 → 读写锁
└── 原子计数 → 原子变量
```

| 原语 | 开销 | 适用场景 | 可重入 |
|:-----|:-----|:---------|:-------|
| 互斥锁 | 中高 | 通用 | 否 |
| 读写锁 | 高 | 读多写少 | 否 |
| 自旋锁 | 低 | 短临界区 | 否 |
| 信号量 | 中 | 资源计数 | 是 |
| 原子操作 | 最低 | 简单操作 | N/A |

## 四、线程安全概念

**安全级别**:

```
Level 4: 线程安全 ← 使用同步机制
Level 3: 可重入 ← 不依赖共享状态
Level 2: 纯函数 ← 无副作用
Level 1: 线程本地 ← 只读参数
```

**实现策略**:

- 互斥同步: mutex保护共享数据
- 原子操作: `_Atomic`类型无锁访问
- 线程本地存储: `_Thread_local`修饰符
- 不可变数据: 创建后只读
- 消息传递: 队列传递避免共享

## 五、死锁条件

**Coffman四条件**（需同时满足）:

1. **互斥条件**: 资源一次只能被一个线程占用
2. **占有且等待**: 持有资源同时等待其他资源
3. **不可剥夺**: 资源只能由持有者主动释放
4. **循环等待**: 线程之间形成循环等待链

**预防策略**:

| 策略 | 方法 |
|:-----|:-----|
| 破坏占有等待 | 一次性申请所有资源 |
| 破坏不可剥夺 | 使用超时(try_lock_for) |
| 破坏循环等待 | 全局定义锁的获取顺序 |

```c
// 按地址排序获取
void lock_ordered(mtx_t *a, mtx_t *b) {
    if (a < b) { mtx_lock(a); mtx_lock(b); }
    else { mtx_lock(b); mtx_lock(a); }
}
```

## 六、性能考虑

**影响因素**:

| 因素 | 对策 |
|:-----|:-----|
| 锁粒度 | 按数据结构/区域分锁 |
| 锁竞争 | 减少临界区、使用无锁结构 |
| 伪共享 | padding隔离、按线程分区 |
| 上下文切换 | 自旋锁、批量处理 |
| 内存序 | 选择更弱的内存序 |

**优化决策**:

```
临界区执行时间?
├── 几条指令 → 自旋锁/原子操作
├── 微秒级 → 互斥锁
└── 毫秒级 → 细粒度锁/异步处理
```

| 模式 | 吞吐量 | 延迟 | 复杂度 |
|:-----|:-------|:-----|:-------|
| 粗粒度锁 | 低 | 高 | 低 |
| 细粒度锁 | 中 | 中 | 中 |
| 无锁结构 | 高 | 低 | 高 |
| 线程池 | 高 | 中 | 中 |
