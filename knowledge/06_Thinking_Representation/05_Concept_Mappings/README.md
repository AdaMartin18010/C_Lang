# 概念映射 - 概念间关联关系

本目录通过可视化的概念映射图，展示C语言核心概念之间的关联关系。
帮助开发者建立完整的知识体系，理解概念间的相互依赖和作用机制。

---

## 使用场景

这些概念映射适用于以下场景：

- **知识体系构建**：从零开始建立C语言的完整知识结构
- **概念关联理解**：理解不同概念如何相互联系、相互影响
- **学习路径规划**：确定学习某个概念前需要掌握的前置知识
- **知识盲区检测**：通过映射图发现自己知识体系中的薄弱环节
- **复杂问题分析**：多概念交织的问题需要理解概念间关系才能解决

## 映射图阅读方法

每个概念映射文档包含以下类型的图示：

```text
1. 概念关系图
   概念A ──[关系类型]──→ 概念B

2. 层次结构图
   顶层概念
   ├── 子概念1
   │   └── 子子概念
   └── 子概念2

3. 流程依赖图
   前置概念1 ──┐
               ├──→ 目标概念 ──→ 衍生概念

   ┌─────────┬───────┬───────┬───────┐

   ├─────────┼───────┼───────┼───────┤

   │ 特性2   │   ✗   │   ✓   │   ✓   │
   └─────────┴───────┴───────┴───────┘
```

## 使用示例

### 示例1：理解指针与内存的关系

```text
阅读 [指针与内存映射](./01_Pointer_Memory_Mapping.md)

概念关系图:

   变量声明 ──→ 内存分配 ──→ 地址确定
       │                          │
       └──────→ 指针变量 ←────────┘
                    │
                    ▼
            ┌───────────────┐
            │  指针运算      │
            │  +, -, [], ->  │
            └───────┬───────┘
                    │
       ┌────────────┼────────────┐
       ▼            ▼            ▼
    数组访问    结构体成员    函数调用
       │            │            │
       └────────────┴────────────┘
                    │
                    ▼
              内存读写操作

关键理解:
- 指针是地址的抽象
- 指针类型决定了解释内存的方式
- 指针运算与类型大小相关
```

### 示例2：类型系统全景理解

```text
阅读 [类型系统矩阵](./02_Type_System_Matrix.md)

C类型系统层次:

基本类型
├── 整型家族
│   ├── char (字符/小整数)
│   ├── short (短整型)
│   ├── int (标准整型)
│   ├── long (长整型)
│   └── long long (扩展长整型)
├── 浮点家族
│   ├── float (单精度)
│   └── double (双精度)
├── 枚举类型
│   └── enum (命名整数常量)
└── void (空类型)

派生类型
├── 数组类型: T[N]
├── 指针类型: T*
├── 函数类型: T(args)
└── 结构体/联合体: struct/union

类型修饰符
├── const (只读)
├── volatile (易变)
├── restrict (C99, 指针优化提示)
└── _Atomic (C11, 原子类型)
```

### 示例3：并发安全层级理解

```text
阅读 [并发安全层级映射](./03_Concurrency_Safety_Layers.md)

并发控制层级金字塔:

           ┌─────────┐
           │ 事务内存 │  ← 最高抽象，自动同步
           ├─────────┤
           │  锁机制  │  ← 互斥锁、读写锁
           ├─────────┤
           │ 内存屏障 │  ← 显式内存序控制
           ├─────────┤
           │原子操作 │  ← 最底层，硬件支持
           └─────────┘

安全层级递进:
无保护 ──→ 原子变量 ──→ 互斥锁 ──→ 读写锁 ──→ 无锁结构
风险高                              风险低    复杂度
低                                    高

关键概念关联:
原子操作 ──[构成基础]──→ 锁实现
内存序 ────[决定行为]──→ 原子操作可见性
线程安全 ──[依赖关系]──→ 锁 + 内存序
```

### 示例5：编译流程理解

```text
~~阅读 编译流程映射~~ (计划中)

完整编译流程:

源文件(.c) ──→ 预处理器(cpp)
                  │
                  ├── 宏展开
                  ├── 头文件包含
                  ├── 条件编译
                  └── 注释删除
                  │
                  ▼
            预处理文件(.i) ──→ 编译器(cc1)
                                      │
                                      ├── 词法分析
                                      ├── 语法分析
                                      ├── 语义分析
                                      ├── 中间代码生成
                                      └── 优化
                                      │
                                      ▼
                                汇编代码(.s) ──→ 汇编器(as)
                                                      │
                                                      └── 机器码生成
                                                      │
                                                      ▼
                                                目标文件(.o) ──→ 链接器(ld)
                                                                      │
                                                                      ├── 符号解析
                                                                      ├── 重定位
                                                                      └── 库文件链接
                                                                      │
                                                                      ▼
                                                                可执行文件

关键概念关系:
符号 ──[定义]──→ 地址空间位置
符号 ──[引用]──→ 需解析的外部依赖
重定位 ──[修正]──→ 相对地址偏移
```
