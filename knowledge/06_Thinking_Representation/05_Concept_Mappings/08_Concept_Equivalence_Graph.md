## 一、指针-数组-偏移等价链

### 1.1 语义等价定理

**定理 1.1** (数组-指针等价性)
对于任意类型 `T`，指针 `p` 和整数 `i`，以下表达式语义等价：

```c
p[i]    ⟺    *(p + i)    ⟺    *(i + p)    ⟺    i[p]
```

**证明**:

```
1. C标准 6.5.2.1: E1[E2] ≡ (*((E1)+(E2)))
2. 加法交换律: (p + i) ≡ (i + p)
3. 因此: p[i] ≡ *(p+i) ≡ *(i+p) ≡ i[p]
```

### 1.2 完整等价矩阵

| 表达式 | 类型 | 汇编形式 | 适用场景 |
|:-------|:-----|:---------|:---------|
| `p[i]` | 数组索引语法 | `[p + i*4]` | 强调数组语义 |
| `*(p+i)` | 指针算术 | `[p + i*4]` | 强调指针运算 |
| `*(i+p)` | 交换律形式 | `[p + i*4]` | 对称表达 |
| `i[p]` | 罕见但合法 | `[p + i*4]` | 代码混淆 |
| `*((char*)p + i*sizeof(T))` | 字节级 | `[p + i*4]` | 底层操作 |

### 1.3 编译映射过程

```
源代码层                    中间表示层                  汇编层
──────────                  ──────────                  ──────
arr[i] = 42;
    ↓                           ↓                         ↓
*(arr + i) = 42;           t1 = arr + i*4            lea eax, [arr]
    ↓                        t2 = *t1                    lea ecx, [i*4]
*(arr + i * 4) = 42;       t2 = 42                     add eax, ecx
    ↓                        store t2, t1                mov [eax], 42
机器码: C7 04 ... 2A 00 00 00
```

## 二、函数抽象等价谱系

### 2.1 回调机制演化链

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     回调抽象层次谱系                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  C (基础)          C (进阶)         C++               Rust              │
│                                                                          │
│  函数指针 ────────► 函数指针+上下文 ────► std::function ────► Fn trait   │
│      │                  │                  │                  │         │
│      │                  │                  │                  │         │
│      ▼                  ▼                  ▼                  ▼         │
│  无状态              有状态              类型擦除              零开销    │
│  回调                回调                多态                  抽象     │
│                                                                          │
│  void (*cb)(int)     struct {            std::function       impl Fn    │
│                      void (*f)(void*,    <void(int)>          for T     │
│                      int);                                               │
│                      void* ctx;                                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 实现对比

**C 基础版 - 无状态回调**:

```c
typedef void (*callback_t)(int event);

void register_handler(callback_t cb) {
    // 无法携带上下文
    cb(EVENT_START);
}

// 使用限制：无法传递额外数据
void my_handler(int e) { /* 只能访问全局状态 */ }
```

**C 进阶版 - 有状态回调**:

```c
typedef struct {
    void (*func)(void* ctx, int event);
    void* context;
} closure_t;

void register_handler_ex(closure_t* c) {
    c->func(c->context, EVENT_START);
}

// 可以传递上下文
void my_handler_ex(void* ctx, int e) {
    MyData* data = ctx;
    data->count++;
}
```

**等价性分析**:

```
基础回调 + 全局变量  ⟺  有状态回调  ⟺  闭包
     (不推荐)              (C标准)       (语言支持)
```

## 五、控制流等价变换

### 5.1 循环结构等价性

**定理 2.1** (循环结构可互换性)
任何循环结构都可转换为其他形式而不改变语义：

```c
// for 循环
for (init; cond; incr) { body }

// 等价于 while 循环
init;
while (cond) {
    body;
    incr;
}

// 等价于 goto 形式
init;
loop:
if (!cond) goto end;
body;
incr;
goto loop;
end:

// 等价于 递归 (如果尾递归优化)
void loop_fn(state) {
    if (!cond(state)) return;
    body(state);
    loop_fn(next_state(state));
}
```

### 5.2 条件分支等价性

```c
// if-else 链
if (cond1) a();
else if (cond2) b();
else if (cond3) c();
else d();

// 等价于 switch (如果条件是整数相等)
switch (val) {
    case V1: a(); break;
    case V2: b(); break;
    case V3: c(); break;
    default: d(); break;
}

// 等价于 跳转表
typedef void (*func_t)(void);
func_t table[] = {d, a, b, c};  // 索引映射
if (index < 4) table[index]();

// 等价于 函数指针数组 (状态机)
(*state_table[current_state][event])();
```

### 6.2 互斥锁与原子操作等价条件

```c
// 方案A: 互斥锁
pthread_mutex_lock(&mtx);
shared_var++;
pthread_mutex_unlock(&mtx);

// 方案B: 原子操作 (等价，如果操作是原子的)
atomic_fetch_add(&shared_var, 1);

// 等价条件:
// 1. 操作是读-修改-写
// 2. 需要顺序一致性
// 3. 无复杂条件判断

// 不等价的情况 (需要锁):
if (shared_var > threshold) {
    shared_var = new_value;  // 条件更新
}
```

## 八、实践应用

### 8.1 代码重构等价变换

```c
// 重构前: 复杂指针算术
void process(int* data, int n) {
    for (int* p = data; p < data + n; p++) {
        *p = *p * 2;
    }
}

// 重构后: 数组索引 (等价但更清晰)
void process_clear(int* data, int n) {
    for (int i = 0; i < n; i++) {
        data[i] = data[i] * 2;
    }
}

// 编译后代码相同 (优化开启时)
```

### 8.2 优化选择依据

| 等价形式 | 优化级别 | 适用场景 |
|:---------|:---------|:---------|
| `p[i]` | -O0 | 调试、可读性优先 |
| `*(p+i)` | -O0 | 强调指针语义 |
| `lea + mov` | -O1 | 编译器自动选择 |
| `SIMD` | -O3 | 批量数据处理 |
