# 优化级别对比矩阵

## 1. GCC/Clang优化级别总览

| 优化级别 | 标志 | 主要目标 | 调试友好度 | 编译时间 | 运行时性能 | 代码大小 |
|---------|-----|---------|-----------|---------|-----------|---------|
| **无优化** | `-O0` | 编译速度、调试 | ⭐⭐⭐⭐⭐ | 最短 | 最低 | 最大 |
| **基本优化** | `-O1` | 平衡优化 | ⭐⭐⭐⭐ | 短 | 中等 | 减小 |
| **标准优化** | `-O2` | 性能优先 | ⭐⭐⭐ | 中等 | 高 | 适中 |
| **激进优化** | `-O3` | 极致性能 | ⭐⭐ | 长 | 最高 | 增大 |
| **大小优化** | `-Os` | 最小代码 | ⭐⭐⭐ | 中等 | 中等 | 最小 |
| **快速数学** | `-Ofast` | 突破标准限制 | ⭐ | 最长 | 理论最高 | 大 |

| 调试特性 | -O0 | -O1 | -O2 | -O3 | -Os |
|---------|-----|-----|-----|-----|-----|
| **源码行对应** | 精确 | 较好 | 可能错位 | 经常错位 | 较好 |
| **变量可见** | 完整 | 大部分 | 部分 | 很少 | 大部分 |

### 6.1 调试建议

| 开发阶段 | 推荐级别 | 理由 |

| **日常开发** | `-O0 -g` | 最佳调试体验 |

### 7.1 -Ofast 的具体放宽

| 标准行为 | -Ofast行为 | 影响 |
|---------|-----------|-----|
| `a + (b + c) == (a + b) + c` | 可重排 | 精度变化 |

## 10. 混合使用策略

### 10.1 文件级别优化

```makefile
# 大部分用 -O2，性能关键用 -O3
CFLAGS = -O2
performance.o: performance.c
    $(CC) -O3 -c $< -o $@

# 调试友好模块用 -O0
debuggable.o: debuggable.c
    $(CC) -O0 -g -c $< -o $@
```

### 10.2 函数级别优化（GCC属性）

```c
// 强制优化级别
__attribute__((optimize("O3")))
int hot_function(int x) {
    // 高性能代码
}

__attribute__((optimize("Os")))
void cold_function(void) {
    // 代码大小敏感
}
```

### 10.3 编译器特定扩展

| 编译器 | 等效选项 | 特殊选项 |
|-------|---------|---------|
| **GCC** | `-O0` 到 `-O3`, `-Os`, `-Ofast` | `-Og` (优化调试) |
| **Clang** | 同GCC | `-Oz` (比-Os更小) |
| **MSVC** | `/Od`, `/O1`, `/O2`, `/Ox` | `/Ot` (速度), `/Os` (大小) |
| **Intel ICC** | `-O0` 到 `-O3` | `-ipo` (过程间优化) |

## 11. 选择决策树

```
开发阶段？
├── 开发/调试 → -O0 -g
├── 测试 → -O1 或 -O2
└── 发布 → 目标？
    ├── 代码大小敏感 → -Os
    ├── 性能关键 → -O3
    │   └── 浮点密集？
    │       ├── 是 → -Ofast (注意精度)
    │       └── 否 → -O3
    └── 平衡 → -O2 (推荐)
```
