## 一、指针基础 (Pointer Basics)

- **指针定义**
  - 存储内存地址的变量
  - 指向特定数据类型的存储位置
  - 大小固定（32位系统4字节，64位系统8字节）

- **声明与初始化**
  - 声明语法：`类型 *指针名;`
  - 取地址：`&变量名`
  - 解引用：`*指针名`
  - 初始化：`int *p = &var;` 或 `int *p = NULL;`

- **核心操作**
  - `&` - 取地址运算符
  - `*` - 解引用运算符（间接访问）
  - `->` - 结构体指针成员访问
  - `.` - 结构体成员访问（配合解引用）

---

## 二、指针类型分支 (Pointer Types)

### 2.1 空指针 (Null Pointer)

- **定义**：值为 NULL 或 0 的指针
- **用途**：初始化指针，表示不指向任何有效地址
- **检查**：`if (p == NULL)` 或 `if (!p)`
- **注意**：解引用空指针会导致段错误

### 2.2 野指针 (Wild Pointer)

- **定义**：未初始化或已释放但未置空的指针
- **危害**：指向不确定的内存区域，导致不可预测行为
- **示例**：

  ```c
  int *p;          // 野指针 - 未初始化
  *p = 10;         // 危险！
  ```

- **预防**：声明时立即初始化

### 2.3 悬垂指针 (Dangling Pointer)

- **定义**：指向已释放内存的指针
- **产生场景**：
  - 指向的内存被 free 后未置 NULL
  - 指向局部变量的指针在函数返回后使用
  - 指向的对象被删除
- **示例**：

  ```c
  int *p = malloc(sizeof(int));
  free(p);
  // p 现在是悬垂指针
  *p = 10;         // 危险！
  ```

- **解决**：释放后立即置 NULL

### 2.4 其他指针类型

- **常量指针**：`int *const p` - 指针本身不可变
- **指向常量的指针**：`const int *p` - 指向的值不可变
- **双重指针**：`int **p` - 指向指针的指针
- **void 指针**：`void *p` - 通用指针，需强制类型转换

## 三、指针运算分支 (Pointer Operations)

- **算术运算**
  - 加法：`p + n`（移动 n 个元素位置）
  - 减法：`p - n`（回退 n 个元素位置）
  - 自增：`p++`（移动到下一个元素）
  - 自减：`p--`（移动到上一个元素）
  - 差值：`p1 - p2`（计算元素个数间隔）

- **关系运算**
  - 比较：`p1 == p2`, `p1 != p2`
  - 大小：`p1 < p2`, `p1 > p2`（同数组内比较）

- **数组与指针**
  - 数组名即首元素地址
  - `a[i]` 等价于 `*(a + i)`
  - 指针数组：`int *arr[10]`
  - 数组指针：`int (*arr)[10]`

- **字符串与指针**
  - 字符串常量返回首字符地址
  - `char *str = "Hello"`

---

## 四、函数指针分支 (Function Pointers)

- **定义与声明**
  - 语法：`返回类型 (*指针名)(参数列表);`
  - 示例：`int (*fp)(int, int);`

- **赋值与调用**
  - 赋值：`fp = &func;` 或 `fp = func;`
  - 调用：`(*fp)(args);` 或 `fp(args);`

- **应用场景**
  - 回调函数（Callback）
  - 函数表/跳转表
  - 实现多态行为
  - 排序算法的比较函数

- **复杂函数指针**
  - 返回指针的函数：`int *func();`
  - 指向返回指针的函数的指针：`int *(*fp)();`

- **typedef 简化**

  ```c
  typedef int (*CompareFunc)(const void *, const void *);
  CompareFunc cmp = strcmp;
  ```

## 五、常见错误分支 (Common Errors)

- **内存访问错误**
  - 解引用 NULL 指针
  - 解引用野指针/悬垂指针
  - 越界访问（数组/缓冲区溢出）

- **内存管理错误**
  - 内存泄漏（malloc 后未 free）
  - 重复释放（double free）
  - 释放非堆内存

- **类型相关错误**
  - 类型不匹配的指针赋值
  - 忽略指针类型大小差异
  - void 指针未转换直接使用

- **逻辑错误**
  - 指针比较时类型不一致
  - 忘记检查 malloc 返回值
  - 混淆指针值和指向的值

---

## 六、使用技巧分支 (Best Practices)

- **安全初始化**
  - 声明时立即初始化
  - 使用 NULL 而非 0
  - 动态分配后检查返回值

- **释放规范**
  - free 后立即置 NULL
  - 遵循"谁分配谁释放"原则
  - 使用智能指针模式（封装管理）

- **防御性编程**
  - 使用前检查指针有效性
  - 使用断言验证指针状态
  - 避免复杂的指针运算

- **调试技巧**
  - 使用调试器观察指针值
  - 打印指针地址追踪问题
  - 使用内存检测工具（Valgrind）

- **设计模式**
  - 浅拷贝 vs 深拷贝
  - 引用计数管理内存
  - RAII 资源获取即初始化
