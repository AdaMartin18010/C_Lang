## 3. 头文件分类详解

### 3.1 输入输出类

```
[<stdio.h>]
    │
    ├──[文件操作]────────────────────────────────────────────►
    │       │
    │       ├──[fopen/fclose]──► [打开/关闭文件]
    │       ├──[fread/fwrite]──► [二进制读写]
    │       ├──[fgetc/fputc]───► [字符读写]
    │       ├──[fgets/fputs]───► [字符串读写]
    │       ├──[fprintf/fscanf]──► [格式化读写]
    │       └──[fseek/ftell]───► [文件定位]
    │
    ├──[标准流]──────────────────────────────────────────────►
    │       │
    │       ├──[stdin]───► [标准输入]
    │       ├──[stdout]──► [标准输出]
    │       └──[stderr]──► [标准错误]
    │
    ├──[格式化IO]────────────────────────────────────────────►
    │       │
    │       ├──[printf]────► [格式化输出到stdout]
    │       ├──[scanf]─────► [格式化输入从stdin]

## 5. 函数依赖关系

### 5.1 malloc家族依赖

```
[malloc家族依赖图]
    │
    ├──[malloc]──► [基础分配]──► [依赖: sbrk/mmap(系统)]
    │
    ├──[calloc]──► [调用malloc]──► [再调用memset清零]
    │
    ├──[realloc]──► [可能调用malloc]──► [可能调用memcpy]──► [调用free]
    │
    └──[free]──► [内存释放]──► [返回内存池/系统]

[使用约束]
    │
    ├──[malloc/calloc/realloc]──► [必须配对使用]──► [free]
    │
    └──[free]──► [只能释放]──► [malloc家族分配的内存]
```

### 5.2 字符串函数依赖

```
[strcpy家族依赖]
    │
    ├──[strcpy]──► [不安全]──► [可能导致缓冲区溢出]
    │
    ├──[strncpy]──► [较安全]──► [可能不以'\0'结尾]
    │
    ├──[strlcpy(BSD)]──► [安全]──► [确保以'\0'结尾]
    │
    └──[strcpy_s(C11)]──► [安全]──► [运行时检查]

[strcat家族依赖]
    │
    ├──[strcat]──► [先调用strlen找结尾]──► [再调用strcpy复制]
    │
    └──[strncat]──► [限制连接长度]

[strtok依赖]
    │
    ├──[首次调用]──► [传入字符串]──► [静态变量保存状态]
    │
    └──[后续调用]──► [传入NULL]──► [继续使用静态状态]
    │
    └──[注意]──► [非线程安全]──► [使用strtok_r(POSIX)]
```

### 5.3 IO函数依赖

```
[printf家族依赖]
    │
    ├──[printf]──► [输出到stdout]
    │
    ├──[fprintf]──► [输出到指定文件]
    │
    ├──[sprintf]──► [输出到字符串]──► [不安全,可能溢出]
    │
    ├──[snprintf]──► [安全版本]──► [带长度限制]
    │
    └──[vfprintf/vsprintf/vsnprintf]──► [变参版本]

[FILE结构依赖]
    │
    ├──[fopen]──► [创建FILE结构]──► [分配缓冲区]
    │
    ├──[fread/fwrite]──► [使用FILE缓冲区]──► [可能触发系统调用]
    │
    ├──[fflush]──► [刷新缓冲区]──► [写入文件/设备]
    │
    └──[fclose]──► [刷新缓冲区]──► [释放FILE结构]
```

```

    [C标准库]
        │
        ├──是──► [语言规范的一部分]
        │
        └──目的──► [提供可移植的基础功能]

    [标准库组件]
        │

## 8. 常见误区澄清

| 误区 | 正确理解 |
|------|---------|
| strcpy是安全的 | strcpy不检查缓冲区大小，应使用strncpy或strlcpy |
| malloc返回的内存已清零 | malloc不初始化内存，calloc才会清零 |
| realloc总是原地扩展 | realloc可能移动数据到新的地址 |
| printf %f用于float | %f用于double，float会自动提升 |
| fflush(stdin)清空输入 | fflush只对输出流有效，对stdin是未定义行为 |
| sizeof(char)总是1 | C标准保证sizeof(char)=1 |
| gets可以安全使用 | gets已在C11中移除，应使用fgets |
| rand()产生真随机数 | rand()是伪随机，需要srand设置种子 |
| fclose后文件指针可用 | fclose后指针无效，应置NULL |
| exit和return等效 | exit立即终止，return可能执行清理代码 |

### 8.1 关键区分

```
[memcpy] vs [memmove]

memcpy:
- 假设内存区域不重叠
- 通常更快
- 重叠时行为未定义

memmove:
- 允许内存区域重叠
- 会先复制到临时区域
- 更安全但可能稍慢
```

```
[strcmp] vs [strncmp]

strcmp:
- 比较整个字符串直到'\0'
- 可能越界读取

strncmp:
- 最多比较n个字符
- 更安全，推荐用于不可信输入
```
