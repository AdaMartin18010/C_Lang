## 3. 详细命题表

### 3.1 空间分区

| 编号 | 概念A | 连接词 | 概念B | 说明 |
|------|-------|--------|-------|------|
| M1 | 虚拟地址空间 | 分为 | 内核空间+用户空间 | 32位系统通常1:3或2:2分割 |
| M2 | 用户空间 | 包含 | 代码段 | 存储程序机器指令 |
| M3 | 用户空间 | 包含 | 数据段 | 存储全局和静态变量 |
| M4 | 用户空间 | 包含 | 堆区 | 动态内存分配 |
| M5 | 用户空间 | 包含 | 栈区 | 函数调用和局部变量 |
| M6 | 内核空间 | 用于 | 系统内核代码 | 用户态无法直接访问 |

### 3.2 段特性

| 编号 | 概念A | 连接词 | 概念B | 说明 |
|------|-------|--------|-------|------|
| S1 | 代码段 | 属性为 | 只读 | 防止程序修改自身指令 |
| S2 | 代码段 | 可以 | 共享 | 多个进程共享同一程序 |
| S3 | 数据段 | 分为 | 已初始化+未初始化 | .data和.bss |
| S4 | .bss段 | 不占用 | 磁盘空间 | 仅记录大小，运行时分配 |
| S5 | 堆区 | 增长方向 | 向上(高地址) | 向高地址扩展 |
| S6 | 栈区 | 增长方向 | 向下(低地址) | 向低地址扩展 |
| S7 | 堆和栈 | 之间是 | 空闲区域 | 动态调整界限 |

### 3.3 生命周期

| 编号 | 概念A | 连接词 | 概念B | 说明 |
|------|-------|--------|-------|------|
| L1 | 代码段 | 生命周期 | 程序运行期 | 编译时确定 |
| L2 | 全局变量 | 生命周期 | 程序运行期 | 从main前到程序结束 |
| L3 | 局部变量 | 生命周期 | 函数执行期 | 进入函数创建，退出销毁 |
| L4 | 动态内存 | 生命周期 | 程序员控制 | malloc创建，free销毁 |
| L5 | 栈帧 | 生命周期 | 函数调用期 | 调用创建，返回销毁 |

### 3.4 管理机制

| 编号 | 概念A | 连接词 | 概念B | 说明 |
|------|-------|--------|-------|------|
| A1 | 堆内存 | 管理使用 | malloc/free | C标准库函数 |
| A2 | 堆内存 | 管理使用 | new/delete | C++方式 |
| A3 | 栈内存 | 管理由 | 编译器自动 | 无需程序员干预 |
| A4 | 内存分配 | 可能 | 失败 | 返回NULL |
| A5 | 内存释放 | 必须 | 配对使用 | 避免内存泄漏 |
| A6 | 野指针 | 导致 | 未定义行为 | 释放后继续使用指针 |

## 4. 交叉连接

### 4.1 与指针的关系

```
[指针]
    │
    ├──指向──► [代码段] (函数指针)
    │
    ├──指向──► [数据段] (全局变量地址)
    │
    ├──指向──► [堆区] (malloc返回地址)
    │
    ├──指向──► [栈区] (局部变量地址)
    │
    └──通过──► [地址运算] ──► [访问内存单元]
```

### 4.2 与变量的关系

```
[变量声明]
    │
    ├──自动变量──► [分配在栈] ──► [函数结束时自动释放]
    │
    ├──静态变量──► [分配在数据段] ──► [程序结束时释放]
    │
    ├──全局变量──► [分配在数据段] ──► [程序结束时释放]
    │
    └──动态分配──► [分配在堆] ──► [程序员手动释放]
```

### 4.3 与函数调用的关系

```
[函数调用]
    │
    ├──创建──► [栈帧] ──► [压入栈顶]
    │              │
    │              ├──保存──► [返回地址]
    │              ├──保存──► [寄存器值]
    │              ├──分配──► [局部变量空间]
    │              └──保存──► [函数参数]
    │
    └──返回──► [弹出栈帧] ──► [恢复调用者状态]
```

## 5. 层次结构

### 5.1 从抽象到具体

```
Level 1: 虚拟内存概念
    │
    ├──是──► [抽象层] ──► [屏蔽物理内存细节]
    └──提供──► [地址空间隔离] ──► [进程间保护]

Level 2: 地址空间分区
    │
    ├──内核空间──► [操作系统专用]
    └──用户空间──► [应用程序使用]

Level 3: 用户空间段
    │
    ├──代码段──► [程序指令]
    ├──数据段──► [静态数据]
    ├──堆─────► [动态数据]
    └──栈─────► [临时数据]

    │

    ├──垃圾回收机制
    └──内存保护机制
```

## 6. 常见问题与误区

### 6.1 误区澄清

| 误区 | 正确理解 |
|------|---------|
| 局部变量在堆中 | 局部变量在栈中分配 |
| malloc在栈分配 | malloc在堆中分配 |
| 全局变量在堆中 | 全局变量在数据段 |
| 栈大小无限 | 栈大小有限(通常几MB) |
| 堆内存自动释放 | 堆内存必须手动free |
| 返回局部变量地址 | 危险！局部变量会销毁 |

### 6.2 典型错误

```
[栈溢出]
    原因──► [递归太深] 或 [局部数组太大]
    解决──► [减少递归] 或 [使用堆内存]

[堆溢出]
    原因──► [分配太多内存未释放]
    解决──► [及时free] 或 [检查返回值]

[内存泄漏]
    原因──► [malloc后未free]
    解决──► [配对使用malloc/free]

[野指针]
    原因──► [释放后继续使用]
    解决──► [释放后置NULL]

[重复释放]
    原因──► [对同一内存free两次]
    解决──► [释放后置NULL]
```

## 7. 32位 vs 64位对比

| 特性 | 32位系统 | 64位系统 |
|------|---------|---------|
| 虚拟地址空间 | 4GB | 16EB (理论) |
| 用户空间 | 约3GB | 128TB+ |
| 指针大小 | 4字节 | 8字节 |
| 最大内存 | 4GB | 几乎无限 |
| long类型 | 4字节 | 8字节 (Linux) |
| 性能 | 较小指针，缓存友好 | 更大地址空间 |
