## 3. 详细命题表

### 3.1 线程与进程关系

| 编号 | 概念A | 连接词 | 概念B | 详细说明 |
|------|-------|--------|-------|----------|
| TP1 | 进程 | 是 | 资源分配的基本单位 | 拥有独立的地址空间和资源 |
| TP2 | 线程 | 是 | CPU调度的基本单位 | 轻量级，共享进程资源 |
| TP3 | 线程 | 属于 | 进程 | 一个进程可包含多个线程 |
| TP4 | 进程 | 拥有 | 独立地址空间 | 进程间内存隔离 |
| TP5 | 线程 | 共享 | 进程地址空间 | 同一进程的线程共享内存 |
| TP6 | 线程 | 拥有 | 独立栈空间 | 每个线程有自己的栈 |
| TP7 | 线程 | 共享 | 堆内存 | 动态分配内存线程共享 |
| TP8 | 进程间 | 通信需要 | IPC机制 | 管道、共享内存、消息队列等 |
| TP9 | 线程间 | 通信通过 | 共享变量 | 直接访问共享内存 |

### 3.2 线程模型层次

| 编号 | 概念A | 连接词 | 概念B | 详细说明 |
|------|-------|--------|-------|----------|
| TM1 | 用户级线程 | 由 | 用户空间库管理 | 如GNU Pth，轻量级但阻塞问题 |
| TM2 | 内核级线程 | 由 | 操作系统调度 | 真正的并行执行能力 |
| TM3 | 1:1模型 | 映射 | 每个用户线程到一个内核线程 | Linux NPTL采用此模型 |
| TM4 | N:1模型 | 映射 | 多个用户线程到一个内核线程 | 用户空间切换，无法并行 |
| TM5 | N:M模型 | 映射 | N个用户线程到M个内核线程 | 复杂但灵活，Solaris使用 |
| TM6 | POSIX线程 | 提供 | 标准化线程接口 | pthread_create, pthread_join等 |
| TM7 | 线程池 | 预先创建 | 多个工作线程 | 减少线程创建销毁开销 |

### 3.3 同步原语层次

| 编号 | 概念A | 连接词 | 概念B | 详细说明 |
|------|-------|--------|-------|----------|
| SP1 | 互斥锁(mutex) | 提供 | 互斥访问 | 保护临界区，同一时刻只有一个线程进入 |
| SP2 | 互斥锁 | 支持 | 递归锁 | 同一线程可多次加锁 |
| SP3 | 读写锁 | 区分 | 读操作和写操作 | 多读单写，提高并发性 |
| SP4 | 条件变量 | 用于 | 线程等待特定条件 | 与互斥锁配合使用 |
| SP5 | 条件变量 | 支持 | 唤醒机制 | pthread_cond_signal/broadcast |
| SP6 | 信号量 | 用于 | 控制访问数量 | 可允许多个线程同时访问 |
| SP7 | 信号量 | 分为 | 二进制信号量和计数信号量 | 互斥锁是二进制信号量的特例 |
| SP8 | 屏障 | 用于 | 线程同步点 | 所有线程到达后才能继续 |
| SP9 | 自旋锁 | 通过 | 忙等待实现 | 适合短期锁定，避免上下文切换 |

### 3.4 内存模型概念

| 编号 | 概念A | 连接词 | 概念B | 详细说明 |
|------|-------|--------|-------|----------|
| MM1 | 内存模型 | 定义 | 内存操作的可见性 | 规定读写操作的执行顺序 |
| MM2 | 原子操作 | 保证 | 不可分割执行 | 操作要么完全执行，要么不执行 |
| MM3 | 内存屏障 | 强制 | 内存操作顺序 | 防止编译器和CPU重排序 |
| MM4 | 可见性 | 指 | 一个线程的修改对其他线程可见 | 需要同步机制保证 |
| MM5 | 有序性 | 指 | 操作按程序顺序执行 | 编译器和CPU可能重排序 |
| MM6 | 原子性 | 指 | 操作不被中断 | 通过原子指令实现 |
| MM7 | 缓存一致性 | 由 | 硬件协议保证 | MESI等协议 |
| MM8 | 内存顺序 | 分为 | 顺序一致、获取-释放、松散 | C11原子操作支持 |

### 3.5 死锁条件

| 编号 | 概念A | 连接词 | 概念B | 详细说明 |
|------|-------|--------|-------|----------|
| DL1 | 死锁 | 需要 | 四个必要条件同时满足 | Coffman条件 |
| DL2 | 互斥条件 | 是 | 死锁的必要条件之一 | 资源独占使用 |
| DL3 | 持有并等待 | 是 | 死锁的必要条件之一 | 持有资源同时等待其他资源 |
| DL4 | 不可抢占 | 是 | 死锁的必要条件之一 | 资源不能被强制释放 |
| DL5 | 循环等待 | 是 | 死锁的必要条件之一 | 形成等待环路 |
| DL6 | 死锁预防 | 通过 | 破坏四个条件之一 | 如资源一次性分配 |
| DL7 | 死锁避免 | 使用 | 银行家算法 | 动态检测安全状态 |
| DL8 | 死锁检测 | 通过 | 资源分配图 | 检测环路并恢复 |

## 6. POSIX线程API详解

### 6.1 线程生命周期管理

```
[pthread_create]
    │
    ├──参数──► [线程ID] [属性] [入口函数] [参数]
    │
    └──返回──► [成功:0] [失败:错误码]

[pthread_exit]
    │
    ├──功能──► [终止当前线程]
    │
    └──注意──► [不终止进程]

[pthread_join]
    │
    ├──功能──► [等待线程结束] [获取返回值]
    │
    └──注意──► [线程资源回收]

[pthread_cancel]
    │
    ├──功能──► [请求取消线程]
    │
    └──注意──► [需要线程设置取消点]
```

### 6.2 互斥锁操作

```
[pthread_mutex_init]
    │
    ├──[静态初始化]──► [PTHREAD_MUTEX_INITIALIZER]
    │
    └──[动态初始化]──► [pthread_mutex_init(&mutex, &attr)]

[pthread_mutex_lock]
    │
    ├──[锁定互斥锁]──► [已被锁则阻塞]
    │
    └──[非阻塞版本]──► [pthread_mutex_trylock]

[pthread_mutex_unlock]
    │
    └──[释放互斥锁]──► [唤醒等待线程]

[pthread_mutex_destroy]
    │
    └──[销毁互斥锁]──► [释放资源]
```

### 6.3 条件变量操作

```
[pthread_cond_init]
    │
    ├──[静态初始化]──► [PTHREAD_COND_INITIALIZER]
    │
    └──[动态初始化]──► [pthread_cond_init(&cond, &attr)]

[pthread_cond_wait]
    │
    ├──[原子操作]──► [解锁mutex] + [等待条件] + [加锁mutex]
    │
    └──[必须]──► [在循环中检查条件]──► [防止虚假唤醒]

[pthread_cond_signal]
    │
    └──[唤醒一个]──► [等待该条件的线程]

    │
    └──[唤醒所有]──► [等待该条件的线程]
```

## 7. 常见误区澄清

| 误区 | 正确理解 |
|------|---------|
| 多线程一定更快 | 线程切换有开销，CPU密集型任务可能更慢 |
| volatile保证线程安全 | volatile只保证可见性，不保证原子性 |
| 单变量操作是原子的 | 取决于平台和变量大小，64位变量在32位系统非原子 |
| 锁保护代码段 | 锁保护的是数据，不是代码 |
| 死锁只发生在多锁情况 | 单锁也可能因递归导致死锁（非递归锁） |
| pthread_cancel立即终止 | cancel只是请求，线程在取消点才响应 |
| 线程越多越好 | 线程数应与CPU核心数匹配，过多导致上下文切换开销 |

### 7.1 关键区分

```
[线程安全] vs [可重入]

线程安全：
- 可在多线程环境安全调用
- 可以使用锁保护共享数据
- 例如：printf（内部使用锁）

可重入：
- 可在执行过程中被中断并安全重入
- 不使用静态/全局变量
- 不使用锁（可能阻塞）
- 例如：strtok_r（可重入版本）
```

```
[互斥锁] vs [自旋锁]

互斥锁：
- 获取失败时阻塞，让出CPU
- 适合临界区较长的情况
- 上下文切换开销

自旋锁：
- 获取失败时忙等待
- 适合临界区极短的情况
- 避免上下文切换
```
