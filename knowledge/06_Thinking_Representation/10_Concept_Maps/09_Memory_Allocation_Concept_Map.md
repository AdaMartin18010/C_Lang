## 8. 常见误区澄清

| 误区 | 正确理解 |
|------|---------|
| malloc返回的内存已清零 | malloc不初始化，calloc才会清零 |
| free后指针变为NULL | free不会修改指针，需手动置NULL |
| realloc失败会释放原内存 | realloc失败保留原内存，需检查返回值 |
| 内存泄漏会立即导致问题 | 内存泄漏是累积性问题，长期运行才显现 |
| 栈内存可以动态分配 | 栈大小固定，VLA和alloca有限制 |
| 大数组应放栈上 | 大数组应放堆上，避免栈溢出 |
| free可以释放栈内存 | free只能释放malloc家族分配的内存 |
| 内存泄漏检测器万能 | 有些泄漏需要特定场景才能检测到 |
| 所有系统malloc实现相同 | 不同系统实现差异很大 |
| 碎片整理自动进行 | C语言通常不进行自动内存整理 |

### 8.1 关键区分

```
[内存泄漏] vs [内存溢出] vs [野指针]

内存泄漏:
- 分配了内存但未释放
- 程序失去对内存的引用
- 内存逐渐耗尽
- 示例: ptr = malloc(100); ptr = malloc(200); // 第一次分配丢失

内存溢出:
- 写入超出分配范围
- 破坏内存结构
- 可能导致崩溃或安全漏洞
- 示例: char buf[10]; strcpy(buf, "hello world!!!");

野指针:
- 指向已释放或未初始化内存
- 解引用导致未定义行为
- 示例: free(ptr); *ptr = 10; // 使用已释放内存
```

```
[内部碎片] vs [外部碎片]

内部碎片:
- 已分配块内部未使用的空间
- 由对齐要求造成
- 无法被其他请求使用

外部碎片:
- 分散的小空闲块
- 总空闲足够但无连续大块
- 可以通过整理解决
```
