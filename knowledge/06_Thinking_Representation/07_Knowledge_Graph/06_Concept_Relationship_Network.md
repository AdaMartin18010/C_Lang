```
│                          指针概念关联网络                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│                              ┌──────────┐                                   │
│                              │  指针    │                                   │
│                              │ (核心)   │                                   │
│                              └────┬─────┘                                   │
│                     ┌─────────────┼─────────────┐                           │
│                     │             │             │                           │
│                     ▼             ▼             ▼                           │
│               ┌──────────┐ ┌──────────┐ ┌──────────┐                       │
│               │ 内存管理 │ │ 数组操作 │ │ 函数调用 │                       │

│                    │             │             │                           │
│           ┌────────┼────────┐   │   ┌────────┼────────┐                   │
│           │        │        │   │   │        │        │                   │
│           ▼        ▼        ▼   │   ▼        ▼        ▼                   │
│       ┌───────┐ ┌───────┐ ┌───┴───┐ ┌───────┐ ┌───────┐                   │
│       │malloc │ │ free  │ │ 下标  │ │ 函数  │ │ 回调  │                   │
│       └───────┘ └───────┘ │ 运算  │ │ 指针  │ │ 函数  │                   │
│                           └───────┘ └───────┘ └───────┘                   │
│                                                                              │
│  关联强度说明: 实线=强关联, 虚线=中等关联, 点线=弱关联                        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

关联详解:
• 指针 ──[直接]──► 内存地址: 指针存储内存地址
• 指针 ──[算术]──► 数组: p[i] 等价于 *(p+i)
• 指针 ──[调用]──► 函数: (*fp)() 调用函数
• 指针 ──[动态]──► 堆内存: malloc返回指针
```

## 8. 知识盲点检测图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        常见知识盲点检测                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  盲点1: 指针与数组的关系                                                      │
│  ├── 误区: "数组就是指针"                                                     │
│  ├── 真相: 数组名在表达式中退化为指针，但数组不是指针                          │
│  ├── sizeof(array) vs sizeof(ptr) 不同                                       │
│  └── 关联: 理解数组指针 vs 指针数组的区别                                      │
│                                                                              │
│  盲点2: const 位置不同含义不同                                                │
│  ├── const int *p: 指向的内容不能改                                            │
│  ├── int * const p: 指针本身不能改                                             │
│  └── 关联: 类型系统、类型限定符                                                │
│                                                                              │
│  盲点3: 栈内存返回问题                                                        │
│  ├── 误区: 返回局部变量地址                                                   │
│  ├── 真相: 栈帧在函数返回后失效                                               │
│  └── 关联: 内存生命周期、栈与堆的区别                                          │
│                                                                              │
│  盲点4: 整数溢出未定义行为                                                    │
│  ├── 有符号整数溢出 = 未定义行为                                              │
│  ├── 无符号整数溢出 = 明确定义(回绕)                                           │
│  └── 关联: C标准、编译器优化、安全检查                                         │
│                                                                              │
│  盲点5: 并发中的内存可见性                                                    │
│  ├── 误区: 修改全局变量其他线程立即可见                                        │
│  ├── 真相: 需要同步原语或原子操作保证可见性                                    │
│  └── 关联: 内存模型、缓存一致性、happens-before                               │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```
