## 1. 性能问题诊断流程

```
程序性能不佳?
│
├─▶ 1. 性能分析 (Profiling)
│   │
│   ├─▶ 使用性能分析工具
│   │   ├─▶ CPU密集型? ──▶ 使用 perf / gprof
│   │   ├─▶ 内存密集型? ──▶ 使用 Valgrind Massif / heaptrack
│   │   ├─▶ IO密集型? ──▶ 使用 iostat / strace
│   │   └─▶ 并发问题? ──▶ 使用 ThreadSanitizer / perf sched
│   │
│   └─▶ 识别热点
│       ├─▶ 函数级热点 ──▶ 查看热点函数列表
│       ├─▶ 代码级热点 ──▶ 查看汇编/源码对应
│       └─▶ 系统级热点 ──▶ 查看系统调用/上下文切换
│
├─▶ 2. 瓶颈分类
│   │
│   ├─▶ CPU瓶颈?
│   │   ├─▶ 是 ──▶ [CPU优化分支]
│   │   │           ├─▶ 算法复杂度问题?
│   │   │           │   ├─▶ O(n^2) 或更差 ──▶ 优化算法
│   │   │           │   │                   ├─▶ 查找: 使用哈希表 O(1)
│   │   │           │   │                   ├─▶ 排序: 使用快速排序 O(nlogn)
│   │   │           │   │                   └─▶ 搜索: 使用二分查找 O(logn)
│   │   │           │
│   │   │           ├─▶ 缓存未命中率高?
│   │   │           │   ├─▶ 是 ──▶ [缓存优化]
│   │   │           │   │           ├─▶ 数据布局优化
│   │   │           │   │           │   ├─▶ 结构体字段重排 (减少填充)
│   │   │           │   │           │   ├─▶ 数组结构体(AoS) → 结构体数组(SoA)
│   │   │           │   │           │   └─▶ 数据对齐 (__attribute__((aligned)))
│   │   │           │   │
│   │   │           │   │           ├─▶ 访问模式优化
│   │   │           │   │           │   ├─▶ 行优先 vs 列优先遍历
│   │   │           │   │           │   ├─▶ 分块 (tiling) 访问
│   │   │           │   │           │   └─▶ 预取 (prefetch) 提示
│   │   │           │
│   │   │           ├─▶ 分支预测失败率高?
│   │   │           │   ├─▶ 是 ──▶ [分支优化]
│   │   │           │   │           ├─▶  likely/unlikely 宏提示
│   │   │           │   │           ├─▶ 分支排序 (常见路径优先)
│   │   │           │   │           └─▶ 查表替代分支 (switch → 函数指针表)
│   │   │           │
│   │   │           └─▶ 向量化未利用?
│   │   │               ├─▶ 是 ──▶ [SIMD优化]
│   │   │                           ├─▶ 循环向量化 (-ftree-vectorize)
│   │   │                           ├─▶ 手动SIMD (SSE/AVX/NEON)
│   │   │                           └─▶ 使用向量化库 (OpenMP SIMD)
│   │
│   ├─▶ 内存瓶颈?
│   │   ├─▶ 是 ──▶ [内存优化分支]
│   │   │           ├─▶ 分配次数过多?
│   │   │           │   ├─▶ 是 ──▶ 内存池/对象池
│   │   │           │   │           ├─▶ 预分配大块内存
│   │   │           │   │           ├─▶ 自定义分配器
│   │   │           │   │           └─▶ 复用已分配内存
│   │   │           │
│   │   │           ├─▶ 内存碎片严重?
│   │   │           │   ├─▶ 是 ──▶ 使用内存池
│   │   │           │   │           ├─▶ 固定大小分配
│   │   │           │   │           └─▶ 定期整理/重启
│   │   │           │
│   │   │           └─▶ 内存泄漏?
│   │   │               ├─▶ 是 ──▶ Valgrind检测
│   │   │                           ├─▶ 智能指针/RAII
│   │   │                           └─▶ 所有权明确化
│   │
│   └─▶ IO瓶颈?
│       ├─▶ 是 ──▶ [IO优化分支]
│       │           ├─▶ 系统调用次数过多?
│       │           │   ├─▶ 是 ──▶ 缓冲IO (setvbuf)
│       │           │   │           ├─▶ 批量读写
│       │           │   │           └─▶ mmap 替代 read/write
│       │           │
│       │           ├─▶ 随机访问模式?
│       │           │   ├─▶ 是 ──▶ 预读 (readahead)
│       │           │   │           ├─▶ 异步IO (aio)
│       │           │   │           └─▶ 使用SSD/缓存
│       │           │
│       │           └─▶ 网络延迟?
│       │               ├─▶ 是 ──▶ 连接池
│       │                           ├─▶ 异步/非阻塞IO
│       │                           └─▶ 批处理请求
│
└─▶ 3. 优化验证
    │
    ├─▶ 测量基准测试
    │   ├─▶ 优化前性能数据
    │   ├─▶ 优化后性能数据
    │   └─▶ 计算加速比
    │
    ├─▶ 回归测试
    │   ├─▶ 功能正确性验证
    │   └─▶ 边界条件测试
    │
    └─▶ 持续监控
        ├─▶ 生产环境性能指标
        └─▶ 异常告警设置
```

## 2. 编译器优化级别选择

```
选择优化级别
│
├─▶ 调试阶段?
│   ├─▶ 是 ──▶ -O0
│   │           ├─▶ 关闭所有优化
│   │           ├─▶ 最快的编译速度
│   │           └─▶ 最好的调试体验
│   │
│   └─▶ 否 ──▶ 发布阶段?
│               ├─▶ 是 ──▶ [选择发布优化]
│               │           │
│               │           ├─▶ 快速优化 ──▶ -O1
│               │           │               ├─▶ 基本优化
│               │           │               ├─▶ 不增加代码大小
│               │           │               └─▶ 编译速度较快
│               │           │
│               │           ├─▶ 平衡优化 ──▶ -O2 (推荐)
│               │           │               ├─▶ 大多数优化开启
│               │           │               ├─▶ 不牺牲代码大小
│               │           │               └─▶ 最佳性价比
│               │           │
│               │           ├─▶ 激进优化 ──▶ -O3
│               │           │               ├─▶ 所有优化开启
│               │           │               ├─▶ 可能增加代码大小
│               │           │               ├─▶ 可能增加编译时间
│               │           │               └─▶ 需要性能测试验证
│               │           │
│               │           ├─▶ 大小优化 ──▶ -Os
│               │           │               ├─▶ 针对代码大小优化
│               │           │               └─▶ 嵌入式系统首选
│               │           │
│               │           └─▶ 极致性能 ──▶ -Ofast
│               │                           ├─▶ 不严格遵循标准
│               │                           ├─▶ 可能破坏严格IEEE754
│               │                           └─▶ 科学计算慎用
│               │
│               └─▶ 否 ──▶ -Og (推荐用于开发)
│                           ├─▶ 优化调试体验
│                           └─▶ 保留部分优化
```

## 3. 内存布局优化决策

```
数据结构性能不佳?
│
├─▶ 缓存行大小 (通常64字节)
│   └─▶ 确保热数据在同一缓存行
│
├─▶ 结构体布局优化
│   ├─▶ 字段排序 (大到小)
│   │   优化前: char a; int b; char c; // 12字节 (含填充)
│   │   优化后: int b; char a; char c; // 8字节
│   │
│   ├─▶ 热字段前置
│   │   ├─▶ 将频繁访问字段放在结构体前部
│   │   └─▶ 减少缓存行数量
│   │
│   └─▶ 对齐控制
│       ├─▶ __attribute__((packed)) // 无填充
│       └─▶ __attribute__((aligned(64))) // 缓存行对齐
│
├─▶ AoS vs SoA
│   ├─▶ AoS (Array of Structs)
│   │   struct { float x, y, z; } points[1000];
│   │   ├─▶ 适合: 面向对象访问, 随机访问单个点
│   │   └─▶ 缓存: 点的所有数据在一起
│   │
│   ├─▶ SoA (Struct of Arrays)
│   │   struct { float x[1000], y[1000], z[1000]; } points;
│   │   ├─▶ 适合: SIMD向量化, 顺序处理所有x再y再z
│   │   └─▶ 缓存: 同类型数据连续
│   │
│   └─▶ 选择:
│       ├─▶ 需要SIMD ──▶ SoA
│       ├─▶ 随机访问对象 ──▶ AoS
│       └─▶ 混合 ──▶ AoSoA (Array of Struct of Arrays)
│
└─▶ 动态分配优化
    ├─▶ 内存池
    │   ├─▶ 固定大小对象池
    │   ├─▶ 生命周期管理简化
    │   └─▶ 减少malloc开销
    │
    ├─▶ 竞技场分配器 (Arena)
    │   ├─▶ 批量分配, 批量释放
    │   └─▶ 适合: 编译器、游戏帧分配
    │
    └─▶ 栈分配
        ├─▶ alloca() 或变长数组
        ├─▶ 自动释放, 无碎片
        └─▶ 注意: 栈大小限制
```

## 4. 并行化决策树

```
考虑并行化?
│
├─▶ 问题是否可并行?
│   ├─▶ 数据依赖分析
│   │   ├─▶ 无依赖 ──▶ 数据并行 (Embarrassingly Parallel)
│   │   ├─▶ 部分依赖 ──▶ 流水线并行
│   │   └─▶ 强依赖 ──▶ 可能不适合并行
│   │
│   └─▶ 阿姆达尔定律评估
│       ├─▶ 串行部分占比小 ──▶ 并行收益大
│       └─▶ 串行部分占比大 ──▶ 并行收益有限
│
├─▶ 选择并行模型
│   ├─▶ 共享内存 (多线程)
│   │   ├─▶ 细粒度并行 ──▶ OpenMP #pragma omp parallel for
│   │   ├─▶ 显式控制 ──▶ pthreads
│   │   └─▶ C11标准 ──▶ <threads.h>
│   │
│   ├─▶ 分布式内存 (多进程)
│   │   └─▶ MPI (科学计算)
│   │
│   ├─▶ GPU加速
│   │   ├─▶ NVIDIA ──▶ CUDA
│   │   └─▶ 通用 ──▶ OpenCL / SYCL
│   │
│   └─▶ 向量化 (SIMD)
│       ├─▶ 自动向量化 ──▶ -ftree-vectorize
│       ├─▶ x86 ──▶ SSE / AVX / AVX-512
│       └─▶ ARM ──▶ NEON / SVE
│
└─▶ 负载均衡
    ├─▶ 静态划分 ──▶ 均匀任务
    ├─▶ 动态调度 ──▶ 任务大小不均 (OpenMP dynamic)
    └─▶ 任务窃取 ──▶ 工作队列
```

## 5. 优化验证清单

```
优化实施前检查
────────────────────────────────────────────────────────────────
□ 1. 测量基线
  □ 记录优化前性能指标
  □ 确定可接受的最低性能目标
  □ 准备可重复的测试用例

□ 2. 功能保障
  □ 确保有完整的单元测试
  □ 确保有集成测试
  □ 准备边界条件测试

□ 3. 代码审查
  □ 优化代码的可读性
  □ 添加注释说明优化逻辑
  □ 避免过度优化导致维护困难

优化实施后检查
────────────────────────────────────────────────────────────────
□ 1. 性能验证
  □ 测量新性能指标
  □ 计算加速比和效率
  □ 检查是否达到目标

□ 2. 功能验证
  □ 通过所有单元测试
  □ 通过所有集成测试
  □ 边界条件测试通过

□ 3. 稳定性验证
  □ 长时间运行测试
  □ 内存泄漏检查 (Valgrind)
  □ 竞态条件检查 (ThreadSanitizer)

□ 4. 可移植性
  □ 在其他平台测试
  □ 检查编译器兼容性
  □ 确认架构无关性
```

## 6. 性能优化工具矩阵

| 工具 | 用途 | 使用场景 |
|:-----|:-----|:---------|
| perf | CPU性能分析 | 热点函数识别、缓存分析 |
| gprof | 函数级分析 | 编译时插桩，详细调用图 |
| Valgrind (Cachegrind) | 缓存模拟 | 缓存命中率分析 |
| AddressSanitizer | 内存错误 | 开发阶段调试 |
| ThreadSanitizer | 竞态检测 | 多线程问题诊断 |
| Heaptrack | 堆分析 | 内存分配追踪 |
| strace | 系统调用 | IO问题诊断 |
|火焰图 (Flame Graph) | 可视化分析 | 复杂性能问题 |
