## 1. 安全开发流程决策树

```
开始安全编码
│
├─▶ 1. 输入处理
│   │
│   ├─▶ 接收外部输入?
│   │   ├─▶ 是 ──▶ [输入验证检查]
│   │   │           ├─▶ 字符串输入?
│   │   │           │   ├─▶ 是 ──▶ 使用 strncpy_s / snprintf
│   │   │           │   │           避免: strcpy, strcat, sprintf
│   │   │           │   │
│   │   │           │   └─▶ 否 ──▶ 数值输入?
│   │   │           │               ├─▶ 是 ──▶ 检查范围和溢出
│   │   │           │               │           使用: strtol 替代 atoi
│   │   │           │               │
│   │   │           │               └─▶ 否 ──▶ 格式化输入?
│   │   │           │                           ├─▶ 是 ──▶ 使用 %s 指定宽度
│   │   │           │                           │           避免: scanf("%s", buf)
│   │   │           │
│   │   │           └─▶ 缓冲区大小固定?
│   │   │               ├─▶ 是 ──▶ 验证输入长度 < 缓冲区大小
│   │   │               │
│   │   │               └─▶ 否 ──▶ 使用动态分配
│   │   │
│   │   └─▶ 否 ──▶ [无外部输入风险]
│   │
│   └─▶ 2. 指针使用检查
│       │
│       ├─▶ 使用指针?
│       │   ├─▶ 是 ──▶ [指针安全检查]
│       │   │           ├─▶ 初始化检查
│       │   │           │   ├─▶ 所有指针初始化?
│       │   │           │   │   ├─▶ 否 ──▶ 初始化为 NULL
│       │   │           │   │
│       │   │           │   └─▶ malloc 返回值检查?
│       │   │               ├─▶ 否 ──▶ 添加: if (ptr == NULL) 错误处理
│       │   │
│       │   │           ├─▶ 边界检查
│       │   │           │   ├─▶ 数组索引访问?
│       │   │           │   │   ├─▶ 是 ──▶ 检查: index >= 0 && index < size
│       │   │           │   │   │           或使用: if (idx < n) arr[idx]
│       │   │           │
│       │   │           └─▶ 释放后使用检查
│       │   │               ├─▶ free 后指针置 NULL?
│       │   │                   ├─▶ 否 ──▶ ptr = NULL; // 防止悬挂指针
│       │   │
│       │   └─▶ 否 ──▶ [无指针风险]
│       │
│       └─▶ 3. 内存管理检查
│           │
│           ├─▶ 动态内存分配?
│           │   ├─▶ 是 ──▶ [内存安全检查]
│           │   │           ├─▶ 配对检查
│           │   │           │   ├─▶ 每个 malloc 有对应的 free?
│           │   │           │   │   ├─▶ 否 ──▶ 添加 free 或使用 RAII 模式
│           │   │           │
│           │   │           ├─▶ 双重释放检查
│           │   │           │   ├─▶ free 前检查 NULL?
│           │   │           │   │   ├─▶ 否 ──▶ if (ptr) { free(ptr); ptr=NULL; }
│           │   │           │
│           │   │           └─▶ 内存泄漏检查
│           │   │               ├─▶ 所有路径都释放内存?
│           │   │                   ├─▶ 否 ──▶ 检查错误处理路径
│           │   │
│           │   └─▶ 否 ──▶ [无动态内存风险]
│           │
│           └─▶ 4. 整数运算检查
│               │
│               ├─▶ 整数运算?
│               │   ├─▶ 是 ──▶ [整数安全检查]
│               │   │           ├─▶ 溢出检查
│               │   │           │   ├─▶ 运算前检查边界?
│               │   │           │   │   ├─▶ 否 ──▶ 使用: if (a > INT_MAX - b) 溢出
│               │   │           │   │           或 C23: stdckdint.h
│               │   │           │
│               │   │           └─▶ 除零检查
│               │   │               ├─▶ 除法前检查除数?
│               │   │                   ├─▶ 否 ──▶ if (divisor == 0) 错误处理
│               │   │
│               │   └─▶ 否 ──▶ [无整数运算风险]
│
└─▶ 5. 并发安全检查
    │
    ├─▶ 多线程代码?
    │   ├─▶ 是 ──▶ [并发安全检查]
    │   │           ├─▶ 共享数据访问
    │   │           │   ├─▶ 多个线程访问共享数据?
    │   │           │   │   ├─▶ 是 ──▶ 使用同步原语保护
    │   │           │   │   │           ├─▶ 读多写少 ──▶ 读写锁
    │   │           │   │   │           ├─▶ 简单互斥 ──▶ mutex
    │   │           │   │   │           └─▶ 高性能 ──▶ 原子操作
    │   │           │
    │   │           ├─▶ 死锁预防
    │   │           │   ├─▶ 多锁加锁顺序一致?
    │   │           │   │   ├─▶ 否 ──▶ 建立全局锁顺序或使用 trylock
    │   │           │
    │   │           └─▶ 竞态条件
    │   │               ├─▶ 检查-修改-写入操作原子?
    │   │                   ├─▶ 否 ──▶ 使用原子操作或临界区保护
    │   │
    │   └─▶ 否 ──▶ [无并发风险]
    │
    └─▶ 6. 代码审查清单
        └─▶ [最终检查]
            ├─▶ 使用静态分析工具 (clang-static-analyzer, cppcheck)
            ├─▶ 启用编译器警告 (-Wall -Wextra -Werror)
            ├─▶ 运行时检测 (AddressSanitizer, Valgrind)
            └─▶ 代码审查 (同行评审)
```

## 2. 漏洞类型识别与修复

```
发现潜在安全问题?
│
├─▶ 缓冲区溢出?
│   ├─▶ 栈溢出 ──▶ 症状: 段错误, 返回地址被覆盖
│   │   ├─▶ 原因: 局部数组越界写入
│   │   └─▶ 修复:
│   │       ├─▶ 使用安全函数 (strncpy_s, snprintf)
│   │       ├─▶ 启用栈保护 (-fstack-protector-strong)
│   │       └─▶ 使用静态分析检测
│   │
│   ├─▶ 堆溢出 ──▶ 症状: 堆损坏, 后续malloc失败
│   │   ├─▶ 原因: 动态分配缓冲区越界
│   │   └─▶ 修复:
│   │       ├─▶ 分配时记录大小, 访问前检查
│   │       ├─▶ 使用 AddressSanitizer 检测
│   │       └─▶ 使用智能指针/RAII模式
│   │
│   └─▶ 格式化字符串 ──▶ 症状: 崩溃或信息泄露
│       ├─▶ 原因: printf(user_input) 直接使用用户输入
│       └─▶ 修复:
│           ├─▶ printf("%s", user_input) // 固定格式
│           └─▶ 禁用 %n 格式说明符
│
├─▶ Use-After-Free?
│   ├─▶ 症状: 随机崩溃, 数据损坏, 可能被利用执行代码
│   ├─▶ 检测: Valgrind, AddressSanitizer
│   └─▶ 修复:
│       ├─▶ free(ptr); ptr = NULL; // 置空指针
│       ├─▶ 使用引用计数
│       └─▶ 使用内存池管理
│
├─▶ 整数溢出?
│   ├─▶ 有符号溢出 ──▶ 未定义行为!
│   │   └─▶ 修复:
│   │       ├─▶ 运算前检查: if (a > INT_MAX - b)
│   │       ├─▶ 使用更大类型
│   │       └─▶ C23: 使用 ckd_add()
│   │
│   └─▶ 无符号回绕 ──▶ 明确定义但可能意外
│       └─▶ 修复:
│           ├─▶ 同样检查边界
│           └─▶ 使用断言: assert(a <= UINT_MAX - b)
│
├─▶ 竞态条件?
│   ├─▶ TOCTOU (检查到使用的时间差)
│   │   ├─▶ 场景: access() 后 open()
│   │   └─▶ 修复: 使用文件描述符而非路径
│   │
│   ├─▶ 非原子操作
│   │   └─▶ 修复: 使用原子类型 (_Atomic, stdatomic.h)
│   │
│   └─▶ 死锁
│       ├─▶ 场景: A锁→B锁 vs B锁→A锁
│       └─▶ 修复:
│           ├─▶ 统一全局锁顺序
│           ├─▶ 使用 trylock + 回退
│           └─▶ 使用层级锁
│
└─▶ 信息泄露?
    ├─▶ 未初始化内存读取
    │   └─▶ 修复: 初始化所有变量, 使用 -ftrivial-auto-var-init=zero
    │
    ├─▶ 敏感数据残留
    │   └─▶ 修复: 使用 memset_s() 或 explicit_bzero() 清零
    │
    └─▶ 调试信息泄露
        └─▶ 修复: 发布版本移除调试符号, 错误消息不暴露内部信息
```

## 3. 安全函数选择矩阵

| 危险函数 | 安全替代 | 说明 | C标准 |
|:---------|:---------|:-----|:------|
| `strcpy` | `strncpy_s` (C11) | 指定目标大小，确保null终止 | C11 Annex K |
| `strcat` | `strncat_s` (C11) | 检查缓冲区溢出 | C11 Annex K |
| `sprintf` | `snprintf` | 限制输出长度 | C99 |
| `gets` | `fgets` | gets 已移除，fgets 安全 | C99 |
| `scanf("%s")` | `scanf("%19s")` | 指定最大宽度 | C89 |
| `memcpy` | `memcpy_s` (C11) | 检查 dest/src 大小 | C11 Annex K |
| `malloc` | 检查返回值 | 必须检查 NULL | - |
| `atoi` | `strtol` | 可检测错误和溢出 | C89 |
| `strlen` | `strnlen` | 限制最大检查长度 | POSIX |

## 4. 编译器安全选项

```
推荐安全编译选项
────────────────────────────────────────────────────────────────

GCC/Clang:
├── 基础警告 (必须启用)
│   -Wall -Wextra -Werror
│   -Wformat=2 -Wformat-security
│   -Wnull-dereference
│   -Wstack-protector
│
├── 栈保护
│   -fstack-protector-strong  # 所有可能溢出的函数
│   -fstack-protector-all     # 所有函数 (性能开销)
│
├── 位置无关代码
│   -fPIC -pie                # ASLR 支持
│
├── 只读 relocations
│   -Wl,-z,relro,-z,now       # GOT 保护
│
├── 链接器保护
│   -Wl,-z,noexecstack        # 栈不可执行
│
└── 运行时检测 (调试)
    -fsanitize=address        # AddressSanitizer
    -fsanitize=undefined      # UBSan
    -fsanitize=thread         # ThreadSanitizer
```

## 5. 安全检查清单

```
发布前安全检查 (Checklist)
────────────────────────────────────────────────────────────────

[ ] 代码层面
  [ ] 所有 strcpy 替换为 strncpy_s
  [ ] 所有 sprintf 替换为 snprintf
  [ ] 所有 malloc 返回值检查
  [ ] 所有数组索引边界检查
  [ ] 所有整数运算溢出检查
  [ ] 所有多线程共享数据加锁
  [ ] 所有敏感数据使用后清零

[ ] 编译选项
  [ ] 启用 -Wall -Wextra -Werror
  [ ] 启用栈保护 -fstack-protector-strong
  [ ] 启用 ASLR (-fPIC -pie)
  [ ] 启用 RELRO (-Wl,-z,relro,-z,now)
  [ ] 栈不可执行 (-Wl,-z,noexecstack)

[ ] 测试验证
  [ ] 通过 AddressSanitizer 测试
  [ ] 通过 Valgrind 内存检查
  [ ] 模糊测试 (fuzzing) 无崩溃
  [ ] 代码审查通过

[ ] 部署配置
  [ ] 生产环境无调试符号
  [ ] 错误消息不泄露敏感信息
  [ ] 日志记录安全事件
```
