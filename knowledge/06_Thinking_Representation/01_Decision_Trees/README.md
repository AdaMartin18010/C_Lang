# 决策树 - 问题诊断与故障排查

本目录包含一系列结构化决策树文档，用于系统化诊断和解决C语言开发中的常见问题。通过逻辑分支结构，帮助开发者快速定位问题根源并找到解决方案。

---

## 使用场景

这些决策树适用于以下场景：

- **生产环境故障**：程序崩溃、异常退出、性能骤降等紧急情况
- **开发阶段调试**：编译失败、运行时错误、逻辑异常等问题
- **代码审查辅助**：预判潜在问题，进行系统性风险评估
- **团队知识沉淀**：将个人调试经验转化为可复用的诊断流程
- **新人培训指导**：帮助新手建立系统化的调试思维模式

## 使用示例

### 示例1：内存泄漏诊断流程

```
程序运行时间越长，占用内存越多 → 进入 [内存泄漏诊断决策树](./01_Memory_Leak_Diagnosis.md)

                    ┌─────────────────────────┐
                    │    内存是否持续增长？    │
                    └───────────┬─────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              │ YES                              │ NO
              ▼                                  ▼
    ┌─────────────────────┐          ┌─────────────────────┐
    │  使用 valgrind 检测  │          │  检查栈空间使用      │
    │  --leak-check=full  │          │  递归/大数组问题     │
    └──────────┬──────────┘          └─────────────────────┘
               │
      ┌────────┴────────┐
      │ Definitely Lost │
      │  修复malloc/free │
      └─────────────────┘
```

### 示例2：段错误快速定位

```c
// 程序崩溃，显示 SIGSEGV
// → 进入 [段错误排查决策树](./02_Segfault_Troubleshooting.md)

步骤1: 启用 core dump
    $ ulimit -c unlimited
    $ ./program
    
步骤2: 使用 gdb 分析
    $ gdb ./program core
    (gdb) bt          # 查看调用栈
    (gdb) info locals # 查看局部变量
```

### 示例3：编译错误分类处理

```
gcc 报错 → 进入 [编译错误处理决策树](./04_Compilation_Error.md)

错误类型判断:
├── 语法错误 (error: expected ';')
│   └── 检查对应行号附近的语法
├── 未声明标识符 (error: undeclared)
│   └── 检查头文件包含、变量声明
├── 类型不匹配 (error: incompatible types)
│   └── 检查函数签名、类型转换
└── 链接错误 (undefined reference)
    └── 检查库文件链接、函数定义
```

## 决策树阅读指南

每个决策树文档遵循以下结构：

1. **问题描述**：定义该决策树要解决的问题类型
2. **诊断起点**：识别问题的关键症状和入口条件
3. **决策节点**：通过是非判断逐步缩小问题范围
4. **解决方案**：每个叶子节点提供具体的修复建议
5. **预防措施**：避免问题再次发生的最佳实践

## 贡献指南

添加新的决策树时，请确保：

1. 覆盖常见的问题场景（影响至少80%的相关问题）
2. 每个决策节点有明确的判断标准
3. 提供可操作的解决方案（而不仅是理论说明）
4. 包含实际的代码示例和命令行操作
5. 定期根据新发现的问题模式进行更新
