## Zig 0.14.0 (2025-03) 重大更新

### 核心特性

| 特性 | 描述 | 影响 |
|------|------|------|
| **增量编译** | 仅重新分析变更代码，500K行项目: 14s → 63ms | 开发效率提升 |
| **Decl Literals** | 扩展枚举字面量语法，支持结构体声明引用 | 代码更简洁 |
| **Labeled Switch** | 带标签的 switch 语句，状态机性能 +13% | 控制流优化 |
| **@branchHint** | 细粒度的分支预测控制 | 性能优化 |
| **移除 @fence** | 简化内存模型，使用更强内存序替代 | 语义更清晰 |

### 标准库更新

| 组件 | 变更 | 说明 |
|------|------|------|
| **DebugAllocator** | 新增堆栈跟踪和泄漏检测 | zig ast-check 性能提升 10.1% |
| **SmpAllocator** | 多线程优化的分配器 | 200行代码匹敌 glibc malloc |
| **ZON 支持** | Zig Object Notation 解析/序列化 | 数据交换格式 |
| **Allocator.remap** | 新增内存重新映射 API | 支持 mremap，避免 memcpy |
| **TLS 支持** | std.crypto.tls 新增 | 安全通信 |
| **页面大小动态化** | std.heap.page_size → pageSize() | 跨平台兼容性 |

### 破坏性变更

```zig
// 1. @fence 移除 - 使用更强的内存序替代
// 旧代码:
@fence(.seq_cst);

// 新代码:
// 使用 SeqCst 原子操作，或升级为 RMW 操作
_ = atomic.fetchAdd(0, .SeqCst);

// 2. @setCold 替换为 @branchHint
// 旧代码:
@setCold(true);

// 新代码:
@branchHint(.cold);

// 3. Unmanaged 容器默认初始化
// 旧代码:
var list: ArrayListUnmanaged(u32) = .{};

// 新代码:
var list: ArrayListUnmanaged(u32) = .empty;
```

### "Writergate" - I/O系统重构

2025年末发生的重大标准库重构，解决旧版I/O在高性能场景下的瓶颈：

| I/O特性 | 旧版本 (0.11-0.14) | 新版本 (0.15.1+) |
|:--------|:------------------:|:----------------:|
| 缓冲方式 | 内置隐式缓冲 | **显式缓冲管理** |
| 性能透明度 | 系统调用时机不透明 | 每次系统调用可见 |
| 接口定义 | 泛型Reader/Writer | 基于Io模块的非泛型接口 |
| 错误处理 | 错误码宽泛 | 精确的`std.Io.Writer.Error` |

#### 破坏性变更示例

```zig
// 旧代码 (0.14及之前):
var writer = file.writer();
writer.print("Hello\n");  // 可能隐式缓冲

// 新代码 (0.15.1+):
var buf_writer = std.io.bufferedWriter(file.writer());
buf_writer.writer().print("Hello\n");
try buf_writer.flush();  // 必须显式刷新
```

## Zig 0.16.0-dev 开发中更新 (2026-03)

> ⚠️ **警告**: Zig 0.16.0 当前为开发版本 (master分支)，尚未发布稳定版。
> 所有API和特性可能变化。生产环境请使用 [Zig 0.14.0](#zig-0140-2025-03-重大更新)。

### 核心特性 (开发中)

| 特性 | 描述 | 影响 |
|:-----|:-----|:-----|
| **类型系统重构** | 30,000行PR，延迟字段分析 | 编译速度提升，依赖循环错误改进 |
| **std.Io接口** | 统一I/O抽象层，支持多种后端 | 无缝切换Threaded/Evented模式 |
| **增量编译** | 默认启用，bug修复 | 小修改毫秒级重编译 |
| **io_uring支持** | Linux高性能异步I/O | 服务器性能大幅提升 |
| **Grand Central Dispatch** | macOS原生异步支持 | Apple平台优化 |

### 异步模型演进

```text
Zig 0.16 异步架构
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

当前实现 (Stackful - Fibers):
  • 无函数着色 (无async fn)
  • 运行时选择执行策略
  • 用户态栈切换
  • 适用于大多数服务器/桌面应用

未来补充 (Stackless - Coroutines):
  • WebAssembly兼容 (栈切换不支持)
  • GPU/SPIR-V兼容
  • 编译时确定帧大小
  • 与JS事件循环集成

统一接口: std.Io
  ├─ std.Io.Threaded   (线程池)
  ├─ std.Io.Evented    (io_uring/GCD)
  └─ std.Io.Stackless  (规划中)
```

### 编译器改进

**类型解析重设计 (2026-03-10)**:

- 延迟字段分析：未使用的类型字段不编译
- 依赖循环错误：详细的循环依赖诊断信息
- 增量编译：消除"过度分析"问题

**zig libc项目**:

- 目标：用Zig替代C实现的libc函数
- 进展：已删除~250个C源文件，剩余2032个
- 收益：减少安装体积、LTO优化、更好的I/O控制
