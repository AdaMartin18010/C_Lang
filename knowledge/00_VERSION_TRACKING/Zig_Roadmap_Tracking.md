## 五、包管理器发展

```zig
// Zig 2026 包管理目标

// 1. 简化依赖声明
.{
    .name = "my-project",
    .version = "1.0.0",
    .dependencies = .{
        .sqlite = .{
            .url = "https://github.com/sqlite/zig-sqlite/archive/v1.0.0.tar.gz",
            .hash = "1220...",
        },
    },
}

// 2. 支持 git 子模块
.{
    .url = "git+https://github.com/user/repo#v1.0.0",
}

// 3. 支持路径依赖 (开发时)
.{
    .path = "../local-package",
}

// 4. 条件依赖
.{
    .target = .{
        .os = .windows,
        .dependency = .{ .name = "winapi" },
    },
}
```

## 六、与 C23 标准对照

| 特性 | C23 | Zig | 兼容性 |
|:-----|:----|:----|:-------|
| `nullptr` | ✅ | `null` (可选类型) | 概念相似 |
| `constexpr` | 有限 | `comptime` (完整) | Zig 更强 |
| `typeof` | ✅ | `@TypeOf` | 相似 |
| `_Static_assert` | ✅ | `@compileError` | 相似 |
| 多线程 | `<threads.h>` | 内置 | Zig 更简洁 |
| 原子操作 | `<stdatomic.h>` | `std.atomic` | 相似 |
| 位操作 | `<stdbit.h>` | 内置 | Zig 更整合 |
| 溢出检查 | `<stdckdint.h>` | 内置 | Zig 更安全 |
| 泛型 | `_Generic` | `comptime` | Zig 更强大 |
