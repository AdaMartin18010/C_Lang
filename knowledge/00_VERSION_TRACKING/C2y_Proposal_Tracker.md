**N3742 Named Loops 详细状态:**

```c
// 提案语法 (N3742)
outer: for (int i = 0; i < N; i++) {
    inner: for (int j = 0; j < M; j++) {
        if (condition)
            break outer;  // 跳出外层循环
    }
}

// 争议点:
// 1. 标签作用域规则
// 2. 与goto标签的命名空间冲突
// 3. 语法风格选择 (冒号位置)
```

**N3472 auto 详细状态:**

```c
// 提案语法
auto x = 1;           // int
auto y = 1.0;         // double
auto z = "hello";     // const char*

auto p = malloc(100); // void* (C) 或需要转换

// 争议点:
// 1. C中auto原本是存储类 (C23已删除此用法)
// 2. 与C++ auto的行为差异
// 3. 指针类型的推导
```

**N3753 Elvis 操作符详细状态:**

```c
// 提案语法 (N3753)
int result = maybe_null ?: default_value;
// 等价于: (maybe_null ? maybe_null : default_value)

// 与GCC扩展对比
// GCC: x ?: y  (如果x非零/非空则返回x，否则y)
// 提案: 类似，但标准化语义

// 链式使用
int val = a ?: b ?: c ?: 0;

// 争议点:
// 1. 与GCC扩展的细微语义差异
// 2. 短路求值规则
// 3. 与三元运算符的关系
```

## 8. 优先级评估详解

### 8.1 评估标准

```
优先级评估矩阵
━━━━━━━━━━━━━━━

每个提案从5个维度评估 (1-5分):

1. 用户需求 (User Demand)
   • 社区呼声
   • 实际项目需求
   • 跨平台需求

2. 实现可行性 (Feasibility)
   • 编译器实现难度
   • 运行时开销
   • 平台兼容性

3. 标准兼容性 (Compatibility)
   • 与C++的协调
   • 向后兼容性
   • 与现有代码的冲突

4. 技术价值 (Technical Value)
   • 解决实际问题的能力
   • 代码质量提升
   • 安全性改进

5. 生态系统影响 (Ecosystem)
   • 对库的影响
   • 工具链支持需求
   • 教育/培训成本

总分 = 加权平均
• 高优先级: 4.0-5.0
• 中优先级: 3.0-3.9
• 低优先级: < 3.0
```
