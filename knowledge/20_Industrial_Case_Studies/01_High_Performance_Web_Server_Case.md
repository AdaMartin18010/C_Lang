## 一、项目背景与需求

### 1.1 业务场景

```
场景：为IoT设备提供HTTP API服务

业务指标：
• 设备数: 100万在线
• QPS: 100万/秒峰值
• 延迟: P99 < 10ms
• 可用性: 99.99%
• 单机成本: < $500/月

技术约束：
• 协议: HTTP/1.1 + WebSocket
• 数据: JSON + Protobuf
• 安全: TLS 1.3 + mTLS
• 部署: Docker + Kubernetes
```

### 1.2 架构选型决策

```
┌─────────────────────────────────────────────────────────────┐
│                    架构决策记录 (ADR)                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  决策1: I/O模型选择                                          │
│  ─────────────────                                           │
│  选项: 阻塞IO / IO多路复用 / io_uring                         │
│  决策: io_uring (Linux 5.1+)                                 │
│  理由:                                                       │
│    • 减少系统调用开销 (80% reduction)                        │
│    • 支持真正异步I/O                                          │
│    • 批处理减少上下文切换                                      │
│  风险: 内核版本依赖，需fallback到epoll                        │
│                                                             │
│  决策2: 并发模型                                              │
│  ─────────────────                                           │
│  选项: 多线程 / 多进程 / 协程 / 事件驱动                      │
│  决策: 事件驱动 + 线程池                                      │
│  理由:                                                       │
│    • 主线程: io_uring事件循环                                 │
│    • 工作线程: CPU密集型任务 (JSON解析)                        │
│    • 避免协程栈内存开销                                        │
│                                                             │
│  决策3: 内存管理                                              │
│  ─────────────────                                           │
│  选项: malloc/jemalloc/自定义分配器                           │
│  决策: 自定义Arena分配器 + jemalloc后端                        │
│  理由:                                                       │
│    • 请求生命周期管理简化                                       │
│    • 减少内存碎片                                              │
│    • 零拷贝支持                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心代码

```c
// io_uring事件循环核心
void event_loop(uring_context_t *ctx) {
    struct io_uring *ring = &ctx->ring;
    struct io_uring_cqe *cqes[32];

    submit_accept(ctx);

    while (running) {
        io_uring_submit(ring);
        int num_cqes = io_uring_wait_cqe_nr(ring, cqes, 32, NULL);

        for (int i = 0; i < num_cqes; i++) {
            struct io_uring_cqe *cqe = cqes[i];
            handle_completion(ctx, cqe);
            io_uring_cqe_seen(ring, cqe);
        }

        check_timeouts(ctx);
    }
}
```
