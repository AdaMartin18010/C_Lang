## 2. 安全启动 (Secure Boot)

### 2.1 信任链形式化定义

**定义 2.1 (信任链)**

```
信任链 = 从不可变根开始逐级验证的软件链

根信任 (Root of Trust):
  - ROM代码 (不可更新)
  - 硬件安全模块 (HSM/TPM)
  - 熔丝中的公钥哈希

验证过程:
  BootROM (可信) → 验证 Bootloader
                      ↓
           Bootloader → 验证 主固件
                            ↓
                 主固件 → 验证 应用

签名验证:
  Verify(PK_root, Sign_SK(SW_i), SW_i) = True/False

  使用 ECDSA/RSA 签名算法

防回滚 (Anti-Rollback):
  版本号存储在安全单调计数器或eFuse中

  条件: Version_new > Version_stored

  否则拒绝启动旧版本 (防止降级攻击)
```

## 4. 密码学硬件加速

### 4.1 算法性能对比

| 算法 | 纯软件 | 硬件加速 | 加速比 | 功耗 |
|------|--------|----------|--------|------|
| AES-128 | 1-5MB/s | 100-500MB/s | 50-100x | 降低80% |
| SHA-256 | 5-20MB/s | 200-800MB/s | 40-100x | 降低85% |
| RSA-2048 | 10-50ms/op | 0.1-1ms/op | 50-100x | 降低90% |
| ECC P-256 | 5-20ms/op | 0.05-0.5ms/op | 100-200x | 降低90% |
| TRNG | <100kbps | 10-100Mbps | N/A | 专用 |

### 4.2 安全密钥存储

```
密钥层次结构
│
├── 设备根密钥 (Device Root Key)
│   ├── 存储: 熔丝/PUF/安全元件
│   ├── 用途: 派生其他密钥
│   └── 属性: 永不离开芯片
│
├── 密钥加密密钥 (KEK)
│   ├── 派生: 从DRK派生
│   ├── 用途: 加密存储其他密钥
│   └── 存储: 仅存在于安全RAM
│
└── 应用密钥
    ├── 传输密钥 (TLS/DTLS)
    ├── 存储密钥 (文件加密)
    └── 签名密钥 (固件签名)

密钥生命周期:
  生成 → 存储 → 使用 → 轮换 → 销毁

  销毁: 物理清除 (多次覆盖)
```
