## 2. Flash存储技术

### 2.1 NOR vs NAND Flash

**定义 2.1 (Flash单元)**

```
浮栅晶体管原理:
  - 通过隧穿效应注入/移除电子改变阈值电压
  - 编程: 注入电子 → Vth升高 → '0'
  - 擦除: 移除电子 → Vth降低 → '1'

擦写限制:
  - 编程/擦除循环次数有限 (耐久性)
  - 必须先擦除后写入
  - 擦除按块进行

NOR Flash:
  - 架构: 并行, 类似SRAM接口
  - 读取: 随机访问, 快 (50-100ns)
  - 写入: 慢 (10μs-1ms)
  - 擦除: 块擦除 (0.5-1s)
  - 容量: 通常 < 256MB
  - 应用: 代码存储, XIP (Execute In Place)

NAND Flash:
  - 架构: 串行, 页式访问
  - 读取: 顺序访问为主, 65μs/页
  - 写入: 页编程 (200μs-1ms)
  - 擦除: 块擦除 (2-3ms)
  - 容量: GB-TB级
  - 应用: 大容量数据存储
```

### 2.2 NAND Flash 管理

```
NAND Flash 关键管理算法

磨损均衡 (Wear Leveling):
  目标: 均匀分布擦写, 延长寿命

  算法类型:
    - 动态WL: 移动频繁修改的数据到空闲块
    - 静态WL: 移动静态数据, 释放高磨损块

  擦写次数均衡:
    σ(擦写次数) / mean(擦写次数) < 阈值

坏块管理 (BBM):
  - 出厂坏块: 标记在Spare Area
  - 运行期坏块: 擦除失败/ECC不可纠正
  - 替换表: 逻辑块号 → 物理块号

ECC (Error Correction Code):
  - SLC: 通常1-4 bit/512B (Hamming/BCH)
  - MLC: 4-8 bit/1KB (BCH)
  - TLC: 24-40 bit/1KB (LDPC)

  纠错能力:
    t = (d-1)/2  (可纠正t位错误)

垃圾回收 (GC):
  触发条件: 空闲块 < 阈值

  过程:
    1. 选择含无效页最多的块 (Victim)
    2. 将有效页复制到空闲块
    3. 擦除Victim块
    4. 加入空闲列表

  写放大 (Write Amplification):
    WA = 实际写入NAND量 / 主机请求写入量
    理想: WA = 1
    实际: WA = 1.1-4.0 (取决于工作负载)
```

## 3. ECC 技术

### 3.1 ECC算法对比

| 算法 | 纠错能力 | 开销 | 复杂度 | 应用 |
|------|----------|------|--------|------|
| **奇偶校验** | 1bit | 1bit/byte | 极低 | SRAM |
| **Hamming** | 1-2bit | 12% | 低 | NOR/早期NAND |
| **RS** | 字节级 | 可变 | 中 | 通信/CD |
| **BCH** | 4-24bit | 13-25% | 中高 | NAND/SSD |
| **LDPC** | 40-60bit | 7-15% | 高 | TLC/QLC NAND |

### 3.2 BCH编码原理

```
BCH (Bose-Chaudhuri-Hocquenghem) 码

参数: BCH(n, k, t)
  n = 码字长度
  k = 数据位长度
  t = 可纠正错误位数

  例: BCH(8191, 7683, 81) 用于NAND

编码过程:
  1. 生成多项式 g(x) = LCM(m1(x), m3(x), ..., m(2t-1)(x))
  2. 数据多项式 m(x)
  3. 码字 c(x) = m(x) × g(x)

解码过程 (Berlekamp-Massey算法):
  1. 计算伴随式 (Syndrome)
  2. 求解错误定位多项式
  3. Chien搜索找错误位置
  4. Forney算法求错误值
  5. 纠正错误

硬件实现:
  - 编码器: 线性反馈移位寄存器 (LFSR)
  - 解码器: 专用硬件加速器或软件
  - 延迟: 编码1周期/byte, 解码10-100周期
```
