## 1. 实时系统基础定义

### 1.1 实时任务模型

**定义 1.1 (周期性任务)**

```
任务 τi = (Ci, Ti, Di, Pi)

Ci = 最坏执行时间 (WCET)
Ti = 周期 (周期任务) 或 最小到达间隔 (偶发任务)
Di = 相对截止期限
Pi = 优先级

利用率:
  Ui = Ci / Ti

总利用率:
  U = Σ Ui = Σ(Ci/Ti)

任务分类:
  硬实时: Di必须满足，违反即系统失效
  软实时: Di应尽量满足，违反可容忍
  固实时: Di通常满足，偶尔违反可接受
```

### 1.2 调度算法对比矩阵

| 算法 | 类型 | 最优性 | 实现复杂度 | 上下文切换 | 适用场景 |
|------|------|--------|------------|------------|----------|
| **RMS** | 静态优先级 | 固定优先级最优 | 低 | 中 | 周期任务 |
| **DMS** | 静态优先级 | 截止期限最优 | 低 | 中 | Di<Ti |
| **EDF** | 动态优先级 | 全局最优 | 中 | 高 | 高利用率 |
| **LLF** | 动态优先级 | 最优 | 高 | 极高 | 理论分析 |
| **FPS** | 静态优先级 | 次优 | 低 | 低 | 通用嵌入式 |

## 2. 速率单调调度 (RMS)

### 2.1 RMS可调度性分析

**定理 2.1 (Liu & Layland, 1973)**

```
定理: 对于n个独立周期任务，RMS可调度的充分条件:

  U = Σ(Ci/Ti) ≤ n × (2^(1/n) - 1)

利用率界限:
  n=1: U ≤ 1.000
  n=2: U ≤ 0.828
  n=3: U ≤ 0.780
  n→∞: U ≤ ln(2) ≈ 0.693

精确分析 (Response Time Analysis):
  Ri = Ci + Σ[j∈hp(i)] ⌈Ri/Tj⌉ × Cj

  迭代计算直到收敛或 Ri > Di

  可调度条件: Ri ≤ Di

示例:
  Task1: C=1, T=4, P=high
  Task2: C=2, T=6, P=medium
  Task3: C=1, T=8, P=low

  U = 1/4 + 2/6 + 1/8 = 0.25 + 0.333 + 0.125 = 0.708

  充分条件: 0.708 > 0.780? 否，无法判断

  精确分析:
    R1 = C1 = 1 ≤ 4 ✓

    R2 = C2 + ⌈R2/T1⌉×C1
       = 2 + ⌈R2/4⌉×1
    迭代: R2=2 → 2+1=3 → 3+1=4 → 4+1=5?
    检查: ⌈5/4⌉=2, R2=2+2=4, 收敛
    R2 = 4 ≤ 6 ✓

    R3 = C3 + ⌈R3/T1⌉×C1 + ⌈R3/T2⌉×C2
    迭代: R3=1 → 1+1+2=4 → 4+1+2=7 → ⌈7/4⌉=2, ⌈7/6⌉=2
    R3=1+2+4=7 > 8? 再算: ⌈7/4⌉=2, ⌈7/6⌉=2, 1+2+4=7
    收敛 R3=7 ≤ 8 ✓

  结论: 可调度
```

### 2.2 响应时间分析算法

```
响应时间分析 (RTA) 算法

输入: 任务集 Γ = {τ1, τ2, ..., τn}, 按优先级排序
输出: 每个任务的可调度性

for each τi in Γ (按优先级从高到低):
    Ri = Ci                          /* 初始值 */

    repeat:
        Ri_prev = Ri
        interference = 0

        for each τj in hp(τi):      /* 高优先级任务 */
            interference += ceil(Ri_prev / Tj) × Cj

        Ri = Ci + interference

    until (Ri == Ri_prev) or (Ri > Di)

    if Ri > Di:
        return NOT_SCHEDULABLE

return SCHEDULABLE

时间复杂度: O(n² × 迭代次数)
迭代次数: 通常几次到几十次收敛

阻塞项扩展 (共享资源):
  Ri = Ci + Bi + Σ[j∈hp(i)] ⌈Ri/Tj⌉ × Cj

  Bi = 任务τi的最大阻塞时间
```

## 3. 最早截止期限优先 (EDF)

### 3.1 EDF最优性证明

**定理 3.1 (EDF最优性)**

```
定理: 对于隐式截止期限任务集 (Di = Ti)，
EDF可调度的充要条件是 U ≤ 1

证明概要 (交换论证):
  假设存在EDF不可调度但其他算法可调度的任务集

  设EDF首次错过截止期限的时刻为t
  则t时刻前的总需求 > t

  即 Σ(Ci × ⌈t/Ti⌉) > t

  因此 Σ(Ci/Ti) = U > 1

  任何算法在此情况下都无法调度

  反之，若U ≤ 1，EDF总能在截止期限前完成最高优先级任务

对比RMS:
  EDF: U ≤ 1 (100%利用率)
  RMS: U ≤ 0.693 (69.3%利用率，n→∞)

  EF利用率高出44%

缺点:
  - 动态优先级实现复杂
  - 运行时开销高
  - 瞬态过载时行为不可预测
```

## 4. 资源访问协议

### 4.1 优先级继承协议 (PIP)

**定义 4.1 (优先级继承)**

```
问题: 优先级反转

场景:
  低优先级任务L持有资源R
  高优先级任务H请求资源R被阻塞
  中优先级任务M运行(优先级介于H和L之间)

  结果: M延迟了H (优先级反转)

PIP解决方案:
  当H被L阻塞时，L临时继承H的优先级
  释放资源后恢复原始优先级

阻塞时间计算:
  Bi = Σ[k∈resources(i)] max(Ck_use)

  即每个资源的最长临界区

定理: PIP保证最大阻塞时间为单个资源临界区
  (而非链式阻塞)

局限性:
  - 死锁仍可能发生
  - 链式阻塞 (多资源时)
```

### 4.2 优先级天花板协议 (PCP)

```
优先级天花板协议规则

定义:
  资源R的天花板优先级 = 所有可能使用R的任务中的最高优先级

协议规则:
  1. 任务τi只能在优先级高于当前被锁定资源的最高天花板时运行

  2. 若τi尝试锁定资源R:
     - 如果R被锁定，τi阻塞
     - 如果R的 ceilings(R) > τi的优先级，τi阻塞
       (即使R未被锁定)

性质:
  - 死锁预防
  - 最大阻塞时间为一个临界区
  - 实现复杂度高

 ceiling(R) = max{priority(τ) | τ uses R}

示例:
  Task H (P=3) uses {R1}
  Task M (P=2) uses {R2}
  Task L (P=1) uses {R1, R2}

  ceiling(R1) = 3
  ceiling(R2) = 2

  L运行，锁定R1 (OK, no ceiling violation)
  H到达，尝试R1 → 阻塞，L继承P=3
  M到达，尝试R2 → 阻塞!
       (即使R2空闲，但L持有R1且ceil(R2)=2 ≥ M)
       L继续运行，完成所有资源后释放
```
