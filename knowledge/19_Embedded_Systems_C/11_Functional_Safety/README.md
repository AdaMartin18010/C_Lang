# 功能安全深度解析 (IEC 61508 / ISO 26262)

> **核心目标**: 系统故障风险降至可接受水平
> **安全生命周期**: 概念、开发、生产、运营、退役
> **安全完整性等级**: SIL (工业) / ASIL (汽车)
> **最后更新**: 2026-03-29

---

## 1. 功能安全基础概念

### 1.1 安全完整性等级 (SIL/ASIL) 定义

**定义 1.1 (SIL等级)**

```
安全完整性等级 = 对安全功能失效概率的离散化分级

IEC 61508 (通用工业):
┌──────┬────────────────┬────────────────┐
│ SIL  │ 低需求模式      │ 高需求/连续模式 │
│      │ (PFD_avg)      │ (PFH)          │
├──────┼────────────────┼────────────────┤
│ SIL4 │ 10^-5 to 10^-4 │ 10^-9 to 10^-8 │
│ SIL3 │ 10^-4 to 10^-3 │ 10^-8 to 10^-7 │
│ SIL2 │ 10^-3 to 10^-2 │ 10^-7 to 10^-6 │
│ SIL1 │ 10^-2 to 10^-1 │ 10^-6 to 10^-5 │
└──────┴────────────────┴────────────────┘

ISO 26262 (汽车电子) ASIL等级:
┌──────┬──────────┬──────────────┬──────────────┐
│ ASIL │ 严重度(S) │ 暴露率(E)    │ 可控性(C)    │
├──────┼──────────┼──────────────┼──────────────┤
│ D    │ 危及生命  │ 高概率暴露   │ 难以控制     │
│ C    │ 严重伤害  │ 中等暴露     │ 一般可控     │
│ B    │ 中度伤害  │ 低暴露       │ 容易控制     │
│ A    │ 轻微伤害  │ 极低暴露     │ 完全可控     │
│ QM   │ 质量管理  │ 不适用       │ 不适用       │
└──────┴──────────┴──────────────┴──────────────┘

危害分析与风险评估 (HARA):
  ASIL = f(S, E, C)

  示例: 刹车系统故障
    S = S3 (危及生命安全)
    E = E4 (高概率暴露)
    C = C3 (难以控制)
    ASIL = D (最高等级)
```

### 1.2 故障类型分类

```
故障分类层次
│
├── 系统性故障 (Systematic Faults)
│   ├── 设计错误
│   │   ├── 需求理解错误
│   │   ├── 架构设计缺陷
│   │   └── 算法错误
│   │
│   └── 制造过程故障
│       ├── 软件编码错误
│       ├── 硬件设计错误
│       └── 生产偏差
│
└── 随机硬件故障 (Random Hardware Faults)
    ├── 永久故障
    │   ├── 开路
    │   ├── 短路
    │   └──  stuck-at故障
    │
    └── 瞬态故障
        ├── 单粒子翻转 (SEU)
        ├── 电磁干扰
        └── 电压瞬态

故障度量指标:
  SFF = (Σλ_safe + Σλ_detected) / Σλ_total
  (Safe Failure Fraction = 安全失效分数)

  要求:
    SIL2: SFF > 90%
    SIL3: SFF > 99%
```

## 4. 软件安全要求

### 4.1 软件ASIL等级与编码规范

| ASIL | 代码度量 | 覆盖率要求 | 静态分析 | 语言子集 |
|------|----------|------------|----------|----------|
| QM | 无 | 无 | 建议 | 无 |
| A | 圈复杂度<20 | 语句覆盖 | 基础 | MISRA-C |
| B | 圈复杂度<15 | 分支覆盖 | 标准 | MISRA-C |
| C | 圈复杂度<10 | MC/DC覆盖 | 深度 | MISRA-C+ |
| D | 圈复杂度<8 | MC/DC+修改条件 | 形式化 | 受限C |

### 4.2 安全关键代码模式

```c
/* 防御性编程模式 (ASIL D示例) */

/* 1. 输入验证 */
ErrorType process_command(uint8_t cmd, uint32_t param) {
    /* 范围检查 */
    if (cmd >= CMD_MAX) {
        log_error("Invalid command: %d", cmd);
        return ERR_INVALID_CMD;
    }

    /* 参数验证 */
    if (param > PARAM_MAX || param < PARAM_MIN) {
        log_error("Invalid parameter: %u", param);
        return ERR_INVALID_PARAM;
    }

    /* 状态检查 */
    if (system_state != STATE_READY) {
        log_error("System not ready");
        return ERR_WRONG_STATE;
    }

    return execute_command(cmd, param);
}

/* 2. 双通道比较 */
BoolType safe_compare(uint32_t val1, uint32_t val2) {
    /*  diversely implemented comparison */
    volatile uint32_t v1 = val1;
    volatile uint32_t v2 = val2;

    BoolType result1 = (v1 == v2);
    BoolType result2 = ((v1 ^ v2) == 0);

    /* 交叉检查 */
    if (result1 != result2) {
        trigger_safe_state();
        return FALSE;
    }

    return result1;
}

/* 3. 程序流监控 */
void safety_critical_function(void) {
    static uint16_t flow_counter = 0;

    /* 入口标记 */
    flow_counter ^= ENTRY_MARKER;

    /* 关键操作 */
    operation_a();
    flow_counter ^= STEP_A_MARKER;

    operation_b();
    flow_counter ^= STEP_B_MARKER;

    operation_c();
    flow_counter ^= STEP_C_MARKER;

    /* 出口检查 */
    if (flow_counter != EXPECTED_FINAL) {
        trigger_safe_state();
    }

    flow_counter = 0;
}
```
